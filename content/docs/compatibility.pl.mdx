LibreChat łączy się z wieloma dostawcami AI poprzez niewielki zestaw **typów endpoint**.
Ta strona podsumowuje, jakie możliwości obsługuje każdy z endpointów. Wiele możliwości
zależy od konkretnego modelu, który wybierzesz, dlatego te komórki są oznaczone jako
zależne od modelu, zamiast jednoznacznego tak lub nie.

## Uwagi [#notes]

- **Custom (OpenAI-compatible)** obejmuje każdego dostawcę skonfigurowanego za pomocą [niestandardowych punktów końcowych](/docs/configuration/librechat_yaml/ai_endpoints), takich jak OpenRouter, Groq, Mistral, Ollama, DeepSeek, Perplexity i inni. Dokładne wsparcie zależy od tego, jak wiernie dany dostawca odwzorowuje API OpenAI oraz od wybranego modelu.
- **Agents** to najbardziej zaawansowany endpoint: dodaje obsługę narzędzi, [MCP](/docs/features/mcp), [Code Interpreter](/docs/features/code_interpreter), wyszukiwanie plików oraz [generowanie obrazów](/docs/features/image_gen) do większości modeli bazowych. Sięgnij po niego, gdy potrzebujesz możliwości, których zwykły endpoint czatu nie oferuje.
- **Vision** oraz **tools / function calling** to właściwości *modelu*, a nie endpointu, dlatego niezależnie od dostawcy wymagany jest odpowiedni model.
- **Assistants** korzysta ze środowiska uruchomieniowego OpenAI Assistants, które zapewnia własne wyszukiwanie plików i wykonywanie kodu, ale nie nakłada na nie funkcji LibreChat, takich jak Memory czy Artifacts.

To jest ręcznie aktualizowana szybka dokumentacja. Jeśli jakaś wartość wydaje się nieprawidłowa dla Twojej wersji, [otwórz issue lub PR](https://github.com/LibreChat-AI/librechat.ai).
