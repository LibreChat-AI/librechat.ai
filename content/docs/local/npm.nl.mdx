Voor de meeste scenario's is Docker Compose de aanbevolen installatiemethode vanwege de eenvoud, het gebruiksgemak en de betrouwbaarheid. Als u liever npm gebruikt, kunt u deze instructies volgen.

## Vereisten [#prerequisites]

- Node.js `v24.16.0`: [https://nodejs.org/en/download](https://nodejs.org/en/download)
- npm `v11.16.0`
  - LibreChat gebruikt CommonJS (CJS) en openid-client v6; Node 24 voldoet aan de vereiste CJS/ESM interop, WebCrypto en Fetch API runtime-ondersteuning.
- Git: https://git-scm.com/download/
- MongoDB (Atlas of Community Server)
  - [MongoDB Atlas](/docs/configuration/mongodb/mongodb_atlas)
  - [MongoDB Community Server](/docs/configuration/mongodb/mongodb_community)

Als je `nvm` gebruikt, installeer en selecteer dan de aanbevolen Node.js-versie en update vervolgens npm:

```bash filename="Use Node.js 24 and npm 11"
nvm install 24.16.0
nvm use 24.16.0
npm install -g npm@11.16.0
node -v
npm -v
```

Je zou `v24.16.0` voor Node.js en `11.16.0` voor npm moeten zien voordat je de LibreChat dependencies installeert.

## Installatiestappen [#installation-steps]

### Voorbereiding [#preparation]

Voer de volgende commando's uit in je terminal:

```bash filename="Clone the Repository"
git clone https://github.com/danny-avila/LibreChat.git
```

```bash filename="Navigate to the LibreChat Directory"
cd LibreChat
```

```bash filename="Create a .env File from .env.example"
cp .env.example .env
```

> **Let op:** **Als je Windows 10 gebruikt, moet je mogelijk `copy` gebruiken in plaats van `cp`.**

```bash filename="Update the MONGO_URI"
Important: Edit the newly created `.env` file to update the `MONGO_URI` with your own MongoDB instance URI.
```

<Callout title="Update de MONGO_URI" emoji="❗">
  Bewerk het zojuist aangemaakte `.env` bestand om de `MONGO_URI` bij te werken met die van jezelf
</Callout>

### Bouwen en Starten [#build-and-start]

Zodra je de voorbereidende stappen hebt voltooid, voer je de volgende commando's uit:

```bash filename="Install dependencies"
npm run reinstall
```

`npm run reinstall` voert een schone installatie van afhankelijkheden uit en bouwt LibreChat. Gebruik dit na het wijzigen van Node.js- of npm-versies, zodat native pakketten opnieuw worden gebouwd voor de actieve runtime.

```bash filename="Start LibreChat!"
npm run backend
```

<Callout type="success" title="Toegang tot LibreChat!" emoji="🎉">
  **Bezoek [http://localhost:3080/](http://localhost:3080/)**
</Callout>

<Callout type="example" title="Tip" emoji="🔥">
  - De volgende keer dat je LibreChat wilt starten, hoef je alleen `npm run backend` uit te voeren.
</Callout>

## LibreChat bijwerken [#update-librechat]

Om LibreChat bij te werken naar de nieuwste versie, voer je de volgende commando's uit:

<Callout type="warning" emoji="">
  Stop eerst LibreChat (als je dat nog niet hebt gedaan).
</Callout>

```bash filename="Pull latest project changes"
git pull
```

```bash filename="Update dependencies"
npm run smart-reinstall
```

Als je tijdens de update van Node.js of npm-versies bent gewisseld, voer dan in plaats daarvan `npm run reinstall` uit.

```bash filename="Start LibreChat!"
npm run backend
```
