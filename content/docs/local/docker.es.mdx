Para la mayoría de los escenarios, Docker Compose es el método de instalación recomendado debido a su simplicidad, facilidad de uso y confiabilidad.

## Requisitos previos [#prerequisites]

- [`Git`](https://git-scm.com/downloads)
- [`Docker`](https://www.docker.com/products/docker-desktop/)

Docker Desktop es recomendado para la mayoría de los usuarios. Para instalaciones en servidores remotos, consulte la [Guía de despliegue de Docker en Ubuntu](/docs/remote/docker_linux).

<Callout type="warn" title="Macs con Apple Silicon (serie M)">

Las computadoras Mac con procesadores Apple Silicon (M1, M2, M3, M4) no son compatibles con las instrucciones AVX, las cuales son requeridas por la imagen de MongoDB predeterminada utilizada en la configuración de Docker Compose de LibreChat. Si utilizas una Mac de la serie M, MongoDB fallará al iniciarse.

**Corrección:** Cree un `docker-compose.override.yml` para usar una imagen de MongoDB más antigua y compatible:

```yaml filename="docker-compose.override.yml"
services:
  mongodb:
    image: mongo:4.4.18
```

Consulta la [guía de Docker Override](/docs/configuration/docker_override) para obtener más detalles.

</Callout>

## Instalación [#installation]

<Steps>
  <Step>

### Clonar el repositorio [#clone-the-repository]

```bash
git clone https://github.com/danny-avila/LibreChat.git
cd LibreChat
```

  </Step>
  <Step>

### Cree su archivo de entorno [#create-your-environment-file]

```bash
cp .env.example .env
```

El archivo `.env` predeterminado funciona de inmediato para una configuración básica. Para una configuración detallada, consulte la [referencia de .env](/docs/configuration/dotenv).

<Callout type="info" title="Windows">

En Windows, utiliza `copy .env.example .env` si `cp` no está disponible.

</Callout>

  </Step>
  <Step>

### Iniciar LibreChat [#start-librechat]

```bash
docker compose up -d
```

El primer inicio descarga las imágenes de Docker y puede tardar unos minutos. Los inicios posteriores son mucho más rápidos.

  </Step>
  <Step>

### Verificar e iniciar sesión [#verify-and-log-in]

Abre tu navegador y visita [http://localhost:3080](http://localhost:3080). Deberías ver la página de inicio de sesión de LibreChat.

<Callout type="info" title="Primera cuenta = Admin">

La primera cuenta que registres se convierte en la cuenta de administrador. No existen credenciales predeterminadas; tú creas tu propio nombre de usuario y contraseña durante el registro.

</Callout>

Haz clic en **Register** para crear tu cuenta y comenzar a usar LibreChat.

  </Step>
</Steps>

## Montaje de librechat.yaml [#mounting-librechatyaml]

Para usar un archivo de configuración `librechat.yaml` personalizado con Docker, necesitas montarlo como un volumen para que el contenedor pueda acceder a él.

Copia el archivo de ejemplo de anulación y edítalo:

```bash
cp docker-compose.override.yml.example docker-compose.override.yml
```

Asegúrate de que el montaje de volumen de librechat.yaml esté descomentado en `docker-compose.override.yml`:

```yaml filename="docker-compose.override.yml"
services:
  api:
    volumes:
      - type: bind
        source: ./librechat.yaml
        target: /app/librechat.yaml
```

Reinicie para que los cambios surtan efecto:

```bash
docker compose down && docker compose up -d
```

Para obtener instrucciones de configuración completas, incluyendo la creación del archivo desde cero, consulta la [guía de librechat.yaml](/docs/configuration/librechat_yaml). Para más opciones de anulación, consulta la [guía de anulación de Docker](/docs/configuration/docker_override).

## Actualizando LibreChat [#updating-librechat]

Los siguientes comandos obtendrán los cambios más recientes del proyecto LibreChat, incluyendo cualquier modificación necesaria en los archivos docker compose, así como las imágenes precompiladas más recientes.

<Callout type="info" title="Permisos">

Es posible que necesites anteponer `sudo` a los comandos según los permisos de tu entorno.

</Callout>

```bash filename="Stop the running container(s)"
docker compose down
```

```bash filename="Remove all existing docker images"
# Linux/Mac
docker images -a | grep "librechat" | awk '{print $3}' | xargs docker rmi

# Windows (PowerShell)
docker images -a --filter "reference=registry.librechat.ai/danny-avila/librechat*" --format "{{.ID}}" | ForEach-Object { docker rmi $_ }
docker images -a --filter "reference=ghcr.io/danny-avila/librechat*" --format "{{.ID}}" | ForEach-Object { docker rmi $_ }
```

```bash filename="Pull latest project changes"
git pull
```

```bash filename="Pull the latest LibreChat image"
docker compose pull
```

```bash filename="Start LibreChat"
docker compose up
```

## Solución de problemas [#troubleshooting]

### Puerto ya en uso [#port-already-in-use]

Si ves un error como `bind: address already in use` para el puerto 3080, otra aplicación está utilizando ese puerto.

O detenga la aplicación en conflicto, o cambie el puerto en `docker-compose.override.yml`:

```yaml filename="docker-compose.override.yml"
services:
  api:
    ports:
      - "3081:3080"
```

Entonces visita `http://localhost:3081` en su lugar.

### El contenedor se cierra al iniciar [#container-crashes-on-startup]

Si los contenedores se cierran inmediatamente después de iniciarse, revisa los registros:

```bash
docker compose logs api
```

Causas comunes:

- Sintaxis de `librechat.yaml` no válida: valídela con el [YAML Validator](/toolkit/yaml_checker)
- Falta el archivo `.env`: asegúrese de que `.env` exista en la raíz del proyecto
- Docker no se está ejecutando -- asegúrate de que Docker Desktop esté abierto y en ejecución

### Variables de entorno faltantes [#missing-environment-variables]

Si las funciones no funcionan como se espera, verifique que las variables de entorno requeridas estén configuradas en su archivo `.env`.

```bash
docker compose exec api env | grep -i "your_variable"
```

Consulta la [.env reference](/docs/configuration/dotenv) para ver todas las variables disponibles y sus valores predeterminados.
