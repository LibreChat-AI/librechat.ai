Helm을 사용하여 Kubernetes에 LibreChat을 배포하려면 이 지침을 따르되, 특정 사용 사례에 맞게 필요에 따라 조정하십시오. 커뮤니티에서 제공한 다른 Helm 차트는 아래 [Community Helm Charts](#community-helm-charts) 섹션에 나열되어 있습니다.

## 필수 조건 [#prerequisites]

* 실행 중인 Kubernetes 클러스터
* `kubectl` 및 Helm의 _로컬_ 설치

## 구성 [#configuration]

1. [Credentials Generator](/toolkit/creds_generator)를 사용하여 `CREDS_KEY`, `JWT_SECRET`, `JWT_REFRESH_SECRET` 및 `MEILI_MASTER_KEY`에 대한 보안 값을 생성하세요.
다음과 같이 Kubernetes Secret에 배치하세요 (Secret 이름을 변경하는 경우 Helm 값을 업데이트하는 것을 잊지 마세요):

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: librechat-credentials-env
  namespace: <librechat-chart-namespace>
type: Opaque
stringData:
  CREDS_KEY: <generated value>
  JWT_SECRET: <generated value>
  JWT_REFRESH_SECRET: <generated value>
  MEILI_MASTER_KEY: <generated value>
```
2. 동일한 secret에 LLM 제공업체에 필요한 모든 API 키를 추가하세요:

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: librechat-credentials-env
  namespace: <librechat-chart-namespace>
. . . .

  OPENAI_API_KEY: <your secret value>
```

## Helm Chart 설치 [#install-helm-chart]

helm chart를 설치하려면 다음을 실행하세요:

`helm install <deployment-name> oci://ghcr.io/danny-avila/librechat-chart/librechat`

<details>
  <summary>개발 버전</summary>

저장소의 루트 디렉토리에서 다음을 실행하세요:

`helm install <deployment-name> ./helm/librechat`
</details>

다른 Helm 차트와 마찬가지로, 기본 설정을 설명하고 수정 가능한 구성 옵션을 나타내는 [values file](https://github.com/danny-avila/LibreChat/blob/main/helm/librechat/values.yaml)이 존재합니다.

기본값에서 수정하려는 값들로 채워진 `values.yaml` 파일을 생성하세요.

Helm 차트 설치: `helm install librechat oci://ghcr.io/danny-avila/librechat-chart/librechat --values <values-override-filel>`

## Helm Chart 제거하기 [#uninstall-the-helm-chart]

Helm Chart를 제거하려면 다음 명령어를 실행하세요: `helm uninstall <deployment-name>`

예시: `helm uninstall librechat`

## 1.x -> 2.x 마이그레이션 [#migrate-1x---2x]
2.x 버전 이전의 차트를 사용했다면 `value` 구조를 업데이트해야 할 수도 있습니다.

1. 설정을 librechat.configEnv로 이동:
```diff
- env:
-     ALLOW_EMAIL_LOGIN: "true"
-     ALLOW_REGISTRATION: "true"
+ librechat:
+   configEnv:
+     ALLOW_REGISTRATION: "true"
+     ALLOW_EMAIL_LOGIN: "true"
```

2. [Configuration Step 1](#configuration)에 설명된 대로 모든 Secret 값을 단일 Secret으로 통합하세요.
3. 외부 MongoDB 인스턴스를 활용하려면 Chart의 [values file](https://github.com/danny-avila/LibreChat/blob/main/helm/librechat/values.yaml)을 참조하여 해당 구성 요소를 비활성화하고 Mongodb 인스턴스의 FQDN을 변경하십시오. 이 방법은 외부에서 관리되는 MongoDB 인스턴스에 이미 데이터가 존재하는 경우 권장됩니다.
