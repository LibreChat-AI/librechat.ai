Für die meisten Szenarien ist Docker Compose aufgrund seiner Einfachheit, Benutzerfreundlichkeit und Zuverlässigkeit die empfohlene Installationsmethode. Wenn Sie die Verwendung von npm bevorzugen, können Sie diesen Anweisungen folgen.

## Voraussetzungen [#prerequisites]

- Node.js `v24.16.0`: [https://nodejs.org/en/download](https://nodejs.org/en/download)
- npm `v11.16.0`
  - LibreChat verwendet CommonJS (CJS) und openid-client v6; Node 24 erfüllt die erforderliche CJS/ESM-Interoperabilität sowie die Laufzeitunterstützung für WebCrypto und die Fetch API.
- Git: https://git-scm.com/download/
- MongoDB (Atlas oder Community Server)
  - [MongoDB Atlas](/docs/configuration/mongodb/mongodb_atlas)
  - [MongoDB Community Server](/docs/configuration/mongodb/mongodb_community)

Wenn Sie `nvm` verwenden, installieren und wählen Sie die empfohlene Node.js-Version aus und aktualisieren Sie dann npm:

```bash filename="Use Node.js 24 and npm 11"
nvm install 24.16.0
nvm use 24.16.0
npm install -g npm@11.16.0
node -v
npm -v
```

Du solltest `v24.16.0` für Node.js und `11.16.0` für npm sehen, bevor du die Abhängigkeiten von LibreChat installierst.

## Installationsschritte [#installation-steps]

### Vorbereitung [#preparation]

Führen Sie die folgenden Befehle in Ihrem Terminal aus:

```bash filename="Clone the Repository"
git clone https://github.com/danny-avila/LibreChat.git
```

```bash filename="Navigate to the LibreChat Directory"
cd LibreChat
```

```bash filename="Create a .env File from .env.example"
cp .env.example .env
```

> **Hinweis:** **Wenn Sie Windows 10 verwenden, müssen Sie möglicherweise `copy` anstelle von `cp` verwenden.**

```bash filename="Update the MONGO_URI"
Important: Edit the newly created `.env` file to update the `MONGO_URI` with your own MongoDB instance URI.
```

<Callout title="Aktualisiere die MONGO_URI" emoji="❗">
  Bearbeiten Sie die neu erstellte `.env` Datei, um die `MONGO_URI` mit Ihren eigenen Daten zu aktualisieren.
</Callout>

### Erstellen und Starten [#build-and-start]

Sobald Sie die Vorbereitungsschritte abgeschlossen haben, führen Sie die folgenden Befehle aus:

```bash filename="Install dependencies"
npm run reinstall
```

`npm run reinstall` führt eine saubere Abhängigkeitsinstallation durch und baut LibreChat neu. Verwenden Sie diesen Befehl nach dem Ändern von Node.js- oder npm-Versionen, damit native Pakete für die aktive Laufzeitumgebung neu erstellt werden.

```bash filename="Start LibreChat!"
npm run backend
```

<Callout type="success" title="Greifen Sie auf LibreChat zu!" emoji="🎉">
  **Besuchen Sie [http://localhost:3080/](http://localhost:3080/)**
</Callout>

<Callout type="example" title="Tipp" emoji="🔥">
  - Wenn Sie LibreChat das nächste Mal starten möchten, müssen Sie nur `npm run backend` ausführen.
</Callout>

## LibreChat aktualisieren [#update-librechat]

Um LibreChat auf die neueste Version zu aktualisieren, führen Sie die folgenden Befehle aus:

<Callout type="warning" emoji="">
  Stoppen Sie zuerst LibreChat (falls Sie dies noch nicht getan haben).
</Callout>

```bash filename="Pull latest project changes"
git pull
```

```bash filename="Update dependencies"
npm run smart-reinstall
```

Falls du während des Updates die Node.js- oder npm-Versionen geändert hast, führe stattdessen `npm run reinstall` aus.

```bash filename="Start LibreChat!"
npm run backend
```
