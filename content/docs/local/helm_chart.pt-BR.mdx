Siga estas orientações para implantar o LibreChat no Kubernetes usando Helm, ajustando conforme necessário para o seu caso de uso específico. Outros Helm charts contribuídos pela comunidade estão listados abaixo na seção [Community Helm Charts](#community-helm-charts).

## Pré-requisitos [#prerequisites]

* Um cluster Kubernetes em execução
* Instalações _locais_ de `kubectl` e Helm

## Configuração [#configuration]

1. Use o [Credentials Generator](/toolkit/creds_generator) para gerar valores seguros para `CREDS_KEY`, `JWT_SECRET`, `JWT_REFRESH_SECRET` e `MEILI_MASTER_KEY`.
Coloque-os em um Kubernetes Secret como este (se você alterar o nome do secret, lembre-se de atualizar seus valores do Helm):

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: librechat-credentials-env
  namespace: <librechat-chart-namespace>
type: Opaque
stringData:
  CREDS_KEY: <generated value>
  JWT_SECRET: <generated value>
  JWT_REFRESH_SECRET: <generated value>
  MEILI_MASTER_KEY: <generated value>
```
2. Adicione a este mesmo secret quaisquer chaves de API necessárias para provedores de LLM:

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: librechat-credentials-env
  namespace: <librechat-chart-namespace>
. . . .

  OPENAI_API_KEY: <your secret value>
```

## Instalar Helm Chart [#install-helm-chart]

Para instalar o helm chart, execute:

`helm install <deployment-name> oci://ghcr.io/danny-avila/librechat-chart/librechat`

<details>
  <summary>Versão de desenvolvimento</summary>

No diretório raiz do repositório, execute:

`helm install <deployment-name> ./helm/librechat`
</details>

Assim como em outros Helm charts, existe um [values file](https://github.com/danny-avila/LibreChat/blob/main/helm/librechat/values.yaml) que descreve as configurações padrão e indica quais opções de configuração podem ser modificadas.

Crie um arquivo `values.yaml` preenchido com os valores que você deseja modificar a partir do padrão.

Instale o Helm chart: `helm install librechat oci://ghcr.io/danny-avila/librechat-chart/librechat --values <values-override-filel>`

## Desinstalar o Helm Chart [#uninstall-the-helm-chart]

Para desinstalar o Helm Chart: `helm uninstall <deployment-name>`

Exemplo: `helm uninstall librechat`

## Migrar 1.x -> 2.x [#migrate-1x---2x]
Se você usou o chart antes da versão 2.x, talvez precise atualizar a estrutura de `value`.

1. Mover Config para librechat.configEnv:
```diff
- env:
-     ALLOW_EMAIL_LOGIN: "true"
-     ALLOW_REGISTRATION: "true"
+ librechat:
+   configEnv:
+     ALLOW_REGISTRATION: "true"
+     ALLOW_EMAIL_LOGIN: "true"
```

2. Consolide todos os valores de Secret em um único Secret, conforme descrito em [Configuration Step 1](#configuration).
3. Para utilizar uma instância externa do MongoDB, consulte o [values file](https://github.com/danny-avila/LibreChat/blob/main/helm/librechat/values.yaml) do Chart, desative os componentes adequadamente e altere o FQDN da instância do Mongodb. Isso é recomendado caso já existam dados nesta instância do MongoDB gerenciada externamente.
