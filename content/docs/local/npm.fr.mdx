Pour la plupart des scénarios, Docker Compose est la méthode d'installation recommandée en raison de sa simplicité, de sa facilité d'utilisation et de sa fiabilité. Si vous préférez utiliser npm, vous pouvez suivre ces instructions.

## Prérequis [#prerequisites]

- Node.js `v24.16.0` : [https://nodejs.org/en/download](https://nodejs.org/en/download)
- npm `v11.16.0`
  - LibreChat utilise CommonJS (CJS) et openid-client v6 ; Node 24 prend en charge l'interopérabilité CJS/ESM, WebCrypto et l'API Fetch requises.
- Git : https://git-scm.com/download/
- MongoDB (Atlas ou Community Server)
  - [MongoDB Atlas](/docs/configuration/mongodb/mongodb_atlas)
  - [MongoDB Community Server](/docs/configuration/mongodb/mongodb_community)

Si vous utilisez `nvm`, installez et sélectionnez la version de Node.js recommandée, puis mettez à jour npm :

```bash filename="Use Node.js 24 and npm 11"
nvm install 24.16.0
nvm use 24.16.0
npm install -g npm@11.16.0
node -v
npm -v
```

Vous devriez voir `v24.16.0` pour Node.js et `11.16.0` pour npm avant d'installer les dépendances de LibreChat.

## Étapes d'installation [#installation-steps]

### Préparation [#preparation]

Exécutez les commandes suivantes dans votre terminal :

```bash filename="Clone the Repository"
git clone https://github.com/danny-avila/LibreChat.git
```

```bash filename="Navigate to the LibreChat Directory"
cd LibreChat
```

```bash filename="Create a .env File from .env.example"
cp .env.example .env
```

> **Remarque :** **Si vous utilisez Windows 10, vous devrez peut-être utiliser `copy` au lieu de `cp`.**

```bash filename="Update the MONGO_URI"
Important: Edit the newly created `.env` file to update the `MONGO_URI` with your own MongoDB instance URI.
```

<Callout title="Mettre à jour la MONGO_URI" emoji="❗">
  Modifiez le fichier `.env` nouvellement créé pour mettre à jour `MONGO_URI` avec le vôtre
</Callout>

### Construire et démarrer [#build-and-start]

Une fois que vous avez terminé les étapes de préparation, exécutez les commandes suivantes :

```bash filename="Install dependencies"
npm run reinstall
```

`npm run reinstall` effectue une installation propre des dépendances et compile LibreChat. Utilisez-le après avoir modifié les versions de Node.js ou npm afin que les paquets natifs soient recompilés pour l'environnement d'exécution actif.

```bash filename="Start LibreChat!"
npm run backend
```

<Callout type="success" title="Accédez à LibreChat !" emoji="🎉">
  **Visitez [http://localhost:3080/](http://localhost:3080/)**
</Callout>

<Callout type="example" title="Astuce" emoji="🔥">
  - La prochaine fois que vous voudrez démarrer LibreChat, vous aurez seulement besoin d'exécuter `npm run backend`
</Callout>

## Mettre à jour LibreChat [#update-librechat]

Pour mettre à jour LibreChat vers la dernière version, exécutez les commandes suivantes :

<Callout type="warning" emoji="">
  Tout d'abord, arrêtez LibreChat (si ce n'est pas déjà fait).
</Callout>

```bash filename="Pull latest project changes"
git pull
```

```bash filename="Update dependencies"
npm run smart-reinstall
```

Si vous avez modifié les versions de Node.js ou npm pendant la mise à jour, exécutez plutôt `npm run reinstall`.

```bash filename="Start LibreChat!"
npm run backend
```
