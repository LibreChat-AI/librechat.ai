## 前置要求 [#prerequisites]

* 一个正在运行的 Kubernetes 集群
* _本地_ 安装的 `kubectl` 和 Helm

## 配置 [#configuration]

1. 使用 [Credentials Generator](/toolkit/creds_generator) 为 `CREDS_KEY`、`JWT_SECRET`、`JWT_REFRESH_SECRET` 和 `MEILI_MASTER_KEY` 生成安全值。

将它们放入 Kubernetes Secret 中，如下所示（如果您更改了 secret 名称，请记得更新您的 Helm values）：

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: librechat-credentials-env
  namespace: <librechat-chart-namespace>
type: Opaque
stringData:
  CREDS_KEY: <generated value>
  JWT_SECRET: <generated value>
  JWT_REFRESH_SECRET: <generated value>
  MEILI_MASTER_KEY: <generated value>
```
2. 将任何所需的 LLM 提供商 API 密钥添加到此相同的 secret 中：

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: librechat-credentials-env
  namespace: <librechat-chart-namespace>
. . . .

  OPENAI_API_KEY: <your secret value>
```

## 安装 Helm Chart [#install-helm-chart]

要安装 helm chart，请运行：

`helm install <deployment-name> oci://ghcr.io/danny-avila/librechat-chart/librechat`

<details>
  <summary>开发版本</summary>

在仓库的根目录下，运行：

`helm install <deployment-name> ./helm/librechat`
</details>

与其他 Helm chart 类似，这里存在一个 [values file](https://github.com/danny-avila/LibreChat/blob/main/helm/librechat/values.yaml)，它概述了默认设置并指出了可以修改的配置选项。

创建一个 `values.yaml` 文件，并填入你想要从默认配置中修改的值。

安装 Helm chart：`helm install librechat oci://ghcr.io/danny-avila/librechat-chart/librechat --values <values-override-filel>`

## 卸载 Helm Chart [#uninstall-the-helm-chart]

要卸载 Helm Chart，请运行：`helm uninstall <deployment-name>`

示例：`helm uninstall librechat`

## 迁移 1.x -> 2.x [#migrate-1x---2x]
如果您在 2.x 版本之前使用过该 chart，您可能需要更新 `value` 结构。

1. 将配置移动到 librechat.configEnv：
```diff
- env:
-     ALLOW_EMAIL_LOGIN: "true"
-     ALLOW_REGISTRATION: "true"
+ librechat:
+   configEnv:
+     ALLOW_REGISTRATION: "true"
+     ALLOW_EMAIL_LOGIN: "true"
```

2. 将所有 Secret 值合并为一个 Secret，如 [Configuration Step 1](#configuration) 所述。
3. 若要使用外部 MongoDB 实例，请参考该 Chart 的 [values file](https://github.com/danny-avila/LibreChat/blob/main/helm/librechat/values.yaml)，相应地停用组件并更改 Mongodb 实例的 FQDN。如果该外部管理的 MongoDB 实例中已存在数据，则建议采用此方法。
