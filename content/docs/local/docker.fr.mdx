Pour la plupart des scénarios, Docker Compose est la méthode d'installation recommandée en raison de sa simplicité, de sa facilité d'utilisation et de sa fiabilité.

## Prérequis [#prerequisites]

- [`Git`](https://git-scm.com/downloads)
- [`Docker`](https://www.docker.com/products/docker-desktop/)

Docker Desktop est recommandé pour la plupart des utilisateurs. Pour les installations sur serveur distant, consultez le [Guide de déploiement Docker sur Ubuntu](/docs/remote/docker_linux).

<Callout type="warn" title="Macs avec Apple Silicon (série M)">

Les ordinateurs Mac équipés de processeurs Apple Silicon (M1, M2, M3, M4) ne prennent pas en charge les instructions AVX, qui sont requises par l'image MongoDB par défaut utilisée dans la configuration Docker Compose de LibreChat. Si vous utilisez un Mac de la série M, MongoDB plantera au démarrage.

**Correction :** Créez un `docker-compose.override.yml` pour utiliser une image MongoDB plus ancienne et compatible :

```yaml filename="docker-compose.override.yml"
services:
  mongodb:
    image: mongo:4.4.18
```

Consultez le [guide de remplacement Docker](/docs/configuration/docker_override) pour plus de détails.

</Callout>

## Installation [#installation]

<Steps>
  <Step>

### Cloner le dépôt [#clone-the-repository]

```bash
git clone https://github.com/danny-avila/LibreChat.git
cd LibreChat
```

  </Step>
  <Step>

### Créer votre fichier d'environnement [#create-your-environment-file]

```bash
cp .env.example .env
```

Le fichier `.env` par défaut fonctionne immédiatement pour une configuration de base. Pour une configuration approfondie, consultez la [référence .env](/docs/configuration/dotenv).

<Callout type="info" title="Windows">

On Windows, use `copy .env.example .env` if `cp` is not available.

</Callout>

  </Step>
  <Step>

### Démarrer LibreChat [#start-librechat]

```bash
docker compose up -d
```

Le premier lancement télécharge les images Docker et peut prendre quelques minutes. Les démarrages suivants sont beaucoup plus rapides.

  </Step>
  <Step>

### Vérifier et se connecter [#verify-and-log-in]

Ouvrez votre navigateur et visitez [http://localhost:3080](http://localhost:3080). Vous devriez voir la page de connexion de LibreChat.

<Callout type="info" title="Premier compte = Admin">

Le premier compte que vous enregistrez devient le compte administrateur. Il n'y a pas d'identifiants par défaut -- vous créez votre propre nom d'utilisateur et mot de passe lors de l'inscription.

</Callout>

Cliquez sur **Register** pour créer votre compte et commencer à utiliser LibreChat.

  </Step>
</Steps>

## Montage de librechat.yaml [#mounting-librechatyaml]

Pour utiliser un fichier de configuration `librechat.yaml` personnalisé avec Docker, vous devez le monter en tant que volume afin que le conteneur puisse y accéder.

Copiez l'exemple de fichier de remplacement et modifiez-le :

```bash
cp docker-compose.override.yml.example docker-compose.override.yml
```

Assurez-vous que le montage de volume librechat.yaml est décommenté dans `docker-compose.override.yml` :

```yaml filename="docker-compose.override.yml"
services:
  api:
    volumes:
      - type: bind
        source: ./librechat.yaml
        target: /app/librechat.yaml
```

Redémarrez pour que les modifications prennent effet :

```bash
docker compose down && docker compose up -d
```

Pour des instructions de configuration complètes, y compris la création du fichier à partir de zéro, consultez le [guide librechat.yaml](/docs/configuration/librechat_yaml). Pour plus d'options de remplacement, consultez le [guide de remplacement Docker](/docs/configuration/docker_override).

## Mise à jour de LibreChat [#updating-librechat]

Les commandes suivantes récupéreront les dernières modifications du projet LibreChat, y compris tous les changements nécessaires aux fichiers docker compose, ainsi que les dernières images pré-construites.

<Callout type="info" title="Autorisations">

Il se peut que vous deviez faire précéder les commandes de `sudo` en fonction des permissions de votre environnement.

</Callout>

```bash filename="Stop the running container(s)"
docker compose down
```

```bash filename="Remove all existing docker images"
# Linux/Mac
docker images -a | grep "librechat" | awk '{print $3}' | xargs docker rmi

# Windows (PowerShell)
docker images -a --filter "reference=registry.librechat.ai/danny-avila/librechat*" --format "{{.ID}}" | ForEach-Object { docker rmi $_ }
docker images -a --filter "reference=ghcr.io/danny-avila/librechat*" --format "{{.ID}}" | ForEach-Object { docker rmi $_ }
```

```bash filename="Pull latest project changes"
git pull
```

```bash filename="Pull the latest LibreChat image"
docker compose pull
```

```bash filename="Start LibreChat"
docker compose up
```

## Dépannage [#troubleshooting]

### Port déjà utilisé [#port-already-in-use]

Si vous voyez une erreur telle que `bind: address already in use` pour le port 3080, une autre application utilise ce port.

Arrêtez l'application en conflit ou modifiez le port dans `docker-compose.override.yml` :

```yaml filename="docker-compose.override.yml"
services:
  api:
    ports:
      - "3081:3080"
```

Ensuite, visitez `http://localhost:3081` à la place.

### Le conteneur plante au démarrage [#container-crashes-on-startup]

Si les conteneurs s'arrêtent immédiatement après le démarrage, vérifiez les journaux :

```bash
docker compose logs api
```

Causes courantes :

- Syntaxe `librechat.yaml` invalide -- validez-la avec le [YAML Validator](/toolkit/yaml_checker)
- Fichier `.env` manquant -- assurez-vous que `.env` existe à la racine du projet
- Docker n'est pas en cours d'exécution -- assurez-vous que Docker Desktop est ouvert et en cours d'exécution

### Variables d'environnement manquantes [#missing-environment-variables]

Si les fonctionnalités ne fonctionnent pas comme prévu, vérifiez que les variables d'environnement requises sont définies dans votre fichier `.env`.

```bash
docker compose exec api env | grep -i "your_variable"
```

Consultez la [référence .env](/docs/configuration/dotenv) pour toutes les variables disponibles et leurs valeurs par défaut.
