Proszę postępować zgodnie z tą instrukcją, aby wdrożyć LibreChat na Kubernetes przy użyciu Helm, dostosowując ją w razie potrzeby do własnego przypadku użycia. Inne wykresy Helm dostarczone przez społeczność zostały wymienione poniżej w sekcji [Community Helm Charts](#community-helm-charts).

## Wymagania wstępne [#prerequisites]

* Działający klaster Kubernetes
* _Lokalne_ instalacje `kubectl` oraz Helm

## Konfiguracja [#configuration]

1. Użyj [Generatora poświadczeń](/toolkit/creds_generator), aby wygenerować bezpieczne wartości dla `CREDS_KEY`, `JWT_SECRET`, `JWT_REFRESH_SECRET` oraz `MEILI_MASTER_KEY`.
Umieść je w obiekcie Kubernetes Secret w następujący sposób (jeśli zmienisz nazwę secretu, pamiętaj o aktualizacji swoich wartości Helm):

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: librechat-credentials-env
  namespace: <librechat-chart-namespace>
type: Opaque
stringData:
  CREDS_KEY: <generated value>
  JWT_SECRET: <generated value>
  JWT_REFRESH_SECRET: <generated value>
  MEILI_MASTER_KEY: <generated value>
```
2. Dodaj do tego samego secret wymagane klucze API dla dostawców LLM:

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: librechat-credentials-env
  namespace: <librechat-chart-namespace>
. . . .

  OPENAI_API_KEY: <your secret value>
```

## Instalacja Helm Chart [#install-helm-chart]

Aby zainstalować helm chart, uruchom:

`helm install <deployment-name> oci://ghcr.io/danny-avila/librechat-chart/librechat`

<details>
  <summary>Wersja deweloperska</summary>

W katalogu głównym repozytorium uruchom:

`helm install <deployment-name> ./helm/librechat`
</details>

Podobnie jak w przypadku innych wykresów Helm, istnieje [plik values](https://github.com/danny-avila/LibreChat/blob/main/helm/librechat/values.yaml), który określa ustawienia domyślne i wskazuje, które opcje konfiguracji można modyfikować.

Utwórz plik `values.yaml` wypełniony wartościami, które chcesz zmodyfikować względem wartości domyślnych.

Zainstaluj wykres Helm: `helm install librechat oci://ghcr.io/danny-avila/librechat-chart/librechat --values <values-override-filel>`

## Odinstaluj Helm Chart [#uninstall-the-helm-chart]

Aby odinstalować Helm Chart, użyj: `helm uninstall <deployment-name>`

Przykład: `helm uninstall librechat`

## Migracja 1.x -> 2.x [#migrate-1x---2x]
Jeśli korzystałeś z wykresu przed wersją 2.x, być może będziesz musiał zaktualizować strukturę `value`.

1. Przenieś konfigurację do librechat.configEnv:
```diff
- env:
-     ALLOW_EMAIL_LOGIN: "true"
-     ALLOW_REGISTRATION: "true"
+ librechat:
+   configEnv:
+     ALLOW_REGISTRATION: "true"
+     ALLOW_EMAIL_LOGIN: "true"
```

2. Skonsoliduj wszystkie wartości Secret w jeden Secret, zgodnie z opisem w [Kroku konfiguracji 1](#configuration).
3. Aby skorzystać z zewnętrznej instancji MongoDB, zapoznaj się z [plikiem values](https://github.com/danny-avila/LibreChat/blob/main/helm/librechat/values.yaml) dla Chart, odpowiednio dezaktywuj komponenty i zmień FQDN instancji MongoDB. Jest to zalecane, jeśli dane już istnieją w tej zewnętrznie zarządzanej instancji MongoDB.
