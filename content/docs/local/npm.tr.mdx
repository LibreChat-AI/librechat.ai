Çoğu senaryo için, basitliği, kullanım kolaylığı ve güvenilirliği nedeniyle Docker Compose önerilen kurulum yöntemidir. Eğer npm kullanmayı tercih ederseniz, bu talimatları takip edebilirsiniz.

## Ön Koşullar [#prerequisites]

- Node.js `v24.16.0`: [https://nodejs.org/en/download](https://nodejs.org/en/download)
- npm `v11.16.0`
  - LibreChat, CommonJS (CJS) ve openid-client v6 kullanır; Node 24, gerekli CJS/ESM birlikte çalışabilirliğini, WebCrypto ve Fetch API çalışma zamanı desteğini karşılar.
- Git: https://git-scm.com/download/
- MongoDB (Atlas veya Community Server)
  - [MongoDB Atlas](/docs/configuration/mongodb/mongodb_atlas)
  - [MongoDB Community Server](/docs/configuration/mongodb/mongodb_community)

`nvm` kullanıyorsanız, önerilen Node.js sürümünü yükleyip seçin ve ardından npm'i güncelleyin:

```bash filename="Use Node.js 24 and npm 11"
nvm install 24.16.0
nvm use 24.16.0
npm install -g npm@11.16.0
node -v
npm -v
```

LibreChat bağımlılıklarını yüklemeden önce Node.js için `v24.16.0` ve npm için `11.16.0` sürümlerini görmelisiniz.

## Kurulum Adımları [#installation-steps]

### Hazırlık [#preparation]

Terminalinizde aşağıdaki komutları çalıştırın:

```bash filename="Clone the Repository"
git clone https://github.com/danny-avila/LibreChat.git
```

```bash filename="Navigate to the LibreChat Directory"
cd LibreChat
```

```bash filename="Create a .env File from .env.example"
cp .env.example .env
```

> **Not:** **Windows 10 kullanıyorsanız, `cp` yerine `copy` kullanmanız gerekebilir.**

```bash filename="Update the MONGO_URI"
Important: Edit the newly created `.env` file to update the `MONGO_URI` with your own MongoDB instance URI.
```

<Callout title="MONGO_URI değerini güncelleyin" emoji="❗">
  Yeni oluşturulan `.env` dosyasını düzenleyerek `MONGO_URI` değerini kendinizinkiyle güncelleyin
</Callout>

### Derleme ve Başlatma [#build-and-start]

Hazırlık adımlarını tamamladıktan sonra aşağıdaki komutları çalıştırın:

```bash filename="Install dependencies"
npm run reinstall
```

`npm run reinstall` komutu, temiz bir bağımlılık kurulumu gerçekleştirir ve LibreChat'i derler. Node.js veya npm sürümlerini değiştirdikten sonra, yerel paketlerin aktif çalışma zamanına göre yeniden derlenmesi için bu komutu kullanın.

```bash filename="Start LibreChat!"
npm run backend
```

<Callout type="success" title="LibreChat'e erişin!" emoji="🎉">
  **[http://localhost:3080/](http://localhost:3080/) adresini ziyaret edin**
</Callout>

<Callout type="example" title="İpucu" emoji="🔥">
  - Bir dahaki sefere LibreChat'i başlatmak istediğinizde, sadece `npm run backend` komutunu çalıştırmanız yeterlidir.
</Callout>

## LibreChat'i Güncelleme [#update-librechat]

LibreChat'i en son sürüme güncellemek için aşağıdaki komutları çalıştırın:

<Callout type="warning" emoji="">
  Öncelikle, (henüz yapmadıysanız) LibreChat'i durdurun.
</Callout>

```bash filename="Pull latest project changes"
git pull
```

```bash filename="Update dependencies"
npm run smart-reinstall
```

Güncelleme sırasında Node.js veya npm sürümlerini değiştirdiyseniz, bunun yerine `npm run reinstall` komutunu çalıştırın.

```bash filename="Start LibreChat!"
npm run backend
```
