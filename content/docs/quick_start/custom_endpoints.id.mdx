LibreChat mendukung layanan yang kompatibel dengan OpenAI API sebagai endpoint kustom. LibreChat juga mendukung endpoint kustom yang kompatibel dengan Anthropic dengan `provider: "anthropic"`. Anda mengonfigurasi endpoint di `librechat.yaml`, menyimpan kunci API di `.env`, dan memasang konfigurasi tersebut melalui `docker-compose.override.yml` untuk deployment Docker.

<Callout type="info" title="File Mana yang Melakukan Apa?">

Pengaturan endpoint kustom melibatkan tiga file, masing-masing dengan peran tertentu:

1. **`librechat.yaml`** -- Mendefinisikan endpoint kustom Anda (nama, URL API, model, pengaturan tampilan)
2. **`.env`** -- Menyimpan nilai sensitif seperti kunci API (direferensikan dari librechat.yaml menggunakan sintaks `${VAR_NAME}`)
3. **`docker-compose.override.yml`** -- Memasang `librechat.yaml` ke dalam kontainer Docker (khusus pengguna Docker)

Untuk gambaran lengkap tentang bagaimana file-file ini bekerja bersama, lihat [Configuration Overview](/docs/configuration).

</Callout>

<Callout type="warning" title="Sebelum Anda Memulai">

Panduan ini mengasumsikan Anda telah menginstal dan menjalankan LibreChat. Jika belum, selesaikan [Docker setup](/docs/local/docker) terlebih dahulu.

</Callout>

## Langkah 1. Mount librechat.yaml (Khusus Docker) [#step-1-mount-librechatyaml-docker-only]

Pengguna Docker perlu melakukan mount `librechat.yaml` sebagai volume agar container dapat membacanya. Lewati langkah ini jika Anda menjalankan LibreChat secara lokal tanpa Docker.

```bash
cp docker-compose.override.yml.example docker-compose.override.yml
```

Edit `docker-compose.override.yml` dan pastikan mount volume tidak dikomentari:

```yaml filename="docker-compose.override.yml"
services:
  api:
    volumes:
      - type: bind
        source: ./librechat.yaml
        target: /app/librechat.yaml
```

Pelajari lebih lanjut: [Docker Override Guide](/docs/configuration/docker_override)

## Langkah 2. Konfigurasi librechat.yaml [#step-2-configure-librechatyaml]

Buat file `librechat.yaml` di root proyek (jika belum ada) dan tambahkan konfigurasi endpoint Anda. Lihat [panduan librechat.yaml](/docs/configuration/librechat_yaml) untuk instruksi pengaturan yang mendetail.

Berikut adalah contoh dengan **OpenRouter**, **Ollama**, dan gateway yang kompatibel dengan Anthropic:

```yaml filename="librechat.yaml"
version: 1.3.13
cache: true
endpoints:
  custom:
    - name: 'OpenRouter'
      apiKey: '${OPENROUTER_KEY}'
      baseURL: 'https://openrouter.ai/api/v1'
      models:
        default: ['meta-llama/llama-3-70b-instruct']
        fetch: true
      titleConvo: true
      titleModel: 'meta-llama/llama-3-70b-instruct'
      dropParams: ['stop']
      modelDisplayLabel: 'OpenRouter'
    - name: 'Ollama'
      apiKey: 'ollama'
      baseURL: 'http://host.docker.internal:11434/v1/'
      models:
        default: ['llama3:latest', 'command-r', 'mixtral', 'phi3']
        fetch: true
      titleConvo: true
      titleModel: 'current_model'
    - name: 'Claude-Compatible'
      provider: 'anthropic'
      apiKey: '${ANTHROPIC_API_KEY}'
      baseURL: 'https://api.anthropic.com'
      headers:
        anthropic-version: '2023-06-01'
      models:
        default: ['claude-sonnet-4-5']
        fetch: false
      titleConvo: true
      titleModel: 'claude-sonnet-4-5'
```

Telusuri semua penyedia yang kompatibel di bagian [AI Endpoints](/docs/configuration/librechat_yaml/ai_endpoints). Untuk referensi kolom lengkap, lihat [Custom Endpoint Object Structure](/docs/configuration/librechat_yaml/object_structure/custom_endpoint).

<Callout type="note" title="Endpoint yang Kompatibel dengan Anthropic">

Gunakan `provider: "anthropic"` hanya untuk endpoint yang menggunakan Anthropic Messages API asli. Untuk gateway yang kompatibel dengan OpenAI yang hanya mengekspos model Anthropic, hilangkan `provider` dan gunakan bentuk endpoint kustom yang kompatibel dengan OpenAI seperti biasa.

</Callout>

<Callout type="warning" title="Konfigurasi API Key">

Saat mengonfigurasi kunci API di endpoint kustom, Anda memiliki tiga opsi:

1. **Variabel lingkungan** (disarankan): `apiKey: "${OPENROUTER_KEY}"` -- membaca dari `.env`
2. **User provided**: `apiKey: "user_provided"` -- pengguna memasukkan kunci mereka sendiri di UI
3. **Nilai langsung** (tidak disarankan): `apiKey: "sk-your-actual-key"` -- disimpan dalam teks biasa

</Callout>

## Langkah 3. Atur Environment Variables [#step-3-set-environment-variables]

Tambahkan kunci API yang direferensikan dalam `librechat.yaml` Anda ke file `.env`:

```bash filename=".env"
OPENROUTER_KEY=your_openrouter_api_key
```

Setiap `${VARIABLE_NAME}` di dalam librechat.yaml harus memiliki entri yang sesuai di dalam `.env`.

## Langkah 4. Mulai Ulang dan Verifikasi [#step-4-restart-and-verify]

Setelah mengedit file konfigurasi, Anda harus memulai ulang LibreChat agar perubahan diterapkan.

<Tabs items={['Docker', 'Lokal']}>
  <Tabs.Tab>

```bash
docker compose down && docker compose up -d
```

  </Tabs.Tab>
  <Tabs.Tab>

Hentikan proses yang sedang berjalan (Ctrl+C) dan mulai ulang:

```bash
npm run backend
```

  </Tabs.Tab>
</Tabs>

Buka LibreChat di browser Anda. endpoint kustom Anda seharusnya muncul di menu tarik-turun pemilih endpoint.

<Callout type="info" title="Tidak Melihat endpoint Anda?">

Periksa log server untuk kesalahan konfigurasi:

```bash
docker compose logs api
```

Masalah umum: kesalahan sintaksis YAML, variabel lingkungan yang hilang, atau `librechat.yaml` tidak dimuat di Docker. Validasi YAML Anda dengan [YAML Validator](/toolkit/yaml_checker).

</Callout>

### OpenRouter Masih Tidak Muncul [#openrouter-still-does-not-show-up]

Khusus untuk OpenRouter, verifikasi rantai tiga file berikut:

1. `.env` memiliki `OPENROUTER_KEY=...`
2. `librechat.yaml` memiliki `apiKey: "${OPENROUTER_KEY}"` di bawah endpoint kustom OpenRouter
3. Pengguna Docker melakukan mount `librechat.yaml` di `docker-compose.override.yml`

Kemudian mulai ulang dengan:

```bash
docker compose down && docker compose up -d
```

Jika endpoint muncul tetapi mengembalikan `402 Payment Required`, permintaan telah berhasil mencapai OpenRouter dan masalahnya biasanya terkait dengan kredit akun, penagihan, atau ketersediaan model di OpenRouter.
