## Part I: 처음부터 시작하기: [#part-i-starting-from-zero]

### **1. DigitalOcean에서 시작하기** [#1-get-started-on-digitalocean]

[여기를 클릭](https://www.digitalocean.com/?refcode=4486923fcf00&utm_campaign=Referral_Invite&utm_medium=Referral_Program&utm_source=badge)하거나 위 배너를 클릭하여 시작하세요.

로그인하면 [멋진 환영 화면](https://cloud.digitalocean.com/welcome)이 표시됩니다.

![image](https://github.com/danny-avila/LibreChat/assets/110412045/b7a71eae-770e-4c69-a5d4-d21b939d64ed)

### **a) Projects 페이지로 이동** [#a-navigate-to-the-projects-page]

["제어판 살펴보기"](https://cloud.digitalocean.com/projects)를 클릭하거나 [프로젝트 페이지](https://cloud.digitalocean.com/projects)로 이동하세요.

DigitalOcean에서 서버 인스턴스는 **"droplets"**라고 불리며, **"Projects"** 하위에서 관리됩니다.

### **b) "Spin up a Droplet"을 클릭하여 설정을 시작하세요** [#b-click-on-spin-up-a-droplet-to-start-the-setup]

![image](https://github.com/danny-avila/LibreChat/assets/110412045/6046e8cd-ff59-4795-a29a-5f44ab2f0a6d)

필요에 따라 이 설정을 조정하세요. 저는 작동 가능한 최소한의/가장 저렴한 옵션을 선택하고 있습니다.

- **지역/데이터 센터 선택:** 귀하와 사용자에게 가장 가까운 곳
- **이미지 선택:** Ubuntu 22.04 (LTS) x64
- **크기 선택:** 공유 CPU, 기본 플랜
  - CPU 옵션: 일반, 월 6 USD 옵션 (시간당 0.009 USD, 1 GB RAM / 1 CPU / 25 GB SSD / 1000 GB 전송)
  - 추가 저장소 없음
- **인증 방법 선택:** 비밀번호 옵션이 가장 쉽지만 선택은 자유입니다
  - 또는, 기존의 SSH를 설정할 수도 있습니다.
- **권장 사항:** 개선된 메트릭 모니터링 및 알림 기능 추가 (무료)
  - 이 옵션을 선택하지 않으면 $4/mo 옵션으로도 가능할 수 있지만, 아직 테스트되지 않았습니다.
- **세부 정보 마무리:**
  - 호스트 이름을 원하는 대로 변경하세요. 그 외의 설정은 기본값으로 둡니다(1 droplet, 태그 없음).
  - 마지막으로 "Create Droplet"을 클릭하세요.

![image](https://github.com/danny-avila/LibreChat/assets/110412045/ac90d40e-3ac6-482f-885c-58058c5e3f76)

droplet을 생성한 후, 진행률 표시줄과 함께 구동이 시작됩니다.

### **2. 드롭릿(droplet) 콘솔에 액세스** [#2-access-your-droplet-console]

일단 가동되면, **droplet을 클릭**하고 오른쪽에 있는 Console 링크를 클릭하여 콘솔을 시작하세요.

![image](https://github.com/danny-avila/LibreChat/assets/110412045/47c14280-fe48-49b9-9997-ff4d9c83212c)

![image](https://github.com/danny-avila/LibreChat/assets/110412045/d5e518fd-4941-4b35-86cc-69f8f65ec8eb)

이런 방식으로 Droplet 콘솔을 실행하는 것이 가장 쉬운 방법이지만, 이전 단계에서 설정했다면 SSH를 사용할 수도 있습니다.

이 가이드를 간단하게 유지하기 위해, 저는 계속해서 droplet 콘솔을 사용하는 쉬운 방법을 따르겠습니다. 관심이 있으시다면 [SSH에 대한 공식 DigitalOcean 가이드](https://docs.digitalocean.com/products/droplets/how-to/connect-with-ssh/)를 확인해 보세요.

### **3. 로그인한 후, 즉시 루트가 아닌 새 사용자를 생성하십시오:** [#3-once-you-have-logged-in-immediately-create-a-new-non-root-user]

**참고:** 이 가이드에서 부등호(greater/less than signs)를 발견하면 언제든지 제거해야 합니다.

```bash
# example: adduser danny
adduser <yourusername>
# you will then be prompted for a password and user details
```

완료되면 다음 명령어를 실행하여 사용자의 권한을 상승시키세요

```bash
# example: usermod -aG sudo danny
usermod -aG sudo <yourusername>
```

**sudo 권한이 있는지 다시 한번 확인하여 이 작업을 올바르게 수행했는지 확인하세요:**

```bash
getent group sudo | cut -d: -f4
```

**새 사용자로 전환**

```bash
# example: su - danny
su - <yourusername>
```

### **4. 방화벽 설정** [#4-firewall-setup]

설정 환경에 간단한 방화벽을 구성하는 것을 강력히 권장합니다.

프로젝트 페이지에서 다시 드롭릿(droplet)을 클릭하고, 왼쪽 사이드바의 ipv4 아래에 있는 Networking 탭으로 이동하세요:

![image](https://github.com/danny-avila/LibreChat/assets/110412045/20a2f31b-83ec-4052-bca7-27a672c3770a)

방화벽을 생성하고, 해당 방화벽에 Droplet을 추가한 뒤, 다음 인바운드 규칙을 추가하세요(이 가이드에서는 작동하지만, 필요에 따라 구성하세요).

![image](https://github.com/danny-avila/LibreChat/assets/110412045/d9bbdd7b-3702-4d2d-899b-c6457e6d221a)
