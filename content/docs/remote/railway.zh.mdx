Railway 提供了一键安装选项来部署 LibreChat，使整个过程变得更加简单。以下是具体操作方法：

## 步骤 [#steps]

### **访问 LibreChat 仓库** [#visit-the-librechat-repository]

前往 GitHub 上的 [LibreChat 仓库](https://github.com/danny-avila/LibreChat)。

### **创建 Railway 账户** [#create-a-railway-account]

[注册一个 Railway 账户](https://railway.app?referralCode=HI9hWz&utm_medium=integration&utm_source=docs&utm_campaign=librechat)（如果您还没有账户的话；此链接包含一个支持 LibreChat 项目的推荐码）。

### **点击 "Deploy on Railway" 按钮** [#click-the-deploy-on-railway-button]

<p align="left">
    <a href="https://railway.com/deploy/librechat-official?referralCode=HI9hWz&utm_medium=integration&utm_source=docs&utm_campaign=librechat">
        <img src="https://railway.com/button.svg" alt="在 Railway 上部署" height="40"/>
    </a>
</p>

（该按钮也可在仓库的 README 文件中找到）


### **配置环境变量** [#configure-environment-variables]

Railway 将自动检测 LibreChat 所需的环境变量。请检查三个容器的配置，并在检查完每个容器后点击 `Save Config`。

![image](https://github.com/danny-avila/LibreChat/assets/32828263/4417e997-621c-44b6-8d2d-94d7e4e1a2bf)

默认配置即可助您入门，但若需使用更多高级功能，您可以查阅我们关于此主题的文档：[Environment Variables](/docs/configuration/dotenv)

### **部署** [#deploy]

一旦您填好了所需的环境变量，请点击“Deploy”按钮。Railway 将处理剩余的工作，包括设置 PostgreSQL 数据库以及构建/部署您的 LibreChat 实例。

![image](https://github.com/danny-avila/LibreChat/assets/32828263/d94e20c6-0ae7-42af-8937-7fbd34d63a3b)

### **访问您的 LibreChat 实例** [#access-your-librechat-instance]

部署成功后，Railway 将为您提供一个公共 URL，您可以通过该 URL 访问您的 LibreChat 实例。

就是这样！您已成功使用一键安装流程在 Railway 上部署了 LibreChat。现在，您可以根据需要开始使用并自定义您的 LibreChat 实例。

## 额外提示 [#additional-tips]

- 定期检查 LibreChat 仓库以获取更新，并重新部署您的实例，从而获得最新的功能和错误修复。
- 在登录 Railway 后，你可以通过点击“view logs”按钮右侧的 3 个点来找到“redeploy”选项。

有关更详细的说明和故障排除，请参阅官方 LibreChat 文档和 Railway 指南。
