LibreChat maakt verbinding met veel AI-providers via een kleine set **endpoint types**.
Deze pagina vat samen welke mogelijkheden elk endpoint ondersteunt. Veel mogelijkheden
hangen af van het specifieke model dat je selecteert, dus die cellen zijn gemarkeerd als
model-afhankelijk in plaats van een harde ja of nee.

## Opmerkingen [#notes]

- **Custom (OpenAI-compatible)** omvat elke provider die is geconfigureerd via [custom endpoints](/docs/configuration/librechat_yaml/ai_endpoints), zoals OpenRouter, Groq, Mistral, Ollama, DeepSeek, Perplexity en anderen. De exacte ondersteuning hangt af van hoe nauw de provider de OpenAI API spiegelt en van het model dat je kiest.
- **Agents** is het meest capabele endpoint: het voegt toolgebruik, [MCP](/docs/features/mcp), [Code Interpreter](/docs/features/code_interpreter), bestandszoekopdrachten en [Image Generation](/docs/features/image_gen) toe bovenop de meeste basismodellen. Gebruik dit wanneer je mogelijkheden nodig hebt die een standaard chat-endpoint niet biedt.
- **Vision** en **tools / function calling** zijn eigenschappen van het *model*, niet van de endpoint, dus is een capabel model vereist, ongeacht de provider.
- **Assistants** maakt gebruik van de Assistants-runtime van OpenAI, die voorziet in eigen bestandzoekfuncties en code-uitvoering, maar geen LibreChat-functies zoals Memory of Artifacts toevoegt.

Dit is een handmatig bijgehouden snelstartgids. Als een waarde onjuist lijkt voor jouw release, [open dan een issue of PR](https://github.com/LibreChat-AI/librechat.ai).
