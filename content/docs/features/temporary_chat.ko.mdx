임시 채팅을 사용하면 대화 내용을 저장하지 않고 질문할 수 있습니다. 민감한 주제, 간단한 실험 또는 저장할 필요가 없는 모든 작업에 사용하세요. 임시 채팅은 기록 사이드바에 표시되지 않고, 검색 결과에도 나타나지 않으며, 즐겨찾기에 추가할 수 없습니다. 또한 보존 기간(기본값 30일)이 지나면 자동으로 삭제됩니다.

## 임시 채팅 시작하기 [#start-a-temporary-chat]

<Steps>

<Step>

**모델 메뉴를 엽니다.** 채팅 보기 왼쪽 상단에 있는 모델 드롭다운을 선택합니다.

</Step>

<Step>

**임시 채팅(Temporary Chat)을 켭니다.** **Temporary Chat** 스위치를 ON 위치로 전환하세요.

</Step>

## 임시 채팅(Temporary Chat)의 기능 [#what-a-temporary-chat-does]

- 채팅 기록 사이드바에 나타나지 않습니다.
- 검색 결과에서 제외됩니다.
- 북마크할 수 없습니다.
- 보존 기간이 종료될 때까지 데이터베이스에 저장된 후 자동으로 삭제됩니다.

<Callout type="info" title="보존 기간">

기본 보존 기간은 30일입니다. 관리자는 이를 변경할 수 있습니다. 아래의 [Configuration](#configuration-administrators)을 참조하세요.

</Callout>

## 구성 (관리자용) [#configuration-administrators]

Temporary Chat은 기본적으로 사용자에게 제공됩니다. 관리자는 이 기능을 제공할지 여부와 임시 채팅을 얼마나 오래 보관할지 제어할 수 있습니다.

**Availability**는 `TEMPORARY_CHAT` 역할 권한에 의해 제어됩니다. [Admin Panel](/docs/features/admin_panel)에서 각 역할에 대해 이를 관리하세요. `librechat.yaml`의 `interface.temporaryChat` 옵션은 시작 시 기본 `USER` 역할에 대해서만 이 권한을 설정하며, 권한 관리 용도로는 더 이상 사용되지 않습니다(deprecated).

**Retention**은 `interface.temporaryChatRetention`으로 설정합니다(단위: 시간). 최소값은 1시간, 최대값은 8760(1년)이며, 기본값은 720(30일)입니다.

<Tabs items={['librechat.yaml (권장)', '환경 변수 (사용 중단됨)']}>

<Tabs.Tab>

```yaml filename="librechat.yaml"
interface:
  temporaryChat: true
  temporaryChatRetention: 168 # retain temporary chats for 7 days
```

</Tabs.Tab>

<Tabs.Tab>

```bash filename=".env"
# Hours to retain temporary chats (default: 720 = 30 days)
TEMP_CHAT_RETENTION_HOURS=168
```

<Callout type="warning" title="사용 중단됨">

`TEMP_CHAT_RETENTION_HOURS`는 더 이상 사용되지 않습니다. `librechat.yaml`의 `interface.temporaryChatRetention`을 사용하는 것을 권장하며, 이는 환경 변수보다 우선합니다.

</Callout>

</Tabs.Tab>

</Tabs>

일반적인 보존 값:

| 값 | 기간 |
| --- | --- |
| `1` | 1시간 (최소) |
| `24` | 1일 |
| `168` | 1주 |
| `720` | 30일 (기본값) |
| `8760` | 1년 (최대) |

`retentionMode` 및 `retainAgentFiles`를 포함한 전체 옵션 세트는 [인터페이스 참조](/docs/configuration/librechat_yaml/object_structure/interface#temporarychatretention)를 확인하세요.
