## Co to robi [#what-it-does]

Panel administratora jest lekkim klientem: wszystkie dane znajdujД… siДҷ w bazie danych LibreChat, a kaЕјde dziaЕӮanie przechodzi przez wersjonowane punkty koЕ„cowe `/api/admin/*` na serwerze API LibreChat. Zapewnia on administratorom jedno miejsce do:

- **Zastosuj nadpisania dla poszczegГіlnych podmiotГіw (per-principal)**: ogranicz nadpisania konfiguracji do okreЕӣlonych rГіl lub grup, korzystajД…c z kaskady opartej na priorytetach, ktГіra okreЕӣla ostatecznД… wartoЕӣДҮ rozwiД…zywanД… dla kaЕјdego uЕјytkownika w momencie logowania.
- **ZarzД…dzanie uЕјytkownikami**: wyЕӣwietlanie listy, wyszukiwanie i przeglД…danie kaЕјdego konta w instancji.
- **ZarzД…dzaj grupami**: twГіrz i usuwaj grupy, dodawaj/usuwaj czЕӮonkГіw oraz uЕјywaj grup jako gЕӮГіwnych podmiotГіw w ACL i nadpisaniach.
- **ZarzД…dzaj rolami**: twГіrz wЕӮasne role wykraczajД…ce poza wbudowane `USER` / `ADMIN`, edytuj ich macierz uprawnieЕ„ do funkcji i przypisuj uЕјytkownikГіw do rГіl.
- **Przyznawanie uprawnieЕ„ systemowych**: delegowanie moЕјliwoЕӣci administracyjnych (np. `manage:users`, `read:usage`, `manage:mcpservers`) konkretnym uЕјytkownikom, grupom lub rolom bez nadawania im peЕӮnych uprawnieЕ„ administratora.
- **Authenticate**: zaloguj siДҷ za pomocД… lokalnego konta administratora LibreChat lub poprzez OpenID SSO / SAML / wspieranych dostawcГіw OAuth, jeЕӣli sД… oni wЕӮД…czeni w instancji LibreChat.

Aby uzyskaДҮ informacje na temat podstawowego modelu uprawnieЕ„ (podmioty, listy kontroli dostДҷpu do zasobГіw, moЕјliwoЕӣci oraz sposГіb tworzenia warstw), zobacz stronДҷ [Access Control](/docs/features/access_control).

## Wprowadzenie [#getting-started]

### Wymagania wstДҷpne [#prerequisites]

- DziaЕӮajД…ca instancja LibreChat w wersji **v0.8.5 lub nowszej** (API administracyjne nie sД… dostДҷpne we wczeЕӣniejszych wersjach)

- Konto administratora w LibreChat: albo pierwszy zarejestrowany uЕјytkownik (auto-admin), uЕјytkownik z ustawionym `role: 'ADMIN'` w Mongo, albo podmiot, ktГіremu przyznano uprawnienie `access:admin`

### DoЕӮД…czone do LibreChat (zalecane) [#bundled-with-librechat-recommended]

JeЕӣli uruchamiasz LibreChat przy uЕјyciu oficjalnego pliku [`docker-compose.yml`](https://github.com/danny-avila/LibreChat/blob/main/docker-compose.yml) lub [`deploy-compose.yml`](https://github.com/danny-avila/LibreChat/blob/main/deploy-compose.yml), panel administratora jest dostarczany jako usЕӮuga i uruchamia siДҷ automatycznie wraz z LibreChat -- nie jest wymagane oddzielne wdraЕјanie.

| Plik Compose                   | URL panelu administratora | SposГіb serwowania                                                 |
| ------------------------------ | ------------------------ | ----------------------------------------------------------------- |
| `docker-compose.yml` (domyЕӣlny) | `http://localhost:3000`  | Opublikowany na porcie hosta (`ADMIN_PANEL_PORT`, domyЕӣlnie `3000`) |
| `deploy-compose.yml`           | `http://admin.localhost` | Przekierowany przez doЕӮД…czone odwrotne proxy nginx na subdomenie  |

Set the panel's session secret in LibreChat's `.env` before starting the stack; the compose files pass it through as the panel's `SESSION_SECRET`:

```bash filename=".env"
# Min 32 characters. Generate with: openssl rand -hex 32
ADMIN_PANEL_SESSION_SECRET=replace-with-a-32-char-random-string

# Optional: host port for the default docker-compose
# ADMIN_PANEL_PORT=3000

# Optional: set true when the panel is served over HTTPS
# ADMIN_PANEL_SESSION_COOKIE_SECURE=false
```

Pliki compose automatycznie konfigurujД… resztДҷ: `API_SERVER_URL` wskazuje na usЕӮugДҷ `api`, `VITE_API_BASE_URL` podД…Ејa za `DOMAIN_CLIENT` w przypadku przekierowaЕ„ OAuth w przeglД…darce, a `ADMIN_PANEL_URL` jest ustawiony tak, aby LibreChat po SSO przekierowywaЕӮ administratorГіw z powrotem do panelu. Aby zrezygnowaДҮ, usuЕ„ usЕӮugДҷ `admin-panel` lub zabezpiecz jД… za pomocД… wpisu [`profiles`](https://docs.docker.com/compose/how-tos/profiles/) w Compose.

<Callout type="info" title="admin.localhost w prawdziwej domenie">
  Nowoczesne przeglД…darki rozwiД…zujД… `*.localhost` (w tym `admin.localhost`) na `127.0.0.1`, wiДҷc adres URL deploy-compose dziaЕӮa bez koniecznoЕӣci modyfikacji pliku hosts. W przypadku prawdziwej domeny naleЕјy skierowaДҮ rekord DNS na hosta, zaktualizowaДҮ `server_name` dla `admin.localhost` w `client/nginx.conf` oraz ustawiДҮ `ADMIN_PANEL_URL` tak, aby byЕӮ z nim zgodny.
</Callout>

### Standalone (osobne wdroЕјenie) [#standalone-separate-deployment]

Aby hostowaДҮ panel administratora samodzielnie вҖ” wskazujД…c na instancjДҷ LibreChat dziaЕӮajД…cД… w innym miejscu вҖ” uЕјyj opublikowanego obrazu z GHCR:

```bash

#    SESSION_SECRET=<random string, min 32 characters>
#    VITE_API_BASE_URL=http://host.docker.internal:3080
```

Samodzielne `docker run`:

```bash

  --add-host=host.docker.internal:host-gateway \
  -e SESSION_SECRET=replace-with-32-char-random-string \
  -e VITE_API_BASE_URL=http://host.docker.internal:3080 \
```

<Callout type="warning" title="Sieci Docker">
  WewnД…trz kontenera `localhost` odnosi siДҷ do samego kontenera, a nie do hosta. Gdy LibreChat dziaЕӮa na tym samym hoЕӣcie, skieruj `VITE_API_BASE_URL` na `http://host.docker.internal:3080` (Linux: dodaj `--add-host=host.docker.internal:host-gateway`). W Еӣrodowisku produkcyjnym uЕјyj publicznej/wewnДҷtrznej nazwy DNS swojego API LibreChat.
</Callout>

### Adres URL przekierowania LibreChat [#librechat-redirect-url]

Gdy panel administratora jest hostowany pod innym adresem URL niЕј LibreChat, ustaw `ADMIN_PANEL_URL` w Еӣrodowisku API LibreChat. UЕјyj zewnДҷtrznego podstawowego adresu URL panelu administratora, wЕӮД…cznie z dowolnym prefiksem ЕӣcieЕјki, i pomiЕ„ koЕ„cowy ukoЕӣnik:


## Uwierzytelnianie [#authentication]

Panel administratora wykorzystuje ponownie stos uwierzytelniania LibreChat i nie posiada wЕӮasnej bazy danych uЕјytkownikГіw. ObsЕӮugiwane sД… dwie ЕӣcieЕјki logowania:

DostДҷp administratora jest weryfikowany po stronie serwera przez LibreChat dla kaЕјdego ЕјД…dania. Konto musi speЕӮniaДҮ jeden z poniЕјszych warunkГіw:

1. MieДҮ `role: 'ADMIN'` w MongoDB, **lub**
2. PosiadaДҮ uprawnienie systemowe `access:admin` (przypisane innemu podmiotowi za poЕӣrednictwem panelu administratora; zobacz [System Grants](/docs/features/access_control#layer-3-system-grants-admin-capabilities)).

Sesje sД… oparte na plikach cookie, szyfrowane za pomocД… `SESSION_SECRET` i wygasajД… po czasie bezczynnoЕӣci okreЕӣlonym w `ADMIN_SESSION_IDLE_TIMEOUT_MS`.

## ZarzД…dzanie konfiguracjД… [#configuration-management]

Panel renderuje konfiguracjДҷ LibreChat jako dynamiczny formularz sterowany schematem konfiguracji. Ma to dwie przydatne wЕӮaЕӣciwoЕӣci:

- **ZgodnoЕӣДҮ z przyszЕӮymi wersjami**: gdy LibreChat wprowadza nowe pole konfiguracyjne, panel automatycznie pobiera je ze schematu. Nie jest wymagana aktualizacja panelu administratora ani ponowne wdraЕјanie.
- **Warstwowe nadpisania**: konfiguracja podstawowa (z `librechat.yaml`) moЕјe byДҮ przesЕӮoniДҷta przez nadpisania dla poszczegГіlnych podmiotГіw, przypisane do roli lub grupy. Gdy uЕјytkownik siДҷ loguje, nadpisania sД… rozstrzygane w kolejnoЕӣci priorytetГіw i scalane z konfiguracjД… podstawowД…, tworzД…c efektywnД… konfiguracjДҷ widocznД… dla danego uЕјytkownika.

To jest interfejs stojД…cy za [opartym na bazie danych systemem nadpisywania konfiguracji dla poszczegГіlnych podmiotГіw](https://github.com/danny-avila/LibreChat/pull/12354) w LibreChat. Typowe przypadki uЕјycia:

- Nadaj grupie "Research" wyЕјszy `recursionLimit` oraz dodatkowe endpointy
- PozwГіl roli "FinanceAdmins" zarzД…dzaДҮ serwerami MCP, podczas gdy zwykli uЕјytkownicy mogД… ich tylko uЕјywaДҮ
- Ogranicz bardziej rygorystyczne uprawnienia `interface` do grup zewnДҷtrznych wykonawcГіw
