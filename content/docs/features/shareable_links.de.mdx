## Übersicht [#overview]

Mit Shareable Links in LibreChat können Sie Ihre Konversationen über automatisch generierte Links und QR-Codes mit anderen teilen. Egal, ob Sie mit Teamkollegen zusammenarbeiten, KI-Interaktionen präsentieren oder generierte Artefakte teilen möchten – diese Funktion macht es einfach, anderen mit nur einem Klick das Anzeigen Ihrer Konversationen zu ermöglichen.

## Hauptfunktionen [#key-features]

- **Einfaches Teilen**: Erstellen Sie mit nur wenigen Klicks teilbare Links
- **QR-Code-Generierung**: Teilen Sie Konversationen schnell über QR-Codes für den mobilen Zugriff
- **Konversations-Verzweigung**: Empfänger können verzweigte Konversationen in Ihren geteilten Links sehen
- **Artifacts Support**: Geteilte Konversationen enthalten jetzt generierte Artifacts und interaktive Komponenten
- **Anpassbares Erscheinungsbild**: Empfänger können Sprach- und Designeinstellungen anpassen
- **Link-Verwaltung**: Zentrales Dashboard zur Verwaltung all Ihrer geteilten Links

## Konfiguration [#configuration]

Das Verhalten von freigebbaren Links wird durch die folgenden Umgebungsvariablen in Ihrer `.env` Datei gesteuert:

- **`ALLOW_SHARED_LINKS`** (Standardwert: `true`) — Aktiviert oder deaktiviert die Funktion für geteilte Links vollständig. Setzen Sie diesen Wert auf `false`, um zu verhindern, dass Benutzer geteilte Links erstellen können.
- **`ALLOW_SHARED_LINKS_PUBLIC`** (Standardwert: `false`) — Steuert, ob geteilte Links ohne Authentifizierung aufgerufen werden können. Standardmäßig müssen Empfänger angemeldet sein, um geteilte Konversationen anzuzeigen. Setzen Sie dies auf `true`, um öffentlichen (nicht authentifizierten) Zugriff zu ermöglichen.

```bash
# Enable shared links (default)
ALLOW_SHARED_LINKS=true

# Require authentication to view shared links (default)
ALLOW_SHARED_LINKS_PUBLIC=false
```

Rollenberechtigungen steuern, welche Benutzer Links erstellen, Links mit authentifizierten Benutzern teilen oder Links für alle Benutzer auf der Instanz sichtbar machen können:

```yaml filename="librechat.yaml"
interface:
  sharedLinks:
    create: true
    share: true
    public: false
```

`sharedLinks.public` steuert die Fähigkeit des authentifizierten Benutzers, „mit jedem teilen“ ein- oder auszuschalten. `ALLOW_SHARED_LINKS_PUBLIC` steuert weiterhin, ob diese öffentlichen Links ohne Anmeldung eingesehen werden können.

> **Hinweis:** Für eine vollständige Referenz der Umgebungsvariablen siehe die Seite [.env Configuration](/docs/configuration/dotenv#shared-links).

## Häufig gestellte Fragen [#frequently-asked-questions]

**F: Können Empfänger auf geteilte Konversationen antworten?**
A: Nein, geteilte Links sind schreibgeschützt. Empfänger können die Konversation einsehen, aber keine Nachrichten hinzufügen oder mit der KI interagieren.

**F: Was passiert, wenn ich eine Unterhaltung lösche, nachdem ich sie geteilt habe?**
A: Der geteilte Link ist nicht mehr zugänglich und jedem Benutzer, der diesen Link aufruft, wird eine Seite mit der Meldung „Shared link not found“ angezeigt.

**F: Kann ich eine Unterhaltung bearbeiten, nachdem ich sie geteilt habe?**
A: Der geteilte Link erfasst die Unterhaltung zum Zeitpunkt der Erstellung. Zusätzliche Nachrichten werden in der geteilten Ansicht nicht angezeigt – Bearbeitungen an bestehenden Nachrichten werden jedoch übernommen.

**F: Funktionieren geteilte Links offline?**
A: Nein, Empfänger benötigen eine Internetverbindung, um geteilte Konversationen anzusehen.
