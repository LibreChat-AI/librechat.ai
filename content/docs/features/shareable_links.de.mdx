Mit teilbaren Links können Sie eine schreibgeschützte Momentaufnahme einer Unterhaltung veröffentlichen, die andere über eine generierte URL oder einen QR-Code öffnen können. Empfänger sehen die Nachrichten, alle Verzweigungen und alle von Ihnen generierten Artefakte, ohne antworten oder Änderungen vornehmen zu können.

## Hauptfunktionen [#key-features]

- **Einfaches Teilen**: Generieren Sie einen Link mit wenigen Klicks.
- **QR-Codes**: Öffnen Sie die Konversation auf einem Smartphone durch Scannen eines Codes.
- **Branching**: Geteilte Links behalten jeden Zweig der Konversation bei.
- **Artifacts**: React-Komponenten, HTML-Vorschauen und Mermaid-Diagramme bleiben interaktiv.
- **Empfängerpräferenzen**: Betrachter können Sprache und Design wechseln.
- **Link-Verwaltung**: Ein Dashboard, um jeden von Ihnen erstellten Link zu überprüfen oder zu widerrufen.

## Optionen für Freigabelinks [#share-link-options]

Das Erstellen eines Links öffnet ein modales Fenster, in dem Sie steuern können, wie die Konversation für die Empfänger angezeigt wird.

![Modal zum Erstellen von Freigabelinks](/images/shared-links/share-modal-create.png)

![Modal für geteilte Links mit Verwaltungsoptionen](/images/shared-links/share-modal-options.png)

Sobald ein Link existiert, bietet das Modal diese Aktionen an:

- **Link aktualisieren**: Generiert eine neue URL, wodurch die vorherige ungültig wird.
- **QR-Code generieren**: Erstellen Sie einen QR-Code für den mobilen Zugriff.
- **Link in die Zwischenablage kopieren**: Kopiert die teilbare URL.
- **Link löschen**: Entfernen Sie den Link und entziehen Sie jedem, der ihn besitzt, den Zugriff.

## QR-Codes [#qr-codes]

Generieren Sie einen QR-Code für jede geteilte Konversation und scannen Sie diesen anschließend mit der Kamera Ihres Smartphones, um den Link zu öffnen. Der Code verweist auf dieselbe geteilte URL.

![QR-Code, der für einen geteilten Link generiert wurde](/images/shared-links/qr-code.png)

![QR-Code-Downloadoptionen](/images/shared-links/qr-code-download.png)

QR-Codes sind praktisch für Präsentationen, schnellen mobilen Zugriff, gedruckte Handouts und Konferenz-Demos.

## Anzeigen geteilter Konversationen [#viewing-shared-conversations]

Wenn jemand Ihren Link öffnet, sieht er eine fokussierte, schreibgeschützte Ansicht der Konversation, einschließlich des Datums, an dem der Link veröffentlicht wurde, des vollständigen Nachrichtenverlaufs und aller von Ihnen generierten Artefakte.

![Eine geteilte Konversation aus der Sicht des Empfängers](/images/shared-links/shared-view.png)

## Verzweigung [#branching]

Geteilte Links bewahren Konversationsverzweigungen, sodass Empfänger den verschiedenen Pfaden folgen können, die die Diskussion genommen hat. Verwenden Sie die Navigationspfeile für Verzweigungen, um zwischen ihnen zu wechseln.

![Verzweigungs-Navigationsmenü in einer geteilten Konversation](/images/shared-links/branching-menu.png)

![Navigieren durch Zweige in einer geteilten Konversation](/images/shared-links/branching-support.png)

Dies ist nützlich, um zu zeigen, wie unterschiedliche Prompts zu verschiedenen KI-Antworten führen oder wie ein einzelnes Problem auf verschiedene Weise angegangen werden kann.

## Artifacts [#artifacts]

Während der Konversation generierte Artifacts bleiben in geteilten Links voll funktionsfähig.

![Interaktive Artefakte in einer geteilten Konversation](/images/shared-links/artifacts-support.png)

Empfänger können:

- Interaktive React-Komponenten anzeigen.
- Siehe HTML-Vorschauen.
- Untersuche Mermaid-Diagramme.
- Lesen Sie den zugrunde liegenden Code und den Kontext, in dem er generiert wurde.

## Empfängerpräferenzen [#recipient-preferences]

Betrachter können die Benutzeroberfläche an ihre eigenen Vorlieben anpassen, ohne Ihre Einstellungen zu beeinflussen.

![Sprach- und Designeinstellungen auf einem geteilten Link](/images/shared-links/recipient-settings.png)

- **Design**: Hell, dunkel oder System.
- **Sprache**: Die Benutzeroberfläche in der bevorzugten Sprache anzeigen.

## Verwalten Ihrer geteilten Links [#managing-your-shared-links]

Öffnen Sie das Dashboard für geteilte Links unter **Einstellungen → Datenkontrolle**, um alle Konversationen zu überprüfen, die Sie geteilt haben.

![Dashboard zur Verwaltung geteilter Links](/images/shared-links/management-modal.png)

![Details und Optionen für einen einzelnen geteilten Link](/images/shared-links/management-details.png)

Vom Dashboard aus können Sie:

- **Alle Links anzeigen**: Sehen Sie sich jede Konversation an, die Sie geteilt haben.
- **Suchen und Filtern**: Finden Sie einen bestimmten geteilten Link und sehen Sie sich dessen Inhalt in der Vorschau an.
- **Zugriff widerrufen**: Löschen Sie einen Link, um die Freigabe sofort zu beenden.

## Konfiguration [#configuration]

Zwei Umgebungsvariablen in Ihrer `.env` Datei steuern diese Funktion:

- **`ALLOW_SHARED_LINKS`** (Standardwert: `true`): Aktiviert die Funktion für geteilte Links. Auf `false` setzen, um Benutzern das Erstellen von Links zu untersagen.
- **`ALLOW_SHARED_LINKS_PUBLIC`** (Standardwert: `false`): Steuert, ob Links ohne Authentifizierung geöffnet werden können. Standardmäßig müssen Empfänger angemeldet sein. Setzen Sie dies auf `true`, um öffentlichen, nicht authentifizierten Zugriff zu ermöglichen.

```bash title=".env"
# Enable shared links (default)
ALLOW_SHARED_LINKS=true

# Require authentication to view shared links (default)
ALLOW_SHARED_LINKS_PUBLIC=false
```

Rollenberechtigungen legen fest, welche Benutzer Links erstellen, diese für authentifizierte Benutzer freigeben oder sie für alle Benutzer auf der Instanz sichtbar machen können:

```yaml title="librechat.yaml"
interface:
  sharedLinks:
    create: true
    share: true
    public: false
```

`sharedLinks.public` steuert, ob ein Benutzer „für alle freigeben“ umschalten kann. `ALLOW_SHARED_LINKS_PUBLIC` entscheidet weiterhin darüber, ob diese öffentlichen Links ohne Anmeldung einsehbar sind.

<Callout type="info">
Für die vollständige Referenz der Umgebungsvariablen siehe die Seite [.env Configuration](/docs/configuration/dotenv#shared-links).
</Callout>

## Häufig gestellte Fragen [#frequently-asked-questions]

<Accordions type="single" collapsible>
<Accordion title="Können Empfänger auf geteilte Konversationen antworten?">

Nein. Geteilte Links sind schreibgeschützt. Empfänger können die Konversation zwar einsehen, aber keine Nachrichten hinzufügen oder mit der KI interagieren.

</Accordion>
<Accordion title="Was passiert, wenn ich eine Konversation lösche, nachdem ich sie geteilt habe?">

Der Link funktioniert nicht mehr. Jeder, der ihn aufruft, sieht eine Seite mit der Meldung „Shared link not found“.

</Accordion>
<Accordion title="Kann ich eine Konversation bearbeiten, nachdem ich sie geteilt habe?">

Der Link erfasst die Konversation so, wie sie zum Zeitpunkt der Erstellung war. Neue Nachrichten erscheinen nicht in der geteilten Ansicht, aber Bearbeitungen an bestehenden Nachrichten werden übernommen.

</Accordion>
<Accordion title="Funktionieren geteilte Links offline?">

Nein. Empfänger benötigen eine Internetverbindung, um geteilte Konversationen anzusehen.

</Accordion>
</Accordions>
