临时聊天允许您在不保存记录的情况下进行提问。您可以将其用于敏感话题、快速实验或任何您无需保存的内容。临时聊天不会出现在您的历史记录侧边栏中，不会出现在搜索结果中，无法被收藏，并且会在保留期结束后（默认为 30 天）自动删除。

## 开启临时聊天 [#start-a-temporary-chat]

<Steps>

<Step>

**打开模型菜单。** 选择聊天视图左上角的模型下拉菜单。

</Step>

<Step>

**开启临时聊天 (Temporary Chat)。** 将 **Temporary Chat** 开关切换至开启（ON）位置。

<Frame>
![模型菜单中的临时聊天切换开关](https://github.com/user-attachments/assets/ebd5370d-7fac-45af-b0d3-2de59df506e9)

</Step>

<Step>

**确认其处于活动状态。** 当 Temporary Chat 开启时，消息框上方会出现一个横幅。选择横幅上的关闭 (x) 按钮即可将其关闭并返回到正常对话。

## 临时对话的作用 [#what-a-temporary-chat-does]

- 不会出现在聊天记录侧边栏中。
- 被排除在搜索结果之外。
- 无法收藏。
- 存储在数据库中，直到其保留期结束，然后自动删除。

<Callout type="info" title="保留期限">

默认保留期为 30 天。管理员可以更改此设置。请参阅下方的 [Configuration](#configuration-administrators)。

</Callout>

## 配置 (管理员) [#configuration-administrators]

临时聊天功能默认对用户开放。管理员可以控制是否提供此功能，以及临时聊天记录的保留时长。

**可用性**受 `TEMPORARY_CHAT` 角色权限控制。请在 [Admin Panel](/docs/features/admin_panel) 中为每个角色进行管理。`librechat.yaml` 中的 `interface.temporaryChat` 选项仅在启动时为默认的 `USER` 角色初始化此权限，且该选项在权限管理方面已被弃用。

**Retention**（保留时间）通过 `interface.temporaryChatRetention`（以小时为单位）进行设置。最小值为 1 小时，最大值为 8760（1 年），默认值为 720（30 天）。

<Tabs items={['librechat.yaml (推荐)', '环境变量（已弃用）']}>

<Tabs.Tab>

```yaml filename="librechat.yaml"
interface:
  temporaryChat: true
  temporaryChatRetention: 168 # retain temporary chats for 7 days
```

</Tabs.Tab>

<Tabs.Tab>

```bash filename=".env"
# Hours to retain temporary chats (default: 720 = 30 days)
TEMP_CHAT_RETENTION_HOURS=168
```

<Callout type="warning" title="已弃用">

`TEMP_CHAT_RETENTION_HOURS` 已弃用。建议在 `librechat.yaml` 中使用 `interface.temporaryChatRetention`，该配置项的优先级高于环境变量。

</Callout>

</Tabs.Tab>

</Tabs>

常见的保留值：

| 值 | 周期 |
| --- | --- |
| `1` | 1 小时（最小值） |
| `24` | 1 天 |
| `168` | 1 周 |
| `720` | 30 天（默认值） |
| `8760` | 1 年（最大值） |

有关包括 `retentionMode` 和 `retainAgentFiles` 在内的完整选项集，请参阅 [接口参考](/docs/configuration/librechat_yaml/object_structure/interface#temporarychatretention)。
