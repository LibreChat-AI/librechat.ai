Los chats temporales te permiten preguntar algo sin guardarlo. Úsalos para temas confidenciales, experimentos rápidos o cualquier cosa que no necesites conservar. Un chat temporal se mantiene fuera de tu barra lateral de historial, nunca aparece en las búsquedas, no se puede marcar como favorito y se elimina automáticamente una vez que finaliza su período de retención (30 días por defecto).

## Iniciar un chat temporal [#start-a-temporary-chat]

<Steps>

<Step>

**Abre el menú de modelos.** Selecciona el menú desplegable de modelos en la parte superior izquierda de la vista de chat.

</Step>

<Step>

**Activa el Chat temporal.** Cambia el interruptor de **Temporary Chat** a la posición de encendido (ON).

<Frame>
![Alternar Chat temporal en el menú de modelos](https://github.com/user-attachments/assets/ebd5370d-7fac-45af-b0d3-2de59df506e9)
</Frame>

</Step>

<Step>

**Confirma que está activo.** Aparece un banner sobre el cuadro de mensaje mientras el Chat temporal está activado. Selecciona cerrar (x) en el banner para desactivarlo y volver a una conversación normal.

</Step>

</Steps>

## Qué hace un Chat Temporal [#what-a-temporary-chat-does]

- No aparece en la barra lateral del historial de chat.
- Está excluido de los resultados de búsqueda.
- No se puede marcar como favorito.
- Se almacena en la base de datos hasta que finaliza su período de retención, momento en el que se elimina automáticamente.

<Callout type="info" title="Período de retención">

El período de retención predeterminado es de 30 días. Los administradores pueden cambiarlo. Consulte [Configuration](#configuration-administrators) a continuación.

</Callout>

## Configuración (administradores) [#configuration-administrators]

Temporary Chat está disponible para los usuarios de forma predeterminada. Los administradores controlan si se ofrece la función y cuánto tiempo se conservan los chats temporales.

La **disponibilidad** se rige por el permiso de rol `TEMPORARY_CHAT`. Adminístrelo para cada rol desde el [Admin Panel](/docs/features/admin_panel). La opción `interface.temporaryChat` en `librechat.yaml` solo establece este permiso para el rol `USER` predeterminado al inicio y está obsoleta para la gestión de permisos.

**Retention** se configura con `interface.temporaryChatRetention` (en horas). El mínimo es 1 hora, el máximo es 8760 (1 año) y el valor predeterminado es 720 (30 días).

<Tabs items={['librechat.yaml (recomendado)', 'Variable de entorno (obsoleto)']}>

<Tabs.Tab>

```yaml filename="librechat.yaml"
interface:
  temporaryChat: true
  temporaryChatRetention: 168 # retain temporary chats for 7 days
```

</Tabs.Tab>

<Tabs.Tab>

```bash filename=".env"
# Hours to retain temporary chats (default: 720 = 30 days)
TEMP_CHAT_RETENTION_HOURS=168
```

<Callout type="warning" title="Obsoleto">

`TEMP_CHAT_RETENTION_HOURS` está obsoleta. Se prefiere `interface.temporaryChatRetention` en `librechat.yaml`, la cual tiene prioridad sobre la variable de entorno.

</Callout>

</Tabs.Tab>

</Tabs>

Valores de retención comunes:

| Valor | Periodo |
| --- | --- |
| `1` | 1 hora (mínimo) |
| `24` | 1 día |
| `168` | 1 semana |
| `720` | 30 días (predeterminado) |
| `8760` | 1 año (máximo) |

Para ver el conjunto completo de opciones, incluyendo `retentionMode` y `retainAgentFiles`, consulta la [referencia de la interfaz](/docs/configuration/librechat_yaml/object_structure/interface#temporarychatretention).
