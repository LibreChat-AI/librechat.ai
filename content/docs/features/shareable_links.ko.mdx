공유 가능한 링크를 사용하면 대화의 읽기 전용 스냅샷을 게시하여 다른 사람들이 생성된 URL이나 QR 코드를 통해 열어볼 수 있습니다. 수신자는 메시지, 모든 브랜치, 생성된 아티팩트를 볼 수 있지만, 답장을 보내거나 내용을 변경할 수는 없습니다.

## 주요 기능 [#key-features]

- **간편한 공유**: 몇 번의 클릭만으로 링크를 생성하세요.
- **QR codes**: 코드를 스캔하여 휴대폰에서 대화를 엽니다.
- **브랜칭(Branching)**: 공유 링크는 대화의 모든 브랜치를 유지합니다.
- **Artifacts**: React 컴포넌트, HTML 미리보기 및 Mermaid 다이어그램이 대화형으로 유지됩니다.
- **수신자 기본 설정**: 뷰어는 언어와 테마를 전환할 수 있습니다.
- **링크 관리**: 생성한 모든 링크를 검토하거나 취소할 수 있는 단일 대시보드입니다.

## 공유 링크 옵션 [#share-link-options]

링크를 생성하면 대화가 수신자에게 어떻게 표시될지 제어할 수 있는 모달이 열립니다.

![공유 링크 생성 모달](/images/shared-links/share-modal-create.png)

![관리 옵션이 포함된 공유 링크 모달](/images/shared-links/share-modal-options.png)

링크가 생성되면, 모달에서 다음 작업들을 수행할 수 있습니다:

- **새로 고침 링크**: 이전 URL을 무효화하고 새로운 URL을 생성합니다.
- **QR 코드 생성**: 모바일 액세스를 위한 QR 코드를 생성합니다.
- **클립보드에 링크 복사**: 공유 가능한 URL을 복사합니다.
- **링크 삭제**: 링크를 제거하고 해당 링크를 가진 모든 사용자의 액세스 권한을 취소합니다.

## QR 코드 [#qr-codes]

공유된 대화에 대한 QR 코드를 생성한 다음, 휴대폰 카메라로 스캔하여 링크를 엽니다. 이 코드는 동일한 공유 URL로 연결됩니다.

![공유 링크를 위해 생성된 QR 코드](/images/shared-links/qr-code.png)

![QR 코드 다운로드 옵션](/images/shared-links/qr-code-download.png)

QR 코드는 프레젠테이션, 빠른 모바일 액세스, 인쇄물 배포 및 컨퍼런스 데모에 유용합니다.

## 공유된 대화 보기 [#viewing-shared-conversations]

누군가 귀하의 링크를 열면, 링크가 게시된 날짜, 전체 메시지 기록, 생성된 모든 아티팩트를 포함하여 대화의 읽기 전용 뷰를 집중적으로 볼 수 있습니다.

![수신자가 보는 공유 대화](/images/shared-links/shared-view.png)

## 브랜칭 [#branching]

공유 링크는 대화의 분기를 보존하므로, 수신자는 대화가 진행된 다양한 경로를 따라갈 수 있습니다. 분기 탐색 화살표를 사용하여 경로 간을 이동하세요.

![공유된 대화 내 브랜치 탐색 메뉴](/images/shared-links/branching-menu.png)

![공유 대화에서 브랜치 탐색하기](/images/shared-links/branching-support.png)

이는 서로 다른 프롬프트가 어떻게 서로 다른 AI 응답을 이끌어내는지, 또는 하나의 문제를 여러 방식으로 어떻게 접근할 수 있는지 보여주는 데 유용합니다.

## Artifacts [#artifacts]

대화 중에 생성된 Artifacts는 공유 링크에서도 완전히 정상적으로 작동합니다.

![공유된 대화 내의 대화형 아티팩트](/images/shared-links/artifacts-support.png)

수신자는 다음을 수행할 수 있습니다:

- 대화형 React 컴포넌트를 확인하세요.
- HTML 미리보기를 참조하세요.
- Mermaid 다이어그램을 검토하세요.
- 기본 코드와 해당 코드가 생성된 맥락을 읽어보세요.

## 수신자 기본 설정 [#recipient-preferences]

사용자는 본인의 설정을 변경하지 않고도 인터페이스를 원하는 대로 조정할 수 있습니다.

![공유 링크의 언어 및 테마 설정](/images/shared-links/recipient-settings.png)

- **테마**: 라이트, 다크 또는 시스템.
- **언어**: 선호하는 언어로 인터페이스를 확인합니다.

## 공유 링크 관리하기 [#managing-your-shared-links]

**Settings → Data Controls** 아래의 Shared Links 대시보드를 열어 공유한 모든 대화를 검토하세요.

![공유 링크 관리 대시보드](/images/shared-links/management-modal.png)

![개별 공유 링크에 대한 세부 정보 및 옵션](/images/shared-links/management-details.png)

대시보드에서 다음을 수행할 수 있습니다:

- **모든 링크 보기**: 공유한 모든 대화를 확인하세요.
- **검색 및 필터링**: 특정 공유 링크를 찾고 해당 콘텐츠를 미리 봅니다.
- **액세스 취소**: 링크를 삭제하여 즉시 공유를 중단합니다.

## 구성 [#configuration]

`.env` 파일에 있는 두 개의 환경 변수가 이 기능을 제어합니다:

- **`ALLOW_SHARED_LINKS`** (기본값: `true`): 공유 링크 기능을 활성화합니다. 사용자가 링크를 생성하지 못하게 하려면 `false`로 설정하세요.
- **`ALLOW_SHARED_LINKS_PUBLIC`** (기본값: `false`): 인증 없이 링크를 열 수 있는지 여부를 제어합니다. 기본적으로 수신자는 로그인해야 합니다. 공개적이고 인증되지 않은 액세스를 허용하려면 `true`로 설정하세요.

```bash title=".env"
# Enable shared links (default)
ALLOW_SHARED_LINKS=true

# Require authentication to view shared links (default)
ALLOW_SHARED_LINKS_PUBLIC=false
```

역할 권한은 어떤 사용자가 링크를 생성하고, 인증된 사용자와 공유하거나, 인스턴스의 모든 사용자에게 공개할 수 있는지 결정합니다:

```yaml title="librechat.yaml"
interface:
  sharedLinks:
    create: true
    share: true
    public: false
```

`sharedLinks.public`는 사용자가 "모두와 공유"를 토글할 수 있는지 여부를 제어합니다. `ALLOW_SHARED_LINKS_PUBLIC`은 해당 공개 링크를 로그인 없이 볼 수 있는지 여부를 여전히 결정합니다.

<Callout type="info">
전체 환경 변수 참조는 [.env Configuration](/docs/configuration/dotenv#shared-links) 페이지를 참조하세요.
</Callout>

## 자주 묻는 질문 (FAQ) [#frequently-asked-questions]

<Accordions type="single" collapsible>
<Accordion title="공유된 대화에 수신자가 응답할 수 있나요?">

아니요. 공유 링크는 읽기 전용입니다. 수신자는 대화를 볼 수는 있지만 메시지를 추가하거나 AI와 상호 작용할 수는 없습니다.

</Accordion>
<Accordion title="공유한 대화를 삭제하면 어떻게 되나요?">

링크가 작동을 멈춥니다. 해당 링크로 이동하는 모든 사용자는 "Shared link not found" 페이지를 보게 됩니다.

</Accordion>
<Accordion title="공유한 대화를 편집할 수 있나요?">

이 링크는 생성 당시의 대화 내용을 그대로 담고 있습니다. 공유된 보기에는 새로운 메시지가 나타나지 않지만, 기존 메시지에 대한 수정 사항은 반영됩니다.

</Accordion>
<Accordion title="공유 링크는 오프라인에서도 작동하나요?">

아니요. 공유된 대화를 보려면 수신자에게 인터넷 연결이 필요합니다.

</Accordion>
</Accordions>
