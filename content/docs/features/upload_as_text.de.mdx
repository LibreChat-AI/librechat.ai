# Dateien als Text hochladen [#upload-files-as-text]

Wollten Sie schon immer ein PDF, eine Codedatei oder eine Tabelle an die KI übergeben und einfach sagen: _"lies das"_? Genau das macht **Upload as Text**.

Sie hängen eine Datei an, LibreChat extrahiert den Text daraus und der vollständige Inhalt wird direkt in Ihre Unterhaltung eingefügt. Die KI kann dann jedes Wort davon lesen – ohne Plugins, ohne Vektordatenbanken und ohne zusätzliche Dienste, die konfiguriert werden müssen. Es funktioniert sofort nach der Installation.

<Callout type="info" title="Keine Einrichtung erforderlich">
  „Upload as Text“ funktioniert sofort auf jeder LibreChat-Instanz. Es verwendet eine integrierte Textanalyse – Sie benötigen weder OCR, eine RAG-Pipeline noch einen externen Dienst, um loszulegen.
</Callout>

---

## Verwendung [#how-to-use-it]

<Steps>
  <Step>
    ### Klicken Sie auf das Anhang-Symbol [#click-the-attachment-icon]

    Klicken Sie in der Chat-Eingabeleiste auf das **Büroklammer**-Symbol (📎).
  </Step>
  <Step>
    ### "Als Text hochladen" auswählen [#pick-upload-as-text]

    Wählen Sie aus dem Dropdown-Menü **Upload as Text** aus. Dies weist LibreChat an, den Dateiinhalt zu lesen, anstatt ihn als reinen Anhang zu übermitteln.
  </Step>
  <Step>
    ### Wählen Sie Ihre Datei [#choose-your-file]

    Wählen Sie die Datei von Ihrem Gerät aus. LibreChat extrahiert den Text und bettet ihn direkt in Ihre Nachricht ein.
  </Step>
  <Step>
    ### Stellen Sie Ihre Frage [#ask-your-question]

    Geben Sie Ihren Prompt wie gewohnt ein. Die KI verfügt nun über den vollständigen Text Ihrer Datei im Kontext und kann auf jeden Teil davon verweisen.
  </Step>
</Steps>

<Callout type="warn" title="Siehst du die Option nicht?">
  Falls „Upload as Text“ nicht erscheint, wurde die `context`-Funktion möglicherweise von Ihrem Administrator deaktiviert. Sie ist standardmäßig aktiviert – falls die Liste der Funktionen jedoch angepasst wurde, muss `context` explizit hinzugefügt werden. Siehe den [Konfigurationsabschnitt](#the-context-capability) weiter unten.
</Callout>

## Was unter der Haube passiert [#what-happens-under-the-hood]

Wenn Sie eine Datei auf diese Weise hochladen, speichert LibreChat nicht einfach nur Rohdaten im Prompt. Sie durchläuft eine Verarbeitungspipeline, um sauberen, lesbaren Text zu extrahieren:

1. **MIME-Typ-Erkennung** — LibreChat überprüft, welche Art von Datei Sie hochgeladen haben (PDF, Bild, Audio, Quellcode usw.), indem es deren MIME-Typ untersucht.
2. **Methodenauswahl** — Basierend auf dem Dateityp und den verfügbaren Diensten wählt es die beste Extraktionsmethode unter Verwendung dieser Priorität:

<Tabs items={["Prioritätsreihenfolge", "Entscheidungsbeispiele"]}>
  <Tabs.Tab>
    | Priorität | Methode | Wann sie verwendet wird |
    |----------|--------|---------------|
    | 1. | **OCR** | Datei ist ein Bild oder ein gescanntes Dokument _und_ OCR ist konfiguriert |
    | 2. | **STT** (Speech-to-Text) | Datei ist Audio _und_ STT ist konfiguriert |
    | 3. | **Text-Parsing** | Datei entspricht einem bekannten Text-MIME-Typ |
    | 4. | **Fallback** | Nichts der oben genannten hat zugetroffen — versucht trotzdem Text-Parsing |
  </Tabs.Tab>
  <Tabs.Tab>
    **Eine `.pdf` auf einer Instanz mit konfigurierter OCR:**
    → OCR wird aktiviert. Großartig für gescannte Dokumente und komplexe Layouts.

    **Eine `.pdf`-Datei auf einer Standard-Instanz (kein OCR):**
    → Die Textanalyse übernimmt dies. Funktioniert gut bei digital erstellten PDFs.

    **Eine `.py` Python-Datei:**
    → Direkte Textanalyse. Der Quellcode ist bereits Text – keine Konvertierung erforderlich.

    **Eine `.mp3`-Datei auf einer Instanz mit konfiguriertem STT:**
    → Speech-to-Text transkribiert sie in Text für die Konversation.

    **Ein `.png`-Screenshot ohne konfigurierte OCR:**
    → Greift auf Text-Parsing zurück (eingeschränkte Ergebnisse — erwägen Sie die Einrichtung von OCR für Bilder).
  </Tabs.Tab>
</Tabs>

3. **Token-Kürzung** — Der extrahierte Text wird auf das `fileTokenLimit` (Standard: 100.000 Token) gekürzt, damit das Kontextfenster des Modells nicht überschritten wird.
4. **Prompt Injection** — Der Text wird in den Gesprächskontext aufgenommen, direkt neben Ihrer Nachricht.

## Welche Dateien werden unterstützt [#which-files-are-supported]

<Tabs items={["Text & Code", "Dokumente", "Bilder", "Audio"]}>
  <Tabs.Tab>
    Diese werden direkt geparst — sie sind bereits Text, daher ist keine Konvertierung erforderlich.

    - Klartext (`.txt`), Markdown (`.md`), CSV, JSON, XML, HTML, CSS
    - Programmiersprachen — Python, JavaScript, TypeScript, Java, C#, PHP, Ruby, Go, Rust, Kotlin, Swift, Scala, Perl, Lua
    - Konfigurationsdateien — YAML, TOML, INI
    - Shell-Skripte, SQL-Dateien
  </Tabs.Tab>
  <Tabs.Tab>
    Die Textanalyse verarbeitet diese standardmäßig. Wenn OCR konfiguriert ist, übernimmt diese die Aufgabe für eine höhere Genauigkeit bei komplexen Layouts.

    - **PDF** — digital und gescannt (gescannte PDFs profitieren von OCR)
    - **Word** — `.docx`, `.doc`
    - **PowerPoint** — `.pptx`, `.ppt`
    - **Excel** — `.xlsx`, `.xls`
    - **EPUB**-Bücher
  </Tabs.Tab>
  <Tabs.Tab>
    Bilder **erfordern OCR**, um nützlichen Text zu erzeugen. Ohne diese Funktion werden die Ergebnisse mangelhaft sein.

    - JPEG, PNG, GIF, WebP
    - HEIC, HEIF (Apple-Formate)
    - Screenshots, Fotos von Dokumenten, gescannte Seiten
  </Tabs.Tab>
  <Tabs.Tab>
    Audiodateien **erfordern eine konfigurierte STT**. Es gibt kein Fallback – Audio kann nicht „textgeparst“ werden.

    - MP3, WAV, OGG, FLAC
    - M4A, WebM
    - Sprachaufnahmen, Podcast-Clips
  </Tabs.Tab>
</Tabs>

## Upload als Text vs. andere Upload-Optionen [#upload-as-text-vs-other-upload-options]

LibreChat bietet drei Möglichkeiten, Dateien hochzuladen. Jede funktioniert anders und eignet sich für unterschiedliche Situationen:

<Cards>
  <Card title="Als Text hochladen" href="#how-to-use-it">
    Extrahiert den vollständigen Dateiinhalt und fügt ihn in die Konversation ein. Am besten geeignet für kleinere Dateien, bei denen die KI alles lesen soll — Verträge, Codedateien, Artikel. Funktioniert mit allen Modellen, es sind keine zusätzlichen Dienste erforderlich.
  </Card>
  <Card title="Upload für Dateisuche (RAG)" href="/docs/features/rag_api">
    Indiziert die Datei in einer Vektordatenbank und ruft nur die relevanten Abschnitte ab, wenn Sie eine Frage stellen. Besser geeignet für große Dateien oder Dateisammlungen, bei denen das vollständige Einfügen in den Kontext Token verschwenden würde. Erfordert die RAG API.
  </Card>
  <Card title="Standard-Upload" href="/docs/features/agents">
    Übergibt die Datei direkt an das Modell – wird für Vision-Modelle zur Bildanalyse oder für den Code-Interpreter zum Ausführen von Skripten verwendet. Es findet keine Textextraktion statt.
  </Card>
</Cards>

**Schnelle Entscheidungshilfe:**

| Situation | Beste Option |
|-----------|-------------|
| _"Lies diesen 5-seitigen Vertrag und fasse ihn zusammen"_ | **Als Text hochladen** |
| _"Ich habe 50 PDFs, finde die Stellen, an denen Preise erwähnt werden"_ | **Dateisuche (RAG)** |
| _"Was ist auf diesem Screenshot zu sehen?"_ (Vision-Modell) | **Standard-Upload** |
| _"Führe dieses Python-Skript aus"_ (Code-Interpreter) | **Standard-Upload** |
| _"Überprüfe diese Codedatei auf Fehler"_ | **Als Text hochladen** |
| _"Durchsuche unsere Firmendokumente"_ | **Dateisuche (RAG)** |

## Die `context` Fähigkeit [#the-context-capability]

Unter der Haube wird „Upload as Text“ durch die **`context` capability** betrieben. Dies steuert, ob die Funktion in Ihrer Chat-Benutzeroberfläche erscheint.

<Callout type="info">
  Die `context`-Funktion ist **standardmäßig aktiviert**. Sie müssen dies nur anpassen, wenn Ihr Administrator die Liste der Funktionen angepasst und sie versehentlich entfernt hat.
</Callout>

```yaml title="librechat.yaml"
endpoints:
  agents:
    capabilities:
      - "context"  # This is what enables "Upload as Text"
```

Die gleiche `context`-Funktion unterstützt auch den **Agent File Context** (das Hochladen von Dateien über den Agent Builder, um Text in die Systemanweisungen eines Agenten einzubetten). Der Unterschied liegt darin, _wo_ der Text landet:

| | Als Text hochladen | Agent-Dateikontext |
|---|---|---|
| **Wo** | Chat-Eingabe (beliebige Konversation) | Agent Builder-Panel |
| **Geltungsbereich** | Nur aktuelle Konversation | Bleibt in den Anweisungen des Agenten bestehen |
| **Anwendungsfall** | Einmalige Dokumentenfragen | Erstellung spezialisierter Agenten mit integriertem Wissen |

## Token-Limits und Kürzung [#token-limits-and-truncation]

Wenn eine Datei zu lang ist, um in das Kontextfenster des Modells zu passen, kürzt LibreChat den extrahierten Text, um innerhalb der Grenzen zu bleiben. Dies geschieht automatisch – Sie müssen sich keine Gedanken darüber machen, aber es ist gut zu wissen, wie es funktioniert.

```yaml title="librechat.yaml"
fileConfig:
  fileTokenLimit: 100000  # Default: 100,000 tokens
```

<Callout type="warn" title="Kürzung bedeutet verlorener Inhalt">
  Wenn Ihre Datei das Limit überschreitet, wird der Text am Ende abgeschnitten. Wenn Sie unvollständige Antworten erhalten, könnte dies der Grund sein. Sie können `fileTokenLimit` erhöhen, aber bedenken Sie, dass größere Werte mehr Token pro Nachricht verbrauchen – was die Kosten erhöht und dazu führen kann, dass das eigene Kontextlimit des Modells erreicht wird.
</Callout>

**Faustregeln:**
- 100k Token ≈ ein 300-seitiges Buch (mehr als genug für die meisten Anwendungsfälle)
- Wenn Sie mit sehr großen Dateien arbeiten, ziehen Sie stattdessen [File Search (RAG)](/docs/features/rag_api) in Betracht – dies ruft nur die relevanten Abschnitte ab, anstatt alles in den Kontext zu stopfen.

## Optional: Boosting der Extraktion mit OCR [#optional-boosting-extraction-with-ocr]

Die Textanalyse funktioniert bei digital erstellten Dokumenten (PDFs, die aus Word gespeichert wurden, Codedateien, Klartext) einwandfrei. Wenn Sie jedoch **gescannte Dokumente, Fotos von Seiten oder Bilder mit Text** hochladen, wird der integrierte Parser keine optimalen Ergebnisse liefern.

Hier kommt OCR ins Spiel. Wenn es konfiguriert ist, verwendet LibreChat automatisch OCR für Dateitypen, die davon profitieren – als Benutzer müssen Sie nichts anders machen.

<Accordions>
  <Accordion title="Wie du deinen Administrator bittest, OCR einzurichten">
    Verweisen Sie sie auf die [OCR configuration docs](/docs/features/ocr). Die Kurzfassung:

    ```yaml title="librechat.yaml"
    ocr:
      strategy: "mistral_ocr"
      apiKey: "${OCR_API_KEY}"
      baseURL: "https://api.mistral.ai/v1"
      mistralModel: "mistral-ocr-latest"
    ```

    Sobald OCR konfiguriert ist, werden Bilder und gescannte PDFs automatisch verarbeitet – auf Benutzerseite sind keine Änderungen erforderlich.
  </Accordion>
  <Accordion title="So forderst du deinen Administrator auf, STT (für Audiodateien) einzurichten">
    Für die Transkription von hochgeladenen Audiodateien muss der Administrator einen Speech-to-Text-Dienst konfigurieren. Siehe die [STT-Konfigurationsreferenz](/docs/configuration/librechat_yaml/object_structure/file_config#stt).
  </Accordion>
</Accordions>

## Referenz zur Konfiguration der Dateiverarbeitung [#file-handling-configuration-reference]

Dieser Abschnitt ist für Administratoren gedacht, die steuern möchten, welche Dateitypen mit welcher Methode verarbeitet werden. Die Standardeinstellungen funktionieren gut – Sie müssen dies nur anpassen, wenn Sie das Verhalten feinabstimmen möchten.

<Accordions>
  <Accordion title="Vollständiges fileConfig-Beispiel">
    ```yaml title="librechat.yaml"
    fileConfig:
      # Max tokens extracted from a single file before truncation
      fileTokenLimit: 100000

      # Files matching these MIME patterns use OCR (if configured)
      ocr:
        supportedMimeTypes:
          - "^image/(jpeg|gif|png|webp|heic|heif)$"
          - "^application/pdf$"
          - "^application/vnd\\.openxmlformats-officedocument\\.(wordprocessingml\\.document|presentationml\\.presentation|spreadsheetml\\.sheet)$"
          - "^application/vnd\\.ms-(word|powerpoint|excel)$"
          - "^application/epub\\+zip$"

      # Files matching these MIME patterns use text parsing
      text:
        supportedMimeTypes:
          - "^text/(plain|markdown|csv|json|xml|html|css|javascript|typescript|x-python|x-java|x-csharp|x-php|x-ruby|x-go|x-rust|x-kotlin|x-swift|x-scala|x-perl|x-lua|x-shell|x-sql|x-yaml|x-toml)$"

      # Files matching these MIME patterns use STT (if configured)
      stt:
        supportedMimeTypes:
          - "^audio/(mp3|mpeg|mpeg3|wav|wave|x-wav|ogg|vorbis|mp4|x-m4a|flac|x-flac|webm)$"
    ```

    **Prioritätshinweis:** OCR > STT > Text-Parsing > Fallback.

    Für die vollständige Referenz siehe [File Config Object Structure](/docs/configuration/librechat_yaml/object_structure/file_config).
  </Accordion>
</Accordions>

## Fehlerbehebung [#troubleshooting]

<Accordions>
  <Accordion title="Option „Als Text hochladen“ wird nicht angezeigt">
    Die `context`-Funktion wurde wahrscheinlich aus Ihrer Konfiguration entfernt. Bitten Sie Ihren Administrator, sie wieder hinzuzufügen:

    ```yaml
    endpoints:
      agents:
        capabilities:
          - "context"
    ```
  </Accordion>
  <Accordion title="Der Dateiinhalt sieht falsch aus oder ist größtenteils leer">
    Ein paar Dinge, die Sie überprüfen sollten:
    - **Gescannte PDFs / Bilder** — Diese benötigen OCR, um Text korrekt zu extrahieren. Ohne diese Funktion könnte der Parser fehlerhafte oder gar keine Ergebnisse liefern. Bitten Sie Ihren Administrator, [OCR zu konfigurieren](/docs/features/ocr).
    - **Audiodateien** — Diese benötigen STT. Es gibt keinen Text-Fallback für Audio.
    - **Beschädigte Dateien** — Versuchen Sie, die Datei lokal zu öffnen, um sicherzustellen, dass sie nicht beschädigt ist.
    - **Nicht unterstütztes Format** — Wenn der MIME-Typ mit keinem konfigurierten Muster übereinstimmt, versucht LibreChat ein Text-Parsing als Fallback, was bei Binärformaten möglicherweise nicht funktioniert.
  </Accordion>
  <Accordion title="Die KI scheint einen Teil meiner Datei zu übersehen">
    Deine Datei hat wahrscheinlich das `fileTokenLimit` überschritten. Der Text wurde gekürzt.

    **Optionen:**
    - Bitte deinen Administrator, das `fileTokenLimit` in der `librechat.yaml` zu erhöhen.
    - Verwenden Sie stattdessen [File Search (RAG)](/docs/features/rag_api), das relevante Abschnitte abruft, anstatt die gesamte Datei zu laden.
    - Teilen Sie die Datei in kleinere Teile auf und laden Sie diese separat hoch
  </Accordion>
  <Accordion title="Bilder werden hochgeladen, aber die KI kann den Text darin nicht lesen">
    Ohne OCR werden Bilder durch Text-Parsing verarbeitet, das den Text in einem Bild nicht tatsächlich „sehen“ kann. Sie müssen OCR konfigurieren, damit dies funktioniert. Siehe [OCR für Dokumente](/docs/features/ocr).

    Alternativ können Sie ein **Vision-Modell** mit Standard-Datei-Upload verwenden – das Modell selbst kann Text in Bildern lesen.
  </Accordion>
</Accordions>
