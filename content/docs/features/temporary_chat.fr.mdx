Les discussions temporaires vous permettent de poser des questions sans les conserver. Utilisez-les pour des sujets sensibles, des expériences rapides ou tout ce que vous n'avez pas besoin de sauvegarder. Une discussion temporaire n'apparaît pas dans votre barre latérale d'historique, n'est jamais incluse dans les recherches, ne peut pas être mise en favori et est supprimée automatiquement une fois sa période de rétention terminée (30 jours par défaut).

## Démarrer une discussion temporaire [#start-a-temporary-chat]

<Steps>

<Step>

**Ouvrez le menu des modèles.** Sélectionnez le menu déroulant des modèles en haut à gauche de la vue de chat.

</Step>

<Step>

**Activez le Chat temporaire.** Basculez l'interrupteur **Temporary Chat** sur la position ON.

<Frame>
![Bascule de chat temporaire dans le menu des modèles](https://github.com/user-attachments/assets/ebd5370d-7fac-45af-b0d3-2de59df506e9)
</Frame>

</Step>

<Step>

**Confirmez qu'il est actif.** Une bannière apparaît au-dessus de la zone de message lorsque le Temporary Chat est activé. Sélectionnez la croix (x) sur la bannière pour le désactiver et revenir à une conversation normale.

</Step>

</Steps>

## Ce que fait un chat temporaire [#what-a-temporary-chat-does]

- N'apparaît pas dans la barre latérale de l'historique des discussions.
- Est exclu des résultats de recherche.
- Ne peut pas être mis en favori.
- Est stocké dans la base de données jusqu'à la fin de sa période de rétention, puis supprimé automatiquement.

<Callout type="info" title="Période de rétention">

La période de rétention par défaut est de 30 jours. Les administrateurs peuvent la modifier. Voir [Configuration](#configuration-administrators) ci-dessous.

</Callout>

## Configuration (administrateurs) [#configuration-administrators]

Le chat temporaire (Temporary Chat) est disponible par défaut pour les utilisateurs. Les administrateurs contrôlent si cette fonctionnalité est proposée et combien de temps les chats temporaires sont conservés.

La **disponibilité** est régie par l'autorisation de rôle `TEMPORARY_CHAT`. Gérez-la pour chaque rôle depuis le [Admin Panel](/docs/features/admin_panel). L'option `interface.temporaryChat` dans `librechat.yaml` ne fait qu'initialiser cette autorisation pour le rôle `USER` par défaut au démarrage et est obsolète pour la gestion des autorisations.

La **rétention** est définie avec `interface.temporaryChatRetention` (en heures). Le minimum est de 1 heure, le maximum est de 8760 (1 an) et la valeur par défaut est de 720 (30 jours).

<Tabs items={['librechat.yaml (recommandé)', 'Variable d\'environnement (obsolète)']}>

<Tabs.Tab>

```yaml filename="librechat.yaml"
interface:
  temporaryChat: true
  temporaryChatRetention: 168 # retain temporary chats for 7 days
```

</Tabs.Tab>

<Tabs.Tab>

```bash filename=".env"
# Hours to retain temporary chats (default: 720 = 30 days)
TEMP_CHAT_RETENTION_HOURS=168
```

<Callout type="warning" title="Obsolète">

`TEMP_CHAT_RETENTION_HOURS` est obsolète. Préférez `interface.temporaryChatRetention` dans `librechat.yaml`, qui est prioritaire sur la variable d'environnement.

</Callout>

</Tabs.Tab>

</Tabs>

Valeurs de rétention courantes :

| Valeur | Période |
| --- | --- |
| `1` | 1 heure (minimum) |
| `24` | 1 jour |
| `168` | 1 semaine |
| `720` | 30 jours (par défaut) |
| `8760` | 1 an (maximum) |

Pour l'ensemble complet des options, y compris `retentionMode` et `retainAgentFiles`, consultez la [référence de l'interface](/docs/configuration/librechat_yaml/object_structure/interface#temporarychatretention).
