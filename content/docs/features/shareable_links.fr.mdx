Les liens partageables vous permettent de publier un instantané en lecture seule d'une conversation que d'autres personnes peuvent ouvrir via une URL générée ou un code QR. Les destinataires voient les messages, les éventuelles branches et tous les artefacts que vous avez générés, sans pouvoir répondre ni rien modifier.

## Fonctionnalités clés [#key-features]

- **Partage facile** : Générez un lien en quelques clics.
- **Codes QR** : Ouvrez la conversation sur un téléphone en scannant un code.
- **Branching** : Les liens partagés conservent chaque branche de la conversation.
- **Artifacts** : les composants React, les aperçus HTML et les diagrammes Mermaid restent interactifs.
- **Instantanés de fichiers** : Les fichiers de conversation référencés peuvent être prévisualisés ou téléchargés à partir du lien partagé sans accorder aux destinataires un accès à la liste de contrôle d'accès (ACL) du fichier original du propriétaire.
- **Préférences du destinataire** : Les spectateurs peuvent changer la langue et le thème.
- **Gestion des liens** : Un tableau de bord unique pour examiner ou révoquer chaque lien que vous avez créé.

## Partager une conversation [#share-a-conversation]

Vous pouvez commencer le partage à partir de deux endroits.

<Steps>
<Step>

**Depuis le menu de conversation.** Ouvrez le menu à côté d'une conversation dans la barre latérale et choisissez l'option de partage.

![Option de partage dans le menu de conversation](/images/shared-links/share-from-menu.png)

</Step>
<Step>

**Depuis le bouton de partage.** Dans une conversation active, utilisez le bouton de partage dédié situé dans l'en-tête.

![Bouton de partage dans une conversation active](/images/shared-links/share-button.png)

</Step>
</Steps>

## Options de lien de partage [#share-link-options]

La création d'un lien ouvre une fenêtre modale où vous contrôlez la façon dont la conversation apparaît aux destinataires.

![Fenêtre modale de création de lien de partage](/images/shared-links/share-modal-create.png)

![Fenêtre modale de lien de partage avec options de gestion](/images/shared-links/share-modal-options.png)

Une fois qu'un lien existe, la modale propose ces actions :

- **Lien de rafraîchissement** : Génère une nouvelle URL, ce qui invalide la précédente.
- **Générer un code QR** : Créez un code QR pour un accès mobile.
- **Copier le lien dans le presse-papiers** : Copier l'URL partageable.
- **Supprimer le lien** : Supprimez le lien et révoquez l'accès pour toute personne le possédant.

Lorsque les instantanés de fichiers sont activés, les utilisateurs peuvent choisir si le lien inclut les fichiers référencés dans la conversation partagée. Les fichiers instantanés sont épinglés à la version partagée et servis via des routes spécifiques au partage ; si le fichier original est supprimé ou modifié après le partage, les destinataires ne reçoivent plus ce fichier.

## Codes QR [#qr-codes]

Générez un code QR pour toute conversation partagée, puis scannez-le avec l'appareil photo d'un téléphone pour ouvrir le lien. Le code pointe vers la même URL partagée.

![QR code généré pour un lien partagé](/images/shared-links/qr-code.png)

![Options de téléchargement du code QR](/images/shared-links/qr-code-download.png)

Les codes QR sont pratiques pour les présentations, l'accès mobile rapide, les documents imprimés et les démonstrations lors de conférences.

## Visualisation des conversations partagées [#viewing-shared-conversations]

Lorsqu'une personne ouvre votre lien, elle accède à une vue ciblée et en lecture seule de la conversation, incluant la date de publication du lien, l'historique complet des messages et tous les artifacts que vous avez générés.

![Une conversation partagée telle qu'un destinataire la voit](/images/shared-links/shared-view.png)

## Branchement [#branching]

Les liens partagés préservent les branches de conversation, afin que les destinataires puissent suivre les différents chemins empruntés par la discussion. Utilisez les flèches de navigation des branches pour passer de l'une à l'autre.

![Menu de navigation des branches dans une conversation partagée](/images/shared-links/branching-menu.png)

![Navigation entre les branches dans une conversation partagée](/images/shared-links/branching-support.png)

Ceci est utile pour montrer comment différents prompts mènent à des réponses IA différentes, ou comment un seul problème peut être abordé de plusieurs manières.

## Artifacts [#artifacts]

Les Artifacts générés pendant la conversation restent entièrement fonctionnels dans les liens partagés.

![Artefacts interactifs dans une conversation partagée](/images/shared-links/artifacts-support.png)

Les destinataires peuvent :

- Voir les composants React interactifs.
- Voir les aperçus HTML.
- Examiner les diagrammes Mermaid.
- Lisez le code sous-jacent et le contexte dans lequel il a été généré.

## Préférences du destinataire [#recipient-preferences]

Les spectateurs peuvent ajuster l'interface selon leurs propres préférences sans affecter vos paramètres.

![Paramètres de langue et de thème sur un lien partagé](/images/shared-links/recipient-settings.png)

- **Thème** : Clair, sombre ou système.
- **Langue** : Affichez l'interface dans leur langue préférée.

## Gestion de vos liens partagés [#managing-your-shared-links]

Ouvrez le tableau de bord des Liens partagés sous **Paramètres → Contrôles des données** pour examiner toutes les conversations que vous avez partagées.

![Tableau de bord de gestion des liens partagés](/images/shared-links/management-modal.png)

![Détails et options pour un lien partagé individuel](/images/shared-links/management-details.png)

Depuis le tableau de bord, vous pouvez :

- **Voir tous les liens** : Consultez toutes les conversations que vous avez partagées.
- **Recherche et filtrage** : Trouvez un lien partagé spécifique et prévisualisez son contenu.
- **Révoquer l'accès** : Supprimez un lien pour arrêter immédiatement le partage.

## Configuration [#configuration]

Trois variables d'environnement dans votre fichier `.env` contrôlent cette fonctionnalité :

- **`ALLOW_SHARED_LINKS`** (par défaut : `true`) : Active la fonctionnalité de liens partagés. Définissez sur `false` pour empêcher les utilisateurs de créer des liens.
- **`ALLOW_SHARED_LINKS_PUBLIC`** (par défaut : `false`) : Contrôle si les liens s'ouvrent sans authentification. Par défaut, les destinataires doivent être connectés. Définissez sur `true` pour autoriser un accès public et non authentifié.
- **`SHARED_LINKS_SNAPSHOT_FILES`** (par défaut : `true`) : Contrôle si les liens partagés peuvent inclure les fichiers de conversation référencés. Cette variable d'environnement remplace [`interface.sharedLinks.snapshotFiles`](/docs/configuration/librechat_yaml/object_structure/interface#sharedlinks) lorsqu'elle est définie.

```bash title=".env"
# Enable shared links (default)
ALLOW_SHARED_LINKS=true

# Require authentication to view shared links (default)
ALLOW_SHARED_LINKS_PUBLIC=false

# Include referenced files in shared-link snapshots (default)
SHARED_LINKS_SNAPSHOT_FILES=true
```

Les permissions de rôle déterminent quels utilisateurs peuvent créer des liens, les partager avec des utilisateurs authentifiés ou les rendre visibles à tous sur l'instance :

```yaml title="librechat.yaml"
interface:
  sharedLinks:
    create: true
    share: true
    public: false
    snapshotFiles: true
```

`sharedLinks.public` contrôle si un utilisateur peut activer ou désactiver le « partage avec tout le monde ». `ALLOW_SHARED_LINKS_PUBLIC` détermine toujours si ces liens publics sont consultables sans connexion. `sharedLinks.snapshotFiles` contrôle le comportement par défaut des instantanés de fichiers depuis le YAML, et `SHARED_LINKS_SNAPSHOT_FILES` peut le remplacer ou le désactiver globalement.

<Callout type="info">
Pour la référence complète des variables d'environnement, consultez la page [.env Configuration](/docs/configuration/dotenv#shared-links).
</Callout>

## Foire aux questions [#frequently-asked-questions]

<Accordions type="single" collapsible>
<Accordion title="Les destinataires peuvent-ils répondre aux conversations partagées ?">

Non. Les liens partagés sont en lecture seule. Les destinataires peuvent consulter la conversation mais ne peuvent pas ajouter de messages ni interagir avec l'IA.

</Accordion>
<Accordion title="Que se passe-t-il si je supprime une conversation après l'avoir partagée ?">

Le lien cesse de fonctionner. Quiconque y accède voit une page « Shared link not found ».

</Accordion>
<Accordion title="Puis-je modifier une conversation après l'avoir partagée ?">

Le lien capture la conversation telle qu'elle était au moment de sa création. Les nouveaux messages n'apparaîtront pas dans la vue partagée, mais les modifications apportées aux messages existants y seront reflétées.

</Accordion>
<Accordion title="Les liens partagés fonctionnent-ils hors ligne ?">

Non. Les destinataires ont besoin d'une connexion internet pour consulter les conversations partagées.

</Accordion>
</Accordions>
