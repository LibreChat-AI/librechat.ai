LibreChat verfügt über ein Benutzerauthentifizierungssystem, das es Benutzern ermöglicht, sich sicher und einfach zu registrieren und anzumelden. Das System ist skalierbar und kann eine große Anzahl gleichzeitiger Benutzer verarbeiten, ohne die Leistung oder Sicherheit zu beeinträchtigen.

Standardmäßig sind die E-Mail-Registrierung und -Anmeldung aktiviert, was bedeutet, dass Benutzer ein Konto mit ihrer E-Mail-Adresse und einem Passwort erstellen können. Sie können ihr Passwort auch zurücksetzen, falls sie es vergessen haben.

Zusätzlich kann unser System Social Logins von verschiedenen Plattformen wie Google, GitHub, Discord, OpenID und weiteren integrieren. Dies bedeutet, dass Benutzer sich mit ihren bestehenden Konten auf diesen Plattformen anmelden können, ohne ein neues Konto erstellen oder sich ein weiteres Passwort merken zu müssen.

**Weitere Details finden Sie in den hier bereitgestellten Konfigurationsanleitungen: [Authentifizierung](/docs/configuration/authentication)**

<Callout type="warning" title="Wichtig">
- Wenn Sie die App zum ersten Mal ausführen, müssen Sie ein neues Konto erstellen, indem Sie auf der Anmeldeseite auf „Sign up“ klicken. Das erste Konto, das Sie erstellen, ist das Administratorkonto und verfügt über alle administrativen Berechtigungen für die Instanz. Siehe [Access Control](/docs/features/access_control) für das vollständige Autorisierungsmodell, das Benutzer, Gruppen, Rollen, Ressourcen-ACLs und Administratorrechte auf Systemebene abdeckt.
- Das erste erstellte Konto sollte idealerweise ein lokales Konto (E-Mail und Passwort) sein.
</Callout>

**Siehe auch:** [Access Control](/docs/features/access_control), das granulare Berechtigungssystem von LibreChat für Benutzer, Gruppen und Rollen, das die ressourcenbezogene Freigabe von Agents, Prompts, MCP Servern und Berechtigungen auf Funktionsebene abdeckt.
