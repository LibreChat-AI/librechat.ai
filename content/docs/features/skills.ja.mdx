Skillsは、LibreChatのAgent向けの再利用可能な命令バンドルです。Skillは`SKILL.md`ファイルを中心に構成されます。フロントマターにはそのSkillをいつ使用すべきかが記述され、Markdown本文にはAgentが従うべき手順、ルール、例、または参照情報が記載されます。

Skillsは、以下のような反復的な作業に役立ちます：

- ブランドやライティングガイドラインの適用
- 内部レビューチェックリストに従う
- 標準的なリサーチワークフローの実行
- 専門的なツールワークフローの準備
- 再利用可能なスクリプト、リファレンス、およびアセットを指示ファイルと共にパッケージ化する

## スキルを有効にする [#enable-skills]

`skills` エージェント機能はデフォルトで有効になっています。管理者は、ユーザーからSkillsを非表示にするために、agents エンドポイントの機能リストからそれを削除することができます。

```yaml filename="librechat.yaml"
endpoints:
  agents:
    capabilities:
      - 'deferred_tools'
      - 'execute_code'
      - 'file_search'
      - 'web_search'
      - 'artifacts'
      - 'subagents'
      - 'actions'
      - 'context'
      - 'skills'
      - 'tools'
      - 'chain'
      - 'ocr'
```

ロール権限は、スキルを使用、作成、共有、および公開共有できるユーザーを制御します。

## デプロイのスキル [#deployment-skills]

管理者は `DEPLOYMENT_SKILLS_DIR` を使用して、ファイルシステムから読み取り専用の Skills を配布できます。

```bash filename=".env"
DEPLOYMENT_SKILLS_DIR=./skill
```

ディレクトリのデフォルトはプロジェクトルートの `./skill` です。LibreChatは起動時にデプロイされたSkillsを読み込み、Skills機能が有効になっているユーザーに対してそれらを公開します。

デプロイスキル:

- UI上で読み取り専用です
- `deployment` をソースとして使用します
- 同じ名前の永続化されたSkillsよりも優先されます
- ファイルが追加、削除、または変更された後は、LibreChatの再起動が必要です。

## GitHub Skill Sync [#github-skill-sync]

管理者は `librechat.yaml` の `skillSync.github` を使用して、GitHub リポジトリから Skills をミラーリングできます。

```yaml filename="librechat.yaml"
skillSync:
  github:
    enabled: true
    intervalMinutes: 60
    runOnStartup: true
    sources:
      - id: librechat-skills
        owner: your-org
        repo: your-skills-repo
        ref: main
        paths:
          - skills
        skillDiscoveryDepth: 2
        token: '${GITHUB_SKILLS_TOKEN}'
```

GitHub Skill Sync:

- 設定されたリポジトリパスをスキャンして `SKILL.md` を探します
- 各スキルの横にバンドルされたファイルをインポートします
- `source: "github"` を持つミラーリングされたスキルを保存します
- アップストリームのリポジトリが変更された際に、ミラーリングされたSkillsを更新します
- 設定されたソース内に存在しなくなったミラーリングされたSkillsを削除します
- スケジュール実行、起動時実行、および管理者による手動実行をサポートしています

選択したリポジトリに対して、読み取り専用のContentsおよびMetadata権限を持つGitHubのファイングレイン個人アクセストークンを使用してください。すべてのフィールド、認証情報のオプション、テナントのスコープ設定、および管理同期エンドポイントについては、[Skill Sync Object Structure](/docs/configuration/librechat_yaml/object_structure/skill_sync) を参照してください。

## スキルを作成する [#create-a-skill]

サイドパネルから **Skills** を開きます。LibreChat 内で直接スキルを作成するか、`SKILL.md` を含む `.md`、`.zip`、または `.skill` ファイルをアップロードできます。

最小限の `SKILL.md`:

```md filename="SKILL.md"
---
name: brand-guidelines
description: Use when writing public-facing content that must follow the company voice and terminology.
---

# Brand Guidelines

Write in a concise, practical tone.
Prefer active voice.
Use product terminology consistently.
```

## 呼び出しモード (Invocation Modes) [#invocation-modes]

Skillsは、以下の3つの方法でエージェントに到達できます：

- **Manual**: ユーザーがチャットで `$` を入力し、ポップオーバーからスキルを選択します。
- **Model-invoked**: モデルが注入されたスキルカタログからスキルを選択し、スキルツールを呼び出します。
- **Always apply**: 有効な場合、そのスキルはすべてのターンでプライミング（準備）されます。

手動呼び出しは、明示的なユーザーの意図によるものです。`user-invocable` が false でない限り、`disable-model-invocation: true` でモデル呼び出しから隠されているスキルを使用することができます。

## Agent Scope [#agent-scope]

エージェントがカタログを使用するには、事前にSkillsを有効にする必要があります。

- 永続化されたエージェントで `skills_enabled` が false に設定されているか、設定されていない場合、そのエージェントのスキルは無効になります。
- `skills_enabled` が true に設定されており、スキル許可リストが設定されていない場合、エージェントはユーザーに表示されるアクティブなカタログ全体を使用できます。
- `skills_enabled` が true であり、`skills` に特定のスキルIDが含まれている場合、エージェントはそのリストに絞り込まれます。

これにより、管理者はSkills機能をグローバルに公開しつつ、各エージェントが使用可能なスキルセットを絞り込むことができます。

## アクティブおよび共有スキル [#active-and-shared-skills]

ユーザーはスキルを有効または無効に切り替えることができます。所有しているスキルはデフォルトで有効になります。共有スキルは、ユーザーが上書きするまで、管理者が設定したデフォルト設定が使用されます。

非アクティブなスキルは以下から除外されます：

- `$` ポップオーバー
- モデルが呼び出すスキルカタログ
- 常に適用されるプライミング

## Bundled Files [#bundled-files]

アップロードされたスキルバンドルには、`SKILL.md` に加えて以下のようなファイルを含めることができます。

- `references/...`
- `scripts/...`
- `assets/...`

バックエンドはそれらのファイルをスキルと共に保存します。エージェントは、スキルがアクティブでスコープ内にある場合にスキルファイルを解決できます。

<Callout type="note" title="作成のヒント">
  `description` は具体的に記述してください。これはモデルがスキルを呼び出すための最も強力なシグナルとなります。説明が短すぎたり曖昧だったりすると、トリガーがうまく機能しません。
</Callout>
