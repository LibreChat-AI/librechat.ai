## 简介 [#introduction]

LibreChat 的 Code Interpreter API 提供了一种安全且便捷的方式，通过简单的 API 接口来执行代码并管理文件。无论您是通过 LibreChat 的 Agents 使用它，还是将其直接集成到您的应用程序中，该 API 都为您提供了一个强大的沙盒环境，用于运行多种编程语言的代码。

## 使用 API [#using-the-api]

### 在 LibreChat 中 [#in-librechat]

LibreChat 通过两种主要方式为 API 提供了一流的支持：

1. **[AI Agents](/docs/features/agents#code-interpreter)**：在您的 agent 配置中启用 Code Interpreter，以允许其自动执行代码并处理文件。

2. **手动执行**：使用聊天界面中代码块内的“Run Code”按钮，如下所示：

![LibreChat 中的代码解释器](/images/agents/run_code.png)

### 设置 API key [#set-up-api-key]

- **用户个人设置：** 在 LibreChat 提示时输入您的 API 密钥（使用上述方法）
- **全局设置：** 在项目的 .env 文件中使用 `LIBRECHAT_CODE_API_KEY` 环境变量（为所有用户提供访问权限）

### 直接 API 集成 [#direct-api-integration]

Code Interpreter API 可以通过简单的 API 密钥认证集成到任何应用程序中：

1. 从 [code.librechat.ai](https://code.librechat.ai/pricing) 获取您的 API key
2. 在请求中使用 `x-api-key` 请求头包含 API 密钥

### 企业版 [#enterprise]

企业版计划需要使用 `LIBRECHAT_CODE_BASEURL` 环境变量来对应自托管的 API 实例，并配合从仪表板生成的任何 API 密钥使用，其使用方式与上述说明相同。

## 核心功能 [#core-functionality]

### 代码执行 [#code-execution]

- 在支持的语言中运行代码片段
- 接收 stdout/stderr 输出
- 获取执行统计信息（内存使用量、CPU 时间）
- 处理程序参数
- 访问执行状态和结果

### 文件操作 [#file-operations]

- 上传输入文件
- 下载生成的输出
- 列出可用文件
- 删除不必要的文件
- 管理文件会话

### 局限性 [#limitations]
- 代码无法访问网络
- 每次运行最多只能生成 10 个文件
- 执行限制因方案而异：
  - **Hobby**:
    - 每个执行实例 256 MB RAM
    - 每个文件上传限制为 25 MB
    - 每月 750 次请求
  - **爱好者**:
    - 每个执行实例 512 MB RAM
    - 每个文件上传限制 50 MB
    - 每月 3,000 次请求
  - **专业版**:
    - 每个执行实例 512 MB RAM
    - 每个文件上传限制为 150 MB
    - 每月 7,000 次请求
- [Enterprise Plan](https://code.librechat.ai/pricing) 提供自定义限制和功能

## 为什么要使用付费 API？ [#why-a-paid-api]

尽管 LibreChat 在 MIT 许可下保持免费和开源，但 Code Interpreter API 作为付费服务提供，主要出于以下几个关键原因：

1. **项目可持续性**：订阅 API 计划能为项目的开发提供直接支持，其效果甚至比 [GitHub Sponsors](https://github.com/sponsors/danny-avila) 更为显著。您的订阅有助于确保 LibreChat 的持续增长与改进。

2. **技术考量**：在核心项目中包含代码执行功能会增加显著的复杂性和硬件需求，并非所有用户都需要这些功能。API 服务消除了这些顾虑，同时保持了核心应用程序的轻量化。

3. **托管服务优势**：
   - 无需复杂的配置
   - 立即可用
   - 定期更新与维护
   - 专业支持
   - 安全、沙盒化的环境

4. **知识产权保护**：Code Interpreter 的架构代表了在安全、可扩展沙箱技术方面的重大创新。虽然存在类似的解决方案，但它们往往缺乏使此实现独一无二的全面安全措施和可扩展性功能。将此组件作为闭源 API 保留，有助于保护这些创新，并确保该服务维持其高安全性和高性能标准。

即使您只是偶尔使用代码执行功能，订阅也能帮助支持 LibreChat 的开发，同时通过专业级功能提升您的体验。这是一种双赢的模式，既能保持核心项目免费，又能为有需要的用户提供可选的高级功能。
