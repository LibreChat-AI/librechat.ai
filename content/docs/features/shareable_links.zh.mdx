共享链接允许您发布对话的只读快照，其他人可以通过生成的 URL 或二维码打开该快照。接收者可以查看消息、任何分支以及您生成的任何工件，但无法回复或进行任何更改。

## 核心功能 [#key-features]

- **轻松共享**：只需点击几下即可生成链接。
- **QR codes**: 通过扫描二维码在手机上打开对话。
- **分支 (Branching)**：共享链接会保留对话的每一个分支。
- **Artifacts**: React 组件、HTML 预览和 Mermaid 图表保持交互性。
- **接收者偏好设置**：查看者可以切换语言和主题。
- **链接管理**：一个用于查看或撤销您所创建的每一个链接的仪表板。

## 分享对话 [#share-a-conversation]

您可以从两个地方开始共享。

<Steps>
<Step>

**从对话菜单中。** 打开侧边栏中对话旁边的菜单，然后选择分享选项。

![对话菜单中的共享选项](/images/shared-links/share-from-menu.png)

</Step>
<Step>

**通过分享按钮。** 在进行中的对话内，使用页眉中的专用分享按钮。

![活动对话中的分享按钮](/images/shared-links/share-button.png)

</Step>
</Steps>

## 分享链接选项 [#share-link-options]

创建链接会打开一个模态框，您可以在其中控制对话对接收者呈现的方式。

![创建共享链接模态框](/images/shared-links/share-modal-create.png)

![带有管理选项的共享链接模态框](/images/shared-links/share-modal-options.png)

一旦链接存在，模态框将提供以下操作：

- **刷新链接**：生成一个新的 URL，并使之前的 URL 失效。
- **生成二维码**: 为移动端访问创建二维码。
- **复制链接到剪贴板**：复制可共享的 URL。
- **删除链接**：移除该链接并撤销任何拥有此链接的人的访问权限。

## 二维码 [#qr-codes]

为任何共享对话生成二维码，然后使用手机摄像头扫描即可打开链接。该二维码指向与共享链接相同的地址。

![为共享链接生成的二维码](/images/shared-links/qr-code.png)

![QR code download options](/images/shared-links/qr-code-download.png)

QR 码在演示文稿、快速移动端访问、打印讲义和会议演示中非常实用。

## 查看共享对话 [#viewing-shared-conversations]

当有人打开您的链接时，他们会看到一个专注的、只读的对话视图，其中包括链接发布日期、完整的消息历史记录以及您生成的任何工件。

![接收者视角下的共享对话](/images/shared-links/shared-view.png)

## 分支 (Branching) [#branching]

共享链接会保留对话分支，因此接收者可以跟随讨论的不同路径进行查看。使用分支导航箭头可在这些路径之间切换。

![共享对话中的分支导航菜单](/images/shared-links/branching-menu.png)

![在共享对话中导航分支](/images/shared-links/branching-support.png)

这对于展示不同的提示词如何引导出不同的 AI 回复，或者如何用多种方式解决同一个问题非常有用。

## Artifacts [#artifacts]

对话过程中生成的 Artifacts 在共享链接中保持完全可用。

![共享对话中的交互式工件](/images/shared-links/artifacts-support.png)

接收者可以：

- 查看交互式 React 组件。
- 查看 HTML 预览。
- 检查 Mermaid 图表。
- 阅读底层代码及其生成的上下文。

## 接收者偏好设置 [#recipient-preferences]

查看者可以根据自己的偏好调整界面，而不会影响您的设置。

![共享链接上的语言和主题设置](/images/shared-links/recipient-settings.png)

- **主题**：浅色、深色或系统。
- **语言**：以用户首选的语言查看界面。

## 管理您的共享链接 [#managing-your-shared-links]

在 **Settings → Data Controls** 下打开 Shared Links 仪表板，以查看您共享的所有对话。

![共享链接管理仪表板](/images/shared-links/management-modal.png)

![单个共享链接的详细信息和选项](/images/shared-links/management-details.png)

从仪表板中，您可以：

- **查看所有链接**：查看您分享过的所有对话。
- **搜索和筛选**：查找特定的共享链接并预览其内容。
- **撤销访问权限**：删除链接以立即停止共享。

## 配置 [#configuration]

`.env` 文件中的两个环境变量控制此功能：

- **`ALLOW_SHARED_LINKS`** (默认值: `true`): 启用共享链接功能。设置为 `false` 可禁止用户创建链接。
- **`ALLOW_SHARED_LINKS_PUBLIC`** (默认值: `false`): 控制链接是否可以在无需身份验证的情况下打开。默认情况下，接收者必须登录。设置为 `true` 以允许公开、未经身份验证的访问。

```bash title=".env"
# Enable shared links (default)
ALLOW_SHARED_LINKS=true

# Require authentication to view shared links (default)
ALLOW_SHARED_LINKS_PUBLIC=false
```

角色权限决定了哪些用户可以创建链接、与已认证用户共享链接，或使其对实例上的所有人可见：

```yaml title="librechat.yaml"
interface:
  sharedLinks:
    create: true
    share: true
    public: false
```

`sharedLinks.public` 控制用户是否可以切换“与所有人共享”。`ALLOW_SHARED_LINKS_PUBLIC` 仍然决定了这些公开链接是否可以在未登录的情况下被查看。

<Callout type="info">
有关完整的环境变量参考，请参阅 [.env Configuration](/docs/configuration/dotenv#shared-links) 页面。
</Callout>

## 常见问题解答 [#frequently-asked-questions]

<Accordions type="single" collapsible>
<Accordion title="接收者可以回复共享的对话吗？">

不可以。共享链接仅供只读。接收者可以查看对话，但无法添加消息或与 AI 进行交互。

</Accordion>
<Accordion title="如果我在分享对话后将其删除，会发生什么？">

该链接将停止工作。任何访问该链接的人都会看到“Shared link not found”页面。

</Accordion>
<Accordion title="分享对话后，我可以对其进行编辑吗？">

该链接会捕捉您创建它时的对话状态。新消息不会出现在共享视图中，但对现有消息的编辑会同步更新。

</Accordion>
<Accordion title="共享链接可以离线使用吗？">

不是。接收者需要互联网连接才能查看共享的对话。

</Accordion>
</Accordions>
