# Cargar archivos como texto [#upload-files-as-text]

¿Alguna vez has querido entregarle un PDF, un archivo de código o una hoja de cálculo a la IA y simplemente decirle _"lee esto"_? Eso es exactamente lo que hace **Upload as Text**.

Adjuntas un archivo, LibreChat extrae el texto del mismo y el contenido completo se pega directamente en tu conversación. La IA puede leer cada palabra del mismo; sin plugins, sin bases de datos vectoriales, sin servicios adicionales que configurar. Funciona de inmediato.

<Callout type="info" title="No requiere configuración">
  Upload as Text funciona inmediatamente en cualquier instancia de LibreChat. Utiliza un análisis de texto integrado; no necesitas OCR, una canalización RAG ni ningún servicio externo para comenzar.
</Callout>

---

## Cómo usarlo [#how-to-use-it]

<Steps>
  <Step>
    ### Haz clic en el icono de adjuntar [#click-the-attachment-icon]

    En la barra de entrada del chat, haz clic en el icono del **clip** (📎).
  </Step>
  <Step>
    ### Selecciona "Upload as Text" [#pick-upload-as-text]

    Desde el menú desplegable, selecciona **Upload as Text**. Esto le indica a LibreChat que lea el contenido del archivo en lugar de pasarlo como un archivo adjunto sin procesar.
  </Step>
  <Step>
    ### Elige tu archivo [#choose-your-file]

    Selecciona el archivo desde tu dispositivo. LibreChat extraerá el texto y lo insertará directamente en tu mensaje.
  </Step>
  <Step>
    ### Haz tu pregunta [#ask-your-question]

    Escribe tu prompt como de costumbre. La IA ahora tiene el texto completo de tu archivo en contexto y puede hacer referencia a cualquier parte del mismo.
  </Step>
</Steps>

<Callout type="warn" title="¿No ves la opción?">
  Si "Upload as Text" no aparece, es posible que la capacidad `context` haya sido desactivada por su administrador. Está activada de forma predeterminada, pero si la lista de capacidades fue personalizada, `context` debe incluirse explícitamente. Consulte la [sección de configuración](#the-context-capability) a continuación.
</Callout>

---

## Qué sucede bajo el capó [#what-happens-under-the-hood]

Cuando subes un archivo de esta manera, LibreChat no solo vuelca bytes sin procesar en el prompt. Pasa por una canalización de procesamiento para extraer texto limpio y legible:

1. **Detección de tipo MIME** — LibreChat verifica qué tipo de archivo has subido (PDF, imagen, audio, código fuente, etc.) inspeccionando su tipo MIME.
2. **Selección de método** — Según el tipo de archivo y los servicios disponibles, elige el mejor método de extracción utilizando esta prioridad:

<Tabs items={["Orden de prioridad", "Ejemplos de decisión"]}>
  <Tabs.Tab>
    | Prioridad | Método | Cuándo se utiliza |
    |----------|--------|---------------|
    | 1.ª | **OCR** | El archivo es una imagen o un documento escaneado, _y_ el OCR está configurado |
    | 2.ª | **STT** (Speech-to-Text) | El archivo es audio, _y_ el STT está configurado |
    | 3.ª | **Análisis de texto** | El archivo coincide con un tipo MIME de texto conocido |
    | 4.ª | **Reserva** (Fallback) | Ninguna de las anteriores coincidió — intenta el análisis de texto de todos modos |
  </Tabs.Tab>
  <Tabs.Tab>
    **Un `.pdf` en una instancia con OCR configurado:**
    → Se activa el OCR. Ideal para documentos escaneados y diseños complejos.

    **Un `.pdf` en una instancia predeterminada (sin OCR):**
    → El análisis de texto lo gestiona. Funciona bien para archivos PDF creados digitalmente.

    **Un archivo Python `.py`:**
    → Directo al análisis de texto. El código fuente ya es texto; no se requiere conversión.

    **Un `.mp3` en una instancia con STT configurado:**
    → Speech-to-Text lo transcribe a texto para la conversación.

    **Una captura de pantalla `.png` sin OCR configurado:**
    → Recurre al análisis de texto (resultados limitados; considere configurar OCR para imágenes).
  </Tabs.Tab>
</Tabs>

3. **Truncamiento de tokens** — El texto extraído se recorta al `fileTokenLimit` (predeterminado: 100,000 tokens) para que no exceda la ventana de contexto del modelo.
4. **Inyección de prompts** — El texto se incluye en el contexto de la conversación, justo al lado de su mensaje.

## ¿Qué archivos son compatibles [#which-files-are-supported]

<Tabs items={["Texto y código", "Documentos", "Imágenes", "Audio"]}>
  <Tabs.Tab>
    Estos se analizan directamente; ya son texto, por lo que no se necesita conversión.

    - Texto plano (`.txt`), Markdown (`.md`), CSV, JSON, XML, HTML, CSS
    - Lenguajes de programación — Python, JavaScript, TypeScript, Java, C#, PHP, Ruby, Go, Rust, Kotlin, Swift, Scala, Perl, Lua
    - Archivos de configuración — YAML, TOML, INI
    - Scripts de shell, archivos SQL
  </Tabs.Tab>
  <Tabs.Tab>
    El análisis de texto gestiona esto de forma nativa. Si OCR está configurado, este toma el control para una mayor precisión en diseños complejos.

    - **PDF** — digital y escaneado (los PDF escaneados se benefician del OCR)
    - **Word** — `.docx`, `.doc`
    - **PowerPoint** — `.pptx`, `.ppt`
    - **Excel** — `.xlsx`, `.xls`
    - Libros **EPUB**
  </Tabs.Tab>
  <Tabs.Tab>
    Las imágenes **requieren OCR** para producir texto útil. Sin él, los resultados serán deficientes.

    - JPEG, PNG, GIF, WebP
    - HEIC, HEIF (formatos de Apple)
    - Capturas de pantalla, fotos de documentos, páginas escaneadas
  </Tabs.Tab>
  <Tabs.Tab>
    Los archivos de audio **requieren que STT** esté configurado. No existe una alternativa — el audio no puede ser "analizado como texto".

    - MP3, WAV, OGG, FLAC
    - M4A, WebM
    - Grabaciones de voz, clips de podcast
  </Tabs.Tab>
</Tabs>

## Cargar como texto vs. otras opciones de carga [#upload-as-text-vs-other-upload-options]

LibreChat tiene tres formas de subir archivos. Cada una funciona de manera diferente y se adapta a distintas situaciones:

<Cards>
  <Card title="Subir como texto" href="#how-to-use-it">
    Extrae el contenido completo del archivo y lo coloca en la conversación. Es ideal para archivos más pequeños donde deseas que la IA lea todo: contratos, archivos de código, artículos. Funciona con todos los modelos, sin necesidad de servicios adicionales.
  </Card>
  <Card title="Carga para búsqueda de archivos (RAG)" href="/docs/features/rag_api">
    Indexa el archivo en una base de datos vectorial y recupera solo los fragmentos relevantes cuando haces una pregunta. Es mejor para archivos grandes o colecciones de archivos donde incluir todo en el contexto desperdiciaría tokens. Requiere la RAG API.
  </Card>
  <Card title="Carga estándar" href="/docs/features/agents">
    Pasa el archivo directamente al modelo; se utiliza para modelos de visión que analizan imágenes o para el intérprete de código que ejecuta scripts. No se realiza ninguna extracción de texto.
  </Card>
</Cards>

**Guía de decisión rápida:**

| Situación | Mejor opción |
|-----------|-------------|
| _"Lee este contrato de 5 páginas y resúmelo"_ | **Cargar como texto** |
| _"Tengo 50 archivos PDF, busca qué menciona sobre precios"_ | **Búsqueda de archivos (RAG)** |
| _"¿Qué hay en esta captura de pantalla?"_ (modelo de visión) | **Carga estándar** |
| _"Ejecuta este script de Python"_ (intérprete de código) | **Carga estándar** |
| _"Revisa este archivo de código en busca de errores"_ | **Cargar como texto** |
| _"Busca en nuestros documentos de la empresa"_ | **Búsqueda de archivos (RAG)** |

## La capacidad `context` [#the-context-capability]

Internamente, Upload as Text funciona mediante la **capacidad `context`**. Esto es lo que controla si la función aparece en tu interfaz de chat.

<Callout type="info">
  La capacidad `context` está **habilitada de forma predeterminada**. Solo necesitas modificar esto si tu administrador ha personalizado la lista de capacidades y la ha omitido accidentalmente.
</Callout>

```yaml title="librechat.yaml"
endpoints:
  agents:
    capabilities:
      - "context"  # This is what enables "Upload as Text"
```

La misma capacidad de `context` también impulsa el **Agent File Context** (subir archivos a través del Agent Builder para incrustar texto en las instrucciones del sistema de un agente). La diferencia es _dónde_ termina el texto:

| | Cargar como texto | Contexto de archivo del agente |
|---|---|---|
| **Dónde** | Entrada de chat (cualquier conversación) | Panel del Creador de agentes |
| **Alcance** | Solo la conversación actual | Persiste en las instrucciones del agente |
| **Caso de uso** | Preguntas sobre documentos puntuales | Creación de agentes especializados con conocimiento integrado |

## Límites de tokens y truncamiento [#token-limits-and-truncation]

Cuando un archivo es demasiado largo para caber en la ventana de contexto del modelo, LibreChat trunca el texto extraído para mantenerse dentro de los límites. Esto sucede automáticamente; no necesita preocuparse por ello, pero es bueno saber cómo funciona.

```yaml title="librechat.yaml"
fileConfig:
  fileTokenLimit: 100000  # Default: 100,000 tokens
```

<Callout type="warn" title="El truncamiento significa contenido perdido">
  Si tu archivo excede el límite, el texto se corta al final. Si estás obteniendo respuestas incompletas, esta podría ser la razón. Puedes aumentar `fileTokenLimit`, pero ten en cuenta que valores más grandes utilizan más tokens por mensaje, lo cual aumenta el costo y puede alcanzar el límite de contexto del propio modelo.
</Callout>

**Reglas generales:**
- 100k tokens ≈ un libro de 300 páginas (suficiente para la mayoría de los casos de uso)
- Si estás trabajando con archivos muy grandes, considera [File Search (RAG)](/docs/features/rag_api) en su lugar; esto solo recupera las secciones relevantes en lugar de incluir todo en el contexto.

## Opcional: mejorar la extracción con OCR [#optional-boosting-extraction-with-ocr]

El análisis de texto funciona bien para documentos creados digitalmente (PDF guardados desde Word, archivos de código, texto sin formato). Pero si vas a subir **documentos escaneados, fotos de páginas o imágenes con texto**, el analizador integrado no obtendrá resultados óptimos.

Ahí es donde entra en juego el OCR. Cuando está configurado, LibreChat utiliza automáticamente OCR para los tipos de archivo que se benefician de ello; no necesitas hacer nada diferente como usuario.

<Accordions>
  <Accordion title="Cómo pedirle a tu administrador que configure OCR">
    Dirígelos a la [documentación de configuración de OCR](/docs/features/ocr). La versión corta:

    ```yaml title="librechat.yaml"
    ocr:
      strategy: "mistral_ocr"
      apiKey: "${OCR_API_KEY}"
      baseURL: "https://api.mistral.ai/v1"
      mistralModel: "mistral-ocr-latest"
    ```

    Una vez configurado, OCR maneja automáticamente imágenes y PDFs escaneados; no se requieren cambios por parte del usuario.
  </Accordion>
  <Accordion title="Cómo pedirle a tu administrador que configure STT (para archivos de audio)">
    Para transcribir archivos de audio cargados, el administrador debe configurar un servicio de Speech-to-Text. Consulte la [referencia de configuración de STT](/docs/configuration/librechat_yaml/object_structure/file_config#stt).
  </Accordion>
</Accordions>

## Referencia de configuración de manejo de archivos [#file-handling-configuration-reference]

Esta sección es para administradores que desean controlar qué tipos de archivos son procesados por qué método. Los valores predeterminados funcionan bien; solo necesita modificar esto si desea ajustar el comportamiento.

<Accordions>
  <Accordion title="Ejemplo completo de fileConfig">
    ```yaml title="librechat.yaml"
    fileConfig:
      # Max tokens extracted from a single file before truncation
      fileTokenLimit: 100000

      # Files matching these MIME patterns use OCR (if configured)
      ocr:
        supportedMimeTypes:
          - "^image/(jpeg|gif|png|webp|heic|heif)$"
          - "^application/pdf$"
          - "^application/vnd\\.openxmlformats-officedocument\\.(wordprocessingml\\.document|presentationml\\.presentation|spreadsheetml\\.sheet)$"
          - "^application/vnd\\.ms-(word|powerpoint|excel)$"
          - "^application/epub\\+zip$"

      # Files matching these MIME patterns use text parsing
      text:
        supportedMimeTypes:
          - "^text/(plain|markdown|csv|json|xml|html|css|javascript|typescript|x-python|x-java|x-csharp|x-php|x-ruby|x-go|x-rust|x-kotlin|x-swift|x-scala|x-perl|x-lua|x-shell|x-sql|x-yaml|x-toml)$"

      # Files matching these MIME patterns use STT (if configured)
      stt:
        supportedMimeTypes:
          - "^audio/(mp3|mpeg|mpeg3|wav|wave|x-wav|ogg|vorbis|mp4|x-m4a|flac|x-flac|webm)$"
    ```

    **Recordatorio de prioridad:** OCR > STT > análisis de texto > alternativa (fallback).

    Para obtener la referencia completa, consulte [File Config Object Structure](/docs/configuration/librechat_yaml/object_structure/file_config).
  </Accordion>
</Accordions>

## Solución de problemas [#troubleshooting]

<Accordions>
  <Accordion title="La opción Cargar como texto no aparece">
    La capacidad `context` probablemente fue eliminada de su configuración. Pídale a su administrador que la vuelva a añadir:

    ```yaml
    endpoints:
      agents:
        capabilities:
          - "context"
    ```
  </Accordion>
  <Accordion title="El contenido del archivo parece incorrecto o está mayormente vacío">
    Algunas cosas para verificar:
    - **PDFs / imágenes escaneadas** — Estos requieren OCR para extraer el texto correctamente. Sin él, el analizador podría devolver información basura o nada en absoluto. Pide a tu administrador que [configure OCR](/docs/features/ocr).
    - **Archivos de audio** — Estos requieren STT. No existe una alternativa de texto para el audio.
    - **Archivos dañados** — Intenta abrir el archivo localmente para asegurarte de que no esté dañado.
    - **Formato no compatible** — Si el tipo MIME no coincide con ningún patrón configurado, LibreChat intenta un respaldo de análisis de texto, el cual podría no funcionar para formatos binarios.
  </Accordion>
  <Accordion title="La IA parece haber omitido parte de mi archivo">
    Es probable que tu archivo haya excedido el `fileTokenLimit`. El texto fue truncado.

    **Opciones:**
    - Pide a tu administrador que aumente `fileTokenLimit` en `librechat.yaml`
    - Utilice [File Search (RAG)](/docs/features/rag_api) en su lugar, el cual recupera fragmentos relevantes en lugar de cargar el archivo completo
    - Divide el archivo en partes más pequeñas y súbelas por separado
  </Accordion>
  <Accordion title="Las imágenes se suben pero la IA no puede leer el texto en ellas">
    Sin OCR, las imágenes se procesan mediante análisis de texto, el cual no puede "ver" realmente el texto dentro de una imagen. Necesitas tener configurado OCR para que esto funcione. Consulta [OCR for Documents](/docs/features/ocr).

    Alternativamente, utilice un **vision model** con la carga de archivos estándar: el propio modelo puede leer texto en imágenes.
  </Accordion>
</Accordions>
