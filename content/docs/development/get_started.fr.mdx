## Préparer l'environnement [#prepare-the-environment]

### Node.js et npm [#nodejs-and-npm]

Si vous utilisez `nvm`, installez et sélectionnez la version de Node.js recommandée avant d'installer LibreChat :

```sh filename="Use Node.js 24"
nvm install 24.16.0
nvm use 24.16.0
npm install -g npm@11.16.0
```

Vérifiez que votre shell utilise les versions attendues :

```sh filename="Check Node.js and npm"
node -v
npm -v
```

```txt filename="Expected versions"
v24.16.0
11.16.0
```

### GitHub [#github]

- Forkez le dépôt LibreChat : [https://github.com/danny-avila/LibreChat/fork](https://github.com/danny-avila/LibreChat/fork)

- Créez une branche sur votre fork, nommez-la correctement et liez-la au dépôt original.

- Téléchargez votre nouvelle branche sur votre PC local

```sh filename="Download your LibreChat branch"
git clone -b branch-name https://github.com/username/LibreChat.git
```

> Remplacez `branch-name` et `username` par vos informations

### Ouvrir dans VS Code [#open-in-vs-code]

- Après avoir cloné votre branche :
  ```sh filename="Navigate to LibreChat folder"
  cd LibreChat
  ```
  ```sh filename="Open in VS Code"
  code .
  ```

### Préparer LibreChat [#prepare-librechat]

- Open the terminal in VS Code with `ctrl`+`shift`+`` ```

  > Alternativement, utilisez `ctrl`+`j` pour ouvrir le volet inférieur et sélectionnez le terminal.

- ```sh filename="Install LibreChat dependencies"
  npm run smart-reinstall
  ```

  > Si vous venez de changer de version de Node.js ou de npm, utilisez `npm run reinstall` une fois pour une installation propre.

- ```sh filename="Build all compiled code"
  npm run build
  ```

- Configuration de .env
  - Créez le fichier `.env`. Si vous n'en avez pas, dupliquez `.env.example` et configurez-le.

<Callout type="warning" title="Avertissement">
  Les valeurs par défaut dans `.env.example` conviennent généralement, à l'exception de `MONGO_URI`. Fournissez la vôtre.
  Assurez-vous d'installer MongoDB et de configurer `MONGO_URI` correctement pour vous connecter à votre instance MongoDB. Utilisez [MongoDB Community Server](https://www.mongodb.com/try/download/community) ou [MongoDB Atlas Cloud](https://www.mongodb.com/cloud/atlas/register).
</Callout>

### Flux de travail de développement [#development-workflow]

Pour un travail efficace sur LibreChat, utilisez ces commandes :

- **Démarrage du backend :**
  - Utilisez `npm run backend` pour un fonctionnement normal.
  - Pour le développement actif, utilisez `npm run backend:dev` pour surveiller les changements.
  - Accédez à `http://localhost:3080/`.

- **Exécution du frontend en mode développement :**
  - **Assurez-vous que le backend est en cours d'exécution.**
  - Utilisez `npm run frontend:dev` pour surveiller les changements du frontend.
  - Consultez à l'adresse `http://localhost:3090/`.

<Callout type="tip" title="Astuces de pro">
  - Pour des mises à jour en temps réel lors du développement du frontend, exécutez `npm run frontend:dev` afin que les modifications du frontend soient actualisées sur le port `3090`.
  - Définissez `DEBUG_CONSOLE=true` dans `.env` pour obtenir une sortie serveur détaillée dans la console.
</Callout>

## Commit, Push, Pull Request (PR) [#commit-push-pull-request-pr]

### Effectuer un commit [#make-a-commit]

**Commits** marquent des points de contrôle logiques dans le développement. Incluez des messages clairs expliquant les changements.

**Exemple :**

```bash filename=" "
git add .
git commit -m "Add login functionality"
```

### Pousser les modifications [#push-changes]

**Poussez** les modifications vers le dépôt distant après avoir terminé une fonctionnalité ou corrigé un problème.

**Exemple :**

```bash filename=" "
git push origin feature-branch-name
```

### Créer une Pull Request (PR) [#create-a-pull-request-pr]

**Pull Request** fusionne les modifications d'une branche de fonctionnalité vers la branche principale.

1. Récupérez les dernières modifications de la branche main et résolvez les conflits.
2. Poussez la branche de fonctionnalité mise à jour.
3. Assurez-vous que le code respecte les directives du projet.

**Exemple :**

```bash filename=" "
git checkout main
git pull origin main
git checkout feature-branch-name
git rebase main
# Resolve conflicts if any
git push origin feature-branch-name
# Open PR on GitHub
```

Accédez à votre dépôt dans un navigateur et cliquez sur "Contribute".

<Callout type="info" title="Remarque :">
  Fournissez une description détaillée de la PR expliquant les changements et leur valeur. Référencez les problèmes associés.
</Callout>

<Callout type="tip" title="Astuce">
  Utilisez GitHub Desktop pour suivre les modifications.
</Callout>

<Callout type="warning" title="Avertissement">
  Si `git commit` échoue en raison d'erreurs ESLint, comprenez et corrigez le problème.
</Callout>

## Annuler des commits en toute sécurité [#revert-commits-safely]

Pour annuler des modifications dans une branche de fonctionnalité, suivez ces étapes avec prudence :

- ```bash filename="1. Update local repository from feature branch:"
  git pull origin feature-branch-name
  ```

- ```bash filename="2. Review commit history to determine commits to revert:"
  git log
  ```

- ```bash filename="3. Start an interactive rebase for 'N' commits to revert:"
  git rebase -i HEAD~N
  ```

  > Remplacez `pick` par `drop` pour les commits à supprimer. Enregistrez et quittez l'éditeur.

- ```bash filename="4. Force push changes to remote repository:"
  git push --force-with-lease origin feature-branch-name
  ```
