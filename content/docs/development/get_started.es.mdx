### Flujo de trabajo de desarrollo [#development-workflow]

Para un trabajo eficiente en LibreChat, utilice estos comandos:

- **Iniciando el backend:**
  - Utilice `npm run backend` para el funcionamiento normal.
  - Para el desarrollo activo, utilice `npm run backend:dev` para monitorear los cambios.
  - Acceda en `http://localhost:3080/`.

- **Ejecuciﾃｳn del Frontend en modo de desarrollo:**
  - **Asegﾃｺrate de que el backend estﾃｩ ejecutﾃ｡ndose.**
  - Usa `npm run frontend:dev` para monitorear los cambios en el frontend.
  - Ver en `http://localhost:3090/`.

<Callout type="tip" title="Consejos profesionales">
  - Para actualizaciones en tiempo real durante el desarrollo del frontend, ejecuta `npm run frontend:dev` para que los cambios en el frontend se actualicen en el puerto `3090`.
  - Establezca `DEBUG_CONSOLE=true` en `.env` para obtener una salida detallada del servidor en la consola.
</Callout>

## Commit, Push, Pull Request (PR) [#commit-push-pull-request-pr]

### Realizar un commit [#make-a-commit]

**Commits** marcan puntos de control lﾃｳgicos en el desarrollo. Incluya mensajes claros que expliquen los cambios.

**Ejemplo:**

```bash filename=" "
git add .
git commit -m "Add login functionality"
```

### Enviar cambios [#push-changes]

**Push** los cambios al repositorio remoto despuﾃｩs de completar una funcionalidad o corregir un problema.

**Ejemplo:**

```bash filename=" "
git push origin feature-branch-name
```

### Crear una Pull Request (PR) [#create-a-pull-request-pr]

**Pull Request** fusiona los cambios de una rama de caracterﾃｭsticas en la rama principal.

1. Extrae los ﾃｺltimos cambios de la rama main y resuelve los conflictos.
2. Envﾃｭa la rama de caracterﾃｭsticas actualizada.
3. Asegﾃｺrese de que el cﾃｳdigo siga las directrices del proyecto.

**Ejemplo:**

```bash filename=" "
git checkout main
git pull origin main
git checkout feature-branch-name
git rebase main
# Resolve conflicts if any
git push origin feature-branch-name
# Open PR on GitHub
```

Acceda a su repositorio en un navegador y haga clic en "Contribute".

<Callout type="info" title="Nota:">
  Proporciona una descripciﾃｳn detallada del PR explicando los cambios y su valor. Haz referencia a los issues relacionados.
</Callout>

<Callout type="tip" title="Consejo">
  Usa GitHub Desktop para realizar un seguimiento de los cambios.
</Callout>

<Callout type="warning" title="Advertencia">
  Si `git commit` falla debido a errores de ESLint, comprende y soluciona el problema.
</Callout>

## Revertir Commits de forma segura [#revert-commits-safely]

Para deshacer cambios en una rama de caracterﾃｭsticas (feature branch), siga estos pasos con precauciﾃｳn:

- ```bash filename="1. Update local repository from feature branch:"
  git pull origin feature-branch-name
  ```

- ```bash filename="2. Review commit history to determine commits to revert:"
  git log
  ```

- ```bash filename="3. Start an interactive rebase for 'N' commits to revert:"
  git rebase -i HEAD~N
  ```

  > Reemplace `pick` con `drop` para los commits que desea eliminar. Guarde y salga del editor.

- ```bash filename="4. Force push changes to remote repository:"
  git push --force-with-lease origin feature-branch-name
  ```
