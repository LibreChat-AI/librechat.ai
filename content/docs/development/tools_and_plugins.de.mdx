# Eigene Tools/Plugins erstellen [#making-your-own-toolsplugins]

<Callout type="warning" title="Warnung">
  Bitte beziehen Sie sich auf die aktuellsten Tools, die mit Assistants in `api/app/clients/tools/structured/` verwendet werden, da Plugins in naher Zukunft zugunsten von Tools eingestellt werden.
</Callout>

Das Erstellen benutzerdefinierter Plugins fﾃｼr dieses Projekt beinhaltet das Erweitern der `Tool`-Klasse aus dem `langchain/tools`-Modul.

**Hinweis:** Ich verwende das Wort Plugin synonym zu Tool, da Letzteres spezifisch fﾃｼr LangChain ist und wir uns hauptsﾃ､chlich an diese Bibliothek anpassen.

Sie erstellen im Grunde genommen DynamicTools im Sinne von LangChain. Weitere Informationen finden Sie in der **[LangChainJS-Dokumentation](https://js.langchain.com/docs/how_to/custom_tools/)**.

Dieser Leitfaden fﾃｼhrt Sie durch den Prozess der Erstellung Ihrer eigenen benutzerdefinierten Plugins, wobei die Tools `StableDiffusionAPI` und `WolframAlphaAPI` als Beispiele dienen.

Bei der Verwendung des Functions Agent (dem Standardmodus fﾃｼr Plugins) werden Tools in **[OpenAI functions](https://openai.com/blog/function-calling-and-other-api-updates)** konvertiert; in jedem Fall werden Plugins/Tools bedingt aufgerufen, basierend darauf, dass das LLM ein spezifisches Format generiert, das wir parsen.

Die gebrﾃ､uchlichste Implementierung eines Plugins besteht darin, basierend auf der natﾃｼrlichsprachlichen Eingabe der KI einen API-Aufruf zu tﾃ､tigen, aber es gibt praktisch keine Grenzen fﾃｼr den programmatischen Anwendungsfall.

## Wichtige Erkenntnisse [#key-takeaways]

Hier sind die wichtigsten Punkte fﾃｼr die Erstellung Ihres eigenen Plugins:

**1.** [**Erforderliche Module importieren:**](#step-1-import-required-modules) Importieren Sie die notwendigen Module fﾃｼr Ihr Plugin, einschlieﾃ殕ich der `Tool`-Klasse aus `langchain/tools` sowie aller anderen Module, die Ihr Plugin mﾃｶglicherweise benﾃｶtigt.

**2.** [**Define Your Plugin Class:**](#step-2-define-your-tool-class) Define a class for your plugin that extends the `Tool` class. Set the `name` and `description` properties in the constructor. If your plugin requires credentials or other variables, set them from the fields parameter or from a method that retrieves them from your process environment. Note: if your plugin requires long, detailed instructions, you can add a `description_for_model` property and make `description` more general.

**3.** [**Hilfsmethoden definieren:**](#step-3-define-helper-methods) Definieren Sie bei Bedarf Hilfsmethoden innerhalb Ihrer Klasse, um spezifische Aufgaben zu erledigen.

**4.** [**Implementieren der `_call`-Methode:**](#step-4-implement-the-_call-method) Implementieren Sie die `_call`-Methode, in der die Hauptfunktionalitﾃ､t Ihres Plugins definiert ist. Diese Methode wird aufgerufen, wenn das Sprachmodell entscheidet, Ihr Plugin zu verwenden. Sie sollte einen `input`-Parameter entgegennehmen und ein Ergebnis zurﾃｼckgeben. Wenn ein Fehler auftritt, sollte die Funktion einen String zurﾃｼckgeben, der den Fehler darstellt, anstatt einen Fehler auszulﾃｶsen. Wenn Ihr Plugin mehrere Eingaben vom LLM erfordert, lesen Sie den Abschnitt [StructuredTools](#StructuredTools).

**5.** [**Exportieren Sie Ihr Plugin und importieren Sie es in handleTools.js:**](#step-5-export-your-plugin-and-import-into-handletoolsjs) Exportieren Sie Ihr Plugin und importieren Sie es in `handleTools.js`. Fﾃｼgen Sie Ihr Plugin dem `toolConstructors`-Objekt in der `loadTools`-Funktion hinzu. Wenn Ihr Plugin eine fortgeschrittenere Initialisierung erfordert, fﾃｼgen Sie es dem `customConstructors`-Objekt hinzu.

**6.** [**Exportieren Sie Ihr Plugin in die index.js:**](#step-6-export-your-plugin-into-indexjs) Exportieren Sie Ihr Plugin in die `index.js` unter `tools`. Fﾃｼgen Sie Ihr Plugin zu den `module.exports` der `index.js` hinzu, sodass Sie es in dieser Datei auch als `const` deklarieren mﾃｼssen.

**7.** [**Fﾃｼgen Sie Ihr Plugin zu manifest.json hinzu:**](#step-7-add-your-plugin-to-manifestjson) Fﾃｼgen Sie Ihr Plugin zu `manifest.json` hinzu. Befolgen Sie das strikte Format fﾃｼr jedes der Felder des "plugin"-Objekts. Wenn Ihr Plugin eine Authentifizierung erfordert, fﾃｼgen Sie diese Details unter `authConfig` als Array hinzu. Der `pluginKey` sollte mit dem Klassennamen (`name`) der von Ihnen erstellten Tool-Klasse ﾃｼbereinstimmen, und die `authField`-Eigenschaft muss mit dem Namen der process.env-Variable ﾃｼbereinstimmen.

Denken Sie daran: Der Schlﾃｼssel zur Erstellung eines benutzerdefinierten Plugins liegt darin, die `Tool`-Klasse zu erweitern und die `_call`-Methode zu implementieren. In der `_call`-Methode definieren Sie, was Ihr Plugin tut. Sie kﾃｶnnen in Ihrer Klasse auch Hilfsmethoden und Eigenschaften definieren, um die Funktionalitﾃ､t Ihres Plugins zu unterstﾃｼtzen.

**Hinweis: Sie finden alle in dieser Anleitung erwﾃ､hnten Dateien im Ordner `.\api\app\langchain\tools`.**

## Schritt 1: Erforderliche Module importieren [#step-1-import-required-modules]

Beginnen Sie mit dem Importieren der erforderlichen Module. Dies umfasst die `Tool`-Klasse aus `langchain/tools` sowie alle anderen Module, die Ihr Tool mﾃｶglicherweise benﾃｶtigt. Zum Beispiel:

```javascript
const { Tool } = require('langchain/tools')
// ... whatever else you need
```

## Schritt 2: Definieren Sie Ihre Tool-Klasse [#step-2-define-your-tool-class]

Definieren Sie als Nﾃ､chstes eine Klasse fﾃｼr Ihr Plugin, die die `Tool`-Klasse erweitert. Die Klasse sollte einen Konstruktor haben, der die `super()`-Methode aufruft und die Eigenschaften `name` und `description` festlegt. Diese Eigenschaften werden vom Sprachmodell verwendet, um zu bestimmen, wann Ihr Tool aufgerufen werden soll und mit welchen Parametern.

**Wichtig:** Sie sollten die Anmeldedaten/erforderlichen Variablen ﾃｼber den fields-Parameter oder alternativ ﾃｼber eine Methode festlegen, die diese aus Ihrer Prozessumgebung abruft.

```javascript
class StableDiffusionAPI extends Tool {
  constructor(fields) {
    super();
    this.name = 'stable-diffusion';
    this.url = fields.SD_WEBUI_URL || this.getServerURL(); // <--- important!
    this.description = `You can generate images with 'stable-diffusion'. This tool is exclusively for visual content...`;
  }
  ...
}
```

**Optional:** Ab v0.5.8 kﾃｶnnen Sie bei der Verwendung von Functions lﾃ､ngere, detailliertere Anweisungen mit der Eigenschaft `description_for_model` hinzufﾃｼgen. Dabei wird empfohlen, die Eigenschaft `description` allgemeiner zu halten, um Token zu optimieren. Jede Zeile in dieser Eigenschaft wird mit `// ` vorangestellt, um widerzuspiegeln, wie der Prompt fﾃｼr ChatGPT (chat.openai.com) generiert wird. Dieses Format entspricht eher dem Prompt-Engineering offizieller ChatGPT-Plugins.

```js
// ...
this.description_for_model = `// Generate images and visuals using text with 'stable-diffusion'.
// Guidelines:
// - ALWAYS use {{"prompt": "7+ detailed keywords", "negative_prompt": "7+ detailed keywords"}} structure for queries.
// - Visually describe the moods, details, structures, styles, and/or proportions of the image. Remember, the focus is on visual attributes.
// - Craft your input by "showing" and not "telling" the imagery. Think in terms of what you'd want to see in a photograph or a painting.
// - Here's an example for generating a realistic portrait photo of a man:
// "prompt":"photo of a man in black clothes, half body, high detailed skin, coastline, overcast weather, wind, waves, 8k uhd, dslr, soft lighting, high quality, film grain, Fujifilm XT3"
// "negative_prompt":"semi-realistic, cgi, 3d, render, sketch, cartoon, drawing, anime, out of frame, low quality, ugly, mutation, deformed"
// - Generate images only once per human query unless explicitly requested by the user`
this.description =
  "You can generate images using text with 'stable-diffusion'. This tool is exclusively for visual content."
// ...
```

Beachten Sie innerhalb des Konstruktors, dass wir eine sensible Variable entweder aus dem fields-Objekt oder aus der von uns definierten **getServerURL**-Methode abrufen, um auf eine Umgebungsvariable zuzugreifen.

```js
this.url = fields.SD_WEBUI_URL || this.getServerURL()
```

Alle erforderlichen Anmeldedaten werden ﾃｼber `fields` ﾃｼbermittelt, wenn der Benutzer sie vom Frontend aus bereitstellt; andernfalls kann der Administrator das Plugin fﾃｼr alle Benutzer ﾃｼber Umgebungsvariablen "autorisieren". Alle vom Frontend ﾃｼbermittelten Anmeldedaten werden verschlﾃｼsselt.

```js
// It's recommended you follow this convention when accessing environment variables.
  getServerURL() {
    const url = process.env.SD_WEBUI_URL || '';
    if (!url) {
      throw new Error('Missing SD_WEBUI_URL environment variable.');
    }
    return url;
  }
```

## Schritt 3: Hilfsmethoden definieren [#step-3-define-helper-methods]

Sie kﾃｶnnen bei Bedarf Hilfsmethoden innerhalb Ihrer Klasse definieren, um spezifische Aufgaben zu erledigen. Zum Beispiel enthﾃ､lt die `StableDiffusionAPI`-Klasse Methoden wie `replaceNewLinesWithSpaces`, `getMarkdownImageUrl` und `getServerURL`, um verschiedene Aufgaben zu verarbeiten.

```javascript
class StableDiffusionAPI extends Tool {
  ...
  replaceNewLinesWithSpaces(inputString) {
    return inputString.replace(/\r\n|\r|\n/g, ' ');
  }
  ...
}
```

## Schritt 4: Implementierung der `_call`-Methode [#step-4-implement-the-_call-method]

Die `_call`-Methode ist der Ort, an dem die Hauptfunktionalitﾃ､t Ihres Plugins implementiert wird. Diese Methode wird aufgerufen, wenn das Sprachmodell entscheidet, Ihr Plugin zu verwenden. Sie sollte einen `input`-Parameter entgegennehmen und ein Ergebnis zurﾃｼckgeben.

> Bei einem einfachen Tool generiert das LLM einen einzelnen String-Wert als Eingabe. Wenn Ihr Plugin mehrere Eingaben vom LLM erfordert, lesen Sie den Abschnitt **[StructuredTools](#StructuredTools)**.

```javascript
class StableDiffusionAPI extends Tool {
  ...
  async _call(input) {
    // Your tool's functionality goes here
    ...
    return this.result;
  }
}
```

**Wichtig:** Die \_call-Funktion ist das, was der Agent tatsﾃ､chlich aufruft. Wenn ein Fehler auftritt, sollte die Funktion nach Mﾃｶglichkeit einen String zurﾃｼckgeben, der den Fehler darstellt, anstatt einen Fehler auszulﾃｶsen. Dies ermﾃｶglicht es, den Fehler an das LLM weiterzuleiten, damit das LLM entscheiden kann, wie damit umzugehen ist. Wenn ein Fehler ausgelﾃｶst wird, wird die Ausfﾃｼhrung des Agenten gestoppt.

## Beispiel: WolframAlphaAPI Tool [#example-wolframalphaapi-tool]

Hier ist ein weiteres Beispiel fﾃｼr ein benutzerdefiniertes Tool, das `WolframAlphaAPI`-Tool. Dieses Tool verwendet das `axios`-Modul, um HTTP-Anfragen an die Wolfram Alpha API zu stellen.

```javascript
const axios = require('axios')
const { Tool } = require('langchain/tools')

class WolframAlphaAPI extends Tool {
  constructor(fields) {
    super()
    this.name = 'wolfram'
    this.apiKey = fields.WOLFRAM_APP_ID || this.getAppId()
    this.description = `Access computation, math, curated knowledge & real-time data through wolframAlpha...`
  }

  async fetchRawText(url) {
    try {
      const response = await axios.get(url, { responseType: 'text' })
      return response.data
    } catch (error) {
      console.error(`Error fetching raw text: ${error}`)
      throw error
    }
  }

  getAppId() {
    const appId = process.env.WOLFRAM_APP_ID || ''
    if (!appId) {
      throw new Error('Missing WOLFRAM_APP_ID environment variable.')
    }
    return appId
  }

  createWolframAlphaURL(query) {
    const formattedQuery = query.replaceAll(/`/g, '').replaceAll(/\n/g, ' ')
    const baseURL = 'https://www.wolframalpha.com/api/v1/llm-api'
    const encodedQuery = encodeURIComponent(formattedQuery)
    const appId = this.apiKey || this.getAppId()
    const url = `${baseURL}?input=${encodedQuery}&appid=${appId}`
    return url
  }

  async _call(input) {
    try {
      const url = this.createWolframAlphaURL(input)
      const response = await this.fetchRawText(url)
      return response
    } catch (error) {
      if (error.response && error.response.data) {
        console.log('Error data:', error.response.data)
        return error.response.data
      } else {
        console.log(`Error querying Wolfram Alpha`, error.message)
        return 'There was an error querying Wolfram Alpha.'
      }
    }
  }
}

module.exports = WolframAlphaAPI
```

In diesem Beispiel verfﾃｼgt die `WolframAlphaAPI`-Klasse ﾃｼber Hilfsmethoden wie `fetchRawText`, `getAppId` und `createWolframAlphaURL`, um spezifische Aufgaben zu erledigen. Die `_call`-Methode fﾃｼhrt eine HTTP-Anfrage an die Wolfram Alpha API aus und gibt die Antwort zurﾃｼck.
