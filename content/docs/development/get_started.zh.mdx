## 要求 [#requirements]

- [Git](https://git-scm.com/downloads) (必需)
- [Node.js](https://nodejs.org/en/download) `v24.16.0` (必需)
- npm `v11.16.0` (必需)

## 准备环境 [#prepare-the-environment]

### Node.js 和 npm [#nodejs-and-npm]

如果您使用 `nvm`，请在安装 LibreChat 之前安装并选择推荐的 Node.js 版本：

```sh filename="Use Node.js 24"
nvm install 24.16.0
nvm use 24.16.0
npm install -g npm@11.16.0
```

验证您的 shell 是否正在使用预期的版本：

```sh filename="Check Node.js and npm"
node -v
npm -v
```

```txt filename="Expected versions"
v24.16.0
11.16.0
```

### GitHub [#github]

- Fork LibreChat 仓库：[https://github.com/danny-avila/LibreChat/fork](https://github.com/danny-avila/LibreChat/fork)

- 在你的 fork 上创建一个分支，为其命名，并将其关联到原始仓库。

- 将您的新分支下载到本地电脑

```sh filename="Download your LibreChat branch"
git clone -b branch-name https://github.com/username/LibreChat.git
```

> 将 `branch-name` 和 `username` 替换为您的详细信息

### 在 VS Code 中打开 [#open-in-vs-code]

- 克隆您的分支后：
  ```sh filename="Navigate to LibreChat folder"
  cd LibreChat
  ```
  ```sh filename="Open in VS Code"
  code .
  ```

### 准备 LibreChat [#prepare-librechat]

- 在 VS Code 中使用 `ctrl`+`shift`+`` ` `` 打开终端。

  > 或者，使用 `ctrl`+`j` 打开底部面板并选择终端。

- ```sh filename="Install LibreChat dependencies"
  npm run smart-reinstall
  ```

  > 如果您刚刚更改了 Node.js 或 npm 版本，请使用 `npm run reinstall` 执行一次干净的安装。

- ```sh filename="Build all compiled code"
  npm run build
  ```

- .env 配置
  - 创建 `.env` 文件。如果你还没有该文件，请复制 `.env.example` 并进行配置。

<Callout type="warning" title="警告">
  `.env.example` 中的默认值通常可以直接使用，但 `MONGO_URI` 除外。请提供您自己的值。
  请确保安装 MongoDB 并正确配置 `MONGO_URI` 以连接到您的 MongoDB 实例。请使用 [MongoDB Community Server](https://www.mongodb.com/try/download/community) 或 [MongoDB Atlas Cloud](https://www.mongodb.com/cloud/atlas/register)。
</Callout>

### 开发工作流程 [#development-workflow]

为了在 LibreChat 上高效工作，请使用以下命令：

- **启动后端：**
  - 使用 `npm run backend` 进行正常操作。
  - 对于活跃开发，请使用 `npm run backend:dev` 来监控更改。
  - 通过 `http://localhost:3080/` 进行访问。

- **在开发模式下运行前端：**
  - **确保后端正在运行。**
  - 使用 `npm run frontend:dev` 来监控前端更改。
  - 在 `http://localhost:3090/` 查看。

<Callout type="tip" title="实用技巧">
  - 若要在前端开发期间进行实时更新，请运行 `npm run frontend:dev`，这样前端更改就会在 `3090` 端口上刷新。
  - 在 `.env` 中设置 `DEBUG_CONSOLE=true` 以在控制台中获取详细的服务器输出。
</Callout>

## 提交 (Commit)、推送 (Push)、拉取请求 (Pull Request, PR) [#commit-push-pull-request-pr]

### 提交更改 [#make-a-commit]

**Commits** 标记了开发过程中的逻辑检查点。请包含清晰的说明信息来解释所做的更改。

**示例：**

```bash filename=" "
git add .
git commit -m "Add login functionality"
```

### 推送更改 [#push-changes]

在完成功能或修复问题后，将更改 **Push** 到远程仓库。

**示例：**

```bash filename=" "
git push origin feature-branch-name
```

### 创建 Pull Request (PR) [#create-a-pull-request-pr]

**Pull Request** 将更改从功能分支合并到主分支。

1. 从 main 分支拉取最新更改并解决冲突。
2. 推送更新后的功能分支。
3. 确保代码遵循项目指南。

**示例：**

```bash filename=" "
git checkout main
git pull origin main
git checkout feature-branch-name
git rebase main
# Resolve conflicts if any
git push origin feature-branch-name
# Open PR on GitHub
```

在浏览器中访问您的仓库并点击“Contribute”。

<Callout type="info" title="注意：">
  请提供详细的 PR 描述，说明所做的更改及其价值。并引用相关议题 (issues)。
</Callout>

<Callout type="tip" title="提示">
  使用 GitHub Desktop 来跟踪更改。
</Callout>

<Callout type="warning" title="警告">
  如果 `git commit` 因 ESLint 错误而失败，请理解并修复该问题。
</Callout>

## 安全地回滚提交 (Revert Commits) [#revert-commits-safely]

若要撤销功能分支中的更改，请谨慎执行以下步骤：

- ```bash filename="1. Update local repository from feature branch:"
  git pull origin feature-branch-name
  ```

- ```bash filename="2. Review commit history to determine commits to revert:"
  git log
  ```

- ```bash filename="3. Start an interactive rebase for 'N' commits to revert:"
  git rebase -i HEAD~N
  ```

  > 将 `pick` 替换为 `drop` 以移除提交。保存并退出编辑器。

- ```bash filename="4. Force push changes to remote repository:"
  git push --force-with-lease origin feature-branch-name
  ```
