## 要件 [#requirements]

- [Git](https://git-scm.com/downloads) (必須)
- [Node.js](https://nodejs.org/en/download) `v24.16.0` (必須)
- npm `v11.16.0` (必須)
- [MongoDB](https://www.mongodb.com/try/download/community) (必須、データベース用)
- [Git LFS](https://git-lfs.com/) (大きなファイルに便利)
- [GitHub Desktop](https://desktop.github.com/) (オプション)
- [VSCode](https://code.visualstudio.com/Download) （推奨されるソースコードエディタ）

### 推奨されるVSCode拡張機能 [#recommended-vscode-extensions]

VS Codeに以下の拡張機能をインストールしてください：

- [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode)
- [ESLint](https://marketplace.visualstudio.com/items?itemName=dbaeumer.vscode-eslint)
- [GitLens](https://marketplace.visualstudio.com/items?itemName=eamodio.gitlens)

## 環境の準備 [#prepare-the-environment]

### Node.js と npm [#nodejs-and-npm]

`nvm` を使用している場合は、LibreChat をインストールする前に、推奨される Node.js バージョンをインストールして選択してください：

```sh filename="Use Node.js 24"
nvm install 24.16.0
nvm use 24.16.0
npm install -g npm@11.16.0
```

シェルが期待されるバージョンを使用していることを確認してください：

```sh filename="Check Node.js and npm"
node -v
npm -v
```

```txt filename="Expected versions"
v24.16.0
11.16.0
```

### GitHub [#github]

- LibreChat リポジトリをフォークします: [https://github.com/danny-avila/LibreChat/fork](https://github.com/danny-avila/LibreChat/fork)

- フォーク上にブランチを作成し、適切な名前を付け、元のリポジトリにリンクしてください。

- 新しいブランチをローカルPCにダウンロードします

```sh filename="Download your LibreChat branch"
git clone -b branch-name https://github.com/username/LibreChat.git
```

> `branch-name` と `username` をご自身の詳細情報に置き換えてください。

### VS Codeで開く [#open-in-vs-code]

- ブランチをクローンした後：
  ```sh filename="Navigate to LibreChat folder"
  cd LibreChat
  ```
  ```sh filename="Open in VS Code"
  code .
  ```

### LibreChatの準備 [#prepare-librechat]

- VS Codeでターミナルを開くには、`ctrl`+`shift`+`` ` `` を押します。

  > または、`ctrl`+`j` を使用して下部ペインを開き、ターミナルを選択します。

- ```sh filename="Install LibreChat dependencies"
  npm run smart-reinstall
  ```

  > Node.jsやnpmのバージョンを変更したばかりの場合は、クリーンインストールを行うために一度 `npm run reinstall` を実行してください。

- ```sh filename="Build all compiled code"
  npm run build
  ```

- .env 設定
  - `.env` ファイルを作成します。まだ作成していない場合は、`.env.example` を複製して設定を行ってください。

<Callout type="warning" title="警告">
  `.env.example` のデフォルト値は通常そのままで問題ありませんが、`MONGO_URI` は例外です。ご自身の値を設定してください。
  MongoDB をインストールし、`MONGO_URI` を正しく設定して MongoDB インスタンスに接続できるようにしてください。[MongoDB Community Server](https://www.mongodb.com/try/download/community) または [MongoDB Atlas Cloud](https://www.mongodb.com/cloud/atlas/register) を使用してください。
</Callout>

### 開発ワークフロー [#development-workflow]

LibreChatで効率的に作業するには、以下のコマンドを使用してください：

- **バックエンドの起動:**
  - 通常の操作には `npm run backend` を使用してください。
  - アクティブな開発を行うには、`npm run backend:dev` を使用して変更を監視してください。
  - `http://localhost:3080/` にアクセスします。

- **フロントエンドを開発モードで実行する:**
  - **バックエンドが実行されていることを確認してください。**
  - `npm run frontend:dev` を使用して、フロントエンドの変更を監視します。
  - `http://localhost:3090/` で表示します。

<Callout type="tip" title="プロのヒント">
  - フロントエンド開発中にリアルタイムで更新を確認するには、`npm run frontend:dev` を実行してください。これにより、フロントエンドの変更がポート `3090` でリフレッシュされるようになります。
  - `.env` に `DEBUG_CONSOLE=true` を設定すると、コンソールに詳細なサーバー出力が表示されます。
</Callout>

## ローカルテスト [#local-testing]

送信前に、ローカルで更新内容をテストしてください。詳細は [Perform Tests Locally](/docs/development/testing) を参照してください。

## コミット、プッシュ、プルリクエスト (PR) [#commit-push-pull-request-pr]

### コミットを作成する [#make-a-commit]

**Commits** は、開発における論理的なチェックポイントを示します。変更内容を説明する明確なメッセージを含めてください。

**例:**

```bash filename=" "
git add .
git commit -m "Add login functionality"
```

### 変更をプッシュする [#push-changes]

機能の実装や問題の修正が完了したら、変更をリモートリポジトリに **Push** してください。

**例:**

```bash filename=" "
git push origin feature-branch-name
```

### Pull Request (PR) を作成する [#create-a-pull-request-pr]

**Pull Request**は、機能ブランチからメインブランチへ変更をマージします。

1. mainブランチから最新の変更をプルし、競合を解決します。
2. 更新されたフィーチャーブランチをプッシュします。
3. コードがプロジェクトのガイドラインに従っていることを確認してください。

**例:**

```bash filename=" "
git checkout main
git pull origin main
git checkout feature-branch-name
git rebase main
# Resolve conflicts if any
git push origin feature-branch-name
# Open PR on GitHub
```

ブラウザでリポジトリにアクセスし、「Contribute」をクリックしてください。

<Callout type="info" title="注意:">
  変更内容とその価値を説明する詳細なPR記述を提供してください。関連するIssueを参照してください。
</Callout>

<Callout type="tip" title="ヒント">
  GitHub Desktopを使用して変更を追跡します。
</Callout>

<Callout type="warning" title="警告">
  ESLintエラーが原因で `git commit` が失敗した場合は、その問題を理解し、修正してください。
</Callout>

## コミットを安全にリバートする [#revert-commits-safely]

機能ブランチでの変更を取り消すには、慎重に以下の手順に従ってください：

- ```bash filename="1. Update local repository from feature branch:"
  git pull origin feature-branch-name
  ```

- ```bash filename="2. Review commit history to determine commits to revert:"
  git log
  ```

- ```bash filename="3. Start an interactive rebase for 'N' commits to revert:"
  git rebase -i HEAD~N
  ```

  > 削除するコミットの `pick` を `drop` に置き換えます。エディタを保存して終了します。

- ```bash filename="4. Force push changes to remote repository:"
  git push --force-with-lease origin feature-branch-name
  ```
