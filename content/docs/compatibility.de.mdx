LibreChat verbindet sich über eine kleine Anzahl von **endpoint types** mit vielen KI-Anbietern.
Diese Seite fasst zusammen, welche Funktionen jeder endpoint unterstützt. Viele Funktionen
hängen von dem spezifischen Modell ab, das Sie auswählen; daher sind diese Zellen als
modellabhängig anstatt mit einem klaren Ja oder Nein gekennzeichnet.

## Hinweise [#notes]

- **Custom (OpenAI-compatible)** deckt jeden Anbieter ab, der über [custom endpoints](/docs/configuration/librechat_yaml/ai_endpoints) konfiguriert wurde, wie zum Beispiel OpenRouter, Groq, Mistral, Ollama, DeepSeek, Perplexity und andere. Die genaue Unterstützung hängt davon ab, wie eng der Anbieter die OpenAI API widerspiegelt und welches Modell Sie wählen.
- **Agents** ist der leistungsfähigste endpoint: Er fügt Werkzeugnutzung, [MCP](/docs/features/mcp), [Code Interpreter](/docs/features/code_interpreter), Dateisuche und [Bildgenerierung](/docs/features/image_gen) zu den meisten Basismodellen hinzu. Greifen Sie darauf zurück, wenn Sie Funktionen benötigen, die ein einfacher Chat-endpoint nicht bietet.
- **Vision** und **Tools / Function Calling** sind Eigenschaften des *Modells*, nicht des endpoint, daher ist unabhängig vom Anbieter ein fähiges Modell erforderlich.
- **Assistants** verwendet die Assistants-Laufzeitumgebung von OpenAI, die eine eigene Dateisuche und Code-Ausführung bietet, jedoch keine LibreChat-Funktionen wie Memory oder Artifacts integriert.

Dies ist eine manuell gepflegte Kurzübersicht. Falls ein Wert für Ihre Version nicht korrekt erscheint, [eröffnen Sie bitte ein Issue oder einen PR](https://github.com/LibreChat-AI/librechat.ai).
