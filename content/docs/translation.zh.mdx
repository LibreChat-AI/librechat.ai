感谢您有兴趣翻译 LibreChat！我们依靠社区贡献来让全球用户都能使用我们的应用程序。所有翻译均通过 [Locize](https://locize.com) 进行管理，这是一个与我们项目无缝集成的强大翻译管理系统。

## 翻译工作原理 [#how-translations-work]

- **集中化管理：**
LibreChat 的所有翻译字符串均在 Locize 上进行统一维护。这种集中化管理确保了整个应用程序的一致性，并简化了更新流程。

- **自动更新：**
在 Locize 中所做的更改会自动与我们的项目同步。您可以通过我们仓库中的动态徽章监控每种语言的翻译进度。

- **社区驱动：**
我们欢迎各种语言的贡献。您的帮助使 LibreChat 能够触达更广泛的受众，并为使用母语的用户提供支持。

## 处理翻译字符串中的 `{{0}}` 和 `{{1}}` [#handling-0-and-1-in-translation-strings]

有时翻译字符串需要包含动态内容。这些动态部分被称为**插值（interpolations）**，由括在双花括号中的占位符表示（例如 `{{0}}` 或 `{{1}}`）。在翻译此类字符串时，务必将这些占位符保持在正确的位置。

以下是两个示例，旨在为您提供指导：

### 示例 1：单项插值 [#example-1-single-interpolation]

考虑翻译键 `com_assistants_completed_action`。原始英文文本为：

```text
Talked to {{0}}
```

对于德语翻译，请确保占位符保持完整并放置在适当的位置：

```text
Mit {{0}} gesprochen
```

此图像展示了单个插值在翻译字符串中是如何表示的：

![Single Interpolation](https://github.com/user-attachments/assets/384ef7c1-9b02-490c-8ca4-b7f74943893f)

---

### 示例 2：多次插值 [#example-2-multiple-interpolations]

现在，查看翻译键 `com_files_number_selected`，它包含了两个占位符。英文版本为：

```text
{{0}} of {{1}} item(s) selected
```

在德语翻译中，必须保留这两个占位符并将其放置在正确的位置：

```text
{{0}} von {{1}} Datei(en) ausgewählt
```

此图像展示了多个插值在翻译字符串中是如何显示的：

![Multiple Interpolations](https://github.com/user-attachments/assets/f3376487-e092-442a-b849-b6ab5d5b390d)

## 添加新语言 [#adding-a-new-language]

如果您没有在我们的翻译表中看到您的语言，您可以帮助我们扩展我们的语言支持：

1. **创建新 Issue：**
在我们的 GitHub 仓库中开启一个新 Issue：[LibreChat Issues](https://github.com/danny-avila/LibreChat/issues)。

2. **选择新语言请求模板：**
使用 **New Language Request** 模板并提供：
 - 您语言的全称（例如：Spanish, Mandarin）。
 - 您语言的 [ISO 639-1](https://www.w3schools.com/tags/ref_language_codes.asp) 代码（例如，`es` 代表西班牙语）。

3. **与维护者协作：**
我们的维护者将审核您的请求，并与您合作集成新语言。一旦获得批准，您的语言将出现在翻译进度表中，您就可以开始贡献了。
