  ```tsx copy filename="carousel"
<Carousel autoplay animationDuration="1500" showBullets perView="3">
  <img src="slide1.jpg" alt="Slide 1" />
  <img src="slide2.jpg" alt="Slide 2" />
  <img src="slide3.jpg" alt="Slide 3" />
</Carousel>
  ```

Dieses Beispiel zeigt, wie die `Carousel`-Komponente mit den folgenden Props verwendet wird:

- `autoplay`: Aktiviert die automatische Wiedergabe für das Karussell.
- `animationDuration="1500"`: Legt die Animationsdauer auf 1500 Millisekunden fest.
- `showControls`: Zeigt die Zurück- und Weiter-Steuerelemente für das Karussell an.
- `showBullets`: Zeigt die Bullet-Navigation für das Karussell an.
- `perView="3"`: Zeigt drei Folien gleichzeitig im Karussell an.

Innerhalb der `Carousel` Komponente können Sie beliebige Inhalte platzieren, die als Folien angezeigt werden sollen. In diesem Beispiel verwenden wir `<img>` Elemente mit Quellpfaden zu `slide1.jpg`, `slide2.jpg` und `slide3.jpg`.

## Dateibaum [#file-tree]

    <FileTree.File name="main.js" active />
    <FileTree.Folder name="components">
      <FileTree.File name="Header.js" />
      <FileTree.File name="Footer.js" />
    </FileTree.Folder>
  </FileTree.Folder>
  <FileTree.File name="package.json" />
</FileTree>

</Tabs.Tab>

<Tabs.Tab>

- `<FileTree.Folder>`: Repräsentiert einen Ordner. Er kann verschachtelte Folder- oder File-Komponenten enthalten.
- `<FileTree.File>`: Repräsentiert eine Datei. Diese Komponente erfordert nur ein name-Prop, welches den in der UI angezeigten Dateinamen bestimmt.
- `name`: Dies bestimmt den Namen der Datei oder des Ordners, wie er in der UI angezeigt wird.
- `defaultOpen`: Dies zeigt den Anfangszustand eines Ordners an. Wenn dies auf true gesetzt ist, wird der Ordner beim ersten Rendern aufgeklappt.
- `open`: Wenn dieser Prop an einen Folder übergeben wird, bleibt er unabhängig von der Benutzerinteraktion immer geöffnet. Dies ist nützlich, wenn bestimmte Abschnitte des Dateibaums dauerhaft sichtbar bleiben sollen.
- `active`: Wenn diese Prop in einer File verwendet wird, wird der Dateiname hervorgehoben. Dies ist nützlich, wenn Sie Benutzern anzeigen möchten, welche Datei aktuell aktiv oder ausgewählt ist.

```tsx copy filename="file-tree.tsx"
<FileTree>
  <FileTree.Folder name="src" defaultOpen>
    <FileTree.File name="main.js" active />
    <FileTree.Folder name="components">
      <FileTree.File name="Header.js" />
      <FileTree.File name="Footer.js" />
    </FileTree.Folder>
  </FileTree.Folder>
  <FileTree.File name="package.json" />
</FileTree>
```

Dieses Beispiel zeigt die Verwendung der Komponenten `<Features>` und `<Feature>` zur Darstellung von Funktionsbereichen. Die Komponente `<Feature>` kann mit verschiedenen Props angepasst werden, wie z. B. `href` für Verlinkungen, `medium` oder `large` für die Größe, `lightOnly` für den reinen Hellmodus und `centered` für die Zentrierung des Inhalts.

Der bereitgestellte Code enthält eine Mischung aus Markdown-Inhalten und dem `<Features>`-Abschnitt. Innerhalb des `<Features>`-Abschnitts befinden sich vier `<Feature>`-Komponenten mit unterschiedlichen Konfigurationen, um verschiedene Anwendungsfälle zu demonstrieren.

```tsx copy filename="features"
import { Feature, Features } from '@/components/features'
;<Features>
  <Feature href="https://example.com">This is a feature with a link.</Feature>
  <Feature medium lightOnly>
    This is a medium "lightOnly" feature occupying two spaces.
  </Feature>
  <Feature large centered>
    This is a large feature with "centered" attribute.
  </Feature>
  <Feature large>![](/images/banner.png)</Feature>
</Features>
```

## Verschachtelte Tabs [#nested-tabs]
