O LibreChat conecta-se a muitos provedores de IA por meio de um pequeno conjunto de **endpoint types**.
Esta página resume quais recursos cada endpoint suporta. Muitos recursos
dependem do modelo específico que você seleciona, portanto, essas células estão marcadas como
dependentes do modelo, em vez de um sim ou não definitivo.

## Notas [#notes]

- **Custom (OpenAI-compatible)** abrange qualquer provedor configurado através de [custom endpoints](/docs/configuration/librechat_yaml/ai_endpoints), como OpenRouter, Groq, Mistral, Ollama, DeepSeek, Perplexity e outros. O suporte exato depende de quão fielmente o provedor espelha a API da OpenAI e do modelo que você escolher.
- **Agents** é o endpoint mais capaz: ele adiciona uso de ferramentas, [MCP](/docs/features/mcp), [Code Interpreter](/docs/features/code_interpreter), pesquisa de arquivos e [Image Generation](/docs/features/image_gen) sobre a maioria dos modelos base. Utilize-o quando precisar de capacidades que um endpoint de chat simples não oferece.
- **Vision** e **tools / function calling** são propriedades do *model*, não do endpoint, portanto, um modelo capaz é necessário independentemente do provedor.
- **Assistants** utiliza o runtime de Assistants da OpenAI, que fornece sua própria pesquisa de arquivos e execução de código, mas não adiciona recursos do LibreChat, como Memory ou Artifacts.

Esta é uma referência rápida mantida manualmente. Se um valor parecer incorreto para a sua versão, por favor, [abra uma issue ou PR](https://github.com/LibreChat-AI/librechat.ai).
