Docker Composeにおいて、オーバーライドファイルは、メインの `docker-compose.yml` で提供されるデフォルト設定を、ファイル全体を直接編集したり複製したりすることなく変更できる強力な機能です。オーバーライドファイルの主な用途はローカル開発環境のカスタマイズであり、`docker compose up` を実行すると、Docker Composeは `docker-compose.yml` と `docker-compose.override.yml` ファイルの設定をマージします。

`docker-compose.override.yml` の使用方法に関するクイックガイドは以下の通りです：

> 注: その他の例については、`docker-compose.override.yml.example` を参照してください。

- **[docker docs - understanding-multiple-compose-files](https://docs.docker.com/compose/multiple-compose-files/extends/#understanding-multiple-compose-files)**
- **[docker docs - merge-compose-files](https://docs.docker.com/compose/multiple-compose-files/merge/#merge-compose-files)**
- **[docker docs - specifying-multiple-compose-files](https://docs.docker.com/compose/reference/#specifying-multiple-compose-files)**

## ステップ 1: `docker-compose.override.yml` ファイルを作成する [#step-1-create-a-docker-composeoverrideyml-file]

`docker-compose.override.yml` ファイルをまだお持ちでない場合は、オーバーライドのサンプル内容をコピーして作成することができます。

```bash filename="Copying the example override file"
cp docker-compose.override.yml.example docker-compose.override.yml
```

このファイルは、docker-composeコマンドを実行する際にDocker Composeによって自動的に読み込まれます。

## Step 2: overrideファイルを編集する [#step-2-edit-the-override-file]

vscode または任意のテキストエディタで `docker-compose.override.yml` ファイルを開きます。

関連するセクションのコメントアウトを解除し、必要に応じてカスタマイズすることで、希望する変更を行ってください。

> 警告: 各サービス名（api、mongodb、meilisearchなど）は一度しか指定できません。1つのサービス内で複数の設定を上書きしたい場合は、それに応じて編集する必要があります。

### 例 [#examples]

Dockerが `librechat.yaml` ファイルを [Custom Endpoints & Configuration](/docs/configuration/librechat_yaml) に使用できるようにするには、以下のように設定します。

```yaml filename="docker-compose.override.yml"
services:
  api:
    volumes:
      - ./librechat.yaml:/app/librechat.yaml
```

あるいは、`api` サービス用にイメージをローカルでビルドし、LibreChat 設定ファイルを使用し、AVX サポートを必要としない古い Mongo を使用したい場合、`docker-compose.override.yml` は以下のようになるかもしれません：

```yaml filename="docker-compose.override.yml"
services:
  api:
    volumes:
      - ./librechat.yaml:/app/librechat.yaml
    image: librechat
    build:
      context: .
      target: node

  mongodb:
    image: mongo:4.4.18
```

> 注意: MongoDB や Meilisearch のポートを公開する場合、データが脆弱になる可能性があるため、十分注意してください。

## Step 3: 変更を適用する [#step-3-apply-the-changes]

設定の変更を適用するには、通常通り Docker Compose を実行するだけです。Docker Compose は `docker-compose.yml` と `docker-compose.override.yml` の両方のファイルを自動的に考慮します：

```bash filename="Apply the changes"
docker compose up -d
```

## ステップ 4: 変更の確認 [#step-4-verify-the-changes]

変更した設定でサービスを開始した後、`docker ps` コマンドを使用して実行中のコンテナとそのプロパティ（ポートなど）を一覧表示し、変更が適用されたことを確認できます。

## 重要な考慮事項 [#important-considerations]

- **優先順位**: オーバーライドファイルで定義された値は、元の `docker-compose.yml` ファイルで指定された値よりも優先されます。
- **セキュリティ**: ポートをカスタマイズしてサービスを公開する場合は、常にセキュリティへの影響を意識してください。本番環境や機密性の高い環境では、デフォルト設定の使用を避けてください。

これらの手順と考慮事項に従うことで、元の `docker-compose.yml` ファイルを変更することなく、Docker Compose 設定を簡単かつ安全に修正でき、さまざまな環境やローカルのカスタマイズの管理と保守が容易になります。


## `deploy-compose.yml` [#deploy-composeyml]

`deploy-compose.yml` のようなデフォルト以外の Docker Compose ファイルでオーバーライドファイルを使用するには、Docker Compose コマンドを実行する際に両方のファイルを明示的に指定する必要があります。

Docker Composeでは、複数の `-f` または `--file` オプションを指定して複数のcomposeファイルを含めることができ、後のファイルの設定が最初の設定を上書き、または追加します。

`deploy-compose.yml` をメインの Docker Compose 設定として使用しており、`docker-compose.override.yml` という名前のオーバーライドファイルがある場合（オーバーライドファイルの名前は任意ですが、この特定のファイルが既に存在している可能性があります）、Docker Compose コマンドは以下のように実行します。

```bash
docker compose -f deploy-compose.yml -f docker-compose.override.yml pull
docker compose -f deploy-compose.yml -f docker-compose.override.yml up
```
