- Consulta: [Función de búsqueda en LibreChat](/docs/features/search) para obtener una visión general rápida.

MeiliSearch es un potente motor de búsqueda de código abierto que mejora la funcionalidad de LibreChat al permitir la búsqueda de texto completo, tolerancia a errores tipográficos y resultados de búsqueda instantáneos para conversaciones pasadas.

Siga estos pasos para configurar MeiliSearch para LibreChat:

## 1. Descargar MeiliSearch [#1-download-meilisearch]
    - Ve a la página de releases de MeiliSearch en GitHub: https://github.com/meilisearch/meilisearch/releases
    - Descargue la última versión de MeiliSearch para su sistema operativo (por ejemplo, `meilisearch-linux-amd64.tar.gz` para Linux, `meilisearch-macos-amd64` para macOS, o `meilisearch-windows-amd64.zip` para Windows).

## 2. Extraer/Instalar MeiliSearch [#2-extractinstall-meilisearch]
    - Linux/macOS: Extraiga el archivo descargado en un directorio de su elección.
    - Windows: Extraiga el archivo ZIP en un directorio de su elección.

## 3. Hacer que el binario de MeiliSearch sea ejecutable (Linux/macOS) [#3-make-the-meilisearch-binary-executable-linuxmacos]
    - Abra una terminal y navegue al directorio donde extrajo MeiliSearch.
    - Ejecute el siguiente comando para hacer que el binario sea ejecutable:
        ```sh filename="Make the MeiliSearch Binary Executable"
        chmod +x meilisearch
        ```

## 4. Generar una Master Key [#4-generate-a-master-key]
    - Abra una terminal (o el Símbolo del sistema en Windows) y navegue al directorio de MeiliSearch.
    - Ejecuta el siguiente comando para generar una Master Key:
        ```sh filename="Generate a Master Key"
        ./meilisearch --generate-master-key
        ```
    - Copia la Master Key generada, ya que la necesitarás más adelante.

## 5. Iniciar MeiliSearch [#5-start-meilisearch]
    - En la misma terminal, ejecuta el siguiente comando para iniciar MeiliSearch, reemplazando `<your_master_key>` con la Master Key que generaste en el paso anterior:
        ```sh filename="Start MeiliSearch"
        ./meilisearch --master-key=<your_master_key>
        ```
    - MeiliSearch comenzará a ejecutarse ahora en el puerto predeterminado (`7700`).

## 6. Actualizar las variables de entorno de LibreChat [#6-update-librechats-environment-variables]
    - Abra el archivo `.env` en el directorio raíz de su proyecto LibreChat.
    - Agregue o actualice las siguientes líneas con su configuración de MeiliSearch:

        ```sh filename=".env"
        SEARCH=true
        MEILI_NO_ANALYTICS=true
        MEILI_HOST=http://localhost:7700
        MEILI_MASTER_KEY=<your_master_key>
        ```

    - Reemplace `<your_master_key>` con la Master Key que generó anteriormente.

## 7. Iniciar/Reiniciar LibreChat [#7-startrestart-librechat]
    - Inicie o reinicie su aplicación LibreChat.

¡Eso es todo! Con MeiliSearch configurado, LibreChat debería tener ahora habilitada la función de búsqueda de conversaciones, lo que permite a los usuarios realizar búsquedas de texto completo, beneficiarse de la tolerancia a errores tipográficos y experimentar resultados de búsqueda instantáneos para sus conversaciones pasadas.

Nota: Asegúrese de mantener el proceso de MeiliSearch ejecutándose en segundo plano para que la funcionalidad de búsqueda funcione correctamente. Es posible que desee configurar un script o servicio para mantener MeiliSearch ejecutándose de forma persistente.

## 8. Opcional: Deshabilitar la sincronización de Meilisearch/Base de datos en una configuración de múltiples nodos [#8-optional-disable-meilisearchdatabase-sync-in-a-multi-node-setup]

Si está ejecutando LibreChat en un clúster de nodos o en una configuración de múltiples nodos y desea deshabilitar la sincronización de indexación de MeiliSearch, puede establecer la variable de entorno `MEILI_NO_SYNC` en `true` en su archivo `.env`. Esto evitará que los documentos de su base de datos se sincronicen de forma redundante a través de múltiples instancias de LibreChat, lo que también puede generar un consumo innecesario de recursos, ya que solo una instancia debería ser responsable de la indexación.

        ```sh filename=".env"
        MEILI_NO_SYNC=true
        ```

## 9. Restablecimiento de la sincronización de MeiliSearch [#9-resetting-meilisearch-synchronization]

Si encuentra problemas donde los datos de MeiliSearch han sido eliminados o corrompidos, o si LibreChat cree que todos los datos han sido sincronizados cuando no es así (por ejemplo, después de actualizar MeiliSearch o eliminar sus archivos de datos), puede utilizar el script de reinicio de sincronización para forzar una re-sincronización completa.

Este script restablece las banderas de sincronización en MongoDB, lo que provocará que LibreChat vuelva a indexar todas las conversaciones y mensajes en MeiliSearch en el próximo inicio o verificación de sincronización.

### Ejecución del script de restablecimiento [#running-the-reset-script]

```bash filename="To reset MeiliSearch synchronization flags, run:"
# Local Development
npm run reset-meili-sync

# Docker (default setup)
docker compose exec api npm run reset-meili-sync

# Docker (deployment setup)
docker exec -it LibreChat-API /bin/sh -c "cd .. && npm run reset-meili-sync"
```

### Qué hace el script [#what-the-script-does]

1. Restablece el indicador `_meiliIndex` a `false` para todos los mensajes y conversaciones en MongoDB
2. Muestra cuántos documentos fueron restablecidos y el número total de documentos a sincronizar
3. Proporciona opciones avanzadas para controlar el comportamiento de sincronización

### Cuándo usar este script [#when-to-use-this-script]

- Después de eliminar los archivos de datos de MeiliSearch
- Al actualizar MeiliSearch a una nueva versión que requiere reindexación
- Si LibreChat muestra las conversaciones como "fully synced" pero a MeiliSearch le faltan datos
- Después de restaurar una copia de seguridad de MongoDB sin los datos correspondientes de MeiliSearch

### Opciones de sincronización avanzadas [#advanced-sync-options]

Después de ejecutar el script de reinicio, puedes controlar el comportamiento de sincronización con estas variables de entorno:

- `MEILI_SYNC_BATCH_SIZE`: Número de documentos a sincronizar por lote (predeterminado: 100)
- `MEILI_SYNC_DELAY_MS`: Retraso entre lotes de sincronización en milisegundos (predeterminado: 100ms)
- `MEILI_SYNC_THRESHOLD`: Número mínimo de documentos no sincronizados antes de activar una sincronización (predeterminado: 1000)

Nota: Después de restablecer las flags de sincronización, deberás reiniciar LibreChat para que comience la resincronización.
