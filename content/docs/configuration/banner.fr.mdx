Affichez des messages importants à tous vos utilisateurs en haut de l'application. Idéal pour les annonces, les avis de maintenance ou les mises à jour.

<Callout type="info" title="Aperçu rapide">
  - Seule **une bannière** peut être active à la fois
  - Planifiez l'apparition et la disparition automatique des bannières
  - Choisissez si les utilisateurs peuvent ignorer la bannière ou non
  - Afficher des bannières à tout le monde, ou uniquement aux utilisateurs connectés
</Callout>

---

## Création d'une bannière [#creating-a-banner]

Exécutez cette commande pour créer ou mettre à jour une bannière :

```bash
npm run update-banner
```

Vous serez guidé à travers quelques invites simples :

```ansi
--------------------------
Update the banner!
--------------------------
Display From (Format: yyyy-mm-ddTHH:MM:SSZ, Default: now):
> 2025-12-02T09:00:00Z

Display To (Format: yyyy-mm-ddTHH:MM:SSZ, Default: not specified):
> 2025-12-31T23:59:59Z

Enter your message (Enter a single dot "." on a new line to finish):
> 🎉 Welcome to LibreChat! Check out our new features.
> .

Is public (y/N):
> n

Is persistable (cannot be dismissed) (y/N):
> n
```

<Callout type="tip" title="Que signifient ces options ?">
  - **Afficher de/à** : Quand la bannière doit apparaître et disparaître. Laissez vide pour « maintenant » et « pour toujours »
  - **Is public** : Afficher aux visiteurs qui ne sont pas connectés (comme sur la page de connexion)
  - **Is persistable** : Si oui, les utilisateurs ne peuvent pas fermer la bannière — à utiliser pour les avis importants
</Callout>

---

## Suppression d'une bannière [#deleting-a-banner]

```bash
npm run delete-banner
```

Vous verrez la bannière actuelle et il vous sera demandé de confirmer avant la suppression.

## Exemples de bannières [#example-banners]

<Tabs items={['Message de bienvenue', 'Avis de maintenance', 'Alerte de sécurité']}>
  <Tabs.Tab>
    Un message de bienvenue simple que les utilisateurs peuvent ignorer :

    ```bash
    npm run update-banner "" "" "👋 Welcome to LibreChat!" "false" "false"
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    Un avis de maintenance programmée qui ne peut pas être ignoré :

    ```bash
    npm run update-banner "2025-12-20T00:00:00Z" "2025-12-21T06:00:00Z" "⚠️ Scheduled maintenance on Dec 20th, 2-6 AM UTC" "true" "true"
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    Un avis de sécurité urgent visible par tous :

    ```bash
    npm run update-banner "" "" "🔒 Please update your password by January 1st" "true" "true"
    ```
  </Tabs.Tab>
</Tabs>

## Format de date [#date-format]

Utilisez ce format pour les dates : `yyyy-mm-ddTHH:MM:SSZ`

**Exemples :**
- `2025-12-25T09:00:00Z` → 25 décembre 2025 à 9h00 (UTC)
- `2025-01-01T00:00:00Z` → 1er janvier 2025 à minuit (UTC)

Laissez la date vide pour utiliser l'heure actuelle ou aucune date de fin.
