- 詳細については、[LibreChatの検索機能](/docs/features/search)の概要をご覧ください。

MeiliSearchは、LibreChatの機能を強化する強力なオープンソースの検索エンジンであり、過去の会話に対する全文検索、あいまい検索（typo tolerance）、およびインスタント検索結果の表示を可能にします。

LibreChatでMeiliSearchをセットアップするには、以下の手順に従ってください：

## 1. MeiliSearchをダウンロードする [#1-download-meilisearch]
    - MeiliSearch GitHub リリースページに移動します: https://github.com/meilisearch/meilisearch/releases
    - お使いのオペレーティングシステムに適した最新バージョンのMeiliSearchをダウンロードしてください（例：Linuxの場合は `meilisearch-linux-amd64.tar.gz`、macOSの場合は `meilisearch-macos-amd64`、Windowsの場合は `meilisearch-windows-amd64.zip`）。

## 2. MeiliSearchの抽出/インストール [#2-extractinstall-meilisearch]
    - Linux/macOS: ダウンロードしたアーカイブを任意のディレクトリに展開します。
    - Windows: ZIPファイルを任意のディレクトリに展開します。

## 3. MeiliSearch バイナリを実行可能にする (Linux/macOS) [#3-make-the-meilisearch-binary-executable-linuxmacos]
    - ターミナルを開き、MeiliSearchを解凍したディレクトリに移動します。
    - 次のコマンドを実行して、バイナリを実行可能にしてください：
        ```sh filename="Make the MeiliSearch Binary Executable"
        chmod +x meilisearch
        ```

## 4. マスターキーの生成 [#4-generate-a-master-key]
    - ターミナル（Windowsの場合はコマンドプロンプト）を開き、MeiliSearchディレクトリに移動します。
    - 以下のコマンドを実行して、Master Keyを生成します：
        ```sh filename="Generate a Master Key"
        ./meilisearch --generate-master-key
        ```
    - 生成されたMaster Keyは後で使用するため、コピーしておいてください。

## 5. MeiliSearch を起動する [#5-start-meilisearch]
    - 同じターミナルで、以下のコマンドを実行してMeiliSearchを起動します。その際、`<your_master_key>`を前のステップで生成したマスターキーに置き換えてください。
        ```sh filename="Start MeiliSearch"
        ./meilisearch --master-key=<your_master_key>
        ```
    - MeiliSearchは、デフォルトのポート（`7700`）で実行を開始します。

## 6. LibreChatの環境変数を更新する [#6-update-librechats-environment-variables]
    - LibreChat プロジェクトのルートディレクトリにある `.env` ファイルを開きます。
    - MeiliSearchの設定で、以下の行を追加または更新してください：

        ```sh filename=".env"
        SEARCH=true
        MEILI_NO_ANALYTICS=true
        MEILI_HOST=http://localhost:7700
        MEILI_MASTER_KEY=<your_master_key>
        ```

    - `<your_master_key>` を、先ほど生成した Master Key に置き換えてください。

## 7. LibreChatの起動/再起動 [#7-startrestart-librechat]
    - LibreChat アプリケーションを起動または再起動します。

以上です！MeiliSearchのセットアップと設定が完了したことで、LibreChatで会話検索機能が有効になり、ユーザーは過去の会話に対して全文検索を実行したり、タイプミスの許容機能を利用したり、即座に検索結果を得たりできるようになります。

注: 検索機能が正しく動作するように、MeiliSearchプロセスをバックグラウンドで実行し続けてください。MeiliSearchを永続的に実行し続けるためのスクリプトやサービスを設定することをお勧めします。

## 8. オプション: マルチノード構成でMeilisearch/データベース同期を無効にする [#8-optional-disable-meilisearchdatabase-sync-in-a-multi-node-setup]

LibreChatをノードクラスターやマルチノード構成で実行しており、MeiliSearchのインデックス同期を無効にしたい場合は、`.env`ファイルで `MEILI_NO_SYNC` 環境変数を `true` に設定できます。これにより、データベースのドキュメントが複数のLibreChatインスタンス間で冗長に同期されるのを防ぐことができます。インデックス作成は1つのインスタンスのみが担当すべきであるため、この設定は不要なリソース消費を抑えることにもつながります。

        ```sh filename=".env"
        MEILI_NO_SYNC=true
        ```

## 9. MeiliSearch同期のリセット [#9-resetting-meilisearch-synchronization]

MeiliSearchのデータが削除または破損した場合、あるいはLibreChatが実際には同期されていないにもかかわらずすべてのデータが同期済みであると判断した場合（例えば、MeiliSearchのアップグレード後やデータファイルの削除後など）に問題が発生した場合は、同期リセットスクリプトを使用して強制的に完全な再同期を実行できます。

このスクリプトはMongoDB内の同期フラグをリセットします。これにより、次回の起動時または同期チェック時に、LibreChatはすべての会話とメッセージをMeiliSearchへ再インデックスするようになります。

### リセットスクリプトの実行 [#running-the-reset-script]

```bash filename="To reset MeiliSearch synchronization flags, run:"
# Local Development
npm run reset-meili-sync

# Docker (default setup)
docker compose exec api npm run reset-meili-sync

# Docker (deployment setup)
docker exec -it LibreChat-API /bin/sh -c "cd .. && npm run reset-meili-sync"
```

### スクリプトの動作 [#what-the-script-does]

1. MongoDB内のすべてのメッセージと会話の `_meiliIndex` フラグを `false` にリセットします。
2. リセットされたドキュメントの数と、同期されるドキュメントの総数を表示します
3. 同期の動作を制御するための高度なオプションを提供します

### このスクリプトを使用するタイミング [#when-to-use-this-script]

- MeiliSearchのデータファイルを削除した後
- MeiliSearchを再インデックスが必要な新しいバージョンにアップグレードする場合
- LibreChat が会話を「完全に同期済み (fully synced)」と表示しているにもかかわらず、MeiliSearch にデータが欠落している場合
- 対応するMeiliSearchデータなしでMongoDBのバックアップを復元した後

### 高度な同期オプション [#advanced-sync-options]

リセットスクリプトを実行した後、以下の環境変数を使用して同期の動作を制御できます：

- `MEILI_SYNC_BATCH_SIZE`: 1バッチあたりの同期ドキュメント数（デフォルト: 100）
- `MEILI_SYNC_DELAY_MS`: 同期バッチ間の遅延（ミリ秒単位）（デフォルト: 100ms）
- `MEILI_SYNC_THRESHOLD`: 同期をトリガーするまでに必要な未同期ドキュメントの最小数（デフォルト: 1000）

注: 同期フラグをリセットした後、再同期を開始するには LibreChat を再起動する必要があります。
