- Siehe: [Suchfunktion in LibreChat](/docs/features/search) für einen kurzen Überblick.

MeiliSearch ist eine leistungsstarke Open-Source-Suchmaschine, die die Funktionalität von LibreChat durch Volltextsuche, Fehlertoleranz und sofortige Suchergebnisse für vergangene Konversationen erweitert.

Befolgen Sie diese Schritte, um MeiliSearch für LibreChat einzurichten:

## 1. MeiliSearch herunterladen [#1-download-meilisearch]
    - Gehen Sie zur MeiliSearch GitHub Releases-Seite: https://github.com/meilisearch/meilisearch/releases
    - Laden Sie die neueste Version von MeiliSearch für Ihr Betriebssystem herunter (z. B. `meilisearch-linux-amd64.tar.gz` für Linux, `meilisearch-macos-amd64` für macOS oder `meilisearch-windows-amd64.zip` für Windows).

## 2. MeiliSearch extrahieren/installieren [#2-extractinstall-meilisearch]
    - Linux/macOS: Entpacken Sie das heruntergeladene Archiv in ein Verzeichnis Ihrer Wahl.
    - Windows: Entpacken Sie die ZIP-Datei in ein Verzeichnis Ihrer Wahl.

## 3. Die MeiliSearch-Binärdatei ausführbar machen (Linux/macOS) [#3-make-the-meilisearch-binary-executable-linuxmacos]
    - Öffnen Sie ein Terminal und navigieren Sie zu dem Verzeichnis, in das Sie MeiliSearch extrahiert haben.
    - Führen Sie den folgenden Befehl aus, um die Binärdatei ausführbar zu machen:
        ```sh filename="Make the MeiliSearch Binary Executable"
        chmod +x meilisearch
        ```

## 4. Einen Master Key generieren [#4-generate-a-master-key]
    - Öffnen Sie ein Terminal (oder die Eingabeaufforderung unter Windows) und navigieren Sie zum MeiliSearch-Verzeichnis.
    - Führen Sie den folgenden Befehl aus, um einen Master Key zu generieren:
        ```sh filename="Generate a Master Key"
        ./meilisearch --generate-master-key
        ```
    - Kopieren Sie den generierten Master Key, da Sie ihn später benötigen werden.

## 5. MeiliSearch starten [#5-start-meilisearch]
    - Führen Sie im selben Terminal den folgenden Befehl aus, um MeiliSearch zu starten, und ersetzen Sie dabei `<your_master_key>` durch den Master Key, den Sie im vorherigen Schritt generiert haben:
        ```sh filename="Start MeiliSearch"
        ./meilisearch --master-key=<your_master_key>
        ```
    - MeiliSearch wird nun auf dem Standard-Port (`7700`) ausgeführt.

## 6. Aktualisieren der Umgebungsvariablen von LibreChat [#6-update-librechats-environment-variables]
    - Öffnen Sie die `.env` Datei im Stammverzeichnis Ihres LibreChat Projekts.
    - Fügen Sie die folgenden Zeilen mit Ihrer MeiliSearch-Konfiguration hinzu oder aktualisieren Sie diese:

        ```sh filename=".env"
        SEARCH=true
        MEILI_NO_ANALYTICS=true
        MEILI_HOST=http://localhost:7700
        MEILI_MASTER_KEY=<your_master_key>
        ```

    - Ersetzen Sie `<your_master_key>` durch den Master Key, den Sie zuvor generiert haben.

## 7. LibreChat starten/neu starten [#7-startrestart-librechat]
    - Starten oder starten Sie Ihre LibreChat-Anwendung neu.

Das war's! Mit der Einrichtung und Konfiguration von MeiliSearch sollte LibreChat nun über die Funktion zur Konversationssuche verfügen. Dies ermöglicht es Benutzern, Volltextsuchen durchzuführen, von Fehlertoleranz zu profitieren und sofortige Suchergebnisse für ihre vergangenen Konversationen zu erhalten.

Hinweis: Stellen Sie sicher, dass der MeiliSearch-Prozess im Hintergrund läuft, damit die Suchfunktion ordnungsgemäß funktioniert. Sie sollten möglicherweise ein Skript oder einen Dienst einrichten, um MeiliSearch dauerhaft im Hintergrund auszuführen.

## 8. Optional: Deaktivieren der Meilisearch-/Datenbank-Synchronisierung in einem Multi-Node-Setup [#8-optional-disable-meilisearchdatabase-sync-in-a-multi-node-setup]

Wenn Sie LibreChat in einem Node-Cluster oder einer Multi-Node-Umgebung betreiben und die MeiliSearch-Index-Synchronisierung deaktivieren möchten, können Sie die Umgebungsvariable `MEILI_NO_SYNC` in Ihrer `.env`-Datei auf `true` setzen. Dies verhindert, dass Ihre Datenbankdokumente redundant über mehrere LibreChat-Instanzen hinweg synchronisiert werden, was auch zu unnötigem Ressourcenverbrauch führen kann, da nur eine Instanz für die Indizierung verantwortlich sein sollte.

        ```sh filename=".env"
        MEILI_NO_SYNC=true
        ```

## 9. Zurücksetzen der MeiliSearch-Synchronisierung [#9-resetting-meilisearch-synchronization]

Wenn Sie auf Probleme stoßen, bei denen MeiliSearch-Daten gelöscht oder beschädigt wurden, oder wenn LibreChat fälschlicherweise annimmt, dass alle Daten synchronisiert wurden, obwohl dies nicht der Fall ist (zum Beispiel nach einem Upgrade von MeiliSearch oder dem Löschen seiner Datendateien), können Sie das Reset-Sync-Skript verwenden, um eine vollständige erneute Synchronisierung zu erzwingen.

Dieses Skript setzt die Synchronisations-Flags in MongoDB zurück, was LibreChat dazu veranlasst, beim nächsten Start oder Synchronisations-Check alle Konversationen und Nachrichten in MeiliSearch neu zu indizieren.

### Ausführen des Reset-Skripts [#running-the-reset-script]

```bash filename="To reset MeiliSearch synchronization flags, run:"
# Local Development
npm run reset-meili-sync

# Docker (default setup)
docker compose exec api npm run reset-meili-sync

# Docker (deployment setup)
docker exec -it LibreChat-API /bin/sh -c "cd .. && npm run reset-meili-sync"
```

### Was das Skript bewirkt [#what-the-script-does]

1. Setzt das `_meiliIndex`-Flag für alle Nachrichten und Konversationen in MongoDB auf `false` zurück
2. Zeigt Ihnen, wie viele Dokumente zurückgesetzt wurden und die Gesamtzahl der zu synchronisierenden Dokumente
3. Bietet erweiterte Optionen zur Steuerung des Synchronisierungsverhaltens

### Wann dieses Skript zu verwenden ist [#when-to-use-this-script]

- Nach dem Löschen der MeiliSearch-Datendateien
- Beim Upgrade von MeiliSearch auf eine neue Version, die eine Neuindizierung erfordert
- Falls LibreChat Unterhaltungen als "fully synced" anzeigt, aber Daten in MeiliSearch fehlen
- Nach dem Wiederherstellen eines MongoDB-Backups ohne die entsprechenden MeiliSearch-Daten

### Erweiterte Synchronisierungsoptionen [#advanced-sync-options]

Nachdem Sie das Reset-Skript ausgeführt haben, können Sie das Synchronisierungsverhalten mit diesen Umgebungsvariablen steuern:

- `MEILI_SYNC_BATCH_SIZE`: Anzahl der Dokumente, die pro Batch synchronisiert werden (Standard: 100)
- `MEILI_SYNC_DELAY_MS`: Verzögerung zwischen Synchronisierungs-Batches in Millisekunden (Standard: 100ms)
- `MEILI_SYNC_THRESHOLD`: Mindestanzahl an nicht synchronisierten Dokumenten, bevor eine Synchronisierung ausgelöst wird (Standard: 1000)

Hinweis: Nachdem Sie die Synchronisierungs-Flags zurückgesetzt haben, müssen Sie LibreChat neu starten, damit die erneute Synchronisierung beginnt.
