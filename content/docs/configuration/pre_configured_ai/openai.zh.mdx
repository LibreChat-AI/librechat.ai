要获取您的 OpenAI API key，您需要：

- 前往 **[https://platform.openai.com/account/api-keys](https://platform.openai.com/account/api-keys)**
- 创建账户或使用现有账户登录
- 为您的账户添加付款方式
- 前往 https://platform.openai.com/api-keys 获取密钥。
- 您需要将以下环境变量设置为您的密钥，或者将其设置为 `user_provided` 以便用户提供他们自己的密钥。

```bash filename=".env"
OPENAI_API_KEY=user_provided
```

- 您可以使用 `OPENAI_MODELS` 来确定您希望可用的模型。
  - 当 `OPENAI_API_KEY` 设置为 `user_provided` 时 → 只有放入此列表中的模型才可用
    - ⚠️新模型不会自动显示；您需要先将它们添加到此列表中
  - 当 `OPENAI_API_KEY` 设置为实际的 API 密钥值时 → 只要 `OPENAI_MODELS` 保持注释状态，它就会进行 API 调用以查明哪些模型可用，这应该会包含任何新模型。

```bash filename=".env"
OPENAI_MODELS=gpt-5,gpt-5-codex,gpt-5-mini,gpt-5-nano,o3-pro,o3,o4-mini,gpt-4.1,gpt-4.1-mini,gpt-4.1-nano,o3-mini,o1-pro,o1,gpt-4o,gpt-4o-mini
```

**注意：**

- 内部 OpenAI 默认值包含当前的尖端模型目录，包括 `gpt-5.5`、`gpt-5.5-pro`、`chat-latest`、`gpt-5.4`、`gpt-5.4-pro`、`gpt-5.4-mini`、`gpt-5.4-nano`、`gpt-5.3-codex` 和 `gpt-5.2`。已弃用的旧版默认值不再包含在内，除非您通过 `OPENAI_MODELS` 显式添加它们。
- 当消息包含附件时，无需手动选择视觉模型，系统会在后台为您自动切换。如果您没有直接选择视觉模型，它将仅用于视觉请求；在请求成功后，您仍会看到您之前选择的非视觉模型。
- OpenAI Vision 模型允许在没有附件的情况下发送消息
- OpenAI endpoint 支持通过 `librechat.yaml` 配置文件使用所有 [Shared Endpoint Settings](/docs/configuration/librechat_yaml/object_structure/shared_endpoint_settings)，包括 `streamRate`、`headers`、`titleModel`、`titleMethod`、`titlePrompt`、`titlePromptTemplate` 和 `titleEndpoint`。
