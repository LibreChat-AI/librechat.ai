Aby uzyskać swój klucz API OpenAI, musisz:

- Przejdź do **[https://platform.openai.com/account/api-keys](https://platform.openai.com/account/api-keys)**
- Utwórz konto lub zaloguj się za pomocą istniejącego
- Dodaj metodę płatności do swojego konta
- Przejdź do https://platform.openai.com/api-keys, aby uzyskać klucz.
- Będziesz musiał ustawić poniższą zmienną środowiskową na swój klucz lub możesz ustawić ją na `user_provided`, aby użytkownicy mogli podać własny.

```bash filename=".env"
OPENAI_API_KEY=user_provided
```

- Możesz określić, które modele mają być dostępne, używając `OPENAI_MODELS`
  - Gdy `OPENAI_API_KEY` jest ustawione na `user_provided` → dostępne będą tylko modele umieszczone na tej liście
    - ⚠️Nowe modele nie pojawią się automatycznie; musisz najpierw dodać je do tej listy
  - Gdy `OPENAI_API_KEY` jest ustawiony na rzeczywistą wartość klucza API → dopóki `OPENAI_MODELS` pozostaje zakomentowane, zostanie wykonane wywołanie API w celu sprawdzenia, jakie modele są dostępne, co powinno obejmować wszystkie nowe modele.

```bash filename=".env"
OPENAI_MODELS=gpt-5,gpt-5-codex,gpt-5-mini,gpt-5-nano,o3-pro,o3,o4-mini,gpt-4.1,gpt-4.1-mini,gpt-4.1-nano,o3-mini,o1-pro,o1,gpt-4o,gpt-4o-mini
```

**Uwagi:**

- Wewnętrzne wartości domyślne OpenAI obejmują bieżący katalog frontier, w tym `gpt-5.5`, `gpt-5.5-pro`, `chat-latest`, `gpt-5.4`, `gpt-5.4-pro`, `gpt-5.4-mini`, `gpt-5.4-nano`, `gpt-5.3-codex` oraz `gpt-5.2`. Wycofane starsze wartości domyślne nie są już uwzględniane, chyba że dodasz je jawnie za pomocą `OPENAI_MODELS`.
- Wybór modelu wizyjnego dla wiadomości z załącznikami nie jest konieczny, ponieważ zostanie on przełączony automatycznie w tle. Jeśli nie wybrano modelu wizyjnego bezpośrednio, zostanie on użyty tylko do żądania wizyjnego, a po pomyślnym zakończeniu żądania nadal powinieneś widzieć wybrany wcześniej model niewizyjny.
- Modele OpenAI Vision pozwalają na przesyłanie wiadomości bez załączników
- Endpoint OpenAI obsługuje wszystkie [Shared Endpoint Settings](/docs/configuration/librechat_yaml/object_structure/shared_endpoint_settings) za pośrednictwem pliku konfiguracyjnego `librechat.yaml`, w tym `streamRate`, `headers`, `titleModel`, `titleMethod`, `titlePrompt`, `titlePromptTemplate` oraz `titleEndpoint`
