- Utwórz konto na **[https://platform.claude.com/](https://platform.claude.com/)**
- Przejdź do **[https://platform.claude.com/settings/keys](https://platform.claude.com/settings/keys)** i pobierz swój klucz API
- Będziesz musiał ustawić poniższą zmienną środowiskową na swój klucz lub ustawić ją na `user_provided`, aby użytkownicy mogli podać własny.

```bash filename=".env"
ANTHROPIC_API_KEY=user_provided
```

- Możesz określić, które modele mają być dostępne, używając `ANTHROPIC_MODELS`.

```bash filename=".env"
ANTHROPIC_MODELS=claude-fable-5,claude-opus-4-8,claude-opus-4-7,claude-sonnet-4-6,claude-opus-4-6,claude-opus-4-20250514,claude-3-7-sonnet-20250219,claude-3-5-sonnet-20241022,claude-3-5-haiku-20241022
```

**Uwagi:**

- Claude Fable 5 jest zawarty w domyślnym katalogu modeli Anthropic w LibreChat. Modele klasy Fable/Mythos korzystają z nowoczesnego profilu Anthropic w LibreChat: 1 mln kontekstu, 128 tys. maksymalnej długości wyjściowej, adaptacyjne myślenie (adaptive thinking), buforowanie promptów (prompt caching) oraz obsługa `thinkingDisplay` dla podsumowanych lub pominiętych wyników rozumowania.
- Endpoint Anthropic obsługuje wszystkie [Shared Endpoint Settings](/docs/configuration/librechat_yaml/object_structure/shared_endpoint_settings) za pośrednictwem pliku konfiguracyjnego `librechat.yaml`, w tym `streamRate`, `headers`, `titleModel`, `titleMethod`, `titlePrompt`, `titlePromptTemplate` oraz `titleEndpoint`
- Aby skonfigurować Anthropic lub bramę kompatybilną z Anthropic jako oddzielny niestandardowy endpoint, użyj [`provider: "anthropic"`](/docs/configuration/librechat_yaml/object_structure/custom_endpoint#provider).

## Vertex AI [#vertex-ai]

LibreChat obsługuje uruchamianie modeli Anthropic Claude za pośrednictwem **Google Cloud Vertex AI**. Jest to przydatne, jeśli:

- Masz już infrastrukturę Google Cloud
- Chcesz użyć swojego istniejącego rozliczenia i danych uwierzytelniających GCP
- Konieczność przestrzegania regionalnych wymogów dotyczących przechowywania danych

### Szybka konfiguracja (zmienne środowiskowe) [#quick-setup-environment-variables]

Ustaw następujące zmienne środowiskowe, aby włączyć Anthropic przez Vertex AI:

```bash filename=".env"
# Enable Vertex AI mode for Anthropic
ANTHROPIC_USE_VERTEX=true

# Vertex AI region (optional, defaults to 'us-east5')
# Available regions: global, us-east5, us-central1, europe-west1, europe-west4, asia-southeast1
ANTHROPIC_VERTEX_REGION=global

# Path to Google service account key file (optional)
# If not specified, uses default path: api/data/auth.json
GOOGLE_SERVICE_KEY_FILE=/path/to/service-account.json
```

### Wymagania wstępne [#prerequisites]

1. **Projekt Google Cloud** z włączonym interfejsem Vertex AI API
2. **Konto serwisowe** z następującymi rolami:
   - `Vertex AI User` (`roles/aiplatform.user`)
   - Lub `Vertex AI Administrator` dla pełnego dostępu
3. Modele **Claude** włączone w Twoim Vertex AI Model Garden
4. **Klucz konta serwisowego** (plik JSON) pobrany i dostępny dla LibreChat

### Zaawansowana konfiguracja [#advanced-configuration]

Aby uzyskać mapowanie nazw modeli i ustawienia zaawansowane (podobnie jak w konfiguracji Azure OpenAI), użyj pliku `librechat.yaml`:

```yaml filename="librechat.yaml"
endpoints:
  anthropic:
    vertex:
      region: us-east5
      models:
        claude-opus-4.5:
          deploymentName: claude-opus-4-5@20251101
        claude-sonnet-4:
          deploymentName: claude-sonnet-4-20250514
        claude-3.5-haiku:
          deploymentName: claude-3-5-haiku@20241022
```

Aby uzyskać szczegółowe opcje konfiguracji YAML, zobacz [Konfiguracja Anthropic Vertex AI](/docs/configuration/librechat_yaml/object_structure/anthropic_vertex).
