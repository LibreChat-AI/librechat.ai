OpenAI APIキーを取得するには、以下の手順が必要です：

- **[https://platform.openai.com/account/api-keys](https://platform.openai.com/account/api-keys)** にアクセスします。
- アカウントを作成するか、既存のアカウントでログインしてください
- アカウントに支払い方法を追加する
- https://platform.openai.com/api-keys にアクセスしてキーを取得してください。
- 以下の環境変数をキーに設定する必要があります。または、ユーザー自身がキーを入力できるように `user_provided` に設定することも可能です。

```bash filename=".env"
OPENAI_API_KEY=user_provided
```

- `OPENAI_MODELS` を使用して、利用可能にするモデルを決定できます。
  - `OPENAI_API_KEY` が `user_provided` に設定されている場合 → このリストに含まれるモデルのみが利用可能になります
    - ⚠️新しいモデルは自動的には表示されません。最初にこのリストに追加する必要があります。
  - `OPENAI_API_KEY` に実際の API キーの値を設定した場合 → `OPENAI_MODELS` をコメントアウトしたままにしておくと、API 呼び出しが行われて利用可能なモデルが取得されます。これには新しいモデルも含まれるはずです。

```bash filename=".env"
OPENAI_MODELS=gpt-5,gpt-5-codex,gpt-5-mini,gpt-5-nano,o3-pro,o3,o4-mini,gpt-4.1,gpt-4.1-mini,gpt-4.1-nano,o3-mini,o1-pro,o1,gpt-4o,gpt-4o-mini
```

**注記:**

- 内部のOpenAIデフォルトには、`gpt-5.5`、`gpt-5.5-pro`、`chat-latest`、`gpt-5.4`、`gpt-5.4-pro`、`gpt-5.4-mini`、`gpt-5.4-nano`、`gpt-5.3-codex`、および`gpt-5.2`を含む現在のフロンティアカタログが含まれています。非推奨となったレガシーデフォルトは、`OPENAI_MODELS`で明示的に追加しない限り含まれなくなりました。
- 添付ファイル付きのメッセージに対してビジョンモデルを選択する必要はありません。バックグラウンドで自動的に切り替えられるためです。ビジョンモデルを明示的に選択しなかった場合、そのモデルはビジョンリクエストにのみ使用され、リクエストが成功した後は、選択していた非ビジョンモデルが引き続き表示されます。
- OpenAI Visionモデルは、添付ファイルなしのメッセージを許可します
- OpenAI endpointは、`librechat.yaml`設定ファイルを通じて、`streamRate`、`headers`、`titleModel`、`titleMethod`、`titlePrompt`、`titlePromptTemplate`、`titleEndpoint`を含む、すべての[Shared Endpoint Settings](/docs/configuration/librechat_yaml/object_structure/shared_endpoint_settings)をサポートしています。
