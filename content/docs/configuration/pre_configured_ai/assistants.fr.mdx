- L'[Assistants API by OpenAI](https://platform.openai.com/docs/assistants/overview) dispose d'un endpoint dédié.
- L'Assistants API permet la création d'assistants IA, offrant des fonctionnalités telles que l'interprète de code, la récupération de connaissances à partir de fichiers et l'exécution de fonctions.
    - [Lisez ici pour une documentation approfondie](https://platform.openai.com/docs/assistants/overview) sur la fonctionnalité, son fonctionnement et ses capacités.
- Comme pour l'[API OpenAI](/docs/configuration/pre_configured_ai/openai) standard, rendez-vous sur **[https://platform.openai.com/account/api-keys](https://platform.openai.com/account/api-keys)** pour obtenir une clé.
- Vous devrez définir la variable d'environnement suivante avec votre clé, ou vous pouvez la définir sur `user_provided` pour permettre aux utilisateurs de fournir la leur.

```bash filename=".env"
ASSISTANTS_API_KEY=your-key
```

- Vous pouvez déterminer les modèles que vous souhaitez rendre disponibles avec `ASSISTANTS_MODELS` ; sinon, la liste des modèles récupérée depuis OpenAI sera utilisée (seuls les modèles compatibles avec l'Assistants API seront affichés).

```bash filename=".env"
ASSISTANTS_MODELS=gpt-3.5-turbo-0125,gpt-3.5-turbo-16k-0613,gpt-3.5-turbo-16k,gpt-3.5-turbo,gpt-4,gpt-4-0314,gpt-4-32k-0314,gpt-4-0613,gpt-3.5-turbo-0613,gpt-3.5-turbo-1106,gpt-4-0125-preview,gpt-4-turbo-preview,gpt-4-1106-preview
```

- Si nécessaire, vous pouvez également définir une URL de base alternative au lieu de l'officielle avec `ASSISTANTS_BASE_URL`, ce qui est similaire à l'équivalent OpenAI `OPENAI_REVERSE_PROXY`

```bash filename=".env"
ASSISTANTS_BASE_URL=http://your-alt-baseURL:3080/
```

- Il existe une configuration supplémentaire et facultative, selon vos besoins, telle que la désactivation de l'interface utilisateur du générateur d'assistants, disponible via le fichier de configuration personnalisé `librechat.yaml` [custom config file](/docs/configuration/librechat_yaml/object_structure/assistants_endpoint) :
    - Contrôlez la visibilité et l'utilisation de l'interface de création pour les assistants. [Plus d'informations](/docs/configuration/librechat_yaml/object_structure/assistants_endpoint#disablebuilder)
    - Spécifiez l'intervalle d'interrogation en millisecondes pour vérifier les mises à jour des exécutions ou les changements dans les états d'exécution de l'assistant. [Plus d'infos](/docs/configuration/librechat_yaml/object_structure/assistants_endpoint#pollintervalms)
    - Définissez la période de délai d'attente en millisecondes pour les exécutions d'assistant. Aide à gérer la charge du système en limitant la durée totale de l'opération d'exécution. [Plus d'informations](/docs/configuration/librechat_yaml/object_structure/assistants_endpoint#timeoutms)
    - Spécifiez quels IDs d'assistant sont pris en charge ou exclus [Plus d'infos](/docs/configuration/librechat_yaml/object_structure/assistants_endpoint#supportedids)

## Appel de fonction strict [#strict-function-calling]
With librechat you can add add the 'x-strict': true flag at operation-level in the openapi spec for actions.
This will automatically generate function calls with 'strict' mode enabled.
Note that strict mode supports only a partial subset of json. Read https://platform.openai.com/docs/guides/structured-outputs for details.

Par exemple :
```json filename="mathapi.json"
{
  "openapi": "3.1.0",
  "info": {
    "title": "Math.js API",
    "description": "API for performing mathematical operations, such as addition, subtraction, etc.",
    "version": "1.0.0"
  },
  "servers": [
    {
      "url": "https://api.mathjs.org/v4"
    }
  ],
  "paths": {
    "/": {
      "post": {
        "summary": "Evaluate a mathematical expression",
        "description": "Sends a mathematical expression in the request body to evaluate.",
"operationId": "math",
"x-strict": true,
"parameters": [
        "requestBody": {
          "required": true,
          "content": {
            "application/json": {
              "schema": {
                "type": "object",
                "properties": {
                  "expr": {
                    "type": "string",
                    "description": "The mathematical expression to evaluate (e.g., `2+3`)."
                  }
                },
                "required": ["expr"]
              }
            }
          }
        },
        "responses": {
          "200": {
            "description": "The result of the evaluated expression.",
            "content": {
              "application/json": {
                "schema": {
                  "type": "object",
                  "properties": {
                    "result": {
                      "type": "number",
                      "description": "The evaluated result of the expression."
                    }
                  }
                }
              }
            }
          },
          "400": {
            "description": "Invalid expression provided.",
            "content": {
              "application/json": {
                "schema": {
                  "type": "object",
                  "properties": {
                    "error": {
                      "type": "string",
                      "description": "Error message describing the invalid expression."
                    }
                  }
                }
              }
            }
          }
        }
      }
    }
  }
}
```

<Callout type="note" title="Notes">
**Notes :**
- Au moment de la rédaction, seuls les modèles suivants prennent en charge la fonctionnalité [Retrieval](https://platform.openai.com/docs/assistants/tools/knowledge-retrieval) :
    - gpt-3.5-turbo-0125
    - gpt-4-0125-preview
    - gpt-4-turbo-preview
    - gpt-4-1106-preview
    - gpt-3.5-turbo-1106
- La fonctionnalité Vision n'est pas encore prise en charge.
- If you have previously set the [`ENDPOINTS` value in your .env file](./dotenv.md#endpoints), you will need to add the value `assistants`
</Callout>
