Actualmente, esta función está disponible a través de [Agents](/docs/features/agents), así como a través de Custom Endpoints, OpenAI, Azure OpenAI, Anthropic y Google.

Los Assistants de OpenAI tienen su propia implementación de RAG a través de la capacidad "Retrieval". Obtén más información al respecto [aquí.](https://platform.openai.com/docs/assistants/tools/knowledge-retrieval)

Aún será útil implementar el uso de la API de RAG con la Assistants API, ya que OpenAI cobra tanto por el almacenamiento de archivos como por el uso de "Retrieval", y esto se introducirá en una futura actualización.

## Configuración [#setup]

Para configurar la RAG API con LibreChat, sigue estos pasos:

### Configuración de Docker - Inicio rápido [#docker-setup---quick-start]

#### Configuración predeterminada [#default-configuration]

**Usar RAG con OpenAI Embedding (predeterminado)**

1. Agregue lo siguiente a su archivo `.env`:

    ```sh
    RAG_API_URL=http://host.docker.internal:8000
    ```

2. Si tu clave de API de OpenAI está configurada como "user_provided", añade también esto a tu archivo `.env` para proporcionar una clave de API de OpenAI:
    - Nota: Puedes omitir este paso si ya estás proporcionando la clave de API de OpenAI en el archivo .env
    ```sh
    RAG_OPENAI_API_KEY=sk-your-openai-api-key-example
    ```

3. Ejecuta el comando para iniciar los contenedores de Docker:

    ```sh
    docker compose up -d
    ```

¡Eso es todo!

---

#### Configuración personalizada - Hugging Face [#custom-configuration---hugging-face]

**Usar RAG con Hugging Face Embedding**

1. Agregue lo siguiente a su archivo `.env`:

    ```sh
    RAG_API_URL=http://host.docker.internal:8000
    EMBEDDINGS_PROVIDER=huggingface
    HF_TOKEN=hf_xxxxxxxxxxxxxxxxxxxxxxx
    ```

2. Actualice su archivo `docker-compose.override.yml` con:

    ```yaml
    version: '3.4'

    services:
      rag_api:
        image: registry.librechat.ai/danny-avila/librechat-rag-api-dev:latest
    ```

3. Ejecuta el comando para iniciar los contenedores de Docker:

    ```sh
    docker compose up -d
    ```

¡Eso es todo!

---

#### Configuración personalizada - Ollama [#custom-configuration---ollama]

**Usar RAG con Ollama Local Embedding**

**Prerrequisito:** Necesitas Ollama y el modelo de embedding `nomic-embed-text`:

- `ollama pull nomic-embed-text`

1. Agregue lo siguiente a su archivo `.env`:

    ```sh
    RAG_API_URL=http://host.docker.internal:8000
    EMBEDDINGS_PROVIDER=ollama
    OLLAMA_BASE_URL=http://host.docker.internal:11434
    EMBEDDINGS_MODEL=nomic-embed-text
    ```

2. Actualice su archivo `docker-compose.override.yml` con:

    ```yaml
    version: '3.4'

    services:
      rag_api:
        image: registry.librechat.ai/danny-avila/librechat-rag-api-dev:latest
        # If running on Linux
        # extra_hosts:
        #   - "host.docker.internal:host-gateway"
    ```

3. Ejecuta el comando para iniciar los contenedores de Docker:

    ```sh
    docker compose up -d
    ```

¡Eso es todo!

### Configuración de Docker [#docker-setup]

Para Docker, la configuración está preparada para ti tanto en el archivo `docker-compose.yml` predeterminado como en el `deploy-compose.yml`, y solo necesitarás asegurarte de estar utilizando la versión más reciente de la imagen de Docker y de los archivos compose. Asegúrate de leer la [guía de actualización de LibreChat para Docker](/docs/local/docker#update-librechat) si no estás seguro de cómo actualizar tu instancia de Docker.

Docker utiliza la imagen "lite" de la RAG API de forma predeterminada, la cual solo admite embeddings remotos, aprovechando los procesos de embeddings de OpenAI o de un servicio remoto que hayas configurado para HuggingFace/Ollama.

Los embeddings locales son compatibles cambiando la imagen utilizada por el archivo compose predeterminado, de `registry.librechat.ai/danny-avila/librechat-rag-api-dev-lite:latest` a `registry.librechat.ai/danny-avila/librechat-rag-api-dev:latest`.

Los archivos compose predeterminados almacenan los datos de PGVector/PostgreSQL en el volumen `pgdata2` gestionado por Docker. Esto es intencional: los archivos de la base de datos no necesitan ser editados directamente desde el host, y un volumen gestionado evita problemas comunes de propiedad y permisos. Los archivos orientados al usuario y la configuración editable, tales como cargas, registros, imágenes, datos de MongoDB y la configuración de NGINX, están montados mediante bind-mount en carpetas del proyecto donde el acceso directo desde el host es útil.

Como siempre, realiza estos cambios en tus URLs de actualización en el [Docker Compose Override File](/docs/configuration/docker_override). Puedes encontrar un ejemplo de cómo cambiar exactamente la imagen en `docker-compose.override.yml.example` en la raíz del proyecto.

Si desea ver un ejemplo de un archivo compose que solo incluya la base de datos PostgresQL + PGVector y la API de Python, consulte el archivo `rag.yml` en la raíz del proyecto.

**Importante:** Al utilizar la configuración predeterminada de Docker, el archivo .env, donde se pueden establecer las opciones de configuración para la API de RAG, se comparte entre LibreChat y la API de RAG.

### Configuración local [#local-setup]

La configuración local sin contenedores requiere más intervención manual, y para esto puede consultar el [RAG API repo.](https://github.com/danny-avila/rag_api/)

En una configuración local, deberá establecer manualmente la `RAG_API_URL` en su archivo `.env` de LibreChat a la ubicación donde esté disponible desde su configuración.

Esto contrasta con Docker, donde ya está configurado en el archivo `docker-compose.yml` predeterminado.

## Configuración [#configuration]

La API de RAG proporciona varias opciones de configuración que se pueden establecer mediante variables de entorno desde un archivo `.env` accesible para la API. La mayoría de ellas son opcionales, aparte de las credenciales/rutas necesarias para el proveedor que configuró. En la configuración predeterminada, solo se requiere `RAG_OPENAI_API_KEY`.

<Callout type="warning" title="Docker" emoji="🐳">
Al utilizar la configuración predeterminada de Docker, el archivo .env se comparte entre LibreChat y la RAG API. Por esta razón, es importante definir las variables necesarias que se muestran en el [RAG API readme.md](https://github.com/danny-avila/rag_api/blob/main/README.md)
</Callout>

## Uso [#usage]

Una vez que la RAG API esté configurada y en funcionamiento, se integra perfectamente con LibreChat. Cuando un usuario sube archivos a una conversación, la RAG API indexa esos archivos y los utiliza para proporcionar respuestas conscientes del contexto.

**Para utilizar la RAG API de manera efectiva:**

1. Asegúrese de que los archivos necesarios estén cargados en la conversación en LibreChat. Si `RAG_API_URL` no está configurado, o no es accesible, la carga del archivo fallará.
2. A medida que el usuario interactúa con el chatbot, la RAG API recuperará automáticamente información relevante de los archivos indexados basándose en la entrada del usuario.
3. La información recuperada se utilizará para aumentar el prompt del usuario, permitiendo que LibreChat genere respuestas más precisas y contextualmente relevantes.
4. Elabore sus prompts cuidadosamente al adjuntar archivos, ya que el comportamiento predeterminado es consultar el vector store en cada nuevo mensaje de una conversación que tenga un archivo adjunto.
    - Puede deshabilitar el comportamiento predeterminado cambiando la opción "Resend Files" a un estado "off", que se encuentra en la configuración de la conversación.
    - Hacer esto permite realizar consultas de archivos dirigidas, logrando que la "recuperación" (retrieval) solo se realice cuando los archivos se adjuntan explícitamente a un mensaje.
    - ![image](https://github.com/danny-avila/LibreChat/assets/110412045/29a2468d-85ac-40d7-90be-a945301c5729)
5. Solo tienes que subir un archivo una vez para usarlo varias veces para RAG.
    - Puedes adjuntar archivos subidos/indexados a cualquier mensaje o conversación nueva usando el Panel Lateral:
    - ![image](https://github.com/danny-avila/LibreChat/assets/110412045/b40cb3d3-e6e7-46ec-bc74-65d194f55a1e)
    - Nota: Los archivos deben estar en el almacenamiento "Host", ya que los archivos de "OpenAI" se tratan de manera diferente y son exclusivos para Assistants. En otras palabras, no deben haber sido subidos cuando el endpoint de Assistants estaba seleccionado y activo. Puedes ver y gestionar tus archivos haciendo clic aquí desde el Panel Lateral.
    - ![image](https://github.com/danny-avila/LibreChat/assets/110412045/1f27e974-4124-4ee3-8091-13514cb4cbca)


## Solución de problemas [#troubleshooting]

Si encuentra algún problema al configurar o utilizar la RAG API, considere lo siguiente:

- Verifique dos veces que todas las variables de entorno requeridas estén configuradas correctamente en su archivo `.env`.
- Asegúrese de que la base de datos vectorial esté configurada correctamente y sea accesible.
- Verifique que la clave de API de OpenAI u otras credenciales necesarias sean válidas.
- Revisa tanto los registros de LibreChat como los de la API de RAG en busca de mensajes de error o advertencias.

Si el problema persiste, por favor consulte la documentación de la RAG API o busque ayuda en la comunidad de LibreChat en GitHub Discussions o Discord.
