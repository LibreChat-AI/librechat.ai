In Docker Compose ist eine Override-Datei eine leistungsstarke Funktion, die es Ihnen ermöglicht, die Standardkonfiguration der Hauptdatei `docker-compose.yml` zu ändern, ohne die gesamte Datei direkt bearbeiten oder duplizieren zu müssen. Der Hauptzweck der Override-Datei liegt in Anpassungen für die lokale Entwicklung, und Docker Compose führt die Konfigurationen der Dateien `docker-compose.yml` und `docker-compose.override.yml` zusammen, wenn Sie `docker compose up` ausführen.

Hier ist eine Kurzanleitung zur Verwendung der `docker-compose.override.yml`:

> Hinweis: Bitte konsultieren Sie die `docker-compose.override.yml.example` für weitere Beispiele

Weitere Informationen finden Sie in der offiziellen Docker-Dokumentation:

- **[docker docs - understanding-multiple-compose-files](https://docs.docker.com/compose/multiple-compose-files/extends/#understanding-multiple-compose-files)**
- **[docker docs - merge-compose-files](https://docs.docker.com/compose/multiple-compose-files/merge/#merge-compose-files)**
- **[docker docs - specifying-multiple-compose-files](https://docs.docker.com/compose/reference/#specifying-multiple-compose-files)**

## Schritt 1: Erstellen einer `docker-compose.override.yml` Datei [#step-1-create-a-docker-composeoverrideyml-file]

Falls du noch keine `docker-compose.override.yml` Datei hast, kannst du eine erstellen, indem du den Inhalt des Beispiel-Overrides kopierst:

```bash filename="Copying the example override file"
cp docker-compose.override.yml.example docker-compose.override.yml
```

Diese Datei wird von Docker Compose automatisch erkannt, wenn Sie docker-compose-Befehle ausführen.

## Schritt 2: Bearbeiten der Override-Datei [#step-2-edit-the-override-file]

Öffnen Sie Ihre `docker-compose.override.yml` Datei mit vscode oder einem beliebigen Texteditor.

Nehmen Sie die gewünschten Änderungen vor, indem Sie die entsprechenden Abschnitte auskommentieren und diese nach Bedarf anpassen.

> Warnung: Sie können jeden Dienstnamen nur einmal angeben (api, mongodb, meilisearch, ...). Wenn Sie mehrere Einstellungen in einem Dienst überschreiben möchten, müssen Sie dies entsprechend bearbeiten.

### Beispiele [#examples]

Wenn Sie sicherstellen möchten, dass Docker Ihre `librechat.yaml` Datei für [Custom Endpoints & Configuration](/docs/configuration/librechat_yaml) verwenden kann, sieht das wie folgt aus:

```yaml filename="docker-compose.override.yml"
services:
  api:
    volumes:
      - ./librechat.yaml:/app/librechat.yaml
```

Oder, wenn Sie das Image für den `api`-Dienst lokal erstellen, die LibreChat-Konfigurationsdatei verwenden und die ältere Mongo-Version nutzen möchten, die keine AVX-Unterstützung erfordert, könnte Ihre `docker-compose.override.yml` wie folgt aussehen:

```yaml filename="docker-compose.override.yml"
services:
  api:
    volumes:
      - ./librechat.yaml:/app/librechat.yaml
    image: librechat
    build:
      context: .
      target: node

  mongodb:
    image: mongo:4.4.18
```

> Hinweis: Seien Sie vorsichtig, wenn Sie Ports für MongoDB oder Meilisearch öffentlich zugänglich machen, da dies Ihre Daten anfällig machen kann.

## Schritt 3: Änderungen anwenden [#step-3-apply-the-changes]

Um Ihre Konfigurationsänderungen anzuwenden, führen Sie einfach wie gewohnt Docker Compose aus. Docker Compose berücksichtigt automatisch sowohl die `docker-compose.yml` als auch die `docker-compose.override.yml` Dateien:

```bash filename="Apply the changes"
docker compose up -d
```

## Schritt 4: Änderungen überprüfen [#step-4-verify-the-changes]

Nachdem Sie Ihre Dienste mit der geänderten Konfiguration gestartet haben, können Sie überprüfen, ob die Änderungen übernommen wurden, indem Sie den Befehl `docker ps` verwenden, um die laufenden Container und deren Eigenschaften, wie z. B. Ports, aufzulisten.

## Wichtige Überlegungen [#important-considerations]

- **Reihenfolge der Priorität**: Werte, die in der Override-Datei definiert sind, haben Vorrang vor den Werten, die in der ursprünglichen `docker-compose.yml` Datei angegeben sind.
- **Sicherheit**: Seien Sie sich bei der Anpassung von Ports und der öffentlichen Bereitstellung von Diensten stets der sicherheitsrelevanten Auswirkungen bewusst. Vermeiden Sie die Verwendung von Standardeinstellungen für Produktions- oder sensible Umgebungen.

Indem Sie diese Schritte und Überlegungen befolgen, können Sie Ihre Docker Compose-Konfiguration einfach und sicher ändern, ohne die ursprüngliche `docker-compose.yml`-Datei zu verändern. Dies erleichtert die Verwaltung und Pflege verschiedener Umgebungen oder lokaler Anpassungen.


## `deploy-compose.yml` [#deploy-composeyml]

Um eine Override-Datei mit einer nicht standardmäßigen Docker Compose-Datei zu verwenden, wie zum Beispiel `deploy-compose.yml`, müssen Sie beim Ausführen von Docker Compose-Befehlen beide Dateien explizit angeben.

Docker Compose ermöglicht es Ihnen, mehrere `-f` oder `--file` Optionen anzugeben, um mehrere Compose-Dateien einzubinden, wobei Einstellungen in späteren Dateien die in den ersten überschreiben oder ergänzen.

Wenn Sie `deploy-compose.yml` als Ihre Hauptkonfiguration für Docker Compose verwenden und eine Override-Datei namens `docker-compose.override.yml` besitzen (Sie können die Override-Datei beliebig benennen, aber möglicherweise haben Sie diese spezifische Datei bereits), führen Sie Docker Compose-Befehle wie folgt aus:

```bash
docker compose -f deploy-compose.yml -f docker-compose.override.yml pull
docker compose -f deploy-compose.yml -f docker-compose.override.yml up
```
