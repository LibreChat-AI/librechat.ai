在 Docker Compose 中，覆盖文件（override file）是一项强大的功能，它允许您修改由主 `docker-compose.yml` 提供的默认配置，而无需直接编辑或复制整个文件。覆盖文件的主要用途是进行本地开发自定义，当您运行 `docker compose up` 时，Docker Compose 会合并 `docker-compose.yml` 和 `docker-compose.override.yml` 文件的配置。

以下是关于如何使用 `docker-compose.override.yml` 的快速指南：

> 注意：请参考 `docker-compose.override.yml.example` 以获取更多示例

请参阅官方 Docker 文档以获取更多信息：

- **[docker docs - understanding-multiple-compose-files](https://docs.docker.com/compose/multiple-compose-files/extends/#understanding-multiple-compose-files)**
- **[docker docs - merge-compose-files](https://docs.docker.com/compose/multiple-compose-files/merge/#merge-compose-files)**
- **[docker docs - specifying-multiple-compose-files](https://docs.docker.com/compose/reference/#specifying-multiple-compose-files)**

## 第一步：创建 `docker-compose.override.yml` 文件 [#step-1-create-a-docker-composeoverrideyml-file]

如果您还没有 `docker-compose.override.yml` 文件，可以通过复制示例覆盖内容来创建一个：

```bash filename="Copying the example override file"
cp docker-compose.override.yml.example docker-compose.override.yml
```

当你运行 docker-compose 命令时，Docker Compose 会自动读取此文件。

## 第 2 步：编辑覆盖文件 [#step-2-edit-the-override-file]

使用 vscode 或任何文本编辑器打开你的 `docker-compose.override.yml` 文件。

通过取消相关部分的注释并根据需要进行自定义，来进行您想要的更改。

> 警告：每个服务名称（api、mongodb、meilisearch 等）只能指定一次。如果您想在一个服务中覆盖多个设置，则需要进行相应的编辑。

### 示例 [#examples]

如果您想确保 Docker 可以使用您的 `librechat.yaml` 文件进行 [Custom Endpoints & Configuration](/docs/configuration/librechat_yaml) 配置，它看起来应该是这样的：

```yaml filename="docker-compose.override.yml"
services:
  api:
    volumes:
      - ./librechat.yaml:/app/librechat.yaml
```

或者，如果您想为 `api` 服务在本地构建镜像，使用 LibreChat 配置文件，并使用不需要 AVX 支持的旧版 Mongo，您的 `docker-compose.override.yml` 可能如下所示：

```yaml filename="docker-compose.override.yml"
services:
  api:
    volumes:
      - ./librechat.yaml:/app/librechat.yaml
    image: librechat
    build:
      context: .
      target: node

  mongodb:
    image: mongo:4.4.18
```

> 注意：如果将 MongoDB 或 Meilisearch 的端口暴露给公网，请务必谨慎，因为这可能会使您的数据面临风险。

## 步骤 3：应用更改 [#step-3-apply-the-changes]

要应用您的配置更改，只需像往常一样运行 Docker Compose。Docker Compose 会自动考虑 `docker-compose.yml` 和 `docker-compose.override.yml` 文件：

```bash filename="Apply the changes"
docker compose up -d
```

## 第 4 步：验证更改 [#step-4-verify-the-changes]

在启动修改过配置的服务后，您可以使用 `docker ps` 命令列出正在运行的容器及其属性（例如端口），以验证更改是否已应用。

## 重要注意事项 [#important-considerations]

- **优先级顺序**：在覆盖文件中定义的值优先于原始 `docker-compose.yml` 文件中指定的值。
- **安全性**：在自定义端口并公开服务时，请务必注意相关的安全隐患。在生产环境或敏感环境中，请避免使用默认配置。

通过遵循这些步骤和注意事项，您可以轻松且安全地修改 Docker Compose 配置，而无需更改原始的 `docker-compose.yml` 文件，从而更简便地管理和维护不同的环境或本地自定义设置。


## `deploy-compose.yml` [#deploy-composeyml]

若要将覆盖文件与非默认的 Docker Compose 文件（例如 `deploy-compose.yml`）配合使用，您在运行 Docker Compose 命令时必须显式指定这两个文件。

Docker Compose 允许您指定多个 `-f` 或 `--file` 选项以包含多个 compose 文件，其中后续文件中的设置会覆盖或添加到第一个文件中的设置。

如果您使用 `deploy-compose.yml` 作为主要的 Docker Compose 配置，并且拥有一个名为 `docker-compose.override.yml` 的覆盖文件（您可以随意命名覆盖文件，但可能已经存在此特定文件），则可以按如下方式运行 Docker Compose 命令：

```bash
docker compose -f deploy-compose.yml -f docker-compose.override.yml pull
docker compose -f deploy-compose.yml -f docker-compose.override.yml up
```
