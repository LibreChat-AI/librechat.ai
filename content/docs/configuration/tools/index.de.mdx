LibreChat-Tools werden im **Agent Builder** ausgewählt und ausgeführt, wenn ein Agent entscheidet, dass sie nützlich sind. Dieser Abschnitt behandelt integrierte Agenten-Tools wie Bildgenerierung, Suche, Wetter, Berechnung und private Index-Abfragen.

<Callout type="info" title="Nicht dasselbe wie Web Search oder MCP">

Die Suchwerkzeuge auf dieser Seite sind Werkzeuge, die Sie einem bestimmten Agenten hinzufügen. Die integrierte [Web Search](/docs/features/web_search)-Funktion von LibreChat wird separat konfiguriert, und benutzerdefinierte Drittanbieter-Werkzeuge werden normalerweise über [MCP](/docs/features/mcp) oder [Actions](/docs/features/agents#actions) hinzugefügt.

</Callout>

## Schnelleinrichtung [#quick-setup]

<Steps>
  <Step>

### Wähle das Tool [#pick-the-tool]

Wählen Sie ein Tool aus der untenstehenden Tabelle und sammeln Sie alle erforderlichen API-Schlüssel, Service-URLs oder Indexnamen.

  </Step>
  <Step>

### Anmeldedaten hinzufügen [#add-credentials]

Fügen Sie die erforderlichen Werte zu Ihrer `.env` Datei hinzu oder lassen Sie Benutzer ihre eigenen Anmeldedaten über die LibreChat Benutzeroberfläche eingeben, wenn das Tool dazu auffordert.

  </Step>
  <Step>

### LibreChat neu starten [#restart-librechat]

Änderungen an Umgebungsvariablen werden beim Neustart geladen.

| Bereitstellung | Befehl |
|------------|---------|
| Docker | `docker compose down && docker compose up -d` |
| Lokal | Stoppen Sie den Server und führen Sie dann erneut `npm run backend` aus |

  </Step>
  <Step>

### Das Tool zu einem Agenten hinzufügen [#add-the-tool-to-an-agent]

Wählen Sie in LibreChat **Agents** aus, erstellen oder bearbeiten Sie einen Agenten, öffnen Sie die **Tools**-Liste des Agenten, wählen Sie das Tool aus und speichern Sie den Agenten.

  </Step>
  <Step>

### Test im Chat [#test-in-chat]

Starten Sie einen Chat mit diesem Agenten und fragen Sie nach etwas, das das Tool erfordert, wie zum Beispiel eine Suche, eine Berechnung, einen Wetterbericht oder ein Bild.

  </Step>
</Steps>

## Erstellen von benutzerdefinierten Tools [#creating-custom-tools]

Die meisten benutzerdefinierten Tools sollten hinzugefügt werden, ohne den Quellcode von LibreChat zu bearbeiten.

- Verwenden Sie [MCP](/docs/features/mcp), wenn Sie einen Agenten mit einem lokalen Skript, einem internen Dienst, einer Datenbank, einem Browser-Automatisierungsserver oder einem benutzerdefinierten API-Wrapper verbinden möchten. Zum Beispiel kann ein DuckDuckGo-Suchtool über einen MCP-Server bereitgestellt und anschließend im Agent Builder ausgewählt werden.
- Verwenden Sie [Actions](/docs/features/agents#actions), wenn es sich bei dem Tool um eine HTTP-API handelt, die mit einem OpenAPI-Schema beschrieben werden kann.
- Bearbeiten Sie die quellseitigen strukturierten Tools von LibreChat nur, wenn Sie LibreChat selbst weiterentwickeln. Der veraltete [Tools and Plugins development guide](/docs/development/tools_and_plugins) bleibt für Mitwirkende verfügbar, aber MCP und Actions sind die empfohlenen Erweiterungspfade für Bereitstellungen.

## Verfügbarkeit von Tools [#tool-availability]

Tools werden intern durch ihren `pluginKey` aus der `api/app/clients/tools/manifest.json` von LibreChat identifiziert.

Verwenden Sie [`filteredTools`](/docs/configuration/librechat_yaml/object_structure/config#filteredtools), um Tools auszublenden, oder [`includedTools`](/docs/configuration/librechat_yaml/object_structure/config#includedtools), um nur bestimmte Tools zuzulassen:

```yaml filename="librechat.yaml"
includedTools:
  - calculator
  - image_gen_oai
  - google
```

Wenn ein Tool nach einem Neustart nicht im Agent Builder sichtbar ist, überprüfen Sie die Umgebungsvariablen des Tools, `includedTools`, `filteredTools` sowie ob die `tools`-Funktion des Agents aktiviert ist.
