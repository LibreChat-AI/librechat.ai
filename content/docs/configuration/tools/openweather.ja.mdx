OpenWeatherツールを使用すると、エージェントはOpenWeatherのOne Call API 3.0を利用して、現在の状況、予報、過去のデータ、日次サマリーなどの気象データを取得できます。

## 前提条件 [#prerequisites]

- OpenWeatherアカウント
- OpenWeather APIキー（特にOne Call API 3.0用）

## APIキーの取得 [#getting-an-api-key]

1. [OpenWeather](https://home.openweathermap.org/users/sign_up) で OpenWeather アカウントにサインアップしてください。
2. サインイン後、[API keys](https://home.openweathermap.org/api_keys)ページに移動します。
3. APIキーをお持ちでない場合は、新しいAPIキーを生成してください
4. [One Call API 3.0](https://openweathermap.org/api/one-call-3) プランに登録する
5. APIキーが有効化されるまでお待ちください（最大2時間かかる場合があります）

## 設定 [#configuration]

### 環境変数 [#environment-variables]

`.env` ファイルに以下を追加してください：

```bash
OPENWEATHER_API_KEY=your_api_key_here
```

### エージェントにツールを追加する [#add-the-tool-to-an-agent]

`.env` を変更した後に LibreChat を再起動し、任意の [agent](/docs/features/agents) に **OpenWeather** を追加してください。

| デプロイ | コマンド |
|------------|---------|
| Docker | `docker compose down && docker compose up -d` |
| ローカル | サーバーを停止し、再度 `npm run backend` を実行します |

LibreChatで、**Agents**を選択し、エージェントを作成または編集して、そのエージェントの**Tools**リストを開き、**OpenWeather**を選択して、エージェントを保存します。

## 使用方法 [#usage]

OpenWeather ツールは、以下の操作をサポートしています。

- `current_forecast`: 現在の天気と予報データを取得する
- `timestamp`: 特定の日付の過去の気象データを取得します
- `daily_aggregation`: 特定の日付の集計された気象データを取得します
- `overview`: 人間が読みやすい天気予報の概要を取得する

### プロンプトの例 [#example-prompts]

```
What's the current weather in London?
What was the weather like in Paris on 2023-01-01?
Give me a weather summary for Tokyo.
What's the temperature in New York in Fahrenheit?
```

### パラメータ [#parameters]

- `city`: 都市名（緯度/経度が指定されていない場合）
- `lat`: 緯度座標（都市名が指定されている場合は省略可能）
- `lon`: 経度座標（都市が指定されている場合は任意）
- `units`: 温度単位 ("Celsius"、"Kelvin"、または "Fahrenheit")
- `lang`: 天気説明の言語コード（例: "en", "fr", "es"）
- `date`: YYYY-MM-DD 形式の日付（timestamp および daily_aggregation アクションで必須）
- `tz`: タイムゾーン（オプション、daily_aggregation アクション用）

## トラブルシューティング [#troubleshooting]

一般的な問題と解決策:

1. **403 Unauthorized エラー**
   - APIキーが正しいことを確認してください
   - APIキーが有効化されているか確認してください（作成後2時間待機してください）
   - One Call API 3.0に登録していることを確認してください。

2. **都市が見つかりません**
   - 都市名のスペルを確認してください
   - 国コードを追加してみてください（例: "London,UK"）
   - 代わりに緯度と経度の座標を使用してください

3. **無効な日付形式**
   - 日付は YYYY-MM-DD 形式にしてください
   - 履歴データは1979-01-01以降のみ利用可能です
   - 将来のデータは1.5年先までに制限されています

## API Limits [#api-limits]

- [OpenWeatherのサブスクリプション](https://home.openweathermap.org/subscriptions)で、ご自身の制限事項を確認してください。
- トラフィックの多い環境では、レート制限の実装を検討してください。

## 注記 [#notes]

- Temperatureの値は、自動的に最も近い数値に丸められます。
- 指定がない場合、デフォルトの温度単位は摂氏（Celsius）となります。
