Alat LibreChat dipilih dari **Agent Builder** dan dijalankan ketika agen memutuskan bahwa alat tersebut berguna. Bagian ini mencakup alat agen bawaan seperti pembuatan gambar, pencarian, cuaca, komputasi, dan pencarian indeks pribadi.

<Callout type="info" title="Tidak sama dengan Web Search atau MCP">

Alat pencarian di halaman ini adalah alat yang Anda tambahkan ke agen tertentu. Fitur [Web Search](/docs/features/web_search) bawaan LibreChat dikonfigurasi secara terpisah, dan alat pihak ketiga kustom biasanya ditambahkan melalui [MCP](/docs/features/mcp) atau [Actions](/docs/features/agents#actions).

</Callout>

## Pengaturan Cepat [#quick-setup]

<Steps>
  <Step>

### Pilih Tool [#pick-the-tool]

Pilih alat dari tabel di bawah ini dan kumpulkan kunci API, URL layanan, atau nama indeks yang diperlukan.

  </Step>
  <Step>

### Tambahkan Kredensial [#add-credentials]

Tambahkan nilai yang diperlukan ke file `.env` Anda, atau biarkan pengguna memberikan kredensial mereka sendiri dari UI LibreChat saat alat tersebut memintanya.

  </Step>
  <Step>

### Restart LibreChat [#restart-librechat]

Perubahan variabel lingkungan dimuat saat memulai ulang.

| Deployment | Command |
|------------|---------|
| Docker | `docker compose down && docker compose up -d` |
| Local | Hentikan server, lalu jalankan `npm run backend` kembali |

  </Step>
  <Step>

### Menambahkan Tool ke Agent [#add-the-tool-to-an-agent]

Di LibreChat, pilih **Agents**, buat atau edit agen, buka daftar **Tools** agen tersebut, pilih alatnya, lalu simpan agen tersebut.

  </Step>
  <Step>

### Uji di Chat [#test-in-chat]

Mulai obrolan dengan agen tersebut dan mintalah sesuatu yang memerlukan alat, seperti pencarian, perhitungan, laporan cuaca, atau gambar.

  </Step>
</Steps>

## Membuat Custom Tools [#creating-custom-tools]

Sebagian besar alat kustom harus ditambahkan tanpa perlu mengubah kode sumber LibreChat.

- Gunakan [MCP](/docs/features/mcp) saat Anda ingin menghubungkan agen ke skrip lokal, layanan internal, basis data, server otomatisasi peramban, atau pembungkus API kustom. Sebagai contoh, alat pencarian DuckDuckGo dapat diekspos melalui server MCP dan kemudian dipilih dari Agent Builder.
- Gunakan [Actions](/docs/features/agents#actions) ketika tool tersebut adalah API HTTP yang dapat dideskripsikan dengan skema OpenAPI.
- Edit alat terstruktur tingkat sumber LibreChat hanya jika Anda sedang mengembangkan LibreChat itu sendiri. Panduan pengembangan [Tools and Plugins](/docs/development/tools_and_plugins) versi lama tetap tersedia bagi kontributor, namun MCP dan Actions adalah jalur ekstensi yang direkomendasikan untuk deployment.

## Ketersediaan Tool [#tool-availability]

Tools diidentifikasi secara internal berdasarkan `pluginKey` dari `api/app/clients/tools/manifest.json` milik LibreChat.

Gunakan [`filteredTools`](/docs/configuration/librechat_yaml/object_structure/config#filteredtools) untuk menyembunyikan alat, atau [`includedTools`](/docs/configuration/librechat_yaml/object_structure/config#includedtools) untuk hanya mengizinkan alat tertentu:

```yaml filename="librechat.yaml"
includedTools:
  - calculator
  - image_gen_oai
  - google
```

Jika sebuah tool tidak terlihat di Agent Builder setelah dimulai ulang, periksa variabel lingkungan tool tersebut, `includedTools`, `filteredTools`, dan apakah kapabilitas `tools` pada agen sudah diaktifkan.
