Les outils LibreChat sont sélectionnés depuis l'**Agent Builder** et sont exécutés lorsqu'un agent décide qu'ils sont utiles. Cette section couvre les outils d'agent intégrés tels que la génération d'images, la recherche, la météo, le calcul et la consultation d'index privés.

<Callout type="info" title="Pas la même chose que Web Search ou MCP">

Les outils de recherche sur cette page sont des outils que vous ajoutez à un agent spécifique. La fonctionnalité de [Web Search](/docs/features/web_search) intégrée à LibreChat est configurée séparément, et les outils tiers personnalisés sont généralement ajoutés via [MCP](/docs/features/mcp) ou [Actions](/docs/features/agents#actions).

</Callout>

## Configuration rapide [#quick-setup]

<Steps>
  <Step>

### Choisir l'outil [#pick-the-tool]

Choisissez un outil dans le tableau ci-dessous et rassemblez toutes les clés API, URL de service ou noms d'index requis.

  </Step>
  <Step>

### Ajouter des identifiants [#add-credentials]

Ajoutez les valeurs requises à votre fichier `.env`, ou laissez les utilisateurs fournir leurs propres identifiants depuis l'interface utilisateur de LibreChat lorsque l'outil les leur demande.

  </Step>
  <Step>

### Redémarrer LibreChat [#restart-librechat]

Les changements de variables d'environnement sont chargés au redémarrage.

| Déploiement | Commande |
|------------|---------|
| Docker | `docker compose down && docker compose up -d` |
| Local | Arrêtez le serveur, puis exécutez à nouveau `npm run backend` |

  </Step>
  <Step>

### Ajouter l'outil à un Agent [#add-the-tool-to-an-agent]

Dans LibreChat, sélectionnez **Agents**, créez ou modifiez un agent, ouvrez la liste **Tools** de l'agent, sélectionnez l'outil et enregistrez l'agent.

  </Step>
  <Step>

### Tester dans le chat [#test-in-chat]

Démarrez une discussion avec cet agent et demandez-lui quelque chose qui nécessite l'outil, comme une recherche, un calcul, un bulletin météo ou une image.

  </Step>
</Steps>

## Création d'outils personnalisés [#creating-custom-tools]

La plupart des outils personnalisés devraient être ajoutés sans modifier le code source de LibreChat.

- Utilisez [MCP](/docs/features/mcp) lorsque vous souhaitez connecter un agent à un script local, un service interne, une base de données, un serveur d'automatisation de navigateur ou un wrapper d'API personnalisé. Par exemple, un outil de recherche DuckDuckGo peut être exposé via un serveur MCP, puis sélectionné depuis l'Agent Builder.
- Utilisez les [Actions](/docs/features/agents#actions) lorsque l'outil est une API HTTP qui peut être décrite avec un schéma OpenAPI.
- Modifiez les outils structurés au niveau du code source de LibreChat uniquement lorsque vous développez LibreChat lui-même. Le guide de développement des [Tools and Plugins](/docs/development/tools_and_plugins) hérité reste disponible pour les contributeurs, mais MCP et les Actions sont les voies d'extension recommandées pour les déploiements.

## Disponibilité des outils [#tool-availability]

Les outils sont identifiés en interne par leur `pluginKey` provenant du fichier `api/app/clients/tools/manifest.json` de LibreChat.

Utilisez [`filteredTools`](/docs/configuration/librechat_yaml/object_structure/config#filteredtools) pour masquer des outils, ou [`includedTools`](/docs/configuration/librechat_yaml/object_structure/config#includedtools) pour n'autoriser que des outils spécifiques :

```yaml filename="librechat.yaml"
includedTools:
  - calculator
  - image_gen_oai
  - google
```

Si un outil n'est pas visible dans l'Agent Builder après un redémarrage, vérifiez les variables d'environnement de l'outil, `includedTools`, `filteredTools`, et si la capacité `tools` de l'agent est activée.
