LibreChat 도구는 **Agent Builder**에서 선택되며, 에이전트가 유용하다고 판단할 때 실행됩니다. 이 섹션에서는 이미지 생성, 검색, 날씨, 계산, 개인 인덱스 조회와 같은 내장 에이전트 도구에 대해 다룹니다.

<Callout type="info" title="Web Search 또는 MCP와는 다름">

이 페이지의 검색 도구는 특정 에이전트에 추가하는 도구입니다. LibreChat의 내장 [Web Search](/docs/features/web_search) 기능은 별도로 구성되며, 사용자 지정 타사 도구는 일반적으로 [MCP](/docs/features/mcp) 또는 [Actions](/docs/features/agents#actions)를 통해 추가됩니다.

</Callout>

## 빠른 설정 [#quick-setup]

<Steps>
  <Step>

### 도구 선택 [#pick-the-tool]

아래 표에서 도구를 선택하고 필요한 API 키, 서비스 URL 또는 인덱스 이름을 수집하세요.

  </Step>
  <Step>

### 자격 증명 추가 [#add-credentials]

`.env` 파일에 필요한 값을 추가하거나, 도구가 요청할 때 사용자가 LibreChat UI에서 직접 자격 증명을 입력하도록 설정하세요.

  </Step>
  <Step>

### LibreChat 재시작 [#restart-librechat]

환경 변수 변경 사항은 재시작 시 로드됩니다.

| 배포 | 명령어 |
|------------|---------|
| Docker | `docker compose down && docker compose up -d` |
| Local | 서버를 중지한 후, `npm run backend`를 다시 실행하세요 |

  </Step>
  <Step>

### 에이전트에 도구 추가하기 [#add-the-tool-to-an-agent]

LibreChat에서 **Agents**를 선택하고, 에이전트를 생성하거나 편집한 다음, 에이전트의 **Tools** 목록을 열어 도구를 선택하고 에이전트를 저장하세요.

  </Step>
  <Step>

### 채팅에서 테스트하기 [#test-in-chat]

해당 에이전트와 채팅을 시작하고 검색, 계산, 날씨 정보 또는 이미지 생성과 같이 도구가 필요한 작업을 요청하세요.

  </Step>
</Steps>

## 현재 내장된 도구 [#current-built-in-tools]

| 도구 | 용도 | 필수 설정 | 세부 정보 |
|------|------------|------------------------|---------|
| OpenAI Image Tools | OpenAI 이미지 모델로 이미지 생성 및 편집 | `IMAGE_GEN_OAI_API_KEY`; 선택 사항 `IMAGE_GEN_OAI_MODEL` | [이미지 생성](/docs/features/image_gen#1--openai-image-tools-recommended) |
| Gemini Image Tools | Gemini를 사용하여 이미지 생성 및 이미지 컨텍스트 편집 | `GEMINI_API_KEY`, `GOOGLE_KEY`, 또는 `GOOGLE_SERVICE_KEY_FILE`; 선택 사항 `GEMINI_IMAGE_MODEL` | [Gemini 이미지 생성](/docs/configuration/tools/gemini_image_gen) |
| DALL-E-3 | 레거시 OpenAI 이미지 생성 | `DALLE3_API_KEY` 또는 `DALLE_API_KEY` | [DALL-E](/docs/features/image_gen#3--dalle-legacy) |
| Flux | 클라우드 이미지 생성 및 파인튜닝된 이미지 모델 | `FLUX_API_KEY`; 선택 사항 `FLUX_API_BASE_URL` | [Flux](/docs/configuration/tools/flux) |
| Stable Diffusion | Automatic1111을 통한 로컬 또는 자체 호스팅 이미지 생성 | `SD_WEBUI_URL` | [Stable Diffusion](/docs/configuration/tools/stable_diffusion) |
| Google Search | 에이전트를 위한 Google 맞춤 검색 결과 | `GOOGLE_SEARCH_API_KEY` 및 `GOOGLE_CSE_ID` | [Google Search](/docs/configuration/tools/google_search) |
| Tavily Search | 에이전트에 최적화된 최신 웹 결과 | `TAVILY_API_KEY` | [Tavily Search](/docs/configuration/tools/tavily) |
| Traversaal | 출처가 포함된 AI 검색 결과 | `TRAVERSAAL_API_KEY` | [Traversaal](/docs/configuration/tools/traversaal) |
| Azure AI Search | 비공개 Azure AI Search 인덱스 검색 | `AZURE_AI_SEARCH_SERVICE_ENDPOINT`, `AZURE_AI_SEARCH_INDEX_NAME`, `AZURE_AI_SEARCH_API_KEY` | [Azure AI Search](/docs/configuration/tools/azure_ai_search) |
| OpenWeather | 현재, 예보, 과거 및 일일 날씨 데이터 | `OPENWEATHER_API_KEY` | [OpenWeather](/docs/configuration/tools/openweather) |
| Wolfram\|Alpha | 수학, 계산, 단위, 큐레이팅된 지식 및 실시간 데이터 | `WOLFRAM_APP_ID` | [Wolfram\|Alpha](/docs/configuration/tools/wolfram) |
| Calculator | 기본 및 복잡한 계산 | 없음 | [Calculator](/docs/configuration/tools/calculator) |

## 사용자 지정 도구 만들기 [#creating-custom-tools]

대부분의 사용자 지정 도구는 LibreChat 소스 코드를 수정하지 않고 추가해야 합니다.

- 에이전트를 로컬 스크립트, 내부 서비스, 데이터베이스, 브라우저 자동화 서버 또는 사용자 지정 API 래퍼에 연결하려면 [MCP](/docs/features/mcp)를 사용하세요. 예를 들어, DuckDuckGo 검색 도구를 MCP 서버를 통해 노출한 다음 에이전트 빌더(Agent Builder)에서 선택할 수 있습니다.
- 도구가 OpenAPI 스키마로 설명할 수 있는 HTTP API인 경우 [Actions](/docs/features/agents#actions)를 사용하세요.
- LibreChat의 소스 수준 구조화된 도구(structured tools)는 LibreChat 자체를 개발할 때만 수정하십시오. 기존 [Tools and Plugins development guide](/docs/development/tools_and_plugins)는 기여자들을 위해 계속 제공되지만, 배포 시에는 MCP와 Actions를 확장 경로로 사용하는 것을 권장합니다.

## 도구 가용성 [#tool-availability]

도구는 LibreChat의 `api/app/clients/tools/manifest.json`에 있는 `pluginKey`를 통해 내부적으로 식별됩니다.

[`filteredTools`](/docs/configuration/librechat_yaml/object_structure/config#filteredtools)를 사용하여 도구를 숨기거나, [`includedTools`](/docs/configuration/librechat_yaml/object_structure/config#includedtools)를 사용하여 특정 도구만 허용할 수 있습니다:

```yaml filename="librechat.yaml"
includedTools:
  - calculator
  - image_gen_oai
  - google
```

재시작 후에도 Agent Builder에서 도구가 보이지 않는다면, 해당 도구의 환경 변수, `includedTools`, `filteredTools`를 확인하고 에이전트의 `tools` 기능이 활성화되어 있는지 확인하세요.
