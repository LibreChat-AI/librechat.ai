Flux es una potente herramienta de generación de imágenes que puede crear imágenes de alta calidad a partir de descripciones de texto. Es compatible con diversos estilos artísticos y ofrece amplias opciones de personalización.

## Instrucciones de configuración [#setup-instructions]

1. Obtén tu clave de API en [bfl.ml](https://bfl.ml)
2. Establece la variable de entorno `FLUX_API_KEY` en tu archivo `.env`:

```bash
FLUX_API_KEY=your_api_key_here
```

3. Reinicie LibreChat y añada **Flux** a la lista de **Tools** de un agente.

| Despliegue | Comando |
|------------|---------|
| Docker | `docker compose down && docker compose up -d` |
| Local | Detenga el servidor, luego ejecute `npm run backend` de nuevo |

## Características [#features]

### Capacidades principales [#core-capabilities]
- Genera imágenes de alta calidad a partir de descripciones de texto detalladas
- Soporte para múltiples estilos artísticos
- Dimensiones de imagen personalizables
- Parámetros de generación ajustables
- Múltiples opciones de endpoint para diferentes casos de uso
- Soporte para generación por lotes (hasta 24 imágenes)

### Endpoints disponibles [#available-endpoints]
- `/v1/flux-pro` - endpoint estándar (predeterminado)
- `/v1/flux-pro-1.1` - Versión mejorada
- `/v1/flux-dev` - Versión de desarrollo
- `/v1/flux-pro-1.1-ultra` - endpoint de calidad premium

### Parámetros [#parameters]

La herramienta Flux admite tres acciones principales:

1. **generate** - Crear una nueva imagen a partir de una descripción de texto
2. **generate_finetuned** - Crear una imagen utilizando un modelo ajustado (fine-tuned)
3. **list_finetunes** - Listar los modelos personalizados disponibles para el usuario


Para la acción `generate`:

• **prompt** – Descripción de texto para la imagen (obligatorio)  
• **width** – Ancho en píxeles (múltiplo de 32)  
• **height** – Alto en píxeles (múltiplo de 32)  
• **prompt_upsampling** – Si se debe realizar un muestreo ascendente (upsampling) en el prompt (predeterminado: false)  
• **steps** – Número de pasos de difusión (1-50, predeterminado: 40)  
• **seed** – Semilla opcional para la reproducibilidad  
• **safety_tolerance** – Nivel de tolerancia para la moderación (0-6, predeterminado: 6)  
• **endpoint** – endpoint del modelo a utilizar:
  - `/v1/flux-pro-1.1` (predeterminado)
  - `/v1/flux-pro`
  - `/v1/flux-dev`
  - `/v1/flux-pro-1.1-ultra`
• **raw** – Generar imágenes menos procesadas (solo para el endpoint ultra, predeterminado: false)

Para la acción `generate_finetuned`:

• Todos los parámetros de `generate` más:  
• **finetune_id** – ID del modelo ajustado (requerido)  
• **finetune_strength** – Intensidad del efecto de ajuste (0.1-1.2, predeterminado: 1.1)  
• **guidance** – Escala de guía (predeterminado: 2.5)  
• **aspect_ratio** – Relación de aspecto para modelos ultra (predeterminado: "16:9")  
• **endpoint** – Debe ser uno de los siguientes:
  - `/v1/flux-pro-finetuned` (predeterminado)
  - `/v1/flux-pro-1.1-ultra-finetuned`


## Mejores prácticas [#best-practices]

### Redacción de Prompts [#prompt-writing]
1. Sea específico y detallado en las descripciones
2. Incluye elementos clave:
   - Asunto
   - Estilo y enfoque artístico
   - Detalles de composición
   - Iluminación y atmósfera
   - Preferencias de color
   - Especificaciones técnicas

### Consejos para obtener los mejores resultados [#tips-for-best-results]
- Escribe los prompts en inglés
- Equilibra la especificidad con la libertad creativa
- Evitar conceptos contradictorios
- Céntrese en las descripciones visuales
- Considere las capas de composición (primer plano, plano medio, fondo)

## Detalles técnicos [#technical-details]

### Procesamiento de imágenes [#image-processing]
- Las imágenes se guardan y gestionan automáticamente
- Admite el formato de salida PNG
- Incluye funciones de moderación y seguridad integradas
- Generación asíncrona con seguimiento de estado

### Características de integración [#integration-features]
- Integración perfecta con interfaces de chat
- Salida con formato Markdown
- Manejo de errores y registro integrados
- Soporte para procesamiento por lotes

## Ejemplos de uso [#usage-examples]

Aquí hay algunos ejemplos de prompts que funcionan bien con Flux:

> Un paisaje montañoso sereno al atardecer, con picos nevados reflejados en un lago alpino cristalino. Una cálida luz dorada ilumina nubes tenues, creando una atmósfera dramática. Estilo fotorrealista con colores intensos y detalles nítidos.

> Un paisaje urbano futurista de noche, con rascacielos iluminados por luces de neón y vehículos voladores. Estilo cyberpunk con azules y morados profundos, acentuados por colores neón brillantes. Las calles mojadas por la lluvia reflejan las luces de la ciudad, creando una atmósfera melancólica.

## Manejo de errores [#error-handling]

Mensajes de error comunes y soluciones:
- Problemas con la API key: Verifique que su API key esté configurada correctamente en las variables de entorno
- Fallos de generación: Compruebe la longitud del prompt y las directrices de contenido
- Errores de tiempo de espera (Timeout): Pueden ocurrir durante una carga alta del servidor, vuelva a intentarlo después de una breve espera

## Límites de tasa y uso [#rate-limits-and-usage]

- El nivel gratuito incluye límites de uso generosos
- Se pueden generar múltiples imágenes en una sola solicitud
- Considere usar conteos de pasos más bajos para generaciones más rápidas durante las pruebas
