La herramienta OpenWeather permite a los agentes obtener datos meteorológicos, incluyendo condiciones actuales, pronósticos, datos históricos y resúmenes diarios, utilizando la One Call API 3.0 de OpenWeather.

## Requisitos previos [#prerequisites]

- Una cuenta de OpenWeather
- Una clave de API de OpenWeather (específicamente para la One Call API 3.0)

## Obtener una API Key [#getting-an-api-key]

1. Regístrese para obtener una cuenta de OpenWeather en [OpenWeather](https://home.openweathermap.org/users/sign_up)
2. Después de iniciar sesión, ve a tu página de [API keys](https://home.openweathermap.org/api_keys)
3. Genera una nueva clave de API si no tienes una
4. Suscríbase al plan [One Call API 3.0](https://openweathermap.org/api/one-call-3)
5. Espere a que su clave de API sea activada (puede tardar hasta 2 horas)

## Configuración [#configuration]

### Variables de entorno [#environment-variables]

Agregue lo siguiente a su archivo `.env`:

```bash
OPENWEATHER_API_KEY=your_api_key_here
```

### Agregar la herramienta a un Agent [#add-the-tool-to-an-agent]

Reinicie LibreChat después de cambiar `.env`, luego añada **OpenWeather** a cualquier [agente](/docs/features/agents).

| Despliegue | Comando |
|------------|---------|
| Docker | `docker compose down && docker compose up -d` |
| Local | Detenga el servidor, luego ejecute `npm run backend` de nuevo |

En LibreChat, selecciona **Agents**, crea o edita un agente, abre la lista de **Tools** del agente, selecciona **OpenWeather** y guarda el agente.

## Uso [#usage]

La herramienta OpenWeather admite las siguientes acciones:

- `current_forecast`: Obtener datos del clima actual y pronóstico
- `timestamp`: Obtener datos meteorológicos históricos para una fecha específica
- `daily_aggregation`: Obtener datos meteorológicos agregados para una fecha específica
- `overview`: Obtener un resumen meteorológico legible para humanos

### Ejemplo de Prompts [#example-prompts]

```
What's the current weather in London?
What was the weather like in Paris on 2023-01-01?
Give me a weather summary for Tokyo.
What's the temperature in New York in Fahrenheit?
```

### Parámetros [#parameters]

- `city`: Nombre de la ciudad (si no se proporciona lat/lon)
- `lat`: Coordenada de latitud (opcional si se proporciona la ciudad)
- `lon`: Coordenada de longitud (opcional si se proporciona la ciudad)
- `units`: Unidades de temperatura ("Celsius", "Kelvin" o "Fahrenheit")
- `lang`: Código de idioma para las descripciones del clima (p. ej., "en", "fr", "es")
- `date`: Fecha en formato AAAA-MM-DD (requerido para las acciones timestamp y daily_aggregation)
- `tz`: Zona horaria (opcional, para la acción daily_aggregation)

## Solución de problemas [#troubleshooting]

Problemas comunes y soluciones:

1. **Error 403 No autorizado**
   - Verifique que su API key sea correcta
   - Compruebe si su API key ha sido activada (espere 2 horas después de la creación)
   - Asegúrate de haberte suscrito a la One Call API 3.0

2. **Ciudad no encontrada**
   - Comprueba la ortografía del nombre de la ciudad
   - Intenta añadir el código de país (por ejemplo, "London,UK")
   - Use coordenadas de latitud y longitud en su lugar

3. **Formato de fecha no válido**
   - Asegúrese de que las fechas estén en formato YYYY-MM-DD
   - Los datos históricos solo están disponibles desde el 1979-01-01
   - Los datos futuros están limitados a 1.5 años en adelante

## Límites de la API [#api-limits]

- Consulta tu [OpenWeather subscription](https://home.openweathermap.org/subscriptions) para conocer tus límites específicos
- Considere implementar limitación de tasa (rate limiting) en entornos de alto tráfico

## Soporte [#support]

Para problemas con la herramienta:
- Puedes abrir un issue en https://github.com/jmaddington/LibreChat/issues o
- Consulta los [LibreChat Issues](https://github.com/danny-avila/LibreChat/issues)
- Revise la [documentación de la API](https://openweathermap.org/api/one-call-3) de OpenWeather
- Comuníquese con el [soporte](https://openweathermap.org/support-centre) de OpenWeather para problemas específicos de la API

## Notas [#notes]

- Los valores de Temperature se redondean automáticamente al grado más cercano
- La unidad de temperatura predeterminada es Celsius si no se especifica.
