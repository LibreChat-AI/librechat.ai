Gli strumenti di LibreChat vengono selezionati dall'**Agent Builder** e vengono eseguiti quando un agente decide che sono utili. Questa sezione copre gli strumenti integrati per gli agenti, come la generazione di immagini, la ricerca, il meteo, il calcolo e la consultazione di indici privati.

<Callout type="info" title="Non è la stessa cosa di Web Search o MCP">

Gli strumenti di ricerca in questa pagina sono strumenti che aggiungi a un agente specifico. La funzionalità di [Web Search](/docs/features/web_search) integrata di LibreChat viene configurata separatamente, mentre gli strumenti personalizzati di terze parti vengono solitamente aggiunti tramite [MCP](/docs/features/mcp) o [Actions](/docs/features/agents#actions).

</Callout>

## Configurazione rapida [#quick-setup]

<Steps>
  <Step>

### Scegli lo strumento [#pick-the-tool]

Scegli uno strumento dalla tabella sottostante e raccogli le chiavi API, gli URL del servizio o i nomi degli indici richiesti.

  </Step>
  <Step>

### Aggiungi credenziali [#add-credentials]

Aggiungi i valori richiesti al tuo file `.env`, oppure consenti agli utenti di fornire le proprie credenziali dall'interfaccia utente di LibreChat quando lo strumento le richiede.

  </Step>
  <Step>

### Riavviare LibreChat [#restart-librechat]

Le modifiche alle variabili d'ambiente vengono caricate al riavvio.

| Distribuzione | Comando |
|------------|---------|
| Docker | `docker compose down && docker compose up -d` |
| Locale | Arresta il server, quindi esegui nuovamente `npm run backend` |

  </Step>
  <Step>

### Aggiungi lo strumento a un Agent [#add-the-tool-to-an-agent]

In LibreChat, seleziona **Agents**, crea o modifica un agente, apri l'elenco **Tools** dell'agente, seleziona lo strumento e salva l'agente.

  </Step>
  <Step>

### Test in Chat [#test-in-chat]

Avvia una chat con quell'agente e chiedi qualcosa che richieda lo strumento, come una ricerca, un calcolo, un bollettino meteorologico o un'immagine.

  </Step>
</Steps>

## Creazione di Custom Tools [#creating-custom-tools]

La maggior parte degli strumenti personalizzati dovrebbe essere aggiunta senza modificare il codice sorgente di LibreChat.

- Usa [MCP](/docs/features/mcp) quando vuoi connettere un agente a uno script locale, un servizio interno, un database, un server di automazione del browser o un wrapper API personalizzato. Ad esempio, uno strumento di ricerca DuckDuckGo può essere esposto tramite un server MCP e quindi selezionato dall'Agent Builder.
- Usa le [Actions](/docs/features/agents#actions) quando lo strumento è un'API HTTP che può essere descritta con uno schema OpenAPI.
- Modifica gli strumenti strutturati a livello di codice sorgente di LibreChat solo quando stai sviluppando LibreChat stesso. La guida legacy [Tools and Plugins development guide](/docs/development/tools_and_plugins) rimane disponibile per i collaboratori, ma MCP e Actions sono i percorsi di estensione consigliati per le implementazioni.

## Disponibilità degli strumenti [#tool-availability]

I tool vengono identificati internamente tramite il loro `pluginKey` dal file `api/app/clients/tools/manifest.json` di LibreChat.

Usa [`filteredTools`](/docs/configuration/librechat_yaml/object_structure/config#filteredtools) per nascondere strumenti, o [`includedTools`](/docs/configuration/librechat_yaml/object_structure/config#includedtools) per consentire solo strumenti specifici:

```yaml filename="librechat.yaml"
includedTools:
  - calculator
  - image_gen_oai
  - google
```

Se uno strumento non è visibile nel Agent Builder dopo il riavvio, controlla le variabili d'ambiente dello strumento, `includedTools`, `filteredTools` e se la funzionalità `tools` dell'agente è abilitata.
