OpenWeather 工具允许智能体使用 OpenWeather 的 One Call API 3.0 获取天气数据，包括当前状况、预报、历史数据和每日摘要。

## 先决条件 [#prerequisites]

- 一个 OpenWeather 账户
- 一个 OpenWeather API 密钥（专门用于 One Call API 3.0）

## 获取 API Key [#getting-an-api-key]

1. 在 [OpenWeather](https://home.openweathermap.org/users/sign_up) 注册 OpenWeather 账户
2. 登录后，前往您的 [API keys](https://home.openweathermap.org/api_keys) 页面
3. 如果您还没有 API key，请生成一个新的。
4. 订阅 [One Call API 3.0](https://openweathermap.org/api/one-call-3) 计划
5. 等待您的 API key 激活（可能需要长达 2 小时）

## 配置 [#configuration]

### 环境变量 [#environment-variables]

将以下内容添加到您的 `.env` 文件中：

```bash
OPENWEATHER_API_KEY=your_api_key_here
```

### 将工具添加到 Agent [#add-the-tool-to-an-agent]

修改 `.env` 后重启 LibreChat，然后将 **OpenWeather** 添加到任何 [agent](/docs/features/agents) 中。

| 部署方式 | 命令 |
|------------|---------|
| Docker | `docker compose down && docker compose up -d` |
| 本地 | 停止服务器，然后再次运行 `npm run backend` |

在 LibreChat 中，选择 **Agents**，创建或编辑一个 agent，打开该 agent 的 **Tools** 列表，选择 **OpenWeather**，然后保存该 agent。

## 使用方法 [#usage]

OpenWeather 工具支持以下操作：

- `current_forecast`: 获取当前天气和预报数据
- `timestamp`: 获取特定日期的历史天气数据
- `daily_aggregation`: 获取特定日期的汇总天气数据
- `overview`: 获取人类可读的天气摘要

### 示例提示词 [#example-prompts]

```
What's the current weather in London?
What was the weather like in Paris on 2023-01-01?
Give me a weather summary for Tokyo.
What's the temperature in New York in Fahrenheit?
```

### 参数 [#parameters]

- `city`: 城市名称（如果未提供经纬度）
- `lat`: 纬度坐标（如果提供了城市，则可选）
- `lon`: 经度坐标（如果提供了城市，则可选）
- `units`: 温度单位（"Celsius"、"Kelvin" 或 "Fahrenheit"）
- `lang`：天气描述的语言代码（例如 "en", "fr", "es"）
- `date`: YYYY-MM-DD 格式的日期（对于 timestamp 和 daily_aggregation 操作是必需的）
- `tz`: 时区（可选，用于 daily_aggregation 操作）

## 故障排除 [#troubleshooting]

常见问题与解决方案：

1. **403 未经授权错误**
   - 验证您的 API key 是否正确
   - 检查您的 API key 是否已激活（创建后请等待 2 小时）
   - 请确保您已订阅 One Call API 3.0

2. **城市未找到**
   - 检查城市名称的拼写
   - 尝试添加国家代码（例如，“London,UK”）
   - 改用经纬度坐标

3. **无效的日期格式**
   - 确保日期格式为 YYYY-MM-DD
   - 历史数据仅提供自 1979-01-01 起的记录
   - 未来数据限制在 1.5 年内

## API 限制 [#api-limits]

- 请检查您的 [OpenWeather subscription](https://home.openweathermap.org/subscriptions) 以了解您的具体限制
- 在流量较高的环境中，请考虑实施速率限制。

## 注意事项 [#notes]

- Temperature 值会自动四舍五入到最接近的整数。
- 如果未指定，默认温度单位为摄氏度
