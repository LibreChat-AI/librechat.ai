Narzędzia LibreChat są wybierane z poziomu **Agent Builder** i uruchamiane, gdy agent uzna je za przydatne. Ta sekcja obejmuje wbudowane narzędzia agenta, takie jak generowanie obrazów, wyszukiwanie, pogoda, obliczenia oraz przeszukiwanie prywatnego indeksu.

<Callout type="info" title="Nie to samo co Web Search lub MCP">

Narzędzia wyszukiwania na tej stronie to narzędzia, które dodajesz do konkretnego agenta. Wbudowana funkcja [Web Search](/docs/features/web_search) w LibreChat jest konfigurowana oddzielnie, a niestandardowe narzędzia innych firm są zazwyczaj dodawane za pośrednictwem [MCP](/docs/features/mcp) lub [Actions](/docs/features/agents#actions).

</Callout>

## Szybka konfiguracja [#quick-setup]

<Steps>
  <Step>

### Wybierz narzędzie [#pick-the-tool]

Wybierz narzędzie z poniższej tabeli i przygotuj wymagane klucze API, adresy URL usług lub nazwy indeksów.

  </Step>
  <Step>

### Dodaj poświadczenia [#add-credentials]

Dodaj wymagane wartości do pliku `.env` lub pozwól użytkownikom na podanie własnych danych uwierzytelniających z poziomu interfejsu LibreChat, gdy narzędzie o nie poprosi.

  </Step>
  <Step>

### Restart LibreChat [#restart-librechat]

Zmiany zmiennych środowiskowych są wczytywane przy ponownym uruchomieniu.

| Wdrożenie | Polecenie |
|------------|---------|
| Docker | `docker compose down && docker compose up -d` |
| Lokalne | Zatrzymaj serwer, a następnie ponownie uruchom `npm run backend` |

  </Step>
  <Step>

### Dodawanie narzędzia do Agenta [#add-the-tool-to-an-agent]

W LibreChat wybierz **Agents**, utwórz lub edytuj agenta, otwórz listę **Tools** agenta, wybierz narzędzie i zapisz agenta.

  </Step>
  <Step>

### Test w czacie [#test-in-chat]

Rozpocznij czat z tym agentem i poproś o coś, co wymaga użycia narzędzia, na przykład wyszukiwanie, obliczenia, prognozę pogody lub obraz.

  </Step>
</Steps>

## Tworzenie własnych narzędzi (Custom Tools) [#creating-custom-tools]

Większość niestandardowych narzędzi powinna być dodawana bez edytowania kodu źródłowego LibreChat.

- Użyj [MCP](/docs/features/mcp), gdy chcesz połączyć agenta z lokalnym skryptem, usługą wewnętrzną, bazą danych, serwerem automatyzacji przeglądarki lub niestandardowym wrapperem API. Na przykład narzędzie wyszukiwania DuckDuckGo może zostać udostępnione przez serwer MCP, a następnie wybrane w kreatorze Agent Builder.
- Użyj [Actions](/docs/features/agents#actions), gdy narzędzie jest interfejsem API HTTP, który można opisać za pomocą schematu OpenAPI.
- Edytuj narzędzia strukturalne na poziomie źródłowym LibreChat tylko wtedy, gdy rozwijasz samo LibreChat. Starszy [przewodnik tworzenia narzędzi i wtyczek](/docs/development/tools_and_plugins) pozostaje dostępny dla współtwórców, ale MCP oraz Actions są zalecanymi ścieżkami rozszerzeń dla wdrożeń.

## Dostępność narzędzi [#tool-availability]

Narzędzia są identyfikowane wewnętrznie za pomocą `pluginKey` z pliku `api/app/clients/tools/manifest.json` w LibreChat.

Użyj [`filteredTools`](/docs/configuration/librechat_yaml/object_structure/config#filteredtools), aby ukryć narzędzia, lub [`includedTools`](/docs/configuration/librechat_yaml/object_structure/config#includedtools), aby zezwolić tylko na określone narzędzia:

```yaml filename="librechat.yaml"
includedTools:
  - calculator
  - image_gen_oai
  - google
```

Jeśli narzędzie nie jest widoczne w Agent Builder po restarcie, sprawdź zmienne środowiskowe narzędzia, `includedTools`, `filteredTools` oraz to, czy funkcja `tools` agenta jest włączona.
