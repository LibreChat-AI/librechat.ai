As ferramentas do LibreChat são selecionadas a partir do **Agent Builder** e executadas quando um agente decide que elas são úteis. Esta seção aborda as ferramentas integradas do agente, como geração de imagens, pesquisa, clima, computação e consulta de índice privado.

<Callout type="info" title="Não é o mesmo que Web Search ou MCP">

As ferramentas de busca nesta página são ferramentas que você adiciona a um agente específico. O recurso de [Web Search](/docs/features/web_search) integrado do LibreChat é configurado separadamente, e ferramentas personalizadas de terceiros geralmente são adicionadas por meio de [MCP](/docs/features/mcp) ou [Actions](/docs/features/agents#actions).

</Callout>

## Configuração Rápida [#quick-setup]

<Steps>
  <Step>

### Escolha a Tool [#pick-the-tool]

Escolha uma ferramenta da tabela abaixo e colete quaisquer chaves de API, URLs de serviço ou nomes de índice necessários.

  </Step>
  <Step>

### Adicionar Credenciais [#add-credentials]

Adicione os valores necessários ao seu `.env` file, ou permita que os usuários forneçam suas próprias credenciais a partir da interface do LibreChat quando a ferramenta solicitar.

  </Step>
  <Step>

### Reiniciar o LibreChat [#restart-librechat]

As alterações nas variáveis de ambiente são carregadas ao reiniciar.

| Deployment | Command |
|------------|---------|
| Docker | `docker compose down && docker compose up -d` |
| Local | Pare o servidor e, em seguida, execute `npm run backend` novamente |

  </Step>
  <Step>

### Adicionar a Ferramenta a um Agent [#add-the-tool-to-an-agent]

No LibreChat, selecione **Agents**, crie ou edite um agente, abra a lista de **Tools** do agente, selecione a ferramenta e salve o agente.

  </Step>
  <Step>

### Testar no Chat [#test-in-chat]

Inicie um chat com esse agente e peça algo que exija a ferramenta, como uma pesquisa, cálculo, previsão do tempo ou imagem.

  </Step>
</Steps>

## Criando Ferramentas Personalizadas [#creating-custom-tools]

A maioria das ferramentas personalizadas deve ser adicionada sem editar o código-fonte do LibreChat.

- Use o [MCP](/docs/features/mcp) quando quiser conectar um agente a um script local, serviço interno, banco de dados, servidor de automação de navegador ou um wrapper de API personalizado. Por exemplo, uma ferramenta de busca do DuckDuckGo pode ser exposta através de um servidor MCP e, em seguida, selecionada no Agent Builder.
- Use [Actions](/docs/features/agents#actions) quando a ferramenta for uma API HTTP que pode ser descrita com um esquema OpenAPI.
- Edite as ferramentas estruturadas em nível de código-fonte do LibreChat apenas quando você estiver desenvolvendo o próprio LibreChat. O [guia de desenvolvimento de Ferramentas e Plugins](/docs/development/tools_and_plugins) legado permanece disponível para colaboradores, mas MCP e Actions são os caminhos de extensão recomendados para implantações.

## Disponibilidade de Ferramentas [#tool-availability]

As ferramentas são identificadas internamente pelo seu `pluginKey` a partir do arquivo `api/app/clients/tools/manifest.json` do LibreChat.

Use [`filteredTools`](/docs/configuration/librechat_yaml/object_structure/config#filteredtools) para ocultar ferramentas, ou [`includedTools`](/docs/configuration/librechat_yaml/object_structure/config#includedtools) para permitir apenas ferramentas específicas:

```yaml filename="librechat.yaml"
includedTools:
  - calculator
  - image_gen_oai
  - google
```

Se uma ferramenta não estiver visível no Agent Builder após a reinicialização, verifique as variáveis de ambiente da ferramenta, `includedTools`, `filteredTools` e se a capacidade `tools` do agente está habilitada.
