Das OpenWeather-Tool ermöglicht es Agenten, Wetterdaten abzurufen, einschließlich aktueller Bedingungen, Vorhersagen, historischer Daten und täglicher Zusammenfassungen unter Verwendung der One Call API 3.0 von OpenWeather.

## Voraussetzungen [#prerequisites]

- Ein OpenWeather-Konto
- Ein OpenWeather API-Schlüssel (speziell für die One Call API 3.0)

## Einen API-Schlüssel erhalten [#getting-an-api-key]

1. Registrieren Sie sich für ein OpenWeather-Konto unter [OpenWeather](https://home.openweathermap.org/users/sign_up)
2. Nach der Anmeldung gehen Sie zu Ihrer [API keys](https://home.openweathermap.org/api_keys) Seite
3. Generieren Sie einen neuen API-Schlüssel, falls Sie noch keinen haben
4. Abonnieren Sie den [One Call API 3.0](https://openweathermap.org/api/one-call-3) Plan
5. Warten Sie, bis Ihr API-Schlüssel aktiviert wurde (dies kann bis zu 2 Stunden dauern)

## Konfiguration [#configuration]

### Umgebungsvariablen [#environment-variables]

Fügen Sie Folgendes zu Ihrer `.env` Datei hinzu:

```bash
OPENWEATHER_API_KEY=your_api_key_here
```

### Das Tool zu einem Agenten hinzufügen [#add-the-tool-to-an-agent]

Starten Sie LibreChat nach dem Ändern der `.env` neu und fügen Sie dann **OpenWeather** zu einem beliebigen [agent](/docs/features/agents) hinzu.

| Bereitstellung | Befehl |
|------------|---------|
| Docker | `docker compose down && docker compose up -d` |
| Lokal | Stoppen Sie den Server und führen Sie dann erneut `npm run backend` aus |

Wählen Sie in LibreChat **Agents** aus, erstellen oder bearbeiten Sie einen Agenten, öffnen Sie die **Tools**-Liste des Agenten, wählen Sie **OpenWeather** aus und speichern Sie den Agenten.

## Verwendung [#usage]

Das OpenWeather-Tool unterstützt die folgenden Aktionen:

- `current_forecast`: Aktuelle Wetter- und Vorhersagedaten abrufen
- `timestamp`: Historische Wetterdaten für ein bestimmtes Datum abrufen
- `daily_aggregation`: Aggregierte Wetterdaten für ein bestimmtes Datum abrufen
- `overview`: Erhalte eine für Menschen lesbare Wetterzusammenfassung

### Beispiel-Prompts [#example-prompts]

```
What's the current weather in London?
What was the weather like in Paris on 2023-01-01?
Give me a weather summary for Tokyo.
What's the temperature in New York in Fahrenheit?
```

### Parameter [#parameters]

- `city`: Name der Stadt (falls Breitengrad/Längengrad nicht angegeben)
- `lat`: Breitengrad-Koordinate (optional, falls eine Stadt angegeben ist)
- `lon`: Längengrad-Koordinate (optional, falls eine Stadt angegeben ist)
- `units`: Temperatureinheiten ("Celsius", "Kelvin" oder "Fahrenheit")
- `lang`: Sprachcode für Wetterbeschreibungen (z. B. "en", "fr", "es")
- `date`: Datum im Format YYYY-MM-DD (erforderlich für timestamp- und daily_aggregation-Aktionen)
- `tz`: Zeitzone (optional, für die Aktion daily_aggregation)

## Fehlerbehebung [#troubleshooting]

Häufige Probleme und Lösungen:

1. **403 Nicht autorisiert Fehler**
   - Überprüfen Sie, ob Ihr API-Schlüssel korrekt ist
   - Überprüfen Sie, ob Ihr API-Schlüssel aktiviert wurde (warten Sie 2 Stunden nach der Erstellung)
   - Stellen Sie sicher, dass Sie die One Call API 3.0 abonniert haben.

2. **Stadt nicht gefunden**
   - Überprüfen Sie die Schreibweise des Stadtnamens
   - Versuchen Sie, den Ländercode hinzuzufügen (z. B. "London,UK")
   - Verwende stattdessen Breiten- und Längengrade

3. **Ungültiges Datumsformat**
   - Stellen Sie sicher, dass Datumsangaben im Format YYYY-MM-DD vorliegen.
   - Historische Daten sind nur ab dem 01.01.1979 verfügbar
   - Zukünftige Daten sind auf 1,5 Jahre im Voraus begrenzt

## API-Limits [#api-limits]

- Überprüfen Sie Ihr [OpenWeather subscription](https://home.openweathermap.org/subscriptions) auf Ihre spezifischen Limits
- Erwägen Sie die Implementierung von Ratenbegrenzung (Rate Limiting) in Umgebungen mit hohem Verkehrsaufkommen.

## Hinweise [#notes]

- Temperaturwerte werden automatisch auf den nächsten Grad gerundet.
- Die Standard-Temperatureinheit ist Celsius, sofern nicht anders angegeben.
