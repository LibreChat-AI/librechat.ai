A ferramenta OpenWeather permite que agentes obtenham dados meteorológicos, incluindo condições atuais, previsões, dados históricos e resumos diários, usando a One Call API 3.0 da OpenWeather.

## Pré-requisitos [#prerequisites]

- Uma conta OpenWeather
- Uma chave de API da OpenWeather (especificamente para a One Call API 3.0)

## Obtendo uma API Key [#getting-an-api-key]

1. Crie uma conta no OpenWeather em [OpenWeather](https://home.openweathermap.org/users/sign_up)
2. Após fazer login, vá para a sua página de [API keys](https://home.openweathermap.org/api_keys)
3. Gere uma nova chave de API se você não tiver uma
4. Assine o plano [One Call API 3.0](https://openweathermap.org/api/one-call-3)
5. Aguarde a ativação da sua chave de API (pode levar até 2 horas)

## Configuração [#configuration]

### Variáveis de Ambiente [#environment-variables]

Adicione o seguinte ao seu arquivo `.env`:

```bash
OPENWEATHER_API_KEY=your_api_key_here
```

### Adicionar a Ferramenta a um Agent [#add-the-tool-to-an-agent]

Reinicie o LibreChat após alterar o `.env` e, em seguida, adicione o **OpenWeather** a qualquer [agent](/docs/features/agents).

| Deployment | Command |
|------------|---------|
| Docker | `docker compose down && docker compose up -d` |
| Local | Pare o servidor e, em seguida, execute `npm run backend` novamente |

No LibreChat, selecione **Agents**, crie ou edite um agente, abra a lista de **Tools** do agente, selecione **OpenWeather** e salve o agente.

## Uso [#usage]

A ferramenta OpenWeather suporta as seguintes ações:

- `current_forecast`: Obter dados meteorológicos atuais e previsão
- `timestamp`: Obter dados meteorológicos históricos para uma data específica
- `daily_aggregation`: Obter dados meteorológicos agregados para uma data específica
- `overview`: Obtenha um resumo meteorológico legível por humanos

### Exemplos de Prompts [#example-prompts]

```
What's the current weather in London?
What was the weather like in Paris on 2023-01-01?
Give me a weather summary for Tokyo.
What's the temperature in New York in Fahrenheit?
```

### Parâmetros [#parameters]

- `city`: Nome da cidade (se lat/lon não forem fornecidos)
- `lat`: Coordenada de latitude (opcional se a cidade for fornecida)
- `lon`: Coordenada de longitude (opcional se a cidade for fornecida)
- `units`: Unidades de temperatura ("Celsius", "Kelvin" ou "Fahrenheit")
- `lang`: Código de idioma para descrições meteorológicas (por exemplo, "en", "fr", "es")
- `date`: Data no formato AAAA-MM-DD (obrigatório para ações de timestamp e daily_aggregation)
- `tz`: Fuso horário (opcional, para a ação daily_aggregation)

## Solução de problemas [#troubleshooting]

Problemas comuns e soluções:

1. **Erro 403 Não Autorizado**
   - Verifique se sua chave de API está correta
   - Verifique se a sua chave de API foi ativada (aguarde 2 horas após a criação)
   - Certifique-se de ter assinado a One Call API 3.0

2. **Cidade Não Encontrada**
   - Verifique a ortografia do nome da cidade
   - Tente adicionar o código do país (por exemplo, "London,UK")
   - Use coordenadas de latitude e longitude em vez disso

3. **Formato de Data Inválido**
   - Certifique-se de que as datas estejam no formato YYYY-MM-DD
   - Dados históricos estão disponíveis apenas a partir de 1979-01-01
   - Dados futuros são limitados a 1,5 anos à frente

## Limites da API [#api-limits]

- Verifique sua [OpenWeather subscription](https://home.openweathermap.org/subscriptions) para ver seus limites específicos
- Considere implementar limitação de taxa (rate limiting) em ambientes de alto tráfego

## Suporte [#support]

Para problemas com a ferramenta:
- Você pode abrir uma issue em https://github.com/jmaddington/LibreChat/issues ou
- Verifique as [LibreChat Issues](https://github.com/danny-avila/LibreChat/issues)
- Revise a [documentação da API](https://openweathermap.org/api/one-call-3) da OpenWeather
- Entre em contato com o [suporte](https://openweathermap.org/support-centre) da OpenWeather para problemas específicos da API

## Notas [#notes]

- Os valores de temperatura são arredondados automaticamente para o grau mais próximo.
- A unidade de temperatura padrão é Celsius, se não especificada
