Flux est un outil de génération d'images puissant capable de créer des images de haute qualité à partir de descriptions textuelles. Il prend en charge divers styles artistiques et offre de nombreuses options de personnalisation.

## Instructions de configuration [#setup-instructions]

1. Obtenez votre clé API sur [bfl.ml](https://bfl.ml)
2. Définissez la variable d'environnement `FLUX_API_KEY` dans votre fichier `.env` :

```bash
FLUX_API_KEY=your_api_key_here
```

3. Redémarrez LibreChat et ajoutez **Flux** à la liste des **Tools** d'un agent.

| Déploiement | Commande |
|------------|---------|
| Docker | `docker compose down && docker compose up -d` |
| Local | Arrêtez le serveur, puis exécutez à nouveau `npm run backend` |

## Fonctionnalités [#features]

### Capacités principales [#core-capabilities]
- Générez des images de haute qualité à partir de descriptions textuelles détaillées
- Prise en charge de plusieurs styles artistiques
- Dimensions d'image personnalisables
- Paramètres de génération ajustables
- Options d'endpoint multiples pour différents cas d'utilisation
- Prise en charge de la génération par lots (jusqu'à 24 images)

### Endpoints disponibles [#available-endpoints]
- `/v1/flux-pro` - endpoint standard (par défaut)
- `/v1/flux-pro-1.1` - Version améliorée
- `/v1/flux-dev` - Version de développement
- `/v1/flux-pro-1.1-ultra` - endpoint de qualité premium

### Paramètres [#parameters]

L'outil Flux prend en charge trois actions principales :

1. **generate** - Créer une nouvelle image à partir d'une invite textuelle
2. **generate_finetuned** - Créer une image en utilisant un modèle affiné
3. **list_finetunes** - Lister les modèles personnalisés disponibles pour l'utilisateur


Pour l'action `generate` :

• **prompt** – Description textuelle pour l'image (requis)
• **width** – Largeur en pixels (multiple de 32)
• **height** – Hauteur en pixels (multiple de 32)
• **prompt_upsampling** – Indique s'il faut effectuer un suréchantillonnage (upsampling) sur le prompt (par défaut : false)
• **steps** – Nombre d'étapes de diffusion (1-50, par défaut : 40)
• **seed** – Graine optionnelle pour la reproductibilité
• **safety_tolerance** – Niveau de tolérance pour la modération (0-6, par défaut : 6)
• **endpoint** – endpoint de modèle à utiliser :
  - `/v1/flux-pro-1.1` (par défaut)
  - `/v1/flux-pro`
  - `/v1/flux-dev`
  - `/v1/flux-pro-1.1-ultra`
• **raw** – Générer des images moins traitées (uniquement pour l'endpoint ultra, par défaut : false)

Pour l'action `generate_finetuned` :

• Tous les paramètres de `generate` plus :
• **finetune_id** – ID du modèle affiné (requis)
• **finetune_strength** – Intensité de l'effet d'affinage (0.1-1.2, par défaut : 1.1)
• **guidance** – Échelle de guidage (par défaut : 2.5)
• **aspect_ratio** – Format d'image pour les modèles ultra (par défaut : "16:9")
• **endpoint** – Doit être l'un des suivants :
  - `/v1/flux-pro-finetuned` (par défaut)
  - `/v1/flux-pro-1.1-ultra-finetuned`


## Meilleures pratiques [#best-practices]

### Rédaction de prompts [#prompt-writing]
1. Soyez précis et détaillé dans les descriptions
2. Inclure les éléments clés :
   - Sujet
   - Style et approche artistique
   - Détails de la composition
   - Éclairage et atmosphère
   - Préférences de couleur
   - Spécifications techniques

### Conseils pour de meilleurs résultats [#tips-for-best-results]
- Écrivez les prompts en anglais
- Équilibrez la spécificité avec la liberté créative
- Évitez les concepts contradictoires
- Concentrez-vous sur les descriptions visuelles
- Considérez les couches de composition (premier plan, arrière-plan intermédiaire, arrière-plan)

## Détails techniques [#technical-details]

### Traitement d'image [#image-processing]
- Les images sont automatiquement enregistrées et gérées
- Prend en charge le format de sortie PNG
- Inclut des fonctionnalités de modération et de sécurité intégrées
- Génération asynchrone avec suivi de statut

### Fonctionnalités d'intégration [#integration-features]
- Intégration transparente avec les interfaces de chat
- Sortie au format Markdown
- Gestion des erreurs et journalisation intégrées
- Prise en charge du traitement par lots

## Exemples d'utilisation [#usage-examples]

Voici quelques exemples de prompts qui fonctionnent bien avec Flux :

> Un paysage de montagne serein au coucher du soleil, avec des sommets enneigés se reflétant dans un lac alpin cristallin. Une lumière dorée et chaude illumine des nuages légers, créant une atmosphère dramatique. Style photoréaliste avec des couleurs riches et des détails nets.

> Un paysage urbain futuriste de nuit, avec des gratte-ciels illuminés par des néons et des véhicules volants. Style cyberpunk avec des bleus et des violets profonds, accentués par des couleurs néon vives. Les rues détrempées par la pluie reflètent les lumières de la ville, créant une atmosphère mélancolique.

## Gestion des erreurs [#error-handling]

Messages d'erreur courants et solutions :
- Problèmes de clé API : Vérifiez que votre clé API est correctement définie dans les variables d'environnement
- Échecs de génération : Vérifiez la longueur du prompt et les directives de contenu
- Erreurs de délai d'attente (Timeout) : Peuvent survenir lors d'une charge serveur élevée, réessayez après une courte attente.

## Limites de débit et utilisation [#rate-limits-and-usage]

- Le niveau gratuit inclut des limites d'utilisation généreuses
- Plusieurs images peuvent être générées en une seule requête
- Envisagez d'utiliser un nombre d'étapes inférieur pour des générations plus rapides lors des tests.
