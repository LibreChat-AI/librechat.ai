LibreChat tools worden geselecteerd vanuit de **Agent Builder** en uitgevoerd wanneer een agent besluit dat ze nuttig zijn. Dit gedeelte behandelt ingebouwde agent-tools zoals het genereren van afbeeldingen, zoeken, weer, berekeningen en het opzoeken in een privé-index.

<Callout type="info" title="Niet hetzelfde als Web Search of MCP">

De zoektools op deze pagina zijn tools die je toevoegt aan een specifieke agent. De ingebouwde [Web Search](/docs/features/web_search) functie van LibreChat wordt afzonderlijk geconfigureerd, en aangepaste tools van derden worden meestal toegevoegd via [MCP](/docs/features/mcp) of [Actions](/docs/features/agents#actions).

</Callout>

## Snelle installatie [#quick-setup]

<Steps>
  <Step>

### Kies de Tool [#pick-the-tool]

Kies een tool uit de onderstaande tabel en verzamel alle vereiste API-sleutels, service-URL's of indexnamen.

  </Step>
  <Step>

### Inloggegevens toevoegen [#add-credentials]

Voeg de vereiste waarden toe aan je `.env` bestand, of laat gebruikers hun eigen inloggegevens opgeven via de LibreChat UI wanneer de tool daarom vraagt.

  </Step>
  <Step>

### LibreChat herstarten [#restart-librechat]

Wijzigingen in omgevingsvariabelen worden geladen bij een herstart.

| Deployment | Command |
|------------|---------|
| Docker | `docker compose down && docker compose up -d` |
| Lokaal | Stop de server en voer daarna opnieuw `npm run backend` uit |

  </Step>
  <Step>

### De Tool toevoegen aan een Agent [#add-the-tool-to-an-agent]

In LibreChat, selecteer **Agents**, maak of bewerk een agent, open de **Tools**-lijst van de agent, selecteer de tool en sla de agent op.

  </Step>
  <Step>

### Testen in de chat [#test-in-chat]

Start een chat met die agent en vraag om iets waarvoor de tool nodig is, zoals een zoekopdracht, berekening, weerbericht of afbeelding.

  </Step>
</Steps>

## Aangepaste tools maken [#creating-custom-tools]

De meeste aangepaste tools moeten worden toegevoegd zonder de broncode van LibreChat te bewerken.

- Gebruik [MCP](/docs/features/mcp) wanneer je een agent wilt verbinden met een lokaal script, interne service, database, browser-automatiseringsserver of een aangepaste API-wrapper. Een DuckDuckGo-zoektool kan bijvoorbeeld worden blootgesteld via een MCP-server en vervolgens worden geselecteerd in de Agent Builder.
- Gebruik [Actions](/docs/features/agents#actions) wanneer de tool een HTTP API is die kan worden beschreven met een OpenAPI-schema.
- Bewerk de bron-gebaseerde gestructureerde tools van LibreChat alleen wanneer je zelf aan LibreChat ontwikkelt. De verouderde [Tools and Plugins development guide](/docs/development/tools_and_plugins) blijft beschikbaar voor bijdragers, maar MCP en Actions zijn de aanbevolen extensiepaden voor implementaties.

## Tool-beschikbaarheid [#tool-availability]

Tools worden intern geïdentificeerd door hun `pluginKey` uit LibreChat's `api/app/clients/tools/manifest.json`.

Gebruik [`filteredTools`](/docs/configuration/librechat_yaml/object_structure/config#filteredtools) om tools te verbergen, of [`includedTools`](/docs/configuration/librechat_yaml/object_structure/config#includedtools) om alleen specifieke tools toe te staan:

```yaml filename="librechat.yaml"
includedTools:
  - calculator
  - image_gen_oai
  - google
```

Als een tool na een herstart niet zichtbaar is in de Agent Builder, controleer dan de omgevingsvariabelen van de tool, `includedTools`, `filteredTools`, en of de `tools` capability van de agent is ingeschakeld.
