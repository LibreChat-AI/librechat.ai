LibreChat 工具在 **Agent Builder** 中进行选择，并在 Agent 认为有用时运行。本节涵盖了内置的 Agent 工具，例如图像生成、搜索、天气、计算和私有索引查找。

<Callout type="info" title="与 Web Search 或 MCP 不同">

此页面上的搜索工具是您添加到特定 Agent 的工具。LibreChat 内置的 [Web Search](/docs/features/web_search) 功能是单独配置的，而自定义的第三方工具通常通过 [MCP](/docs/features/mcp) 或 [Actions](/docs/features/agents#actions) 添加。

</Callout>

## 快速设置 [#quick-setup]

<Steps>
  <Step>

### 选择工具 [#pick-the-tool]

从下表中选择一个工具，并收集所需的任何 API 密钥、服务 URL 或索引名称。

  </Step>
  <Step>

### 添加凭据 [#add-credentials]

将所需的值添加到您的 `.env` 文件中，或者在工具提示时让用户从 LibreChat UI 中提供他们自己的凭据。

  </Step>
  <Step>

### 重启 LibreChat [#restart-librechat]

重启后会加载环境变量的更改。

| 部署方式 | 命令 |
|------------|---------|
| Docker | `docker compose down && docker compose up -d` |
| 本地 | 停止服务器，然后再次运行 `npm run backend` |

  </Step>
  <Step>

### 将工具添加到 Agent [#add-the-tool-to-an-agent]

在 LibreChat 中，选择 **Agents**，创建或编辑一个 agent，打开该 agent 的 **Tools** 列表，选择相应的工具，然后保存该 agent。

  </Step>
  <Step>

### 在聊天中测试 [#test-in-chat]

与该 Agent 开始对话，并要求其执行需要该工具的任务，例如搜索、计算、天气预报或生成图像。

  </Step>
</Steps>

## 创建自定义工具 [#creating-custom-tools]

大多数自定义工具都应在不编辑 LibreChat 源代码的情况下添加。

- 当您想要将 Agent 连接到本地脚本、内部服务、数据库、浏览器自动化服务器或自定义 API 包装器时，请使用 [MCP](/docs/features/mcp)。例如，可以通过 MCP 服务器公开 DuckDuckGo 搜索工具，然后从 Agent Builder 中进行选择。
- 当工具是一个可以通过 OpenAPI 模式描述的 HTTP API 时，请使用 [Actions](/docs/features/agents#actions)。
- 仅在开发 LibreChat 本身时，才编辑 LibreChat 的源码级结构化工具。旧版的 [Tools and Plugins development guide](/docs/development/tools_and_plugins) 仍然对贡献者开放，但对于部署而言，MCP 和 Actions 是推荐的扩展路径。

## 工具可用性 [#tool-availability]

工具通过 LibreChat 的 `api/app/clients/tools/manifest.json` 中的 `pluginKey` 在内部进行标识。

使用 [`filteredTools`](/docs/configuration/librechat_yaml/object_structure/config#filteredtools) 来隐藏工具，或使用 [`includedTools`](/docs/configuration/librechat_yaml/object_structure/config#includedtools) 来仅允许特定的工具：

```yaml filename="librechat.yaml"
includedTools:
  - calculator
  - image_gen_oai
  - google
```

如果在重启后 Agent Builder 中看不到某个工具，请检查该工具的环境变量、`includedTools`、`filteredTools`，以及是否启用了该 agent 的 `tools` 功能。
