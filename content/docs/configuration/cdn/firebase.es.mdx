Firebase Storage se integra con la CDN global de Firebase Hosting, lo que le permite servir archivos almacenados en Firebase Storage a través de ubicaciones perimetrales en todo el mundo. Es una de las opciones de almacenamiento de archivos respaldadas por CDN de LibreChat, junto con [CloudFront for S3](/docs/configuration/cdn/cloudfront).

<Callout type="info" title="Lo que necesitarás">
Una cuenta de Google y aproximadamente 10 minutos. Creará un proyecto de Firebase, habilitará Cloud Storage, registrará una aplicación web para obtener las credenciales y luego configurará LibreChat para que las utilice.
</Callout>

<Step>

<Frame>
![Registrar la aplicación](https://github.com/danny-avila/LibreChat/assets/81851188/0a1b0a75-7285-4f03-95cf-bf971bd7d874)
</Frame>

</Step>
<Step>

**Copia tus valores de `firebaseConfig`.** Guarda la configuración mostrada en un lugar seguro.

<Frame>
![Valores de configuración de Firebase](https://github.com/danny-avila/LibreChat/assets/81851188/056754ad-9d36-4662-888e-f189ddb38fd3)
</Frame>

</Step>
<Step>

**Agregue los valores a su archivo `.env`.** Asigne cada valor de `firebaseConfig` a la variable correspondiente:

```bash filename=".env"
FIREBASE_API_KEY=api_key                          # apiKey
FIREBASE_AUTH_DOMAIN=auth_domain                  # authDomain
FIREBASE_PROJECT_ID=project_id                    # projectId
FIREBASE_STORAGE_BUCKET=storage_bucket            # storageBucket
FIREBASE_MESSAGING_SENDER_ID=messaging_sender_id  # messagingSenderId
FIREBASE_APP_ID=1:your_app_id                     # appId
```

</Step>
</Steps>

## Actualizar reglas de almacenamiento [#update-storage-rules]

<Steps>
<Step>

**Reglas de Open Storage.** Regrese a la **Project Overview**, seleccione **Storage**, luego abra la pestaña **Rules**.

<Frame>
![Open Storage](https://github.com/danny-avila/LibreChat/assets/32828263/16a0f850-cdd4-4875-8342-ab67bfb59804)
</Frame>

</Step>
<Step>

**Permitir acceso de lectura y escritura.** Cambie `allow read, write: if false;` a `if true;` para que coincida con las reglas a continuación:

```js filename="storage.rules"
rules_version = '2';

service firebase.storage {
  match /b/{bucket}/o {
    match /images/{userId}/{fileName} {
      allow read, write: if true;
    }
  }
}
```

## Configurar LibreChat [#configure-librechat]

Establezca `fileStrategy` en `firebase` en su archivo de configuración `librechat.yaml` para que LibreChat utilice Firebase para el almacenamiento de archivos:

```yaml filename="librechat.yaml"
version: 1.3.5
cache: true
fileStrategy: 'firebase'
```

Para obtener más información sobre este archivo, consulta la [guía de librechat.yaml](/docs/configuration/librechat_yaml).

## Habilitar CORS para exportaciones PNG [#enable-cors-for-png-exports]

<Callout type="warning" title="Solo es necesario para exportaciones PNG">
La exportación de conversaciones como PNG obtiene las imágenes directamente desde Firebase Storage en el navegador. Sin una política de CORS que permita su dominio, esas solicitudes serán bloqueadas. Omita esta sección si no exporta conversaciones como PNG.
</Callout>

<Steps>
<Step>

**Cree el archivo de configuración CORS.** En un editor de texto, cree `cors.json` y permita el acceso desde su dominio:

```json filename="cors.json"
[
  {
    "origin": ["https://ai.example.com"],
    "method": ["GET", "POST", "DELETE", "PUT"],
    "maxAgeSeconds": 3600
  }
]
```

</Step>
<Step>

**Aplique la configuración.** Desde el directorio que contiene `cors.json`, ejecute el siguiente comando, reemplazando `<your-cloud-storage-bucket>` con el nombre de su bucket:

```shell
gsutil cors set cors.json gs://<your-cloud-storage-bucket>
```

</Step>
<Step>

**Verifique la configuración.** Recupere la política activa y confirme que coincide con `cors.json`:

```shell
gsutil cors get gs://<your-cloud-storage-bucket>
```

</Step>
<Step>

**Pruébelo.** Exporte una conversación como PNG desde su origen permitido. Si todo está configurado correctamente, la exportación se realizará con éxito sin errores de CORS.

</Step>
</Steps>

<Callout type="info" title="Consejo de seguridad">
Solo permita CORS para orígenes de confianza y limite los métodos y encabezados a lo que su despliegue realmente necesite.
</Callout>
