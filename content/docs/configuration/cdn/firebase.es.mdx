Firebase Storage se integra con la CDN global de Firebase Hosting, lo que le permite servir archivos almacenados en Firebase Storage a través de ubicaciones perimetrales en todo el mundo. Es una de las opciones de almacenamiento de archivos respaldadas por CDN de LibreChat, junto con CloudFront para S3.

## Pasos para configurar Firebase [#steps-to-set-up-firebase]

1. Abre el [sitio web de Firebase](https://firebase.google.com/).
2. Haz clic en "Get started."
3. Inicia sesión con tu cuenta de Google.

- Rellena todas las variables `firebaseConfig` en el archivo `.env`.

```bash
FIREBASE_API_KEY=api_key #apiKey
FIREBASE_AUTH_DOMAIN=auth_domain #authDomain
FIREBASE_PROJECT_ID=project_id #projectId
FIREBASE_STORAGE_BUCKET=storage_bucket #storageBucket
FIREBASE_MESSAGING_SENDER_ID=messaging_sender_id #messagingSenderId

```

![image](https://github.com/danny-avila/LibreChat/assets/32828263/16a0f850-cdd4-4875-8342-ab67bfb59804)

- Seleccione `Rules` y elimine `: if false;` en esta línea: `allow read, write: if false;`
  - tus reglas actualizadas deberían verse así:

  ```bash
  rules_version = '2';

  service firebase.storage {
    match /b/{bucket}/o {
      match /images/{userId}/{fileName} {
        allow read, write: if true;
      }
    }
  }
  ```

### Configurar `fileStrategy` en `librechat.yaml` [#configure-filestrategy-in-librechatyaml]

Finalmente, para permitir que la aplicación use Firebase, debe configurar lo siguiente en su archivo de configuración `librechat.yaml`.

```yaml
version: 1.3.5
cache: true
fileStrategy: 'firebase'
```

Para obtener más información sobre el archivo de configuración `librechat.yaml`, consulta la guía aquí: [Custom Endpoints & Configuration](/docs/configuration/librechat_yaml).

### Guía paso a paso para configurar CORS para Firebase Storage [#step-by-step-guide-to-set-up-cors-for-firebase-storage]

---

#### **Paso 1: Crear el archivo de configuración CORS** [#step-1-create-the-cors-configuration-file]

- Abra un editor de texto de su elección.
- Cree un nuevo archivo y asígnele el nombre `cors.json`.
- Agregue la siguiente configuración para permitir el acceso desde "https://ai.example.com":

```json
[
  {
    "origin": ["https://ai.example.com"],
    "method": ["GET", "POST", "DELETE", "PUT"],
    "maxAgeSeconds": 3600
  }
]
```

- Guarda el archivo.

---

#### **Paso 2: Aplicar la configuración de CORS** [#step-2-apply-the-cors-configuration]

- Abre tu terminal o símbolo del sistema.
- Navegue al directorio donde guardó el archivo `cors.json`.
- Ejecute el siguiente comando, reemplazando `<your-cloud-storage-bucket>` con el nombre de su bucket de Firebase Storage:

```shell
gsutil cors set cors.json gs://<your-cloud-storage-bucket>
```

---

#### **Paso 3: Verificar la configuración de CORS** [#step-3-verify-the-cors-settings]

- Para confirmar que la configuración de CORS se ha aplicado correctamente, puede recuperar la configuración actual de CORS con el siguiente comando:

```shell
gsutil cors get gs://<your-cloud-storage-bucket>
```

- La salida debe reflejar los ajustes que especificó en el archivo `cors.json`.

---

#### **Paso 4: Probar la configuración** [#step-4-test-the-configuration]

- Intenta exportar una conversación como png desde el origen permitido ("https://ai.example.com").
- Si todo está configurado correctamente, no deberías encontrar ningún problema de CORS.

---

> **Nota:** Asegúrate siempre de aplicar la configuración de CORS solo para orígenes de confianza para mantener la seguridad de tu aplicación. Ajusta los métodos y encabezados permitidos según tus necesidades específicas.

---

¡Eso es todo! Has configurado correctamente CORS para tu bucket de Firebase Storage para permitir solicitudes desde un origen específico. Recuerda reemplazar `<your-cloud-storage-bucket>` con el nombre real de tu bucket y `https://ai.example.com` con tu propio dominio al aplicar la configuración.
