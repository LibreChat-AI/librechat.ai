Firebase StorageはFirebase HostingのグローバルCDNと統合されており、Firebase Storageに保存されたファイルを世界中のエッジロケーションを通じて配信できます。これは、S3向けのCloudFrontと並び、LibreChatがサポートするCDNバックエンドのファイルストレージオプションの一つです。

## Firebase をセットアップする手順 [#steps-to-set-up-firebase]

1. [Firebaseのウェブサイト](https://firebase.google.com/)を開きます。
2. 「Get started」をクリックします。
3. Googleアカウントでサインインします。

### 新しいプロジェクトを作成する [#create-a-new-project]

- プロジェクトに名前を付けます（Google OAuthと同じプロジェクトを使用できます）。

![Project Name](https://github.com/danny-avila/LibreChat/assets/81851188/dccce3e0-b639-41ef-8142-19d24911c65c)

- オプションで、Google Analyticsを無効にすることができます。

![Google Analytics](https://github.com/danny-avila/LibreChat/assets/81851188/5d4d58c5-451c-498b-97c0-f123fda79514)

- プロジェクトの準備が整うまで20〜30秒ほど待ち、「Continue」をクリックしてください。

![Continue](https://github.com/danny-avila/LibreChat/assets/81851188/6929802e-a30b-4b1e-b124-1d4b281d0403)

- 「All Products」をクリックします。

![All Products](https://github.com/danny-avila/LibreChat/assets/81851188/92866c82-2b03-4ebe-807e-73a0ccce695e)

- 「Storage」を選択します。

![Storage](https://github.com/danny-avila/LibreChat/assets/81851188/b22dcda1-256b-494b-a835-a05aeea02e89)

- 「Get Started」をクリックします。

![Get Started](https://github.com/danny-avila/LibreChat/assets/81851188/c3f0550f-8184-4c79-bb84-fa79655b7978)

- 「Next」をクリックします。

![Next](https://github.com/danny-avila/LibreChat/assets/81851188/2a65632d-fe22-4c71-b8f1-aac53ee74fb6)

- 「Cloud Storage location」を選択してください。

![Cloud Storage Location](https://github.com/danny-avila/LibreChat/assets/81851188/c094d4bc-8e5b-43c7-96d9-a05bcf4e2af6)

- プロジェクトの概要に戻る。

![Project Overview](https://github.com/danny-avila/LibreChat/assets/81851188/c425f4bb-a494-42f2-9fdc-ff2c8ce005e1)

- プロジェクト名の下にある「+ Add app」をクリックし、「Web」をクリックします。

![Web](https://github.com/danny-avila/LibreChat/assets/81851188/22dab877-93cb-4828-9436-10e14374e57e)

- アプリを登録します。

![Register App](https://github.com/danny-avila/LibreChat/assets/81851188/0a1b0a75-7285-4f03-95cf-bf971bd7d874)

- この情報をすべてテキストファイルに保存してください。

![Save Information](https://github.com/danny-avila/LibreChat/assets/81851188/056754ad-9d36-4662-888e-f189ddb38fd3)

- `.env` ファイル内のすべての `firebaseConfig` 変数を入力してください。

```bash
FIREBASE_API_KEY=api_key #apiKey
FIREBASE_AUTH_DOMAIN=auth_domain #authDomain
FIREBASE_PROJECT_ID=project_id #projectId
FIREBASE_STORAGE_BUCKET=storage_bucket #storageBucket
FIREBASE_MESSAGING_SENDER_ID=messaging_sender_id #messagingSenderId
FIREBASE_APP_ID=1:your_app_id #appId
```

- もう一度、Project Overviewに戻ります。

![Project Overview](https://github.com/danny-avila/LibreChat/assets/81851188/c425f4bb-a494-42f2-9fdc-ff2c8ce005e1)

- `Storage` を選択

![image](https://github.com/danny-avila/LibreChat/assets/32828263/16a0f850-cdd4-4875-8342-ab67bfb59804)

- `Rules` を選択し、この行の `: if false;` を削除してください: `allow read, write: if false;`
  - 更新されたルールは以下のようになります：

  ```bash
  rules_version = '2';

  service firebase.storage {
    match /b/{bucket}/o {
      match /images/{userId}/{fileName} {
        allow read, write: if true;
      }
    }
  }
  ```

![image](https://github.com/danny-avila/LibreChat/assets/32828263/c190011f-c1a6-47c7-986e-8d309b5f8704)

- 更新したルールを公開する

![image](https://github.com/danny-avila/LibreChat/assets/32828263/5e6a17c3-5aba-419a-a18f-be910b1f25d5)

### `librechat.yaml` で `fileStrategy` を設定する [#configure-filestrategy-in-librechatyaml]

最後に、アプリでFirebaseを使用できるようにするには、`librechat.yaml`設定ファイルで以下を設定する必要があります。

```yaml
version: 1.3.5
cache: true
fileStrategy: 'firebase'
```

`librechat.yaml` 設定ファイルの詳細については、こちらのガイドを参照してください: [Custom Endpoints & Configuration](/docs/configuration/librechat_yaml)

<Callout type="warning" title="Firebase使用時に会話をPNGとしてエクスポートする">

### Firebase StorageでCORSを設定するためのステップバイステップガイド [#step-by-step-guide-to-set-up-cors-for-firebase-storage]

---

#### **Step 1: CORS設定ファイルを作成する** [#step-1-create-the-cors-configuration-file]

- お好みのテキストエディタを開いてください。
- 新しいファイルを作成し、`cors.json` という名前を付けます。
- "https://ai.example.com" からのアクセスを許可するには、以下の設定を追加してください：

```json
[
  {
    "origin": ["https://ai.example.com"],
    "method": ["GET", "POST", "DELETE", "PUT"],
    "maxAgeSeconds": 3600
  }
]
```

- ファイルを保存します。

---

#### **Step 2: CORS設定を適用する** [#step-2-apply-the-cors-configuration]

- ターミナルまたはコマンドプロンプトを開きます。
- `cors.json` ファイルを保存したディレクトリに移動します。
- 以下のコマンドを実行し、`<your-cloud-storage-bucket>` をご自身の Firebase Storage バケット名に置き換えてください：

```shell
gsutil cors set cors.json gs://<your-cloud-storage-bucket>
```

---

#### **Step 3: CORS設定を確認する** [#step-3-verify-the-cors-settings]

- CORS設定が正しく適用されていることを確認するには、次のコマンドを使用して現在のCORS設定を取得できます：

```shell
gsutil cors get gs://<your-cloud-storage-bucket>
```

- 出力は、`cors.json`ファイルで指定した設定を反映したものになります。

---

#### **ステップ 4: 設定のテスト** [#step-4-test-the-configuration]

- 許可されたオリジン ("https://ai.example.com") から会話を png としてエクスポートしてみてください。
- すべてが正しく設定されていれば、CORSの問題は発生しないはずです。

---

> **注:** アプリケーションのセキュリティを維持するため、CORS設定は必ず信頼できるオリジンに対してのみ適用するようにしてください。許可するメソッドやヘッダーは、特定のニーズに合わせて調整してください。

---

以上で完了です！Firebase StorageバケットのCORS設定が完了し、特定のオリジンからのリクエストが許可されました。設定を適用する際は、`<your-cloud-storage-bucket>` を実際のバケット名に、`https://ai.example.com` をご自身のドメインに置き換えることを忘れないでください。

</Callout>
