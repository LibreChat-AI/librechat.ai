Firebase StorageはFirebase HostingのグローバルCDNと統合されており、世界中のエッジロケーションを通じてFirebase Storageに保存されたファイルを配信できます。これは、[CloudFront for S3](/docs/configuration/cdn/cloudfront)と並び、LibreChatのCDNを活用したファイルストレージオプションの一つです。

<Callout type="info" title="必要なもの">
Googleアカウントと約10分。Firebaseプロジェクトを作成し、Cloud Storageを有効化し、ウェブアプリを登録して認証情報を取得した後、LibreChatでそれを指定します。
</Callout>

</Step>
<Step>

</Step>
<Step>

## Webアプリの登録 [#register-a-web-app]

<Steps>
<Step>

**Webアプリを追加する。** プロジェクトの概要（Project Overview）で、プロジェクト名の下にある **+ Add app** をクリックし、**Web** を選択します。

<Frame>
![Add a web app](https://github.com/danny-avila/LibreChat/assets/81851188/22dab877-93cb-4828-9436-10e14374e57e)
</Frame>

</Step>
<Step>

**アプリを登録**し、ニックネームを付けます。

<Frame>
![Register the app](https://github.com/danny-avila/LibreChat/assets/81851188/0a1b0a75-7285-4f03-95cf-bf971bd7d874)
</Frame>

</Step>
<Step>

**`firebaseConfig` の値をコピーします。** 表示された設定を安全な場所に保存してください。

<Frame>
![Firebaseの設定値](https://github.com/danny-avila/LibreChat/assets/81851188/056754ad-9d36-4662-888e-f189ddb38fd3)
</Frame>

</Step>
<Step>

**`.env` ファイルに値を追加します。** 各 `firebaseConfig` の値を対応する変数にマッピングしてください：

```bash filename=".env"
FIREBASE_API_KEY=api_key                          # apiKey
FIREBASE_AUTH_DOMAIN=auth_domain                  # authDomain
FIREBASE_PROJECT_ID=project_id                    # projectId
FIREBASE_STORAGE_BUCKET=storage_bucket            # storageBucket
FIREBASE_MESSAGING_SENDER_ID=messaging_sender_id  # messagingSenderId
FIREBASE_APP_ID=1:your_app_id                     # appId
```

</Step>
</Steps>

## ストレージルールの更新 [#update-storage-rules]

<Steps>
<Step>

**Open Storage rules.** **Project Overview** に戻り、**Storage** を選択してから、**Rules** タブを開きます。

<Frame>
![Open Storage](https://github.com/danny-avila/LibreChat/assets/32828263/16a0f850-cdd4-4875-8342-ab67bfb59804)
</Frame>

</Step>
<Step>

**読み取りおよび書き込みアクセスを許可します。** `allow read, write: if false;` を `if true;` に変更し、以下のルールと一致するようにします。

```js filename="storage.rules"
rules_version = '2';

service firebase.storage {
  match /b/{bucket}/o {
    match /images/{userId}/{fileName} {
      allow read, write: if true;
    }
  }
}
```

<Frame>
![Updated storage rules](https://github.com/danny-avila/LibreChat/assets/32828263/c190011f-c1a6-47c7-986e-8d309b5f8704)
</Frame>

</Step>

**変更を公開します。**

<Frame>

</Step>
</Steps>

## LibreChat の設定 [#configure-librechat]

`librechat.yaml` 設定ファイルで `fileStrategy` を `firebase` に設定し、LibreChat がファイルストレージに Firebase を使用するようにします：

```yaml filename="librechat.yaml"
version: 1.3.5
cache: true
fileStrategy: 'firebase'
```

このファイルの詳細については、[librechat.yaml guide](/docs/configuration/librechat_yaml) を参照してください。

## PNGエクスポートのためにCORSを有効にする [#enable-cors-for-png-exports]

<Callout type="warning" title="PNGエクスポートにのみ必要">
会話をPNGとしてエクスポートすると、ブラウザからFirebase Storageの画像が直接取得されます。ドメインを許可するCORSポリシーがない場合、これらのリクエストはブロックされます。会話をPNGとしてエクスポートしない場合は、このセクションをスキップしてください。
</Callout>

<Steps>
<Step>

**CORS設定ファイルを作成します。** テキストエディタで `cors.json` を作成し、自身のドメインからのアクセスを許可します：

```json filename="cors.json"
[
  {
    "origin": ["https://ai.example.com"],
    "method": ["GET", "POST", "DELETE", "PUT"],
    "maxAgeSeconds": 3600
  }
]
```

</Step>
<Step>

**設定を適用します。** `cors.json` が含まれるディレクトリから、以下のコマンドを実行してください。その際、`<your-cloud-storage-bucket>` を実際のバケット名に置き換えてください：

```shell
gsutil cors set cors.json gs://<your-cloud-storage-bucket>
```

</Step>
<Step>

**設定を確認します。** アクティブなポリシーを取得し、それが `cors.json` と一致していることを確認してください。

```shell
gsutil cors get gs://<your-cloud-storage-bucket>
```

</Step>
<Step>

**テストします。** 許可されたオリジンから会話をPNGとしてエクスポートしてください。すべてが正しく設定されていれば、CORSエラーなしでエクスポートが成功します。

</Step>
</Steps>

<Callout type="info" title="セキュリティのヒント">
信頼できるオリジンに対してのみCORSを許可し、メソッドとヘッダーをデプロイメントで実際に必要なものに制限してください。
</Callout>
