Firebase Storage 与 Firebase Hosting 的全球 CDN 集成，让您可以通过世界各地的边缘节点提供存储在 Firebase Storage 中的文件。它是 LibreChat 支持 CDN 的文件存储选项之一，另一个是 [CloudFront for S3](/docs/configuration/cdn/cloudfront)。

<Callout type="info" title="您需要准备什么">
一个 Google 账号和大约 10 分钟的时间。你需要创建一个 Firebase 项目，启用 Cloud Storage，注册一个 Web 应用以获取凭据，然后将其指向 LibreChat。
</Callout>

![注册应用](https://github.com/danny-avila/LibreChat/assets/81851188/0a1b0a75-7285-4f03-95cf-bf971bd7d874)
</Frame>

</Step>
<Step>

**复制您的 `firebaseConfig` 值。** 将显示的配置保存到安全的地方。

<Frame>
![Firebase 配置值](https://github.com/danny-avila/LibreChat/assets/81851188/056754ad-9d36-4662-888e-f189ddb38fd3)
</Frame>

</Step>
<Step>

**将这些值添加到您的 `.env` 文件中。** 将每个 `firebaseConfig` 值映射到对应的变量：

```bash filename=".env"
FIREBASE_API_KEY=api_key                          # apiKey
FIREBASE_AUTH_DOMAIN=auth_domain                  # authDomain
FIREBASE_PROJECT_ID=project_id                    # projectId
FIREBASE_STORAGE_BUCKET=storage_bucket            # storageBucket
FIREBASE_MESSAGING_SENDER_ID=messaging_sender_id  # messagingSenderId
FIREBASE_APP_ID=1:your_app_id                     # appId
```

</Step>
</Steps>

## 更新存储规则 [#update-storage-rules]

<Steps>
<Step>

**Open Storage rules.** 返回 **Project Overview**，选择 **Storage**，然后打开 **Rules** 选项卡。

<Frame>
![Open Storage](https://github.com/danny-avila/LibreChat/assets/32828263/16a0f850-cdd4-4875-8342-ab67bfb59804)
</Frame>

</Step>
<Step>

**允许读写访问。** 将 `allow read, write: if false;` 修改为 `if true;`，使其符合下方的规则：

```js filename="storage.rules"
rules_version = '2';

service firebase.storage {
  match /b/{bucket}/o {
    match /images/{userId}/{fileName} {
      allow read, write: if true;
    }
  }
}
```

## 配置 LibreChat [#configure-librechat]

在你的 `librechat.yaml` 配置文件中将 `fileStrategy` 设置为 `firebase`，以便 LibreChat 使用 Firebase 进行文件存储：

```yaml filename="librechat.yaml"
version: 1.3.5
cache: true
fileStrategy: 'firebase'
```

有关此文件的更多信息，请参阅 [librechat.yaml 指南](/docs/configuration/librechat_yaml)。

## 启用 PNG 导出的 CORS [#enable-cors-for-png-exports]

<Callout type="warning" title="仅在 PNG 导出时需要">
将对话导出为 PNG 会在浏览器中直接从 Firebase Storage 获取图像。如果没有允许您域名的 CORS 策略，这些请求将被拦截。如果您不将对话导出为 PNG，则可以跳过此部分。
</Callout>

<Steps>
<Step>

**创建 CORS 配置文件。** 在文本编辑器中创建 `cors.json`，并允许来自您域名的访问：

```json filename="cors.json"
[
  {
    "origin": ["https://ai.example.com"],
    "method": ["GET", "POST", "DELETE", "PUT"],
    "maxAgeSeconds": 3600
  }
]
```

</Step>
<Step>

**应用配置。** 在包含 `cors.json` 的目录下，运行以下命令，并将 `<your-cloud-storage-bucket>` 替换为您的存储桶名称：

```shell
gsutil cors set cors.json gs://<your-cloud-storage-bucket>
```

</Step>
<Step>

**验证设置。** 获取当前生效的策略并确认其与 `cors.json` 一致：

```shell
gsutil cors get gs://<your-cloud-storage-bucket>
```

</Step>
<Step>

**进行测试。** 从你允许的来源将对话导出为 PNG。如果配置正确，导出将成功且不会出现 CORS 错误。

</Step>
</Steps>

<Callout type="info" title="安全提示">
仅允许受信任的源进行 CORS，并将方法和标头限制为您部署实际所需的内容。
</Callout>
