Firebase Storage 与 Firebase Hosting 的全球 CDN 集成，允许您通过世界各地的边缘节点提供存储在 Firebase Storage 中的文件。它是 LibreChat 支持 CDN 的文件存储选项之一，与用于 S3 的 CloudFront 并列。

## 设置 Firebase 的步骤 [#steps-to-set-up-firebase]

1. 打开 [Firebase 网站](https://firebase.google.com/)。
2. 点击“Get started”。
3. 使用您的 Google 账号登录。

### 在 `librechat.yaml` 中配置 `fileStrategy` [#configure-filestrategy-in-librechatyaml]

最后，要启用应用的 Firebase 功能，你必须在 `librechat.yaml` 配置文件中设置以下内容。

```yaml
version: 1.3.5
cache: true
fileStrategy: 'firebase'
```

有关 `librechat.yaml` 配置文件的更多信息，请参阅此处的指南：[自定义端点与配置](/docs/configuration/librechat_yaml)。

### 设置 Firebase Storage CORS 的分步指南 [#step-by-step-guide-to-set-up-cors-for-firebase-storage]

---

#### **第一步：创建 CORS 配置文件** [#step-1-create-the-cors-configuration-file]

- 打开您选择的文本编辑器。
- 创建一个新文件并将其命名为 `cors.json`。
- 添加以下配置以允许来自 "https://ai.example.com" 的访问：

```json
[
  {
    "origin": ["https://ai.example.com"],
    "method": ["GET", "POST", "DELETE", "PUT"],
    "maxAgeSeconds": 3600
  }
]
```

- 保存文件。

---

#### **Step 2: 应用 CORS 配置** [#step-2-apply-the-cors-configuration]

- 打开您的终端或命令提示符。
- 导航到您保存 `cors.json` 文件的目录。
- 执行以下命令，并将 `<your-cloud-storage-bucket>` 替换为您的 Firebase Storage 存储桶名称：

```shell
gsutil cors set cors.json gs://<your-cloud-storage-bucket>
```

---

#### **Step 3: 验证 CORS 设置** [#step-3-verify-the-cors-settings]

- 为了确认 CORS 设置已正确应用，您可以使用以下命令获取当前的 CORS 配置：

```shell
gsutil cors get gs://<your-cloud-storage-bucket>
```

- 输出应反映您在 `cors.json` 文件中指定的设置。

---

#### **第 4 步：测试配置** [#step-4-test-the-configuration]

- 尝试从允许的来源 ("https://ai.example.com") 将对话导出为 png。
- 如果一切配置正确，您应该不会遇到任何 CORS 问题。

---

> **注意：** 请务必确保仅为受信任的源应用 CORS 设置，以维护应用程序的安全性。请根据您的具体需求调整允许的方法和标头。

---

就是这样！您已成功为您的 Firebase Storage 存储桶配置了 CORS，以允许来自特定来源的请求。在应用配置时，请记得将 `<your-cloud-storage-bucket>` 替换为您实际的存储桶名称，并将 `https://ai.example.com` 替换为您自己的域名。

</Callout>
