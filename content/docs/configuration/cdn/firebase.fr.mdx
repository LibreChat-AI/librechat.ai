Firebase Storage s'intègre au CDN mondial de Firebase Hosting, vous permettant de servir les fichiers stockés dans Firebase Storage via des emplacements périphériques (edge locations) dans le monde entier. C'est l'une des options de stockage de fichiers supportées par un CDN pour LibreChat, aux côtés de CloudFront pour S3.

## Étapes pour configurer Firebase [#steps-to-set-up-firebase]

1. Ouvrez le [site web de Firebase](https://firebase.google.com/).
2. Cliquez sur « Get started. »
3. Connectez-vous avec votre compte Google.

### Configurer `fileStrategy` dans `librechat.yaml` [#configure-filestrategy-in-librechatyaml]

Enfin, pour permettre à l'application d'utiliser Firebase, vous devez définir les éléments suivants dans votre fichier de configuration `librechat.yaml`.

```yaml
version: 1.3.5
cache: true
fileStrategy: 'firebase'
```

Pour plus d'informations sur le fichier de configuration `librechat.yaml`, consultez le guide ici : [Custom Endpoints & Configuration](/docs/configuration/librechat_yaml).

### Guide étape par étape pour configurer CORS pour Firebase Storage [#step-by-step-guide-to-set-up-cors-for-firebase-storage]

---

#### **Étape 1 : Créer le fichier de configuration CORS** [#step-1-create-the-cors-configuration-file]

- Ouvrez l'éditeur de texte de votre choix.
- Créez un nouveau fichier et nommez-le `cors.json`.
- Ajoutez la configuration suivante pour autoriser l'accès depuis "https://ai.example.com" :

```json
[
  {
    "origin": ["https://ai.example.com"],
    "method": ["GET", "POST", "DELETE", "PUT"],
    "maxAgeSeconds": 3600
  }
]
```

- Enregistrez le fichier.

---

#### **Étape 2 : Appliquer la configuration CORS** [#step-2-apply-the-cors-configuration]

- Ouvrez votre terminal ou invite de commande.
- Accédez au répertoire où vous avez enregistré le fichier `cors.json`.
- Exécutez la commande suivante, en remplaçant `<your-cloud-storage-bucket>` par le nom de votre bucket Firebase Storage :

```shell
gsutil cors set cors.json gs://<your-cloud-storage-bucket>
```

---

#### **Étape 3 : Vérifier les paramètres CORS** [#step-3-verify-the-cors-settings]

- Pour confirmer que les paramètres CORS ont été appliqués correctement, vous pouvez récupérer la configuration CORS actuelle avec la commande suivante :

```shell
gsutil cors get gs://<your-cloud-storage-bucket>
```

- La sortie doit refléter les paramètres que vous avez spécifiés dans le fichier `cors.json`.

---

#### **Étape 4 : Tester la configuration** [#step-4-test-the-configuration]

- Essayez d'exporter une conversation au format png depuis l'origine autorisée ("https://ai.example.com").
- Si tout est configuré correctement, vous ne devriez rencontrer aucun problème de CORS.

---

> **Note :** Assurez-vous toujours d'appliquer les paramètres CORS uniquement pour les origines de confiance afin de maintenir la sécurité de votre application. Ajustez les méthodes et les en-têtes autorisés en fonction de vos besoins spécifiques.

---

C'est tout ! Vous avez configuré avec succès CORS pour votre bucket Firebase Storage afin d'autoriser les requêtes provenant d'une origine spécifique. N'oubliez pas de remplacer `<your-cloud-storage-bucket>` par le nom réel de votre bucket et `https://ai.example.com` par votre propre domaine lors de l'application de la configuration.

</Callout>
