<Callout type="info" title="Hinweis">
  LibreChat unterstützt lokalen Speicher, Objektspeicher-Backends und CDN-gestützte Bereitstellung. Verwenden Sie ein Objektspeicher-Backend wie S3 oder Azure Blob Storage für dauerhaften Dateispeicher und fügen Sie dann ein CDN wie CloudFront hinzu, wenn Sie stabile Medienlinks, Edge-Caching, signierte Cookies oder signierte Download-URLs benötigen.
</Callout>

## Dateispeicherung [#file-storage]

Durable Objektspeicher-Backends für Benutzer-Uploads, generierte Bilder und andere Dateien.

<Cards num={2}>
  <Cards.Card icon={<S3Icon />} title="Amazon S3" href="/docs/configuration/cdn/s3" arrow>
    Richten Sie einen Amazon S3-Bucket als dauerhaftes, skalierbares Objektspeicher-Backend ein.
  </Cards.Card>
  <Cards.Card icon={<AzureIcon />} title="Azure Blob Storage" href="/docs/configuration/cdn/azure" arrow>
    Verwenden Sie Azure Blob Storage-Container als Ihr Backend für die Dateispeicherung.
  </Cards.Card>
</Cards>

## CDN [#cdn]

Edge-gecachte Bereitstellung für stabile Medien-Links, signierte Cookies und signierte Download-URLs.

<Cards num={2}>
  <Cards.Card icon={<AWSIcon />} title="CloudFront mit S3" href="/docs/configuration/cdn/cloudfront" arrow>
    Stellen Sie S3-basierte Dateien über CloudFront für Edge-Caching und signierte URLs bereit.
  </Cards.Card>
  <Cards.Card icon={<FirebaseIcon />} title="Firebase CDN" href="/docs/configuration/cdn/firebase" arrow>
    Stellen Sie Dateien über Firebase Storage bereit, das durch das CDN von Google unterstützt wird.
  </Cards.Card>
</Cards>
