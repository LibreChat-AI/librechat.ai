<Callout type="info" title="Uwaga">
  LibreChat obsługuje pamięć lokalną, backendy pamięci obiektowej oraz dostarczanie treści przez CDN. Użyj backendu pamięci obiektowej, takiego jak S3 lub Azure Blob Storage, do trwałego przechowywania plików, a następnie dodaj CDN, taki jak CloudFront, gdy potrzebujesz stabilnych linków do mediów, buforowania brzegowego (edge caching), podpisanych plików cookie lub podpisanych adresów URL do pobierania.
</Callout>

## Przechowywanie plików [#file-storage]

Trwałe backendy przechowywania obiektów dla plików przesyłanych przez użytkowników, wygenerowanych obrazów oraz innych plików.

<Cards num={2}>
  <Cards.Card icon={<S3Icon />} title="Amazon S3" href="/docs/configuration/cdn/s3" arrow>
    Skonfiguruj zasobnik Amazon S3 jako trwały i skalowalny backend pamięci obiektowej.
  </Cards.Card>
  <Cards.Card icon={<AzureIcon />} title="Azure Blob Storage" href="/docs/configuration/cdn/azure" arrow>
    Użyj kontenerów Azure Blob Storage jako backendu do przechowywania plików.
  </Cards.Card>
</Cards>

## CDN [#cdn]

Dostarczanie z pamięci podręcznej typu edge dla stabilnych linków do mediów, podpisanych plików cookie oraz podpisanych adresów URL do pobierania.

<Cards num={2}>
  <Cards.Card icon={<AWSIcon />} title="CloudFront z S3" href="/docs/configuration/cdn/cloudfront" arrow>
    Serwuj pliki z S3 za pośrednictwem CloudFront w celu buforowania brzegowego (edge caching) i używania podpisanych adresów URL (signed URLs).
  </Cards.Card>
  <Cards.Card icon={<FirebaseIcon />} title="Firebase CDN" href="/docs/configuration/cdn/firebase" arrow>
    Dostarczaj pliki za pośrednictwem Firebase Storage wspieranego przez Google CDN.
  </Cards.Card>
</Cards>
