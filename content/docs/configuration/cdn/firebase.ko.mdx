Firebase Storage는 Firebase Hosting의 글로벌 CDN과 통합되어 전 세계 엣지 로케이션을 통해 Firebase Storage에 저장된 파일을 제공할 수 있게 해줍니다. 이는 [CloudFront for S3](/docs/configuration/cdn/cloudfront)와 더불어 LibreChat에서 지원하는 CDN 기반 파일 저장소 옵션 중 하나입니다.

<Callout type="info" title="준비물">
Google 계정과 약 10분의 시간이 필요합니다. Firebase 프로젝트를 생성하고, Cloud Storage를 활성화하고, 웹 앱을 등록하여 자격 증명을 얻은 다음, LibreChat에서 해당 설정을 지정하면 됩니다.
</Callout>

## Firebase 프로젝트 생성 [#create-a-firebase-project]

<Steps>
<Step>

**Firebase를 열고 로그인합니다.** [Firebase 웹사이트](https://firebase.google.com/)로 이동하여 **시작하기(Get started)**를 클릭하고 Google 계정으로 로그인합니다.

</Step>
<Step>

![웹 앱 추가](https://github.com/danny-avila/LibreChat/assets/81851188/22dab877-93cb-4828-9436-10e14374e57e)

![앱 등록](https://github.com/danny-avila/LibreChat/assets/81851188/0a1b0a75-7285-4f03-95cf-bf971bd7d874)

<Step>

**`firebaseConfig` 값을 복사하세요.** 표시된 구성을 안전한 곳에 저장하세요.

<Frame>
![Firebase 구성 값](https://github.com/danny-avila/LibreChat/assets/81851188/056754ad-9d36-4662-888e-f189ddb38fd3)
</Frame>

</Step>
<Step>

**Add the values to your `.env` file.** Map each `firebaseConfig` value to the matching variable:

```bash filename=".env"
FIREBASE_API_KEY=api_key                          # apiKey
FIREBASE_AUTH_DOMAIN=auth_domain                  # authDomain
FIREBASE_PROJECT_ID=project_id                    # projectId
FIREBASE_STORAGE_BUCKET=storage_bucket            # storageBucket
FIREBASE_MESSAGING_SENDER_ID=messaging_sender_id  # messagingSenderId
FIREBASE_APP_ID=1:your_app_id                     # appId
```

</Step>
</Steps>

## 스토리지 규칙 업데이트 [#update-storage-rules]

<Steps>
<Step>

**Open Storage rules.** **Project Overview**로 돌아가서 **Storage**를 선택한 다음, **Rules** 탭을 엽니다.

<Frame>
![Open Storage](https://github.com/danny-avila/LibreChat/assets/32828263/16a0f850-cdd4-4875-8342-ab67bfb59804)
</Frame>

</Step>
<Step>

**읽기 및 쓰기 권한을 허용합니다.** `allow read, write: if false;`를 아래 규칙과 일치하도록 `if true;`로 변경하세요:

```js filename="storage.rules"
rules_version = '2';

service firebase.storage {
  match /b/{bucket}/o {
    match /images/{userId}/{fileName} {
      allow read, write: if true;
    }
  }
}
```

## LibreChat 구성하기 [#configure-librechat]

`librechat.yaml` 설정 파일에서 `fileStrategy`를 `firebase`로 설정하여 LibreChat이 파일 저장소로 Firebase를 사용하도록 하세요:

```yaml filename="librechat.yaml"
version: 1.3.5
cache: true
fileStrategy: 'firebase'
```

이 파일에 대한 자세한 내용은 [librechat.yaml 가이드](/docs/configuration/librechat_yaml)를 참조하세요.

## PNG 내보내기를 위한 CORS 활성화 [#enable-cors-for-png-exports]

<Callout type="warning" title="PNG 내보내기에만 필요함">
대화를 PNG로 내보내는 기능은 브라우저에서 Firebase Storage로부터 이미지를 직접 가져옵니다. 귀하의 도메인을 허용하는 CORS 정책이 없으면 해당 요청은 차단됩니다. 대화를 PNG로 내보내지 않는다면 이 섹션은 건너뛰어도 됩니다.
</Callout>

<Steps>
<Step>

**CORS 구성 파일을 생성합니다.** 텍스트 편집기에서 `cors.json` 파일을 생성하고 귀하의 도메인에서 액세스를 허용하도록 설정합니다:

```json filename="cors.json"
[
  {
    "origin": ["https://ai.example.com"],
    "method": ["GET", "POST", "DELETE", "PUT"],
    "maxAgeSeconds": 3600
  }
]
```

</Step>
<Step>

**구성을 적용합니다.** `cors.json`이 포함된 디렉토리에서 아래 명령어를 실행하고, `<your-cloud-storage-bucket>`을 본인의 버킷 이름으로 변경하세요:

```shell
gsutil cors set cors.json gs://<your-cloud-storage-bucket>
```

</Step>
<Step>

**설정을 확인합니다.** 활성 정책을 검색하고 `cors.json`과 일치하는지 확인합니다:

```shell
gsutil cors get gs://<your-cloud-storage-bucket>
```

</Step>
<Step>

**테스트해 보세요.** 허용된 출처(origin)에서 대화를 PNG로 내보냅니다. 모든 설정이 올바르게 구성되었다면, CORS 오류 없이 내보내기가 성공할 것입니다.

</Step>
</Steps>

<Callout type="info" title="보안 팁">
신뢰할 수 있는 출처에 대해서만 CORS를 허용하고, 배포 환경에서 실제로 필요한 메서드와 헤더로 제한하십시오.
</Callout>
