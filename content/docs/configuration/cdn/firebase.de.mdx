Firebase Storage lässt sich in das globale CDN von Firebase Hosting integrieren, wodurch Sie Dateien, die in Firebase Storage gespeichert sind, über Edge-Standorte auf der ganzen Welt bereitstellen können. Es ist neben CloudFront für S3 eine der CDN-gestützten Dateispeicheroptionen von LibreChat.

## Schritte zur Einrichtung von Firebase [#steps-to-set-up-firebase]

1. Öffnen Sie die [Firebase website](https://firebase.google.com/).
2. Klicken Sie auf "Get started."
3. Melden Sie sich mit Ihrem Google-Konto an.

### Konfigurieren von `fileStrategy` in `librechat.yaml` [#configure-filestrategy-in-librechatyaml]

Schließlich müssen Sie Folgendes in Ihrer `librechat.yaml` Konfigurationsdatei festlegen, damit die App Firebase verwenden kann.

```yaml
version: 1.3.5
cache: true
fileStrategy: 'firebase'
```

Weitere Informationen zur Konfigurationsdatei `librechat.yaml` finden Sie im Leitfaden hier: [Custom Endpoints & Configuration](/docs/configuration/librechat_yaml).

<Callout type="warning" title="Konversationen als PNG exportieren bei Verwendung von Firebase">

### Schritt-für-Schritt-Anleitung zur Einrichtung von CORS für Firebase Storage [#step-by-step-guide-to-set-up-cors-for-firebase-storage]

---

#### **Schritt 1: Erstellen der CORS-Konfigurationsdatei** [#step-1-create-the-cors-configuration-file]

- Öffnen Sie einen Texteditor Ihrer Wahl.
- Erstellen Sie eine neue Datei und nennen Sie sie `cors.json`.
- Fügen Sie die folgende Konfiguration hinzu, um den Zugriff von "https://ai.example.com" zu erlauben:

```json
[
  {
    "origin": ["https://ai.example.com"],
    "method": ["GET", "POST", "DELETE", "PUT"],
    "maxAgeSeconds": 3600
  }
]
```

- Speichern Sie die Datei.

---

#### **Schritt 2: Die CORS-Konfiguration anwenden** [#step-2-apply-the-cors-configuration]

- Öffnen Sie Ihr Terminal oder Ihre Eingabeaufforderung.
- Navigieren Sie zu dem Verzeichnis, in dem Sie die Datei `cors.json` gespeichert haben.
- Führen Sie den folgenden Befehl aus und ersetzen Sie dabei `<your-cloud-storage-bucket>` durch den Namen Ihres Firebase Storage-Buckets:

```shell
gsutil cors set cors.json gs://<your-cloud-storage-bucket>
```

---

#### **Schritt 3: Überprüfen der CORS-Einstellungen** [#step-3-verify-the-cors-settings]

- Um zu bestätigen, dass die CORS-Einstellungen korrekt angewendet wurden, können Sie die aktuelle CORS-Konfiguration mit dem folgenden Befehl abrufen:

```shell
gsutil cors get gs://<your-cloud-storage-bucket>
```

- Die Ausgabe sollte die Einstellungen widerspiegeln, die Sie in der Datei `cors.json` festgelegt haben.

---

#### **Schritt 4: Konfiguration testen** [#step-4-test-the-configuration]

- Versuchen Sie, eine Konversation als PNG von dem erlaubten Ursprung ("https://ai.example.com") zu exportieren.
- Wenn alles korrekt eingerichtet ist, sollten keine CORS-Probleme auftreten.

---

> **Hinweis:** Stellen Sie immer sicher, dass Sie CORS-Einstellungen nur für vertrauenswürdige Ursprünge anwenden, um die Sicherheit Ihrer Anwendung zu gewährleisten. Passen Sie die erlaubten Methoden und Header an Ihre spezifischen Anforderungen an.

---

Das war's! Sie haben CORS für Ihren Firebase Storage-Bucket erfolgreich konfiguriert, um Anfragen von einem bestimmten Ursprung zuzulassen. Denken Sie daran, `<your-cloud-storage-bucket>` durch Ihren tatsächlichen Bucket-Namen und `https://ai.example.com` durch Ihre eigene Domain zu ersetzen, wenn Sie die Konfiguration anwenden.

</Callout>
