---
title: MongoDB Community Server
icon: MongoDB
description: Configurando um MongoDB Community Server para o seu banco de dados LibreChat.
---

<Steps>
<Step>

### Download do MongoDB Community Server [#download-mongodb-community-server]

- Vá para o site oficial do MongoDB: [https://www.mongodb.com/try/download/community](https://www.mongodb.com/try/download/community)
- Selecione seu sistema operacional e baixe o pacote apropriado.

</Step>
<Step>

### Instalar o MongoDB Community Server [#install-mongodb-community-server]

Siga as instruções de instalação para o seu sistema operacional para instalar o MongoDB Community Server.

</Step>
<Step>

### Criar um Diretório de Dados [#create-a-data-directory]

O MongoDB requer um diretório de dados para armazenar seus arquivos de dados. Crie um diretório em seu sistema onde você deseja armazenar os arquivos de dados do MongoDB (por exemplo, `/path/to/data/directory`).

</Step>
<Step>

### Inicie o servidor MongoDB [#start-the-mongodb-server]

- Abra um terminal ou prompt de comando.
- Navegue até o diretório de instalação do MongoDB (por exemplo, `/path/to/mongodb/bin`).
- Execute o seguinte comando para iniciar o servidor MongoDB, substituindo `/path/to/data/directory` pelo caminho para o diretório de dados que você criou na etapa anterior:

```sh filename="Start the MongoDB Server"
./mongod --dbpath=/path/to/data/directory
```

</Step>
<Step>

### Configurar o MongoDB para Acesso Remoto (Opcional) [#configure-mongodb-for-remote-access-optional]

Se você planeja acessar o servidor MongoDB a partir de um local remoto (por exemplo, uma máquina diferente ou uma instância do LibreChat hospedada em outro lugar), você precisa configurar o MongoDB para acesso remoto:

- Crie um arquivo de configuração (por exemplo, `/path/to/mongodb/config/mongodb.conf`) com o seguinte conteúdo:

```yaml filename="mongodb.conf"
# Network interfaces
net:
  port: 27017
  bindIp: 0.0.0.0
```
- Pare o servidor MongoDB se ele estiver em execução.
- Inicie o servidor MongoDB com o arquivo de configuração:

```yaml filename="Start the MongoDB server"
./mongod --config /path/to/mongodb/config/mongodb.conf
```

</Step>
<Step>

### Obtenha a Connection String [#get-the-connection-string]

A string de conexão para o seu MongoDB Community Server estará no seguinte formato:

```sh filename="Connection String"
mongodb://[hostname]:[port]
```
Substitua `[hostname]` pelo endereço IP ou hostname da máquina onde o MongoDB está sendo executado, e `[port]` pelo número da porta (geralmente 27017).

</Step>
<Step>

### Atualize o arquivo .env [#update-the-env-file]

- No seu projeto LibreChat, abra o arquivo `.env`.
- Encontre a variável `MONGO_URI` e cole sua string de conexão:

```sh filename=".env"
MONGO_URI=mongodb://[hostname]:[port]
```

</Step>
</Steps>

É isso! Você configurou agora um MongoDB Community Server para o LibreChat. Sua aplicação LibreChat deve ser capaz de se conectar à instância local do MongoDB usando a string de conexão que você forneceu.

## Nota sobre o Docker [#note-about-docker]

<Callout type="note" title="Docker">
**Nota:** Se você estiver usando o LibreChat com Docker, precisará utilizar o arquivo `docker-compose.override.yml`. Este arquivo de substituição permite que você impeça a instalação da instância do MongoDB incluída. Em vez disso, seu contêiner Docker do LibreChat usará o banco de dados local MongoDB Community Server que você acabou de configurar. Para obter mais informações sobre como usar o arquivo de substituição, consulte nosso [Guia de Substituição do Docker](/docs/configuration/docker_override).

**Exemplo:**
```yaml filename="docker-compose.override.yml"
services:
  api:
    environment:
    - MONGO_URI=mongodb://user:pass@host1:27017,host2:27017,host3:27017/LibreChat?authSource=admin&replicaSet=setname
```
</Callout>
