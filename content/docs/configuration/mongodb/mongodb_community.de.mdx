## 1. MongoDB Community Server herunterladen [#1-download-mongodb-community-server]
- Gehen Sie zur offiziellen MongoDB-Website: [https://www.mongodb.com/try/download/community](https://www.mongodb.com/try/download/community)
- Wählen Sie Ihr Betriebssystem aus und laden Sie das entsprechende Paket herunter.

## 2. MongoDB Community Server installieren [#2-install-mongodb-community-server]
Befolgen Sie die Installationsanweisungen für Ihr Betriebssystem, um den MongoDB Community Server zu installieren.

## 3. Erstellen eines Datenverzeichnisses [#3-create-a-data-directory]
MongoDB benötigt ein Datenverzeichnis, um seine Datendateien zu speichern. Erstellen Sie ein Verzeichnis auf Ihrem System, in dem Sie die MongoDB-Datendateien speichern möchten (z. B. `/path/to/data/directory`).

## 4. Starten des MongoDB-Servers [#4-start-the-mongodb-server]
- Öffnen Sie ein Terminal oder eine Eingabeaufforderung.
- Navigieren Sie zum Installationsverzeichnis von MongoDB (z. B. `/path/to/mongodb/bin`).
- Führen Sie den folgenden Befehl aus, um den MongoDB-Server zu starten, und ersetzen Sie dabei `/path/to/data/directory` durch den Pfad zu dem Datenverzeichnis, das Sie in Schritt 3 erstellt haben:

```sh filename="Start the MongoDB Server"
./mongod --dbpath=/path/to/data/directory
```

## 5. MongoDB für den Fernzugriff konfigurieren (Optional) [#5-configure-mongodb-for-remote-access-optional]
Wenn Sie planen, von einem entfernten Standort (z. B. einem anderen Rechner oder einer anderswo gehosteten LibreChat-Instanz) auf den MongoDB-Server zuzugreifen, müssen Sie MongoDB für den Fernzugriff konfigurieren:

- Erstellen Sie eine Konfigurationsdatei (z. B. `/path/to/mongodb/config/mongodb.conf`) mit folgendem Inhalt:

```yaml filename="mongodb.conf"
# Network interfaces
net:
  port: 27017
  bindIp: 0.0.0.0
```
- Stoppen Sie den MongoDB-Server, falls er läuft.
- Starten Sie den MongoDB-Server mit der Konfigurationsdatei:

```yaml filename="Start the MongoDB server"
./mongod --config /path/to/mongodb/config/mongodb.conf
```

## 6. Verbindungszeichenfolge abrufen [#6-get-the-connection-string]
Die Verbindungszeichenfolge für Ihren MongoDB Community Server hat das folgende Format:

```sh filename="Connection String"
mongodb://[hostname]:[port]
```
Ersetzen Sie `[hostname]` durch die IP-Adresse oder den Hostnamen des Rechners, auf dem MongoDB läuft, und `[port]` durch die Portnummer (üblicherweise 27017).

## 7. Aktualisieren der .env Datei [#7-update-the-env-file]
- Öffnen Sie in Ihrem LibreChat-Projekt die `.env` Datei.
- Suchen Sie die Variable `MONGO_URI` und fügen Sie Ihren Verbindungs-String ein:

```sh filename=".env"
MONGO_URI=mongodb://[hostname]:[port]
```

Das war's! Sie haben nun einen MongoDB Community Server für LibreChat eingerichtet. Ihre LibreChat-Anwendung sollte in der Lage sein, sich mit der lokalen MongoDB-Instanz unter Verwendung des von Ihnen bereitgestellten Verbindungs-Strings zu verbinden.

## Hinweis zu Docker [#note-about-docker]

<Callout type="note" title="Docker">
**Hinweis:** Wenn Sie LibreChat mit Docker verwenden, müssen Sie die Datei `docker-compose.override.yml` nutzen. Diese Override-Datei ermöglicht es Ihnen, die Installation der enthaltenen MongoDB-Instanz zu verhindern. Stattdessen wird Ihr LibreChat Docker-Container die lokale MongoDB Community Server-Datenbank verwenden, die Sie gerade eingerichtet haben. Weitere Informationen zur Verwendung der Override-Datei finden Sie in unserem [Docker Override Guide](/docs/configuration/docker_override).

**Beispiel:**
```yaml filename="docker-compose.override.yml"
services:
  api:
    environment:
    - MONGO_URI=mongodb://user:pass@host1:27017,host2:27017,host3:27017/LibreChat?authSource=admin&replicaSet=setname
```
</Callout>
