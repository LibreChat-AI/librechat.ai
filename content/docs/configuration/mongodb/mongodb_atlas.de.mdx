## 1. Erstellen eines MongoDB Atlas-Kontos [#1-create-a-mongodb-atlas-account]
1. Öffnen Sie einen neuen Tab in Ihrem Webbrowser und gehen Sie zu [account.mongodb.com/account/register](https://account.mongodb.com/account/register).
2. Geben Sie die erforderlichen Informationen ein und erstellen Sie Ihr Konto.

## 2. Neues Projekt erstellen [#2-create-a-new-project]
1. Nachdem Sie Ihr Konto eingerichtet haben, klicken Sie auf die Schaltfläche „New Project“ und geben Sie ihm einen Namen (z. B. „LibreChat“).

## 3. Datenbank erstellen [#3-build-a-database]
1. Klicken Sie auf die Schaltfläche "Build a Database".

## 4. Wählen Sie den Free Tier [#4-choose-the-free-tier]
1. Wählen Sie die Option "Shared Clusters", bei der es sich um den kostenlosen Tarif handelt.

## 5. Benennen Sie Ihren Cluster [#5-name-your-cluster]
1. Geben Sie Ihrem Cluster einen Namen (z. B. "LibreChat-Cluster") und klicken Sie auf "Create Cluster".

## 6. Datenbank-Anmeldedaten einrichten [#6-set-up-database-credentials]
1. Klicken Sie auf die Option "Database & Network Access" in der Seitenleiste.
2. Klicken Sie auf die Schaltfläche "Add New Database User".
3. Geben Sie einen Benutzernamen und ein sicheres Passwort ein und klicken Sie dann auf "Add User".

## 7. Netzwerkkonfiguration konfigurieren [#7-configure-network-access]
1. Klicken Sie auf die Option "IP Access List" in der Seitenleiste.
2. Klicken Sie auf die Schaltfläche "Add IP Address".
3. Geben Sie "0.0.0.0/0" ein und klicken Sie auf "Confirm".

## 8. Verbindungszeichenfolge abrufen [#8-get-your-connection-string]
1. Klicken Sie auf die Option "Project Overview" in der Seitenleiste.
2. Klicken Sie unter "Application Development" auf "Get connection string".
2. Klicken Sie auf die Schaltfläche "Connect".
3. Wählen Sie "Connect Your Application".
4. Kopieren Sie die bereitgestellte Verbindungszeichenfolge.
5. Ersetzen Sie `<db_username>` und `<db_password>` im Verbindungs-String durch den Benutzernamen und das Passwort, die Sie in Schritt 6 festgelegt haben. Entfernen Sie die `<>`-Zeichen um das Passwort.

Ihr finaler Verbindungs-String sollte in etwa so aussehen:

```sh filename="Connection String"
mongodb+srv://username:password@cluster-url.mongodb.net/LibreChat?retryWrites=true
```

## 9. Aktualisieren der .env Datei [#9-update-the-env-file]
1. Öffnen Sie in Ihrem LibreChat-Projekt die `.env` Datei.
2. Suchen Sie die Variable `MONGO_URI` und fügen Sie Ihren Verbindungs-String ein:

```sh filename=".env"
MONGO_URI=mongodb+srv://username:password@cluster-url.mongodb.net/LibreChat?retryWrites=true
```

Das war's! Sie haben nun eine Online-MongoDB-Datenbank für LibreChat mit MongoDB Atlas eingerichtet und Ihre LibreChat-Anwendung aktualisiert, um diese Datenbankverbindung zu nutzen. Ihre Anwendung sollte nun in der Lage sein, eine Verbindung zur Online-MongoDB-Datenbank herzustellen.

## Hinweis zu Docker [#note-about-docker]

<Callout type="note" title="Docker">
**Hinweis:** Wenn Sie LibreChat mit Docker verwenden, müssen Sie die Datei `docker-compose.override.yml` nutzen. Diese Override-Datei ermöglicht es Ihnen, die Installation der enthaltenen MongoDB-Instanz zu verhindern. Stattdessen wird Ihr LibreChat Docker-Container die von Ihnen gerade eingerichtete Online-Datenbank MongoDB Atlas verwenden. Weitere Informationen zur Verwendung der Override-Datei finden Sie in unserem [Docker Override Guide](/docs/configuration/docker_override).
</Callout>
