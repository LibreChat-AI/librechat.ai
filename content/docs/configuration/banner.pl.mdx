Wyświetlaj ważne komunikaty wszystkim użytkownikom na górze aplikacji. Idealne rozwiązanie do ogłoszeń, powiadomień o konserwacji lub aktualizacjach.

<Callout type="info" title="Szybki przegląd">
  - Tylko **jeden baner** może być aktywny jednocześnie
  - Zaplanuj automatyczne wyświetlanie i ukrywanie banerów
  - Wybierz, czy użytkownicy mogą zamknąć baner, czy nie
  - Wyświetlaj banery wszystkim lub tylko zalogowanym użytkownikom
</Callout>

---

## Tworzenie banera [#creating-a-banner]

Uruchom to polecenie, aby utworzyć lub zaktualizować baner:

```bash
npm run update-banner
```

Zostaniesz przeprowadzony przez kilka prostych monitów:

```ansi
--------------------------
Update the banner!
--------------------------
Display From (Format: yyyy-mm-ddTHH:MM:SSZ, Default: now):
> 2025-12-02T09:00:00Z

Display To (Format: yyyy-mm-ddTHH:MM:SSZ, Default: not specified):
> 2025-12-31T23:59:59Z

Enter your message (Enter a single dot "." on a new line to finish):
> 🎉 Welcome to LibreChat! Check out our new features.
> .

Is public (y/N):
> n

Is persistable (cannot be dismissed) (y/N):
> n
```

<Callout type="tip" title="Co oznaczają te opcje?">
  - **Wyświetl od/do**: Kiedy baner ma się pojawić i zniknąć. Pozostaw puste dla ustawień „teraz” i „zawsze”
  - **Is public**: Pokaż odwiedzającym, którzy nie są zalogowani (np. na stronie logowania)
  - **Is persistable**: Jeśli tak, użytkownicy nie mogą zamknąć banera — użyj w przypadku ważnych powiadomień
</Callout>

## Usuwanie banera [#deleting-a-banner]

```bash
npm run delete-banner
```

Zobaczysz bieżący baner i zostaniesz poproszony o potwierdzenie przed usunięciem.

## Format daty [#date-format]

Użyj tego formatu dla dat: `yyyy-mm-ddTHH:MM:SSZ`

**Przykłady:**
- `2025-12-25T09:00:00Z` → 25 grudnia 2025 r. o godz. 9:00 (UTC)
- `2025-01-01T00:00:00Z` → 1 stycznia 2025 o północy (UTC)

Pozostaw datę pustą, aby użyć bieżącego czasu lub braku daty końcowej.
