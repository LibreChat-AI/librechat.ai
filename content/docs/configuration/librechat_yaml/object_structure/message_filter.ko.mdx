## 개요 [#overview]

`messageFilter` 객체는 서버 측 메시지 필터를 구성합니다. v1.3.13 버전에서 지원되는 필터 유형은 `messageFilter.pii`입니다.

`messageFilter.pii`는 제출된 메시지 텍스트가 조정(moderation), 모델 요청 또는 메시지 지속성(persistence)에 도달하기 전에 자격 증명 형태의 시작 패턴이나 운영자가 정의한 정규 표현식과 일치하는 경우 해당 텍스트를 거부합니다.

## 예시 [#example]

```yaml filename="librechat.yaml"
messageFilter:
  pii:
    starterPatterns:
      - sk_prefix
      - bearer_header
      - api_key_header
    customPatterns:
      - id: anthropic_api_key
        label: Anthropic API key
        regex: 'sk-ant-[A-Za-z0-9_-]{20,}'
```

## 동작 [#behavior]

제출된 메시지 텍스트가 구성된 PII 패턴과 일치하면, LibreChat은 `message_filter_pii_block`과 함께 HTTP `400` 응답을 반환하며 호출자에게 일치하는 자격 증명 유형을 제거하도록 요청하는 메시지를 보냅니다.

필터는 다음을 확인합니다:

- LibreChat UI를 통해 제출된 채팅 경로 사용자 텍스트입니다.
- OpenAI 호환 Chat Completions 요청에서 호출자가 제공한 모든 메시지 역할에 걸친 텍스트 콘텐츠입니다.
- OpenAI 호환 응답 요청의 모든 호출자 제공 메시지 역할에 걸친 텍스트 콘텐츠입니다.

이 필터는 요청 본문(request body)에 제공된 텍스트를 평가합니다. 파일 내용은 스캔하지 않습니다.
