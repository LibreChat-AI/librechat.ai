Chaque élément sous `models` fait partie d'une liste d'enregistrements, soit une valeur booléenne, soit un Objet :

**Lors de la spécification d'un modèle en tant qu'objet :**

Un objet permet une configuration détaillée du modèle, incluant son `deploymentName` et/ou sa `version`. Ce mode est utilisé pour un contrôle plus granulaire sur les modèles, particulièrement lors de l'utilisation de plusieurs versions ou déploiements sous une même instance ou un même groupe de ressources.

**Exemple** :
```yaml filename="endpoints / azureOpenAI / groups / {group_item} / models / {model_item=Object}"
models:
  gpt-4-vision-preview:
    deploymentName: "gpt-4-vision-preview"
    version: "2024-02-15-preview"
```

**Notes :**
- Les **Deployment Names** et les **Versions** sont essentiels pour garantir que le modèle correct est utilisé.
    - Vérifiez deux fois ces valeurs pour en assurer l'exactitude et éviter tout comportement inattendu.
