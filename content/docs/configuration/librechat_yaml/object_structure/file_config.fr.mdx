## Vue d'ensemble [#overview]

L'objet `fileConfig` vous permet de configurer les paramètres de gestion des fichiers pour l'application, y compris les limites de taille et les restrictions de type MIME. Cette section fournit une analyse détaillée de la structure de l'objet `fileConfig`.

Il y a 8 champs principaux sous `fileConfig` :

  - `endpoints`
  - `serverFileSizeLimit`
  - `avatarSizeLimit`
  - `imageGeneration`
  - `fileTokenLimit`
  - `ocr`
  - `text`
  - `stt`

**Notes :**

- Au moment de la rédaction, l'endpoint Assistants [prend en charge les types de fichiers de cette liste](https://platform.openai.com/docs/assistants/tools/file-search#supported-files).
- Les endpoints OpenAI, Azure OpenAI, Google et Custom prennent en charge les fichiers via la [RAG API.](../../rag_api.mdx)
- Les endpoints OpenAI, Azure OpenAI, Anthropic, Google et AWS Bedrock prennent en charge le téléchargement direct de fichiers via [Upload to Provider](/docs/features/ocr#5-upload-files-to-provider-direct).
- Les sections `ocr`, `text` et `stt` contrôlent le traitement des fichiers pour des fonctionnalités telles que [Upload as Text](/docs/features/upload_as_text) et [OCR](/docs/features/ocr)
- Tous les autres endpoints non mentionnés, comme les Plugins, ne prennent pas (encore) en charge le téléchargement de fichiers.
- L'endpoint Assistants a une valeur d'endpoint définie sur `assistants`. Tous les autres endpoints utilisent la valeur définie `default`.
  - Pour les endpoints autres que les assistants, vous pouvez ajuster les paramètres de fichiers pour chacun d'entre eux sous `default`
  - Si vous souhaitez ajuster les paramètres pour un endpoint spécifique, vous pouvez lister leurs noms d'endpoint correspondants :
    - `assistants`
        - n'utilise pas "default" car il possède des valeurs par défaut définies séparément des autres.
    - `openAI`
    - `azureOpenAI`
    - `google`
    - `bedrock`
    - `YourCustomEndpointName`
- Vous pouvez omettre des valeurs, auquel cas l'application utilisera les valeurs par défaut telles que définies pour chaque type d'endpoint listé ci-dessous.
- LibreChat compte 1 mégaoctet comme suit : `1 x 1024 x 1024`

## Exemple [#example]

```yaml filename="fileConfig"
fileConfig:
  endpoints:
    assistants:
      fileLimit: 5
      fileSizeLimit: 10
      totalSizeLimit: 50
      supportedMimeTypes:
        - "image/.*"
        - "application/pdf"
    openAI:
      disabled: true
    default:
      totalSizeLimit: 20
    YourCustomEndpointName:
      fileLimit: 5
      fileSizeLimit: 1000
      supportedMimeTypes:
        - "image/.*"
  serverFileSizeLimit: 1000
  avatarSizeLimit: 2
  fileTokenLimit: 100000
  imageGeneration:
    percentage: 100
    px: 1024
  ocr:
    supportedMimeTypes:
      - "^image/(jpeg|gif|png|webp|heic|heif)$"
      - "^application/pdf$"
      - "^application/vnd\\.openxmlformats-officedocument\\.(wordprocessingml\\.document|presentationml\\.presentation|spreadsheetml\\.sheet)$"
      - "^application/vnd\\.ms-(word|powerpoint|excel)$"
      - "^application/epub\\+zip$"
  text:
    supportedMimeTypes:
      - "^text/(plain|markdown|csv|json|xml|html|css|javascript|typescript|x-python|x-java|x-csharp|x-php|x-ruby|x-go|x-rust|x-kotlin|x-swift|x-scala|x-perl|x-lua|x-shell|x-sql|x-yaml|x-toml)$"
  stt:
    supportedMimeTypes:
      - "^audio/(mp3|mpeg|mpeg3|wav|wave|x-wav|ogg|vorbis|mp4|x-m4a|flac|x-flac|webm)$"
```

## imageGeneration

<OptionTable
  options={[
    ['imageGeneration', 'Object', 'Paramètres liés à la qualité et aux dimensions de la sortie de génération d\'images.', 'Allows configuration of either output size as a percentage relative to some base size or as an explicit pixel dimension.'],
  ]}
/>

`imageGeneration` prend en charge les paramètres suivants :

- `percentage` (Entier)
  - La taille de sortie de l'image générée exprimée en pourcentage (par ex., `100` signifie 100 % de la taille de base).
  - Utilisez ceci pour mettre à l'échelle l'image de sortie par rapport à une taille par défaut ou originale.

- `px` (Entier)
  - Spécifie la dimension de l'image de sortie en pixels (par ex. `1024`).
  - Utilisez ceci pour définir explicitement la taille de sortie de l'image générée, indépendamment de la taille de base.

Vous pouvez définir un seul de ces paramètres (`percentage` ou `px`), et non les deux, en fonction de votre cas d'utilisation.

Exemple de configuration :

```yaml filename="fileConfig / imageGeneration"
fileConfig:
  imageGeneration:
    percentage: 100
    px: 1024
```

## text

<OptionTable
  options={[
    ['text', 'Object', 'Paramètres pour l\'analyse directe de fichiers texte sans OCR.', 'Configures which file types are processed as plain text files for direct content extraction.'],
  ]}
/>

**Description :** La section `text` configure les types de fichiers qui doivent être traités en utilisant l'extraction directe de texte.

**Note :** L'analyse de texte est la méthode par défaut utilisée par la fonctionnalité « Téléverser en tant que texte » (contrôlée par la capacité `context`). Elle tente d'abord d'utiliser la bibliothèque d'analyse de texte de l'API RAG, et si l'API RAG n'est pas connectée, elle utilise une méthode d'extraction de texte plus simple sans nécessiter de services externes. Voir [Upload as Text](/docs/features/upload_as_text) pour plus d'informations.

## stt

<OptionTable
  options={[
    ['stt', 'Object', 'Paramètres pour le traitement des fichiers audio de synthèse vocale (STT).', 'Configures which audio file types are processed using STT for transcription.'],
  ]}
/>

**Description :** La section `stt` configure les types de fichiers audio qui doivent être traités à l'aide de la fonctionnalité Speech-to-Text pour convertir l'audio en texte.

### supportedMimeTypes [#supportedmimetypes-2]

<OptionTable
  options={[
    ['supportedMimeTypes', 'Array of Strings', 'Liste des modèles de types MIME pour les fichiers audio qui doivent être transcrits avec STT.', 'Uses regular expressions to match audio file types.'],
  ]}
/>

**Par défaut :** Formats audio courants

```yaml filename="fileConfig / stt / supportedMimeTypes"
fileConfig:
  stt:
    supportedMimeTypes:
      - "^audio/(mp3|mpeg|mpeg3|wav|wave|x-wav|ogg|vorbis|mp4|x-m4a|flac|x-flac|webm)$"
```

**Notes :**
- Les fichiers correspondant aux modèles `text` sont traités avec une extraction de texte simple
- Les fichiers correspondant aux modèles `ocr` sont traités avec le service OCR fourni
- Les fichiers correspondant aux modèles `stt` sont traités avec la transcription Speech-to-Text.
- **Priorité de traitement : OCR > STT > analyse de texte > repli**
- Les fichiers ne correspondant à aucun modèle seront traités par une analyse textuelle par défaut.

## Priorité de traitement des fichiers [#file-processing-priority]

LibreChat traite les fichiers téléchargés en fonction de la correspondance du type MIME selon l'**ordre de priorité** suivant :

1. **OCR** - Si le fichier correspond à `ocr.supportedMimeTypes` ET que l'OCR est configuré
2. **STT** - Si le fichier correspond à `stt.supportedMimeTypes` ET que STT est configuré
3. **Analyse de texte** - Si le fichier correspond à `text.supportedMimeTypes`
4. **Fallback** - Analyse de texte en dernier recours

Cet ordre de traitement garantit une qualité d'extraction optimale tout en conservant les fonctionnalités, même lorsque les services spécialisés (OCR/STT) ne sont pas configurés.

### Exemples de traitement [#processing-examples]

**Fichier PDF avec OCR configuré :**
- Le fichier correspond à `ocr.supportedMimeTypes`
- **Utilise l'OCR** pour extraire du texte
- Meilleure qualité pour les PDF numérisés et les images

**Fichier PDF sans OCR configuré :**
- Le fichier correspond à `text.supportedMimeTypes` (ou utilise le repli)
- **Utilise la bibliothèque** de parsing de texte
- Fonctionne bien pour les PDF numériques avec du texte sélectionnable

**Fichier Python :**
- Le fichier correspond à `text.supportedMimeTypes`
- **Utilise l'analyse de texte** (aucune OCR nécessaire)
- Extraction directe de texte

**Fichier audio avec STT configuré :**
- Le fichier correspond à `stt.supportedMimeTypes`
- **Utilise STT** pour transcrire l'audio en texte

**Fichier image sans OCR configuré :**
- Le fichier correspond à `ocr.supportedMimeTypes` mais l'OCR n'est pas disponible
- **Revient à l'analyse de texte**
- Capacité d'extraction limitée sans OCR

Ce système de priorité permet à des fonctionnalités telles que « Upload as Text » de fonctionner sans nécessiter de configuration OCR, tout en tirant parti de l'OCR lorsqu'il est disponible pour une meilleure qualité d'extraction.

## endpoints

<OptionTable
  options={[
    ['endpoints', 'Record/Object', 'Configure les paramètres de gestion des fichiers pour chaque endpoint, permettant une personnalisation par endpoint.', 'Specifies file handling configurations for individual endpoints, allowing customization per endpoint basis.'],
  ]}
/>

**Description :** Chaque objet sous endpoints est un enregistrement qui peut comporter les paramètres suivants :

### Vue d'ensemble [#overview-1]

  - `disabled`
      - Si la gestion des fichiers est désactivée pour l'endpoint.
  - `fileLimit`
      - Le nombre maximum de fichiers autorisés par requête de téléchargement.
  - `fileSizeLimit`
      - La taille maximale pour un seul fichier. En unités de Mo (par exemple, utilisez `20` pour 20 mégaoctets)
  - `totalSizeLimit`
      - La taille maximale totale pour tous les fichiers dans une seule requête. En unités de Mo (par exemple, utilisez `20` pour 20 mégaoctets)
  - `supportedMimeTypes`
      - Une liste d'[Expressions régulières](https://en.wikipedia.org/wiki/Regular_expression) spécifiant quels types MIME sont autorisés pour le téléchargement. Ceci peut être personnalisé pour restreindre les types de fichiers.
