Cada elemento bajo `models` es parte de una lista de registros, ya sea un valor booleano o un Object:

**Al especificar un modelo como un objeto:**

Un objeto permite una configuración detallada del modelo, incluyendo su `deploymentName` y/o `version`. Este modo se utiliza para un control más granular sobre los modelos, especialmente al trabajar con múltiples versiones o despliegues bajo una misma instancia o grupo de recursos.

**Ejemplo**:
```yaml filename="endpoints / azureOpenAI / groups / {group_item} / models / {model_item=Object}"
models:
  gpt-4-vision-preview:
    deploymentName: "gpt-4-vision-preview"
    version: "2024-02-15-preview"
```

**Notas:**
- Los **Deployment Names** y **Versions** son fundamentales para garantizar que se utilice el modelo correcto.
    - Verifica dos veces estos valores para asegurar su precisión y evitar comportamientos inesperados.
