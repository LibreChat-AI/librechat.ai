## 概要 [#overview]

`interface` オブジェクトを使用すると、メニュー、パネル、リンクなどのコンポーネントの表示設定や動作設定を含め、アプリケーション内のさまざまなユーザーインターフェース要素をカスタマイズできます。このセクションでは、`interface` オブジェクトの構造について詳しく解説します。

これらは `interface` 配下のフィールドです：

- `mcpServers`
- `privacyPolicy`
- `termsOfService`
- `modelSelect`
- `parameters`
- `contextUsage`
- `contextCost`
- `currency`
- `presets`
- `prompts`
- `bookmarks`
- `memories`
- `multiConvo`
- `agents`
- `remoteAgents`
- `skills`
- `sharedLinks`
- `temporaryChat`
- `temporaryChatRetention`
- `retentionMode`
- `retainAgentFiles`
- `autoSubmitFromUrl`
- `customWelcome`
- `runCode`
- `webSearch`
- `fileSearch`
- `fileCitations`
- `defaultPinnedTools`
- `peoplePicker`
- `marketplace`

**注記:**

- `interface` 設定は、アプリケーション全体にグローバルに適用されます。
- ほとんどの設定にはデフォルト値が提供されていますが、特定の要件や条件に基づいて上書きすることが可能です。
- アプリケーション内の条件付きロジックは、Model Specsのような他の設定に基づいて、これらの設定をさらに変更することができます。

<Callout type="warning" title="非推奨：権限の副作用フィールド">
以下のいくつかのフィールド（`mcpServers`、`prompts`、`bookmarks`、`memories`、`multiConvo`、`agents`、`remoteAgents`、`skills`、`sharedLinks`、`temporaryChat`、`runCode`、`webSearch`、`fileSearch`、`fileCitations`、`peoplePicker`、`marketplace`）は、単にUIを切り替えるだけでなく、起動時にデータベース内の組み込みの `USER` ロールに対してロール権限をシード（初期設定）します。

継続的な管理には、各ロール（カスタムロールを含む）の権限マトリックスを直接編集できる [**LibreChat Admin Panel**](/docs/features/admin_panel) を使用してください。これらのYAMLフィールドは、新規インスタンスのブートストラップや完全にファイル駆動型のデプロイメントでは引き続きサポートされますが、機能権限を管理する主要な方法として使用すべきではありません。

完全な権限モデルについては、[Access Control](/docs/features/access_control) を参照してください。

</Callout>

## 例 [#example]

```yaml filename="interface"
interface:
  mcpServers:
    placeholder: 'MCP Servers'
    use: true
    create: true
    share: false
    public: false
    configureObo: false
    trustCheckbox:
      label: 'I trust this server'
      subLabel: 'Only enable servers you trust'
  privacyPolicy:
    externalUrl: 'https://example.com/privacy'
    openNewTab: true
  termsOfService:
    externalUrl: 'https://example.com/terms'
    openNewTab: true
    modalAcceptance: true
    modalTitle: 'Terms of Service'
    modalContent: |
      # Terms of Service
      ## Introduction
      Welcome to LibreChat!
  modelSelect: false
  parameters: true
  contextUsage: true
  contextCost: true
  currency:
    code: EUR
    rate: 0.92
  presets: false
  prompts:
    use: true
    create: true
    share: true
    public: false
  bookmarks: true
  multiConvo: true
  agents:
    use: true
    create: true
    share: true
    public: false
  skills:
    use: true
    create: true
    share: false
    public: false
    defaultActiveOnShare: false
  sharedLinks:
    create: true
    share: true
    public: false
    snapshotFiles: true
  customWelcome: 'Hey {{user.name}}! Welcome to LibreChat'
  runCode: true
  webSearch: true
  fileSearch: true
  fileCitations: true
  defaultPinnedTools:
    - artifacts
    - execute_code
    - mcp
```

## prompts

> **権限管理のため非推奨となりました。** 起動時にデフォルトの `USER` ロールに対してのみ `PROMPTS` ロールの権限をシードします。ロール、グループ、ユーザーごとのプロンプト権限の管理には、[Admin Panel](/docs/features/admin_panel) を使用することを推奨します。

**キー:**

<OptionTable
  options={[
    [
      'prompts',
      'Boolean or Object',
      '全ユーザーのプロンプト関連機能を制御します。単純な有効/無効の切り替えにはブール値を、使用、作成、共有、公開範囲のきめ細かな制御にはオブジェクトを使用できます。',
      'When set to `false`, users will not have access to create, edit, or use custom prompts.',
    ],
  ]}
/>

**デフォルト:** `true`

**重要: ブール値とオブジェクトの設定について**

- **Boolean (`prompts: true`)**: `use` 権限のみを更新します。既存の `create`、`share`、および `public` 権限の値はデータベースから**保持**されます。管理パネルを通じて設定された他の設定に影響を与えることなく、シンプルな機能トグルとしてこれを使用してください。

- **Object**: 明示的に指定されたサブ権限のみを更新します。設定に含まれていない権限は、データベース内の既存の値が保持されます。

オブジェクト構造を使用する場合：

**サブキー:**

<OptionTable
  options={[
    ['use', 'Boolean', 'ユーザーがプロンプトを使用できるかどうかを制御します。', 'true'],
    ['create', 'Boolean', 'ユーザーが新しいプロンプトを作成できるかどうかを制御します。', 'true'],
    [
      'share',
      'Boolean',
      'ユーザーが特定のユーザーやグループとプロンプトを共有できるかどうかを制御します。',
      'false',
    ],
    [
      'public',
      'Boolean',
      'ユーザーがプロンプトを公開（全ユーザーに表示）できるかどうかを制御します。',
      'false',
    ],
  ]}
/>

**例 (boolean - シンプルな機能トグル):**

```yaml filename="interface / prompts (boolean)"

  prompts: true # Only updates USE; create/share/public remain unchanged
```

**例 (オブジェクト - 詳細な制御):**

```yaml filename="interface / prompts (object)"

  prompts:
    use: true
    create: false # Disable creation while allowing use
    # share and public not specified - preserves existing values
```

**例 (オブジェクト - 完全な制御):**

```yaml filename="interface / prompts (object)"

  prompts:
    use: true
    create: true
    share: true
    public: false
```

## autoSubmitFromUrl

`/c/new` のURLクエリパラメータ経由で提供されたプロンプトを、モデルに対して自動送信するかどうかを制御します。

認証済みのユーザーが `/c/new?prompt=…&submit=true` を開くと、LibreChatは通常、URLで指定されたプロンプトをコンポーザーに自動入力し、即座に送信します。これは、作成されたディープリンクや共有チャットURLのための便利な機能です。

外部ソースから作成されたリンクをユーザーが受け取る可能性があり、かつプロンプトインジェクションのペイロードがモデルに到達した場合にメモリやツールが有効なモデルが機密コンテキストを漏洩する可能性があるデプロイメントでは、オペレーターは自動送信を無効にすることができます。このフラグを `false` に設定すると、プロンプトはコンポーザーに事前入力された状態になりますが、ユーザーは明示的に **Send** を押す必要があります。

**キー:**

<OptionTable
  options={[
    [
      'autoSubmitFromUrl',
      'Boolean',
      '`/c/new?prompt=…&submit=true` がモデルに対して自動送信されるかどうかを制御します。',
      'When `false`, the prompt is pre-filled in the composer but not submitted.',
    ],
  ]}
/>

**デフォルト:** `true` (明示的に無効化されない限り、既存の動作が維持されます)。

**注記:**

- この設定は、URLベースのModel Specsの選択やその他のURLベースの設定には影響しません。自動送信のステップのみに影響します。
- クエリパラメータは、プロンプトのソースとして `prompt` と `q` の両方を受け付けますが、`prompt` が優先されます。`submit=true` がトリガーとなります。
- 機密性の高いメモリやツールデータを扱うインスタンスに推奨されます。このような環境では、ワンクリックのプロンプトインジェクション攻撃を防ぐため、ユーザーによる明示的な確認を必須とすべきです。

**例:**

```yaml filename="interface / autoSubmitFromUrl"
interface:
  autoSubmitFromUrl: false
```
