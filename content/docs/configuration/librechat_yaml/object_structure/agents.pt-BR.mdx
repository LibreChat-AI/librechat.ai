Esta página se aplica ao endpoint [`agents`](/docs/features/agents).

## Exemplo [#example]

```yaml filename="Agents Endpoint"
endpoints:
  agents:
    recursionLimit: 50
    maxRecursionLimit: 100
    disableBuilder: false
    # (optional) Agent Capabilities available to all users. Omit the ones you wish to exclude. Defaults to list below.
    # capabilities: ["deferred_tools", "execute_code", "file_search", "web_search", "artifacts", "subagents", "actions", "context", "skills", "tools", "chain", "ocr"]
    # (optional) File citation configuration for file_search capability
    maxCitations: 30 # Maximum total citations in responses (1-50)
    maxCitationsPerFile: 7 # Maximum citations from each file (1-10)
    minRelevanceScore: 0.45 # Minimum relevance score threshold (0.0-1.0)
    titleTiming: immediate
    skills:
      maxCatalogSkills: 20
    remoteApi:
      auth:
        oidc:
          enabled: false
```

> Esta configuração habilita a interface do construtor para agentes.

## titleTiming

<OptionTable
  options={[
    [

      'String',

      'Defaults to "immediate".',
    ],

**Padrão:** `"immediate"`

- **`"immediate"`**: Gera o título assim que a solicitação é iniciada, em paralelo com a resposta do modelo, usando a primeira mensagem do usuário.

**Exemplo:**

```yaml filename="endpoints / agents / titleTiming"

```

### Exemplos de Configuração de Citação de Arquivos [#file-citation-configuration-examples]

**Configuração Padrão (Balanceada)**

```yaml
endpoints:
  agents:
    maxCitations: 30
    maxCitationsPerFile: 7
    minRelevanceScore: 0.45
```

Fornece citações abrangentes, evitando respostas excessivas e filtrando correspondências de baixa qualidade.

**Configuração Estrita (Alta Qualidade)**

```yaml
endpoints:
  agents:
    maxCitations: 10
    maxCitationsPerFile: 3
    minRelevanceScore: 0.7
```

Inclui apenas citações altamente relevantes com limites rigorosos para respostas focadas.

**Configuração Abrangente (Pesquisa)**

```yaml
endpoints:
  agents:
    maxCitations: 50
    maxCitationsPerFile: 10
    minRelevanceScore: 0.0
```

Extração máxima de informações para tarefas de pesquisa exaustivas, incluindo todas as fontes, independentemente da relevância.

## Capacidades dos Agentes [#agent-capabilities]

O campo `capabilities` permite que você habilite ou desabilite funcionalidades específicas para agentes. As capacidades disponíveis são:

- **deferred_tools**: Permite que agentes descubram ferramentas MCP adiadas em tempo de execução, em vez de carregar todas as ferramentas no contexto antecipadamente.
- **execute_code**: Permite que o agente execute código.
- **file_search**: Permite que o agente pesquise e interaja com arquivos. Quando ativado, o comportamento de citação é controlado pelas configurações `maxCitations`, `maxCitationsPerFile` e `minRelevanceScore`.
- **web_search**: Habilita a funcionalidade de pesquisa na web para agentes, permitindo que eles pesquisem e recuperem informações da internet.
- **artifacts**: Permite que o agente gere artefatos interativos (componentes React, HTML, diagramas Mermaid).
- **subagents**: Habilita execuções de agentes filhos em contexto isolado. Veja [Subagents](/docs/features/subagents).
- **actions**: Permite que o agente execute ações predefinidas.
- **context**: Habilita a funcionalidade "Upload as Text" no chat e "File Context" para agentes, permitindo que os usuários enviem arquivos e tenham seu conteúdo extraído e incluído diretamente na conversa.
- **skills**: Habilita Skills no painel lateral, invocação manual com `$`, skills invocadas pelo modelo e listas de permissão de skills de agentes. Veja [Skills](/docs/features/skills).
- **tools**: Concede ao agente acesso a várias ferramentas.
- **chain**: Habilita o recurso Beta para encadeamento de agentes, também conhecido como fluxos de trabalho Mixture-of-Agents (MoA).
- **ocr**: Opcionalmente aprimora o "Upload as Text" no chat e o "File Context" para agentes, permitindo que arquivos sejam enviados e processados com OCR. **Requer que um serviço de OCR seja configurado.**

Ao especificar as capabilities, você pode controlar os recursos disponíveis para os usuários ao interagir com agentes.

## Exemplo de Configuração [#example-configuration]

Aqui está um exemplo de configuração do endpoint `agents` com capacidades personalizadas e configurações de citação de arquivos:

```yaml filename="Agents Endpoint"
endpoints:
  agents:
    disableBuilder: false
    # File citation configuration
    maxCitations: 20
    maxCitationsPerFile: 5
    minRelevanceScore: 0.6
    # Custom capabilities
    capabilities:
      - 'execute_code'
      - 'file_search'
      - 'skills'
      - 'subagents'
      - 'actions'
      - 'artifacts'
      - 'context'
      - 'ocr'
      - 'web_search'
```

Neste exemplo:

- A interface do construtor está habilitada
- As citações de arquivos são limitadas a um total de 20, com no máximo 5 por arquivo
- Apenas fontes com 60%+ de relevância são incluídas
- Os agentes do LibreChat têm acesso à execução de código, pesquisa de arquivos (com citações), Skills, Subagentes, ações, artefatos, contexto de arquivos, serviços de OCR (se configurados) e recursos de pesquisa na web.

**Padrão:** `enabled: false`

**Exemplo - Apenas OIDC:**

```yaml filename="endpoints / agents / remoteApi"
endpoints:
  agents:
    remoteApi:
      auth:
        apiKey:
          enabled: false
        oidc:
          enabled: true
          issuer: https://auth.example.com/realms/myrealm
          audience: my-client-id
```

**Exemplo - OIDC com fallback de chave de API:**

```yaml filename="endpoints / agents / remoteApi"
endpoints:
  agents:
    remoteApi:
      auth:
        apiKey:
          enabled: true
        oidc:
          enabled: true
          issuer: https://auth.example.com/realms/myrealm
          # jwksUri is optional and auto-discovered if omitted
          jwksUri: https://auth.example.com/realms/myrealm/protocol/openid-connect/certs
          audience: my-client-id
```

<Callout type="info">
  A prioridade de resolução do JWKS URI é o `jwksUri` explícito, seguido por `OPENID_JWKS_URL` e, por fim, a descoberta automática via `{issuer}/.well-known/openid-configuration`.
</Callout>

<Callout type="info">
  O mapeamento de usuários OIDC usa a claim `sub` como busca primária, com fallback para as claims `email`, `preferred_username` ou `upn`. O usuário correspondente já deve existir no LibreChat.
</Callout>

## Notas [#notes]

- Não é recomendado desativar a interface do construtor, a menos que você esteja usando [modelSpecs](/docs/configuration/librechat_yaml/object_structure/model_specs) para definir uma lista de agentes para escolha.
- A configuração de citação de arquivos (`maxCitations`, `maxCitationsPerFile`, `minRelevanceScore`) só se aplica quando a capacidade `file_search` está habilitada.
- A pontuação de relevância é calculada usando similaridade vetorial, onde 1.0 representa uma correspondência perfeita e 0.0 representa nenhuma similaridade.
- Os limites de citação ajudam a equilibrar a recuperação abrangente de informações com a qualidade e o desempenho da resposta.
- A capacidade `context` funciona sem configuração de OCR usando métodos de análise de texto. O OCR aprimora a qualidade da extração quando configurado.
- A funcionalidade `ocr` requer que um serviço de OCR seja configurado (veja [Configuração de OCR](/docs/configuration/librechat_yaml/object_structure/ocr)).
