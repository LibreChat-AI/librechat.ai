**Catatan**
- Tujuan dari bagian dokumentasi ini adalah untuk membantu memahami fungsi `addParams` dan `dropParams`. Anda **TIDAK DAPAT** mengonfigurasi parameter dan nilainya secara global yang dikirim oleh LibreChat secara default, hal tersebut hanya dapat dikonfigurasi di dalam satu endpoint.

Endpoint kustom berbagi logika dengan endpoint OpenAI, sehingga memiliki parameter default yang disesuaikan untuk API OpenAI.

```yaml filename="Default Parameters"
{
  "model": "your-selected-model",
  "user": "LibreChat_User_ID",
  "stream": true,
  "messages": [
    {
      "role": "user",
      "content": "hi how are you",
    },
  ],
}
```

### Rincian [#breakdown]
- `model`: Model yang dipilih dari daftar model.
- `user`: Pengidentifikasi unik yang mewakili pengguna akhir Anda, yang dapat membantu OpenAI untuk [memantau dan mendeteksi penyalahgunaan](https://platform.openai.com/docs/api-reference/chat/create#chat-create-user).
- `stream`: Jika diatur, delta pesan parsial akan dikirim, seperti di ChatGPT. Jika tidak, pembuatan hanya akan tersedia setelah selesai.
- `messages`: [format OpenAI untuk pesan](https://platform.openai.com/docs/api-reference/chat/create#chat-create-messages); field `name` ditambahkan ke pesan dengan peran `system` dan `assistant` ketika nama kustom ditentukan melalui preset.

**Catatan:** Bidang `max_tokens` tidak dikirim untuk menggunakan jumlah token maksimum yang tersedia, yang merupakan perilaku default API OpenAI. Beberapa API alternatif memerlukan bidang ini, atau mungkin secara default bernilai sangat rendah sehingga respons Anda mungkin tampak terpotong; dalam kasus ini, Anda harus menambahkannya ke bidang `addParams` seperti yang ditunjukkan pada [Struktur Objek Endpoint Kustom](/docs/configuration/librechat_yaml/object_structure/custom_endpoint).
