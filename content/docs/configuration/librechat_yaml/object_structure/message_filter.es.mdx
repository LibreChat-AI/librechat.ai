## Descripción general [#overview]

El objeto `messageFilter` configura los filtros de mensajes del lado del servidor. En la v1.3.13, el tipo de filtro admitido es `messageFilter.pii`.

`messageFilter.pii` rechaza el texto del mensaje enviado que coincida con patrones iniciales con formato de credenciales o expresiones regulares definidas por el operador antes de que el texto llegue a la moderación, a una solicitud de modelo o a la persistencia del mensaje.

## Ejemplo [#example]

```yaml filename="librechat.yaml"
messageFilter:
  pii:
    starterPatterns:
      - sk_prefix
      - bearer_header
      - api_key_header
    customPatterns:
      - id: anthropic_api_key
        label: Anthropic API key
        regex: 'sk-ant-[A-Za-z0-9_-]{20,}'
```

- `sk_prefix`: coincide con prefijos de token de estilo `sk-`.
- `bearer_header`: coincide con tokens bearer en el texto.
- `api_key_header`: coincide con texto con formato de encabezado `api-key`.

## Comportamiento [#behavior]

Cuando el texto del mensaje enviado coincide con un patrón de PII configurado, LibreChat devuelve un HTTP `400` con `message_filter_pii_block` y un mensaje solicitando al emisor que elimine el tipo de credencial coincidente.

El filtro comprueba:

- Texto de usuario de la ruta de chat enviado a través de la interfaz de usuario de LibreChat.
- Contenido de texto a través de todos los roles de mensaje proporcionados por el llamador en solicitudes de Chat Completions compatibles con OpenAI.
- Contenido de texto a través de todos los roles de mensaje proporcionados por el llamador en las solicitudes de respuestas compatibles con OpenAI.

El filtro evalúa el texto proporcionado en el cuerpo de la solicitud. No escanea el contenido de los archivos.
