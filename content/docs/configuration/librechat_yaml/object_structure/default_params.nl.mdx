**Opmerking**
- Het doel van dit deel van de documentatie is om te helpen begrijpen wat `addParams` en `dropParams` doen. Je **KUNT** de parameters en hun waarden die LibreChat standaard verstuurt niet globaal configureren; dit kan alleen worden geconfigureerd binnen een enkel endpoint.

Aangepaste endpoints delen logica met het OpenAI endpoint en hebben daarom standaardparameters die zijn afgestemd op de OpenAI API.

```yaml filename="Default Parameters"
{
  "model": "your-selected-model",
  "user": "LibreChat_User_ID",
  "stream": true,
  "messages": [
    {
      "role": "user",
      "content": "hi how are you",
    },
  ],
}
```

### Uitsplitsing [#breakdown]
- `model`: Het geselecteerde model uit de lijst met modellen.
- `user`: Een unieke identificatiecode die uw eindgebruiker vertegenwoordigt, wat OpenAI kan helpen om [misbruik te monitoren en te detecteren](https://platform.openai.com/docs/api-reference/chat/create#chat-create-user).
- `stream`: Indien ingesteld, zullen gedeeltelijke bericht-delta's worden verzonden, zoals in ChatGPT. Anders is de generatie pas beschikbaar wanneer deze is voltooid.
- `messages`: [OpenAI-indeling voor berichten](https://platform.openai.com/docs/api-reference/chat/create#chat-create-messages); het `name`-veld wordt toegevoegd aan berichten met `system`- en `assistant`-rollen wanneer een aangepaste naam is opgegeven via een preset.

**Let op:** Het `max_tokens` veld wordt niet verzonden om het maximale aantal beschikbare tokens te gebruiken, wat het standaardgedrag van de OpenAI API is. Sommige alternatieve API's vereisen dit veld, of het kan standaard op een zeer lage waarde staan waardoor je antwoorden mogelijk afgebroken worden; in dit geval moet je het toevoegen aan het `addParams` veld, zoals getoond in de [Custom Endpoint Object Structure](/docs/configuration/librechat_yaml/object_structure/custom_endpoint).
