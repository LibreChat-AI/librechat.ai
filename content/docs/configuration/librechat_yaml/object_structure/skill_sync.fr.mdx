## Vue d'ensemble [#overview]

L'objet `skillSync` permet aux administrateurs de refléter des Skills provenant de sources externes. Dans la v1.3.13, GitHub est le fournisseur pris en charge.

GitHub Skill Sync lit les fichiers `SKILL.md` et leurs fichiers groupés à partir des chemins de dépôt configurés, les stocke en tant que Skills avec `source: "github"`, et maintient les lignes mises en miroir alignées avec le dépôt amont lors des synchronisations ultérieures.

## Exemple [#example]

```yaml filename="librechat.yaml"
skillSync:
  github:
    enabled: true
    intervalMinutes: 60
    runOnStartup: true
    sources:
      - id: librechat-skills
        owner: your-org
        repo: your-skills-repo
        ref: main
        paths:
          - skills
        skillDiscoveryDepth: 2
        token: '${GITHUB_SKILLS_TOKEN}'
        # credentialKey: production-skills
        # tenantId: your-tenant-id
```

<Callout type="warning" title="Identifiants">
  Utilisez soit `token`, soit `credentialKey` pour chaque source, mais pas les deux. `token` doit être une référence de variable d'environnement telle que `${GITHUB_SKILLS_TOKEN}`. Utilisez un jeton d'accès personnel GitHub à grain fin limité au dépôt sélectionné avec des autorisations de lecture seule pour Contents et Metadata.
</Callout>

### Identité de la source [#source-identity]

The `id` field is part of the mirror identity. Keep it stable. Repointing a source to a renamed repository or a new ref keeps mirrored Skills linked to the same source id, while changing the source id creates a new mirror.

Lorsque `tenantId` est défini, les lectures et écritures de synchronisation s'exécutent dans le contexte d'isolation de ce tenant. Considérez `tenantId` comme immuable pour un identifiant source : le modifier, l'ajouter ou le supprimer ultérieurement laisse les Skills précédemment mis en miroir dans l'ancien tenant.

## Comportement de synchronisation [#sync-behavior]

Synchronisation des compétences GitHub :

- Découvre les fichiers `SKILL.md` sous chaque chemin configuré.
- Importe le corps, le frontmatter et les fichiers groupés de `SKILL.md`.
- Stocke les métadonnées de la source telles que source id, owner, repo, ref, skill path, commit SHA et blob SHA.
- Met à jour les Skills en miroir lorsque le fichier amont est modifié.
- Supprime les Skills et les fichiers mis en miroir qui ont été retirés de la source configurée.
- Utilise un verrou de synchronisation afin qu'une seule exécution de synchronisation GitHub ne puisse se dérouler à la fois.
