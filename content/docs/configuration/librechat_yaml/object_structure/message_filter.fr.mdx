## Vue d'ensemble [#overview]

L'objet `messageFilter` configure les filtres de message côté serveur. Dans la version v1.3.13, le type de filtre pris en charge est `messageFilter.pii`.

`messageFilter.pii` rejette le texte des messages soumis qui correspond à des modèles de type identifiants ou à des expressions régulières définies par l'opérateur avant que le texte n'atteigne la modération, une requête de modèle ou la persistance des messages.

## Exemple [#example]

```yaml filename="librechat.yaml"
messageFilter:
  pii:
    starterPatterns:
      - sk_prefix
      - bearer_header
      - api_key_header
    customPatterns:
      - id: anthropic_api_key
        label: Anthropic API key
        regex: 'sk-ant-[A-Za-z0-9_-]{20,}'
```

- `sk_prefix` : correspond aux préfixes de jetons de style `sk-`.

## Comportement [#behavior]

Lorsque le texte du message soumis correspond à un modèle PII configuré, LibreChat renvoie une erreur HTTP `400` avec `message_filter_pii_block` et un message demandant à l'appelant de supprimer le type d'identifiant correspondant.

Le filtre vérifie :

- La route de chat achemine le texte utilisateur soumis via l'interface utilisateur de LibreChat.
- Contenu textuel à travers tous les rôles de message fournis par l'appelant dans les requêtes de Chat Completions compatibles avec OpenAI.
- Contenu textuel à travers tous les rôles de message fournis par l'appelant dans les requêtes de réponses compatibles avec OpenAI.

Le filtre évalue le texte fourni dans le corps de la requête. Il n'analyse pas le contenu des fichiers.
