## Visão geral [#overview]

O objeto `messageFilter` configura filtros de mensagem no lado do servidor. Na v1.3.13, o tipo de filtro suportado é `messageFilter.pii`.

`messageFilter.pii` rejeita o texto da mensagem enviada que corresponda a padrões iniciais com formato de credenciais ou expressões regulares definidas pelo operador antes que o texto chegue à moderação, a uma solicitação de modelo ou à persistência da mensagem.

## Exemplo [#example]

```yaml filename="librechat.yaml"
messageFilter:
  pii:
    starterPatterns:
      - sk_prefix
      - bearer_header
      - api_key_header
    customPatterns:
      - id: anthropic_api_key
        label: Anthropic API key
        regex: 'sk-ant-[A-Za-z0-9_-]{20,}'
```

**Ids de padrão inicial integrados:**

- `sk_prefix`: corresponde a prefixos de token no estilo `sk-`.
- `bearer_header`: corresponde a tokens bearer no texto.
- `api_key_header`: corresponde a texto no formato de cabeçalho `api-key`.

## Comportamento [#behavior]

Quando o texto da mensagem enviada corresponde a um padrão de PII configurado, o LibreChat retorna um HTTP `400` com `message_filter_pii_block` e uma mensagem solicitando que o autor da chamada remova o tipo de credencial correspondente.

O filtro verifica:

- Rota de chat para o texto do usuário enviado através da interface do LibreChat.
- Conteúdo de texto em todas as funções de mensagem fornecidas pelo chamador em solicitações de Chat Completions compatíveis com OpenAI.
- Conteúdo de texto em todas as funções de mensagem fornecidas pelo chamador em solicitações de Respostas compatíveis com OpenAI.

O filtro avalia o texto fornecido no corpo da requisição. Ele não verifica o conteúdo de arquivos.
