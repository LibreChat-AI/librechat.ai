## 概述 [#overview]

`skillSync` 对象允许管理员从外部源镜像 Skills。在 v1.3.13 版本中，GitHub 是受支持的提供商。

GitHub Skill Sync 会从配置的仓库路径中读取 `SKILL.md` 文件及其捆绑文件，将它们存储为 `source: "github"` 的 Skills，并在后续同步时保持镜像行与上游仓库同步。

## 示例 [#example]

```yaml filename="librechat.yaml"
skillSync:
  github:
    enabled: true
    intervalMinutes: 60
    runOnStartup: true
    sources:
      - id: librechat-skills
        owner: your-org
        repo: your-skills-repo
        ref: main
        paths:
          - skills
        skillDiscoveryDepth: 2
        token: '${GITHUB_SKILLS_TOKEN}'
        # credentialKey: production-skills
        # tenantId: your-tenant-id
```

<Callout type="warning" title="凭据">
  每个源请使用 `token` 或 `credentialKey` 中的一个，不要同时使用。`token` 必须是环境变量引用，例如 `${GITHUB_SKILLS_TOKEN}`。请使用针对所选仓库设置的 GitHub 精细化个人访问令牌（fine-grained personal access token），并授予其只读的 Contents 和 Metadata 权限。
</Callout>

### 源身份 [#source-identity]

`id` 字段是镜像标识的一部分。请保持其稳定。将源指向重命名后的仓库或新的 ref 会使镜像的 Skills 继续链接到同一个源 id，而更改源 id 则会创建一个新的镜像。

当设置了 `tenantId` 时，同步读写操作将在该租户的隔离上下文中运行。请将 `tenantId` 视为特定源 ID 的不可变属性：后续对其进行更改、添加或移除，会导致之前镜像的 Skills 保留在旧租户中。

## 同步行为 [#sync-behavior]

GitHub Skill Sync:

- 发现每个配置路径下的 `SKILL.md` 文件。
- 导入 `SKILL.md` 正文、frontmatter 和捆绑文件。
- 存储源元数据，例如 source id、owner、repo、ref、skill path、commit SHA 和 blob SHA。
- 当上游文件发生更改时，更新镜像的 Skills。
- 删除已从配置源中移除的镜像 Skills 和文件。
- 使用同步锁，确保同一时间只有一个 GitHub 同步任务在运行。
