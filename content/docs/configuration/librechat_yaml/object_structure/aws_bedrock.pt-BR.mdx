Integrar o AWS Bedrock à sua aplicação permite que você utilize perfeitamente múltiplos modelos de IA hospedados na AWS. Esta seção detalha como configurar o endpoint do AWS Bedrock para suas necessidades.

## Exemplo de Configuração [#example-configuration]

```yaml filename="Example AWS Bedrock Object Structure"
endpoints:
  bedrock:
    titleModel: 'anthropic.claude-3-haiku-20240307-v1:0'
    streamRate: 35
    availableRegions:
      - 'us-east-1'
      - 'us-west-2'
    guardrailConfig:
      guardrailIdentifier: 'your-guardrail-id'
      guardrailVersion: '1'
      trace: 'enabled'
      streamProcessingMode: 'sync'
```

> **Nota:** O endpoint do AWS Bedrock suporta todas as [Shared Endpoint Settings](/docs/configuration/librechat_yaml/object_structure/shared_endpoint_settings), incluindo `streamRate`, `titleModel`, `titleMethod`, `titlePrompt`, `titlePromptTemplate` e `titleEndpoint`. As configurações mostradas abaixo são específicas para o Bedrock ou possuem padrões específicos do Bedrock.

## inferenceProfiles

**Chave:**

<OptionTable
  options={[
    [
      'inferenceProfiles',
      'Object (Record)',
      'Mapeia IDs de modelo para ARNs de perfil de inferência para inferência entre regiões. As chaves são IDs de modelo e os valores são o ARN do perfil de inferência ou uma referência de variável de ambiente.',
      'When a selected model matches a key, the corresponding ARN is used as the application inference profile.',
    ],
  ]}
/>

**Padrão:** Não especificado

**Exemplo:**

```yaml filename="inferenceProfiles"
endpoints:
  bedrock:
    inferenceProfiles:
      'us.anthropic.claude-sonnet-4-20250514-v1:0': '${BEDROCK_INFERENCE_PROFILE_CLAUDE_SONNET}'
      'anthropic.claude-3-7-sonnet-20250219-v1:0': 'arn:aws:bedrock:us-west-2:123456789012:application-inference-profile/abc123'
```

**Notas:**

- Perfis de inferência permitem inferência entre regiões, permitindo que você encaminhe solicitações para modelos em diferentes regiões da AWS.
- Os valores suportam interpolação de variáveis de ambiente com a sintaxe `${ENV_VAR}`
- O ID do modelo na chave deve corresponder ao modelo selecionado pelo usuário na UI
- Use com o campo `models` para disponibilizar IDs de modelos entre regiões no seletor de modelos
- Para um guia completo sobre como criar e gerenciar perfis de inferência, veja [AWS Bedrock Inference Profiles](/docs/configuration/pre_configured_ai/bedrock_inference_profiles)

**Exemplo Combinado:**

```yaml filename="Bedrock with inference profiles"
endpoints:
  bedrock:
    models:
      - 'us.anthropic.claude-sonnet-4-20250514-v1:0'
      - 'us.anthropic.claude-haiku-4-20250514-v1:0'
    inferenceProfiles:
      'us.anthropic.claude-sonnet-4-20250514-v1:0': '${BEDROCK_CLAUDE_SONNET_PROFILE}'
      'us.anthropic.claude-haiku-4-20250514-v1:0': '${BEDROCK_CLAUDE_HAIKU_PROFILE}'
```

**Chave:**

**Sub-chaves:**

<OptionTable
  options={[
    [
      'guardrailIdentifier',

      'O identificador único do guardrail a ser aplicado.',

    [
      'guardrailVersion',
      'String',
      'A versão do guardrail a ser utilizada.',
      'Required when using guardrails.',
    ],
    [
      'trace',
      'String',
      'Controla a saída de rastreamento do guardrail para depuração. Opções: "enabled", "enabled_full" ou "disabled".',
      'Optional. Default: "disabled"',
    ],
    [
      'streamProcessingMode',
      'String',
      'Controla o modo de processamento de fluxo do guardrail. Opções: "sync" ou "async".',
      'Optional. Default: "sync"',
    ],
  ]}
/>

**Exemplo:**

```yaml filename="guardrailConfig"
endpoints:
  bedrock:
    guardrailConfig:
      guardrailIdentifier: 'abc123xyz'
      guardrailVersion: '1'
      trace: 'enabled'
      streamProcessingMode: 'sync'
```

**Notas:**

- Guardrails ajudam a garantir o uso responsável da IA ao filtrar conteúdo prejudicial, PII e outras informações sensíveis.
- O `guardrailIdentifier` pode ser encontrado no console do AWS Bedrock em Guardrails
- Defina `trace` como `"enabled"` ou `"enabled_full"` durante o desenvolvimento para ver quais políticas de guardrail são acionadas
- Defina `streamProcessingMode` como `"async"` para transmitir respostas mais rapidamente (ao custo de o guardrail possivelmente permitir a passagem de conteúdo inadequado até que sua verificação seja concluída)
- Para produção, defina `trace` como `"disabled"` para reduzir o tamanho do payload da resposta.

## Notas [#notes]

- A autenticação do AWS Bedrock é configurada por meio de variáveis de ambiente. Você pode usar `BEDROCK_AWS_PROFILE`, a cadeia de provedores de credenciais padrão do AWS SDK, `BEDROCK_AWS_BEARER_TOKEN` para chaves de API do Bedrock ou credenciais estáticas específicas do Bedrock. Consulte o [guia de configuração do AWS Bedrock](/docs/configuration/pre_configured_ai/bedrock#authentication) para obter detalhes.
