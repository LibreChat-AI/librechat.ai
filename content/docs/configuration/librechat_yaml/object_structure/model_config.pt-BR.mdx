Cada item sob `models` faz parte de uma lista de registros, sendo um valor booleano ou um Object:

**Ao especificar um modelo como um objeto:**

Um objeto permite uma configuração detalhada do modelo, incluindo seu `deploymentName` e/ou `version`. Este modo é usado para um controle mais granular sobre os modelos, especialmente ao trabalhar com múltiplas versões ou implantações sob uma única instância ou grupo de recursos.

**Exemplo**:
```yaml filename="endpoints / azureOpenAI / groups / {group_item} / models / {model_item=Object}"
models:
  gpt-4-vision-preview:
    deploymentName: "gpt-4-vision-preview"
    version: "2024-02-15-preview"
```

**Notas:**
- **Deployment Names** e **Versions** são fundamentais para garantir que o modelo correto seja utilizado.
    - Verifique novamente esses valores para garantir a precisão e evitar comportamentos inesperados.
