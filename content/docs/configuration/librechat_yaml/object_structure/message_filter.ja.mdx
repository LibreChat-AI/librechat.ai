## 概要 [#overview]

`messageFilter` オブジェクトは、サーバーサイドのメッセージフィルターを設定します。v1.3.13 では、サポートされているフィルタータイプは `messageFilter.pii` です。

`messageFilter.pii` は、送信されたメッセージテキストがモデレーション、モデルへのリクエスト、またはメッセージの永続化に到達する前に、認証情報のような形式のパターンやオペレーターが定義した正規表現に一致する場合、そのテキストを拒否します。

## 例 [#example]

```yaml filename="librechat.yaml"
messageFilter:
  pii:
    starterPatterns:
      - sk_prefix
      - bearer_header
      - api_key_header
    customPatterns:
      - id: anthropic_api_key
        label: Anthropic API key
        regex: 'sk-ant-[A-Za-z0-9_-]{20,}'
```

## 動作 [#behavior]

送信されたメッセージテキストが設定済みのPIIパターンと一致した場合、LibreChatは `message_filter_pii_block` とともにHTTP `400` エラーを返し、呼び出し元に対して一致した認証情報の種類を削除するよう求めるメッセージを表示します。

フィルターは以下をチェックします：

- LibreChat UIを通じて送信されたチャットルートのユーザーテキスト。
- OpenAI互換のChat Completionsリクエストにおける、すべての呼び出し元が提供するメッセージロール全体にわたるテキストコンテンツ。
- OpenAI互換のResponsesリクエストにおける、すべての呼び出し元が提供するメッセージロールにわたるテキストコンテンツ。

このフィルターは、リクエストボディで提供されたテキストを評価します。ファイルの内容はスキャンしません。
