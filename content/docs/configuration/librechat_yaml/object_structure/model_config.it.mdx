Ogni elemento sotto `models` fa parte di un elenco di record, che può essere un valore booleano o un Object:

**Quando si specifica un modello come oggetto:**

Un oggetto consente una configurazione dettagliata del modello, inclusi il suo `deploymentName` e/o `version`. Questa modalità viene utilizzata per un controllo più granulare sui modelli, specialmente quando si lavora con più versioni o distribuzioni sotto una singola istanza o gruppo di risorse.

**Esempio**:
```yaml filename="endpoints / azureOpenAI / groups / {group_item} / models / {model_item=Object}"
models:
  gpt-4-vision-preview:
    deploymentName: "gpt-4-vision-preview"
    version: "2024-02-15-preview"
```

**Note:**
- I **Deployment Names** e le **Versions** sono fondamentali per garantire che venga utilizzato il modello corretto.
    - Controlla attentamente questi valori per verificarne l'accuratezza e prevenire comportamenti imprevisti.
