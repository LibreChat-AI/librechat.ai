## Übersicht [#overview]

Das `transactions`-Objekt steuert, ob Token-Nutzungsdatensätze in LibreChat in der Datenbank gespeichert werden. Dies ermöglicht es Administratoren, die Transaktionsverfolgung unabhängig vom Guthabensystem zu aktivieren oder zu deaktivieren.

**Felder unter `transactions`:**

- `enabled`

**Hinweise:**

- Die Transaktionsaufzeichnung ist für die Nachverfolgung der historischen Token-Nutzung unerlässlich.
- Wenn `balance.enabled` auf `true` gesetzt ist, werden Transaktionen unabhängig von dieser Einstellung automatisch aktiviert.
- Der Standardwert ist `true`, um sicherzustellen, dass die Token-Nutzung standardmäßig erfasst wird.
- Das Deaktivieren von Transaktionen kann den Speicherbedarf der Datenbank reduzieren, verhindert jedoch die Analyse der historischen Nutzung.

## Beispiel [#example]

```yaml filename="transactions"
transactions:
  enabled: false
```

## enabled

**Schlüssel:**

    options={[
        ['enabled', 'Boolean', 'Steuert, ob Transaktionsdatensätze in der Datenbank gespeichert werden sollen.', 'Default: true. Set to false to disable transaction recording (unless balance.enabled is true).'],
    ]}
/>

**Beschreibung:**

Das Feld `enabled` bestimmt, ob LibreChat detaillierte Transaktionsdatensätze für jedes Token-Nutzungsereignis speichert. Diese Datensätze beinhalten:

- Token-Anzahl für Prompts und Completions
- Zugehörige Kosten und Tarife
- Benutzer- und Konversationskennungen
- Zeitstempel für jede Transaktion

**Wichtiges Verhalten:**

Wenn das Guthabensystem aktiviert ist (`balance.enabled: true`), wird die Transaktionsaufzeichnung unabhängig von der Einstellung `transactions.enabled` automatisch aktiviert. Dies stellt sicher, dass:

1. Funktionen zur Guthabenverfolgung funktionieren korrekt mit einem vollständigen Audit-Trail
2. Die Token-Nutzung kann präzise berechnet und vom Guthaben der Benutzer abgezogen werden.
3. Historische Daten stehen für den Abgleich des Guthabens zur Verfügung

**Anwendungsfälle:**

- **Transaktionen aktivieren** (`true`): Wenn Sie Nutzungsmuster nachverfolgen, Berichte erstellen oder einen Prüfpfad (Audit Trail) führen müssen
- **Transaktionen deaktivieren** (`false`): Wenn Sie den Datenbankspeicher reduzieren möchten und keine historischen Nutzungsdaten benötigen (funktioniert nur, wenn auch die Guthabenverfolgung deaktiviert ist)

## Beziehung zum Balance System [#relationship-with-balance-system]

Die Transaktions- und Guthabensysteme arbeiten zusammen:

```yaml filename="Example: Transactions with Balance"
# When balance is enabled, transactions are always enabled
balance:
  enabled: true
  startBalance: 20000

transactions:
  enabled: false  # This will be overridden to true because balance.enabled is true
```

```yaml filename="Example: Standalone Transaction Tracking"
# Track transactions without balance management
balance:
  enabled: false

transactions:
  enabled: true  # Records all token usage without enforcing balance limits
```

## Auswirkungen auf die Datenbank [#database-impact]

Wenn Transaktionen aktiviert sind, erstellt jeder API-Aufruf, der Token verbraucht, einen Datensatz in der "Transactions"-Sammlung mit den folgenden Informationen:

- Benutzer-ID und E-Mail
- Konversations-ID
- Verwendetes Modell
- Token-Anzahl (Prompt und Completion)
- Token-Werte und -Raten
- Zeitstempel
- Transaktionstyp (Gutschrift oder Lastschrift)

Berücksichtigen Sie die Auswirkungen auf den Speicher, wenn Sie Transaktionen für Bereitstellungen mit hohem Volumen aktivieren.
