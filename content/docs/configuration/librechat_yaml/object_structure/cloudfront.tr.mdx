title: CloudFront Nesne Yapısı
icon: AWS
---

`cloudfront` nesnesi, S3'te depolanan dosyalar için CloudFront teslimatını yapılandırır. `fileStrategy` veya herhangi bir `fileStrategies` girdisi `"cloudfront"` kullandığında bu gereklidir.

## Örnek [#example]

```yaml filename="cloudfront"
fileStrategies:
  avatar: 'cloudfront'
  image: 'cloudfront'
  document: 's3'

cloudfront:
  domain: 'https://cdn.example.com'
  distributionId: 'E1234ABCD'
  invalidateOnDelete: false
  imageSigning: 'cookies'
  cookieDomain: '.example.com'
  cookieExpiry: 1800
  urlExpiry: 3600
  storageRegion: 'us-east-2'
  includeRegionInPath: false
  requireSignedAccess: true
```

## Doğrulama Kuralları [#validation-rules]

- `invalidateOnDelete` değeri `true` olduğunda `distributionId` gereklidir.
- `imageSigning` `"cookies"` olarak ayarlandığında `cookieDomain` gereklidir.
- `cookieDomain` bir nokta ile başlamalıdır, örneğin `.example.com`.
- `requireSignedAccess: true` ayarı, `imageSigning: "cookies"` ayarını gerektirir.
