Jedes Element unter `models` ist Teil einer Liste von Datensätzen, entweder ein boolescher Wert oder ein Object:

**Wenn ein Modell als Objekt angegeben wird:**

Ein Objekt ermöglicht eine detaillierte Konfiguration des Modells, einschließlich seines `deploymentName` und/oder seiner `version`. Dieser Modus wird für eine präzisere Steuerung der Modelle verwendet, insbesondere bei der Arbeit mit mehreren Versionen oder Bereitstellungen unter einer Instanz oder Ressourcengruppe.

**Beispiel**:
```yaml filename="endpoints / azureOpenAI / groups / {group_item} / models / {model_item=Object}"
models:
  gpt-4-vision-preview:
    deploymentName: "gpt-4-vision-preview"
    version: "2024-02-15-preview"
```

**Hinweise:**
- **Deployment Names** und **Versions** sind entscheidend, um sicherzustellen, dass das korrekte Modell verwendet wird.
    - Überprüfen Sie diese Werte sorgfältig auf ihre Richtigkeit, um unerwartetes Verhalten zu vermeiden.
