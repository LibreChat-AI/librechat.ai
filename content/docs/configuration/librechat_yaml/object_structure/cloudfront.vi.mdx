Đối tượng `cloudfront` cấu hình việc phân phối qua CloudFront cho các tệp được lưu trữ trong S3. Đối tượng này là bắt buộc khi `fileStrategy` hoặc bất kỳ mục nào trong `fileStrategies` sử dụng `"cloudfront"`.

## Ví dụ [#example]

```yaml filename="cloudfront"
fileStrategies:
  avatar: 'cloudfront'
  image: 'cloudfront'
  document: 's3'

cloudfront:
  domain: 'https://cdn.example.com'
  distributionId: 'E1234ABCD'
  invalidateOnDelete: false
  imageSigning: 'cookies'
  cookieDomain: '.example.com'
  cookieExpiry: 1800
  urlExpiry: 3600
  storageRegion: 'us-east-2'
  includeRegionInPath: false
  requireSignedAccess: true
```

## Các quy tắc xác thực [#validation-rules]

- `distributionId` là bắt buộc khi `invalidateOnDelete` là `true`.
- `cookieDomain` là bắt buộc khi `imageSigning` là `"cookies"`.
- `cookieDomain` phải bắt đầu bằng một dấu chấm, ví dụ: `.example.com`.
- `requireSignedAccess: true` yêu cầu `imageSigning: "cookies"`.
