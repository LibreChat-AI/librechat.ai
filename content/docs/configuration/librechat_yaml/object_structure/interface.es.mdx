## Descripción general [#overview]

El objeto `interface` permite la personalización de varios elementos de la interfaz de usuario dentro de la aplicación, incluyendo la visibilidad y los ajustes de comportamiento para componentes como menús, paneles y enlaces. Esta sección proporciona un desglose detallado de la estructura del objeto `interface`.

Estos son los campos bajo `interface`:

- `mcpServers`
- `privacyPolicy`
- `termsOfService`
- `modelSelect`
- `parameters`
- `contextUsage`
- `contextCost`
- `currency`
- `presets`
- `prompts`
- `bookmarks`
- `memories`
- `multiConvo`
- `agents`
- `remoteAgents`
- `skills`
- `sharedLinks`
- `temporaryChat`
- `temporaryChatRetention`
- `retentionMode`
- `retainAgentFiles`
- `autoSubmitFromUrl`
- `customWelcome`
- `runCode`
- `webSearch`
- `fileSearch`
- `fileCitations`
- `peoplePicker`
- `marketplace`

**Notas:**

- Las configuraciones de `interface` se aplican globalmente dentro de la aplicación.
- Se proporcionan valores predeterminados para la mayoría de los ajustes, pero pueden ser anulados según requisitos o condiciones específicas.
- La lógica condicional en la aplicación puede modificar aún más estos ajustes basándose en otras configuraciones como Model Specs.

<Callout type="warning" title="Obsoleto: campos de efectos secundarios de permisos">
Varios campos a continuación (`mcpServers`, `prompts`, `bookmarks`, `memories`, `multiConvo`, `agents`, `remoteAgents`, `skills`, `sharedLinks`, `temporaryChat`, `runCode`, `webSearch`, `fileSearch`, `fileCitations`, `peoplePicker`, `marketplace`) no solo alternan la interfaz de usuario, sino que también establecen permisos de rol en la base de datos al inicio, y solo para el rol `USER` integrado.

Para la gestión continua, utilice el [**LibreChat Admin Panel**](/docs/features/admin_panel), que edita la matriz de permisos directamente en cada rol (incluidos los roles personalizados). Estos campos YAML siguen siendo compatibles para el arranque de una instancia nueva o implementaciones totalmente basadas en archivos, pero ya no deben utilizarse como la forma principal de gestionar los permisos de las funciones.

Consulta [Access Control](/docs/features/access_control) para conocer el modelo de permisos completo.

</Callout>

## Ejemplo [#example]

```yaml filename="interface"
interface:
  mcpServers:
    placeholder: 'MCP Servers'
    use: true
    create: true
    share: false
    public: false
    configureObo: false
    trustCheckbox:
      label: 'I trust this server'
      subLabel: 'Only enable servers you trust'
  privacyPolicy:
    externalUrl: 'https://example.com/privacy'
    openNewTab: true
  termsOfService:
    externalUrl: 'https://example.com/terms'
    openNewTab: true
    modalAcceptance: true
    modalTitle: 'Terms of Service'
    modalContent: |
      # Terms of Service
      ## Introduction
      Welcome to LibreChat!
  modelSelect: false
  parameters: true
  contextUsage: true
  contextCost: true
  currency:
    code: EUR
    rate: 0.92
  presets: false
  prompts:
    use: true
    create: true
    share: true
    public: false
  bookmarks: true
  multiConvo: true
  agents:
    use: true
    create: true
    share: true
    public: false
  skills:
    use: true
    create: true
    share: false
    public: false
    defaultActiveOnShare: false
  sharedLinks:
    create: true
    share: true
    public: false
  customWelcome: 'Hey {{user.name}}! Welcome to LibreChat'
  runCode: true
  webSearch: true
  fileSearch: true
  fileCitations: true
```

## prompts

> **Obsoleto para la gestión de permisos.** Inicializa los permisos de rol `PROMPTS` al inicio solo para el rol `USER` predeterminado. Se prefiere el [Admin Panel](/docs/features/admin_panel) para gestionar los permisos de prompts por rol/grupo/usuario.

**Clave:**

<OptionTable
  options={[
    [
      'prompts',
      'Boolean or Object',
      'Controla las funciones relacionadas con los prompts para todos los usuarios. Puede ser un booleano para habilitar/deshabilitar de forma sencilla, o un objeto para un control granular sobre el uso, la creación, el uso compartido y la visibilidad pública.',
      'When set to `false`, users will not have access to create, edit, or use custom prompts.',
    ],
  ]}

**Predeterminado:** `true`

**Importante: Configuración booleana frente a objeto**

- **Objeto**: Actualiza solo los sub-permisos que se especifican explícitamente. Cualquier permiso que no esté incluido en la configuración se conserva de la base de datos.

## temporaryChatRetention

La configuración `temporaryChatRetention` le permite personalizar cuánto tiempo se conservan los chats temporales antes de ser eliminados automáticamente.

**Clave:**

<OptionTable
  options={[
    [
      'temporaryChatRetention',
      'Number',
      'Establece el período de retención para los chats temporales en horas.',
      'temporaryChatRetention: 168',
    ],
  ]}
/>

**Reglas de validación:**

- **Mínimo**: 1 hora (evita la eliminación inmediata)
- **Máximo**: 8760 horas (retención máxima de 1 año)
- **Predeterminado**: 720 horas (30 días)

**Métodos de configuración:**

1. **LibreChat.yaml** (recomendado): `interface.temporaryChatRetention: 168`
2. **Variable de entorno** (obsoleta): `TEMP_CHAT_RETENTION_HOURS=168`

> **Nota:** La variable de entorno `TEMP_CHAT_RETENTION_HOURS` está obsoleta. Por favor, utilice la opción de configuración `interface.temporaryChatRetention` en `librechat.yaml` en su lugar. El valor del archivo de configuración tiene prioridad sobre la variable de entorno.

**Ejemplo:**

```yaml filename="interface / temporaryChatRetention"
interface:
  temporaryChatRetention: 168 # Retain temporary chats for 7 days
  retentionMode: 'temporary'
```

**Períodos de retención comunes:**

- **1 hora**: `temporaryChatRetention: 1` (retención mínima)
- **24 horas**: `temporaryChatRetention: 24` (1 día)
- **168 horas**: `temporaryChatRetention: 168` (1 semana)
- **720 horas**: `temporaryChatRetention: 720` (30 días - valor predeterminado)
- **8760 horas**: `temporaryChatRetention: 8760` (1 año - máximo)

## autoSubmitFromUrl

Controla si un prompt proporcionado a través de parámetros de consulta de URL en `/c/new` se envía automáticamente al modelo.

Cuando un usuario autenticado abre `/c/new?prompt=…&submit=true`, LibreChat normalmente rellena previamente el campo de redacción con el prompt proporcionado en la URL y lo envía inmediatamente. Esta es una función de conveniencia para enlaces profundos (deeplinks) personalizados y URLs de chat compartidas.

Para despliegues donde los usuarios puedan recibir enlaces manipulados de fuentes externas —y donde los modelos con memoria o herramientas habilitadas podrían filtrar contexto sensible si una carga útil de inyección de prompts llega al modelo—, los operadores pueden desactivar el envío automático. Con el flag establecido en `false`, el prompt se sigue rellenando previamente en el editor, pero el usuario debe presionar **Enviar** explícitamente.

**Clave:**

<OptionTable
  options={[
    [
      'autoSubmitFromUrl',
      'Boolean',
      'Controla si `/c/new?prompt=…&submit=true` se envía automáticamente al modelo.',
      'When `false`, the prompt is pre-filled in the composer but not submitted.',
    ],
  ]}
/>

**Predeterminado:** `true` (el comportamiento existente se conserva a menos que se deshabilite explícitamente).

**Notas:**

- Esta configuración no afecta la selección de Model Specs basada en URL ni otras configuraciones basadas en URL, solo el paso de envío automático.
- El parámetro de consulta acepta tanto `prompt` como `q` como fuente del prompt, teniendo `prompt` prioridad. `submit=true` es el activador.
- Recomendado para instancias que manejan datos sensibles de memoria o herramientas, donde un vector de inyección de prompts de un solo clic debería requerir la confirmación explícita del usuario.

**Ejemplo:**

```yaml filename="interface / autoSubmitFromUrl"
interface:
  autoSubmitFromUrl: false
```

## fileCitations

Controla la disponibilidad global de la funcionalidad de citas de archivos. Cuando está desactivada, elimina efectivamente el permiso `FILE_CITATIONS` para todos los usuarios, evitando que se muestren citas de archivos al utilizar la búsqueda de archivos, independientemente de los permisos individuales de cada usuario.

> **Obsoleto para la gestión de permisos.** Establece/restringe globalmente el permiso de rol `FILE_CITATIONS` al inicio. Se prefiere el [Panel de administración](/docs/features/admin_panel) para gestionar los permisos de citas por rol/grupo/usuario.

**Nota:**

- Este ajuste actúa como un interruptor global para el sistema de permisos `FILE_CITATIONS` en toda la aplicación.
- Cuando se establece en `false`, ningún usuario verá las citas de archivos, incluso si se les ha otorgado el permiso a través de roles.
- Las citas de archivos requieren que la función `fileSearch` esté habilitada.
- Al utilizar agentes con capacidad de búsqueda de archivos, el comportamiento de las citas (cantidad y calidad) puede configurarse a través de la [configuración del endpoint de Agents](/docs/configuration/librechat_yaml/object_structure/agents#file-citation-configuration-examples).
