## Übersicht [#overview]

Das `messageFilter`-Objekt konfiguriert serverseitige Nachrichtenfilter. In v1.3.13 ist der unterstützte Filtertyp `messageFilter.pii`.

`messageFilter.pii` lehnt übermittelten Nachrichtentext ab, der mit anmeldedatenähnlichen Startmustern oder vom Betreiber definierten regulären Ausdrücken übereinstimmt, bevor der Text die Moderation, eine Modellanfrage oder die Nachrichtenspeicherung erreicht.

## Beispiel [#example]

```yaml filename="librechat.yaml"
messageFilter:
  pii:
    starterPatterns:
      - sk_prefix
      - bearer_header
      - api_key_header
    customPatterns:
      - id: anthropic_api_key
        label: Anthropic API key
        regex: 'sk-ant-[A-Za-z0-9_-]{20,}'
```

**Integrierte Starter-Muster-IDs:**

- `sk_prefix`: entspricht Token-Präfixen im `sk-`-Stil.
- `bearer_header`: gleicht Bearer-Token im Text ab.
- `api_key_header`: entspricht Text im Format des `api-key`-Headers.

## Verhalten [#behavior]

Wenn der übermittelte Nachrichtentext mit einem konfigurierten PII-Muster übereinstimmt, gibt LibreChat den HTTP-Status `400` mit `message_filter_pii_block` sowie eine Nachricht zurück, die den Aufrufer auffordert, den übereinstimmenden Anmeldedatentyp zu entfernen.

Der Filter prüft:

- Die Chat-Route verarbeitet Benutzereingaben, die über die LibreChat UI übermittelt werden.
- Textinhalt über alle vom Aufrufer bereitgestellten Nachrichtenrollen in OpenAI-kompatiblen Chat Completions-Anfragen.
- Textinhalt über alle vom Aufrufer bereitgestellten Nachrichtenrollen in OpenAI-kompatiblen Responses-Anfragen.

Der Filter wertet den im Request-Body bereitgestellten Text aus. Er scannt keine Dateiinhalte.
