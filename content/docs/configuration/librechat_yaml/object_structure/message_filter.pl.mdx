## Przegląd [#overview]

Obiekt `messageFilter` konfiguruje serwerowe filtry wiadomości. W wersji v1.3.13 obsługiwanym typem filtra jest `messageFilter.pii`.

`messageFilter.pii` odrzuca przesłany tekst wiadomości, który pasuje do wzorców przypominających dane uwierzytelniające lub wyrażeń regularnych zdefiniowanych przez operatora, zanim tekst trafi do moderacji, żądania do modelu lub utrwalenia wiadomości.

## Przykład [#example]

```yaml filename="librechat.yaml"
messageFilter:
  pii:
    starterPatterns:
      - sk_prefix
      - bearer_header
      - api_key_header
    customPatterns:
      - id: anthropic_api_key
        label: Anthropic API key
        regex: 'sk-ant-[A-Za-z0-9_-]{20,}'
```

## Zachowanie [#behavior]

Gdy przesłany tekst wiadomości pasuje do skonfigurowanego wzorca PII, LibreChat zwraca kod HTTP `400` wraz z `message_filter_pii_block` oraz komunikatem proszącym użytkownika o usunięcie dopasowanego typu danych uwierzytelniających.

Filtr sprawdza:

- Tekst użytkownika trasy czatu przesłany przez interfejs użytkownika LibreChat.
- Treść tekstu we wszystkich rolach wiadomości dostarczonych przez wywołującego w żądaniach Chat Completions zgodnych z OpenAI.
- Treść tekstowa we wszystkich rolach wiadomości dostarczonych przez wywołującego w żądaniach odpowiedzi zgodnych z OpenAI.

Filtr ocenia tekst dostarczony w treści żądania. Nie skanuje on zawartości plików.
