## Panoramica [#overview]

L'oggetto `messageFilter` configura i filtri dei messaggi lato server. Nella v1.3.13, il tipo di filtro supportato è `messageFilter.pii`.

`messageFilter.pii` rifiuta il testo del messaggio inviato che corrisponde a pattern di credenziali o a espressioni regolari definite dall'operatore prima che il testo raggiunga la moderazione, una richiesta al modello o la persistenza del messaggio.

## Esempio [#example]

```yaml filename="librechat.yaml"
messageFilter:
  pii:
    starterPatterns:
      - sk_prefix
      - bearer_header
      - api_key_header
    customPatterns:
      - id: anthropic_api_key
        label: Anthropic API key
        regex: 'sk-ant-[A-Za-z0-9_-]{20,}'
```

**ID dei pattern di avvio integrati:**

- `sk_prefix`: corrisponde ai prefissi dei token in stile `sk-`.
- `bearer_header`: corrisponde ai token bearer nel testo.
- `api_key_header`: corrisponde al testo in formato header `api-key`.

## Comportamento [#behavior]

Quando il testo del messaggio inviato corrisponde a un pattern PII configurato, LibreChat restituisce un HTTP `400` con `message_filter_pii_block` e un messaggio che richiede al chiamante di rimuovere il tipo di credenziale corrispondente.

Il filtro controlla:

- Instrada il testo dell'utente inviato tramite l'interfaccia utente di LibreChat.
- Contenuto del testo in tutti i ruoli dei messaggi forniti dal chiamante nelle richieste di Chat Completions compatibili con OpenAI.
- Contenuto del testo attraverso tutti i ruoli dei messaggi forniti dal chiamante nelle richieste di Risposte compatibili con OpenAI.

Il filtro valuta il testo fornito nel corpo della richiesta. Non esegue la scansione del contenuto dei file.
