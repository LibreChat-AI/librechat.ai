## 概述 [#overview]

`messageFilter` 对象用于配置服务器端消息过滤器。在 v1.3.13 版本中，支持的过滤器类型为 `messageFilter.pii`。

`messageFilter.pii` 会在文本到达审核、模型请求或消息持久化之前，拒绝匹配凭据类起始模式或操作员定义的正则表达式的已提交消息文本。

## 示例 [#example]

```yaml filename="librechat.yaml"
messageFilter:
  pii:
    starterPatterns:
      - sk_prefix
      - bearer_header
      - api_key_header
    customPatterns:
      - id: anthropic_api_key
        label: Anthropic API key
        regex: 'sk-ant-[A-Za-z0-9_-]{20,}'
```

**内置的 starter pattern id：**

- `sk_prefix`：匹配 `sk-` 样式的令牌前缀。
- `bearer_header`：匹配文本中的 bearer tokens。
- `api_key_header`：匹配 `api-key` 标头格式的文本。

## 行为 [#behavior]

当提交的消息文本匹配到已配置的 PII 模式时，LibreChat 将返回 HTTP `400` 状态码以及 `message_filter_pii_block`，并提示调用者移除匹配到的凭据类型。

过滤器检查：

- 通过 LibreChat UI 提交的聊天路由用户文本。
- OpenAI 兼容的 Chat Completions 请求中所有调用者提供的消息角色对应的文本内容。
- 在 OpenAI 兼容的 Responses 请求中，跨所有调用者提供的消息角色的文本内容。

该过滤器会对请求体中提供的文本进行评估。它不会扫描文件内容。
