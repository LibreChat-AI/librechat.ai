Każdy element w `models` jest częścią listy rekordów, będącą wartością logiczną (boolean) lub obiektem:

**Podczas określania modelu jako obiektu:**

Obiekt pozwala na szczegółową konfigurację modelu, w tym jego `deploymentName` i/lub `version`. Ten tryb jest używany w celu uzyskania bardziej precyzyjnej kontroli nad modelami, szczególnie podczas pracy z wieloma wersjami lub wdrożeniami w ramach jednej instancji lub grupy zasobów.

**Przykład**:
```yaml filename="endpoints / azureOpenAI / groups / {group_item} / models / {model_item=Object}"
models:
  gpt-4-vision-preview:
    deploymentName: "gpt-4-vision-preview"
    version: "2024-02-15-preview"
```

**Uwagi:**
- **Deployment Names** oraz **Versions** mają kluczowe znaczenie dla zapewnienia użycia właściwego modelu.
    - Sprawdź dokładnie te wartości pod kątem poprawności, aby zapobiec nieoczekiwanemu zachowaniu.
