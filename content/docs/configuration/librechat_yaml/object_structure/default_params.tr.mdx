**Not**
- Dokümantasyonun bu bölümünün amacı, `addParams` ve `dropParams` işlevlerinin ne işe yaradığını anlamanıza yardımcı olmaktır. LibreChat'in varsayılan olarak gönderdiği parametreleri ve değerlerini küresel olarak yapılandır**AMAZSINIZ**; bu yalnızca tek bir endpoint içerisinde yapılandırılabilir.

Özel endpoint'ler, OpenAI endpoint'i ile mantığı paylaşır ve bu nedenle OpenAI API'sine göre uyarlanmış varsayılan parametrelere sahiptir.

```yaml filename="Default Parameters"
{
  "model": "your-selected-model",
  "user": "LibreChat_User_ID",
  "stream": true,
  "messages": [
    {
      "role": "user",
      "content": "hi how are you",
    },
  ],
}
```

### Döküm [#breakdown]
- `model`: Model listesinden seçilen model.
- `user`: Son kullanıcınızı temsil eden ve OpenAI'ın [kötüye kullanımı izlemesine ve tespit etmesine](https://platform.openai.com/docs/api-reference/chat/create#chat-create-user) yardımcı olabilecek benzersiz bir tanımlayıcı.
- `stream`: Ayarlanırsa, ChatGPT'de olduğu gibi kısmi mesaj deltaları gönderilir. Aksi takdirde, oluşturma işlemi yalnızca tamamlandığında kullanılabilir olacaktır.
- `messages`: [OpenAI format for messages](https://platform.openai.com/docs/api-reference/chat/create#chat-create-messages); `name` alanı, hazır ayar (preset) aracılığıyla özel bir isim belirtildiğinde `system` ve `assistant` rollerine sahip mesajlara eklenir.

**Not:** `max_tokens` alanı, varsayılan OpenAI API davranışında olduğu gibi mevcut maksimum token miktarını kullanmak için gönderilmez. Bazı alternatif API'ler bu alanı gerektirebilir veya bu alan çok düşük bir değere varsayılan olarak ayarlanabilir ve yanıtlarınız kesilmiş görünebilir; bu durumda, [Özel Uç Nokta Nesne Yapısı](/docs/configuration/librechat_yaml/object_structure/custom_endpoint) bölümünde gösterildiği gibi bunu `addParams` alanına eklemelisiniz.
