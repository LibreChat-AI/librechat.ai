## Übersicht [#overview]

Das `fileConfig`-Objekt ermöglicht es Ihnen, die Einstellungen für die Dateiverarbeitung der Anwendung zu konfigurieren, einschließlich Größenbeschränkungen und MIME-Typ-Einschränkungen. Dieser Abschnitt bietet eine detaillierte Aufschlüsselung der Struktur des `fileConfig`-Objekts.

Es gibt 8 Hauptfelder unter `fileConfig`:

  - `endpoints`
  - `serverFileSizeLimit`
  - `avatarSizeLimit`
  - `imageGeneration`
  - `fileTokenLimit`
  - `ocr`
  - `text`
  - `stt`

**Hinweise:**

- Zum Zeitpunkt der Erstellung dieses Dokuments unterstützt der Assistants endpoint [Dateitypen aus dieser Liste](https://platform.openai.com/docs/assistants/tools/file-search#supported-files).
- OpenAI, Azure OpenAI, Google und benutzerdefinierte endpoints unterstützen Dateien über die [RAG API.](../../rag_api.mdx)
- OpenAI, Azure OpenAI, Anthropic, Google und AWS Bedrock endpoints unterstützen direkte Datei-Uploads über [Upload to Provider](/docs/features/ocr#5-upload-files-to-provider-direct).
- Die Abschnitte `ocr`, `text` und `stt` steuern die Dateiverarbeitung für Funktionen wie [Upload as Text](/docs/features/upload_as_text) und [OCR](/docs/features/ocr)
- Alle anderen nicht erwähnten endpoints, wie Plugins, unterstützen (noch) keine Datei-Uploads.
- Der Assistants endpoint hat einen definierten endpoint-Wert von `assistants`. Alle anderen endpoints verwenden den definierten Wert `default`.
  - Für Endpunkte, die keine Assistants sind, können Sie die Dateieinstellungen für alle unter `default` anpassen.
  - Wenn Sie Einstellungen für ein bestimmtes endpoint anpassen möchten, können Sie die entsprechenden endpoint-Namen auflisten:
    - `assistants`
        - verwendet nicht "default", da es separate Standardwerte definiert, die sich von den anderen unterscheiden.
    - `openAI`
    - `azureOpenAI`
    - `google`
    - `bedrock`
    - `YourCustomEndpointName`
- Sie können Werte weglassen; in diesem Fall verwendet die App die Standardwerte, wie sie für den jeweiligen endpoint-Typ unten aufgeführt sind.
- LibreChat berechnet 1 Megabyte wie folgt: `1 x 1024 x 1024`

## Beispiel [#example]

```yaml filename="fileConfig"
fileConfig:
  endpoints:
    assistants:
      fileLimit: 5
      fileSizeLimit: 10
      totalSizeLimit: 50
      supportedMimeTypes:
        - "image/.*"
        - "application/pdf"
    openAI:
      disabled: true
    default:
      totalSizeLimit: 20
    YourCustomEndpointName:
      fileLimit: 5
      fileSizeLimit: 1000
      supportedMimeTypes:
        - "image/.*"
  serverFileSizeLimit: 1000
  avatarSizeLimit: 2
  fileTokenLimit: 100000
  imageGeneration:
    percentage: 100
    px: 1024
  ocr:
    supportedMimeTypes:
      - "^image/(jpeg|gif|png|webp|heic|heif)$"
      - "^application/pdf$"
      - "^application/vnd\\.openxmlformats-officedocument\\.(wordprocessingml\\.document|presentationml\\.presentation|spreadsheetml\\.sheet)$"
      - "^application/vnd\\.ms-(word|powerpoint|excel)$"
      - "^application/epub\\+zip$"
  text:
    supportedMimeTypes:
      - "^text/(plain|markdown|csv|json|xml|html|css|javascript|typescript|x-python|x-java|x-csharp|x-php|x-ruby|x-go|x-rust|x-kotlin|x-swift|x-scala|x-perl|x-lua|x-shell|x-sql|x-yaml|x-toml)$"
  stt:
    supportedMimeTypes:
      - "^audio/(mp3|mpeg|mpeg3|wav|wave|x-wav|ogg|vorbis|mp4|x-m4a|flac|x-flac|webm)$"
```

## imageGeneration

<OptionTable
  options={[
    ['imageGeneration', 'Object', 'Einstellungen bezüglich der Qualität und Dimensionen der Bildgenerierungsausgabe.', 'Allows configuration of either output size as a percentage relative to some base size or as an explicit pixel dimension.'],
  ]}
/>

`imageGeneration` unterstützt die folgenden Parameter:

- `percentage` (Ganzzahl)
  - Die Ausgabegröße des generierten Bildes, ausgedrückt als Prozentsatz (z. B. bedeutet `100` 100 % der Basisgröße).
  - Verwenden Sie dies, um die Ausgabegröße des Bildes relativ zu einer Standard- oder Originalgröße zu skalieren.

- `px` (Ganzzahl)
  - Gibt die Ausgabebilddimension in Pixeln an (z. B. `1024`).
  - Verwenden Sie dies, um die Ausgabegröße des generierten Bildes unabhängig von der Basisgröße explizit festzulegen.

Sie können nur einen dieser Parameter (`percentage` oder `px`) festlegen, nicht beide, abhängig von Ihrem Anwendungsfall.

Beispielkonfiguration:

```yaml filename="fileConfig / imageGeneration"
fileConfig:
  imageGeneration:
    percentage: 100
    px: 1024
```

## ocr

<OptionTable
  options={[
    ['ocr', 'Object', 'Einstellungen für die OCR-Dateiverarbeitung (Optical Character Recognition).', 'Configures which file types are processed using OCR.'],
  ]}
/>

**Beschreibung:** Der `ocr`-Abschnitt konfiguriert, welche Dateitypen mithilfe der OCR-Funktionalität verarbeitet werden sollen, um Text aus visuellen Dokumenten zu extrahieren.

**Hinweis:** Dieser Abschnitt steuert den Abgleich von Dateitypen für die OCR-Verarbeitung. Um Agent-Funktionen zu aktivieren und OCR-Dienste zu konfigurieren, siehe:
- [Agents-Konfiguration](/docs/configuration/librechat_yaml/object_structure/agents#capabilities) für die `ocr`- und `context`-Funktionen
- [OCR-Konfiguration](/docs/configuration/librechat_yaml/object_structure/ocr) für die Einrichtung des OCR-Dienstes

### supportedMimeTypes

<OptionTable
  options={[
    ['supportedMimeTypes', 'Array of Strings', 'Liste der MIME-Typ-Muster für Dateien, die mit OCR verarbeitet werden sollen.', 'Uses regular expressions to match file types.'],
  ]}
/>

**Standard:** Bilder, PDFs und Office-Dokumente

```yaml filename="fileConfig / ocr / supportedMimeTypes"
fileConfig:
  ocr:
    supportedMimeTypes:
      - "^image/(jpeg|gif|png|webp|heic|heif)$"
      - "^application/pdf$"
      - "^application/vnd\\.openxmlformats-officedocument\\.(wordprocessingml\\.document|presentationml\\.presentation|spreadsheetml\\.sheet)$"
      - "^application/vnd\\.ms-(word|powerpoint|excel)$"
      - "^application/epub\\+zip$"
```

## text

<OptionTable
  options={[
    ['text', 'Object', 'Einstellungen für die direkte Textdateianalyse ohne OCR.', 'Configures which file types are processed as plain text files for direct content extraction.'],
  ]}
/>

**Beschreibung:** Der `text`-Abschnitt konfiguriert, welche Dateitypen mittels direkter Textextraktion verarbeitet werden sollen.

**Hinweis:** Die Textanalyse ist die Standardmethode, die von der Funktion „Als Text hochladen“ verwendet wird (gesteuert durch die `context`-Fähigkeit). Sie versucht zuerst, die Textanalyse-Bibliothek der RAG API zu verwenden. Wenn die RAG API nicht verbunden ist, greift sie auf eine einfachere Textextraktionsmethode zurück, ohne dass externe Dienste erforderlich sind. Weitere Informationen finden Sie unter [Als Text hochladen](/docs/features/upload_as_text).

**Standard:** Alle Textdateien und gängigen Programmiersprachen

## stt

<OptionTable
  options={[
    ['stt', 'Object', 'Einstellungen für die Verarbeitung von Speech-to-Text (STT)-Audiodateien.', 'Configures which audio file types are processed using STT for transcription.'],
  ]}
/>

**Beschreibung:** Der `stt` Abschnitt konfiguriert, welche Audiodateitypen mithilfe der Speech-to-Text-Funktionalität für die Umwandlung von Audio in Text verarbeitet werden sollen.

### supportedMimeTypes [#supportedmimetypes-2]

<OptionTable
  options={[
    ['supportedMimeTypes', 'Array of Strings', 'Liste der MIME-Typ-Muster für Audiodateien, die mit STT transkribiert werden sollen.', 'Uses regular expressions to match audio file types.'],
  ]}
/>

**Standard:** Gängige Audioformate

```yaml filename="fileConfig / stt / supportedMimeTypes"
fileConfig:
  stt:
    supportedMimeTypes:
      - "^audio/(mp3|mpeg|mpeg3|wav|wave|x-wav|ogg|vorbis|mp4|x-m4a|flac|x-flac|webm)$"
```

**Hinweise:**
- Dateien, die `text`-Mustern entsprechen, werden mit einfacher Textextraktion verarbeitet
- Dateien, die den `ocr`-Mustern entsprechen, werden mit dem bereitgestellten OCR-Dienst verarbeitet
- Dateien, die den `stt`-Mustern entsprechen, werden mit Speech-to-Text-Transkription verarbeitet
- **Verarbeitungspriorität: OCR > STT > Text-Parsing > Fallback**
- Dateien, die keinem Muster entsprechen, werden auf die Textanalyse zurückgreifen.

## Priorität der Dateiverarbeitung [#file-processing-priority]

LibreChat verarbeitet hochgeladene Dateien basierend auf dem MIME-Typ-Abgleich in der folgenden **Prioritätsreihenfolge**:

1. **OCR** – Wenn die Datei `ocr.supportedMimeTypes` entspricht UND OCR konfiguriert ist
2. **STT** - Wenn die Datei `stt.supportedMimeTypes` entspricht UND STT konfiguriert ist
3. **Text-Parsing** – Wenn die Datei `text.supportedMimeTypes` entspricht
4. **Fallback** – Text-Parsing als letzter Ausweg

Diese Verarbeitungsreihenfolge stellt eine optimale Extraktionsqualität sicher und erhält gleichzeitig die Funktionalität aufrecht, selbst wenn spezialisierte Dienste (OCR/STT) nicht konfiguriert sind.

### Verarbeitungsbeispiele [#processing-examples]

**PDF-Datei mit konfigurierter OCR:**
- Datei entspricht `ocr.supportedMimeTypes`
- **Verwendet OCR**, um Text zu extrahieren
- Bessere Qualität für gescannte PDFs und Bilder

**PDF-Datei ohne konfigurierte OCR:**
- Die Datei entspricht `text.supportedMimeTypes` (oder verwendet einen Fallback)
- **Verwendet die** Text-Parsing-Bibliothek
- Funktioniert gut für digitale PDFs mit auswählbarem Text

**Python-Datei:**
- Datei entspricht `text.supportedMimeTypes`
- **Verwendet Text-Parsing** (kein OCR erforderlich)
- Direkte Textextraktion

**Audiodatei mit konfiguriertem STT:**
- Datei entspricht `stt.supportedMimeTypes`
- **Verwendet STT**, um Audio in Text zu transkribieren

**Bilddatei ohne konfiguriertes OCR:**
- Datei entspricht `ocr.supportedMimeTypes`, aber OCR ist nicht verfügbar
- **Fällt auf Text-Parsing zurück**
- Eingeschränkte Extraktionsfähigkeit ohne OCR

Dieses Prioritätssystem ermöglicht es Funktionen wie „Upload as Text“, ohne OCR-Konfiguration zu funktionieren, während gleichzeitig OCR genutzt wird, sofern verfügbar, um die Extraktionsqualität zu verbessern.

## endpoints

<OptionTable
  options={[
    ['endpoints', 'Record/Object', 'Konfiguriert die Dateiverarbeitungseinstellungen für einzelne endpoints und ermöglicht eine Anpassung auf Basis des jeweiligen endpoints.', 'Specifies file handling configurations for individual endpoints, allowing customization per endpoint basis.'],
  ]}
/>

**Beschreibung:** Jedes Objekt unter endpoints ist ein Datensatz, der die folgenden Einstellungen enthalten kann:

### Übersicht [#overview-1]

  - `disabled`
      - Ob die Dateiverarbeitung für den endpoint deaktiviert ist.
  - `fileLimit`
      - Die maximale Anzahl an Dateien, die pro Upload-Anfrage erlaubt sind.
  - `fileSizeLimit`
      - Die maximale Größe für eine einzelne Datei. In Einheiten von MB (z. B. `20` für 20 Megabyte verwenden)
  - `totalSizeLimit`
      - Die maximale Gesamtgröße für alle Dateien in einer einzelnen Anfrage. In Einheiten von MB (verwenden Sie z. B. `20` für 20 Megabyte)
  - `supportedMimeTypes`
      - Eine Liste von [Regular Expressions](https://en.wikipedia.org/wiki/Regular_expression), die festlegt, welche MIME-Typen für den Upload zulässig sind. Dies kann angepasst werden, um Dateitypen einzuschränken.
