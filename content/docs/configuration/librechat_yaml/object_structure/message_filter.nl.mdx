## Overzicht [#overview]

Het `messageFilter` object configureert server-side berichtfilters. In v1.3.13 is het ondersteunde filtertype `messageFilter.pii`.

`messageFilter.pii` weigert ingediende berichttekst die overeenkomt met patronen van inloggegevens of door de operator gedefinieerde reguliere expressies voordat de tekst moderatie, een modelverzoek of berichtpersistentie bereikt.

## Voorbeeld [#example]

```yaml filename="librechat.yaml"
messageFilter:
  pii:
    starterPatterns:
      - sk_prefix
      - bearer_header
      - api_key_header
    customPatterns:
      - id: anthropic_api_key
        label: Anthropic API key
        regex: 'sk-ant-[A-Za-z0-9_-]{20,}'
```

- `sk_prefix`: komt overeen met token-prefixen in `sk-` stijl.
- `bearer_header`: komt overeen met bearer tokens in tekst.
- `api_key_header`: komt overeen met `api-key` header-vormige tekst.

## Gedrag [#behavior]

Wanneer de ingediende berichttekst overeenkomt met een geconfigureerd PII-patroon, retourneert LibreChat een HTTP `400` met `message_filter_pii_block` en een bericht waarin de aanroeper wordt gevraagd het overeenkomende type inloggegevens te verwijderen.

De filter controleert:

- Chat-route voor gebruikerstekst ingediend via de LibreChat UI.
- Tekstinhoud in alle door de aanroeper opgegeven berichtrollen in OpenAI-compatibele Chat Completions-verzoeken.
- Tekstinhoud over alle door de aanroeper opgegeven berichtrollen in OpenAI-compatibele Responses-verzoeken.

Het filter evalueert tekst die in de request body wordt aangeleverd. Het scant geen bestandsinhoud.
