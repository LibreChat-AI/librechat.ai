**Nota**
- O objetivo desta parte da documentação é ajudar a entender o que `addParams` e `dropParams` fazem. Você **NÃO PODE** configurar globalmente os parâmetros e seus valores que o LibreChat envia por padrão; isso só pode ser configurado dentro de um único endpoint.

Endpoints personalizados compartilham a lógica com o endpoint da OpenAI e, portanto, possuem parâmetros padrão adaptados para a API da OpenAI.

```yaml filename="Default Parameters"
{
  "model": "your-selected-model",
  "user": "LibreChat_User_ID",
  "stream": true,
  "messages": [
    {
      "role": "user",
      "content": "hi how are you",
    },
  ],
}
```

### Detalhamento [#breakdown]
- `model`: O modelo selecionado a partir da lista de modelos.
- `user`: Um identificador único que representa seu usuário final, o que pode ajudar a OpenAI a [monitorar e detectar abusos](https://platform.openai.com/docs/api-reference/chat/create#chat-create-user).
- `stream`: Se definido, deltas de mensagens parciais serão enviados, como no ChatGPT. Caso contrário, a geração só estará disponível quando concluída.
- `messages`: [formato de mensagens da OpenAI](https://platform.openai.com/docs/api-reference/chat/create#chat-create-messages); o campo `name` é adicionado a mensagens com as funções `system` e `assistant` quando um nome personalizado é especificado via preset.

**Nota:** O campo `max_tokens` não é enviado para usar a quantidade máxima de tokens disponível, o que é o comportamento padrão da API da OpenAI. Algumas APIs alternativas exigem este campo, ou podem definir um valor padrão muito baixo, fazendo com que suas respostas pareçam cortadas; neste caso, você deve adicioná-lo ao campo `addParams` conforme mostrado na [Estrutura de Objeto de Endpoint Personalizado](/docs/configuration/librechat_yaml/object_structure/custom_endpoint).
