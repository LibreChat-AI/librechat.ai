## Descripción general [#overview]

El objeto `skillSync` permite a los administradores replicar Skills desde fuentes externas. En la versión v1.3.13, GitHub es el proveedor compatible.

GitHub Skill Sync lee los archivos `SKILL.md` y sus archivos adjuntos desde las rutas del repositorio configuradas, los almacena como Skills con `source: "github"` y mantiene las filas reflejadas alineadas con el repositorio original en sincronizaciones posteriores.

## Ejemplo [#example]

```yaml filename="librechat.yaml"
skillSync:
  github:
    enabled: true
    intervalMinutes: 60
    runOnStartup: true
    sources:
      - id: librechat-skills
        owner: your-org
        repo: your-skills-repo
        ref: main
        paths:
          - skills
        skillDiscoveryDepth: 2
        token: '${GITHUB_SKILLS_TOKEN}'
        # credentialKey: production-skills
        # tenantId: your-tenant-id
```

<Callout type="warning" title="Credenciales">
  Utilice `token` o `credentialKey` para cada fuente, pero no ambos. `token` debe ser una referencia a una variable de entorno como `${GITHUB_SKILLS_TOKEN}`. Utilice un personal access token de GitHub de tipo fine-grained con alcance al repositorio seleccionado y con permisos de solo lectura para Contents y Metadata.
</Callout>

### Identidad de origen [#source-identity]

El campo `id` es parte de la identidad del espejo. Mantenlo estable. Redirigir una fuente a un repositorio renombrado o a una nueva referencia mantiene las Skills espejadas vinculadas al mismo id de fuente, mientras que cambiar el id de la fuente crea un nuevo espejo.

Cuando `tenantId` está configurado, las lecturas y escrituras de sincronización se ejecutan dentro del contexto de aislamiento de ese inquilino. Trate `tenantId` como inmutable para un id de origen: cambiarlo, añadirlo o eliminarlo posteriormente deja las Skills reflejadas anteriormente en el inquilino antiguo.

## Comportamiento de sincronización [#sync-behavior]

Sincronización de habilidades de GitHub:

- Descubre archivos `SKILL.md` debajo de cada ruta configurada.
- Importa el cuerpo, el frontmatter y los archivos incluidos de `SKILL.md`.
- Almacena metadatos de origen como source id, owner, repo, ref, skill path, commit SHA y blob SHA.
- Actualiza las Skills reflejadas cuando el archivo original cambia.
- Elimina las Skills y archivos reflejados que fueron eliminados de la fuente configurada.
- Utiliza un bloqueo de sincronización para que solo se ejecute una sincronización de GitHub a la vez.
