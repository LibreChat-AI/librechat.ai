## ¿Qué es librechat.yaml? [#what-is-librechatyaml]

El archivo `librechat.yaml` es el archivo de configuración principal de LibreChat para endpoints de IA personalizados, ajustes de modelos, opciones de interfaz y funciones avanzadas como servidores MCP y agentes. Es opcional: LibreChat funciona con valores predeterminados razonables si el archivo no existe.

Siga los pasos a continuación para crear el archivo, montarlo para su tipo de despliegue y verificar que funcione.

<Callout type="info" title="Si solo recuerdas una cosa">

Para instalaciones de Docker, editar `librechat.yaml` no es suficiente. El archivo debe existir en la raíz del proyecto, estar montado en el contenedor de la API y LibreChat debe reiniciarse antes de que los cambios aparezcan en la UI.

</Callout>

<Callout type="info" title="¿Prefieres una interfaz de usuario? Usa el Admin Panel">

El [**Panel de administración de LibreChat**](/docs/features/admin_panel) gestiona esta misma configuración desde un navegador, incluyendo anulaciones por rol y por grupo que entran en vigor al iniciar sesión sin necesidad de reiniciar LibreChat. Se incluye con las pilas oficiales de Docker Compose. Utilice `librechat.yaml` para la configuración basada en archivos o de arranque, y el panel de administración para la gestión continua.

</Callout>

## Configuración [#setup]

<Steps>
  <Step>

### Localizar o crear el archivo [#locate-or-create-the-file]

Cree un nuevo `librechat.yaml` en la raíz de su proyecto (el mismo directorio que su archivo `.env`):

```bash
touch librechat.yaml
```

También puedes copiar la [configuración de ejemplo](/docs/configuration/librechat_yaml/example) como punto de partida:

```bash
cp librechat.example.yaml librechat.yaml
```

<Callout type="info" title="Ruta de archivo alternativa">

Puede establecer una ruta de archivo personalizada utilizando la variable de entorno `CONFIG_PATH`:

```bash filename=".env"
CONFIG_PATH="/alternative/path/to/librechat.yaml"
```

</Callout>

  </Step>
  <Step>

### Montar el archivo de configuración [#mount-the-config-file]

<Tabs items={['Docker', 'Local']}>
  <Tabs.Tab>

Docker necesita un montaje de volumen para acceder a su archivo `librechat.yaml` dentro del contenedor.

**Copie el archivo de anulación de ejemplo:**

```bash
cp docker-compose.override.yml.example docker-compose.override.yml
```

**Edita `docker-compose.override.yml`** y asegúrate de que el montaje de volumen de librechat.yaml no esté comentado:

```yaml filename="docker-compose.override.yml"
services:
  api:
    volumes:
      - type: bind
        source: ./librechat.yaml
        target: /app/librechat.yaml
```

Esto utiliza el patrón [docker-compose.override.yml](/docs/configuration/docker_override): Docker Compose lo fusiona automáticamente con el `docker-compose.yml` principal, por lo que tus personalizaciones sobreviven a las actualizaciones.

  </Tabs.Tab>
  <Tabs.Tab>

Coloque `librechat.yaml` en el directorio raíz del proyecto (el mismo directorio que su archivo `.env`). No se necesita montaje adicional para instalaciones locales.

  </Tabs.Tab>
</Tabs>

  </Step>
  <Step>

### Reiniciar LibreChat [#restart-librechat]

<Tabs items={['Docker', 'Local']}>
  <Tabs.Tab>

```bash
docker compose down && docker compose up -d
```

  </Tabs.Tab>
  <Tabs.Tab>

Detenga el proceso en ejecución (Ctrl+C) y reinicie:

```bash
npm run backend
```

  </Tabs.Tab>
</Tabs>

  </Step>
  <Step>

### Verificar que funciona [#verify-it-works]

Abra LibreChat en su navegador. Si su configuración incluye endpoints personalizados, debería verlos en el menú desplegable del selector de modelos.

Si el servidor no logra iniciarse, revisa los registros en busca de errores de validación:

```bash
docker compose logs api
```

  </Step>
</Steps>

## Ejemplo: Agregar OpenRouter [#example-adding-openrouter]

Este ejemplo explica cómo añadir [OpenRouter](https://openrouter.ai/) como un endpoint personalizado, una de las configuraciones más populares.

**1. Obtén una API key** en [openrouter.ai/keys](https://openrouter.ai/keys).

**2. Agregue la clave a su archivo `.env`:**

```bash filename=".env"
OPENROUTER_KEY=sk-or-v1-your-key-here
```

<Callout type="warning" title="Nombre de la variable de entorno">

Use `OPENROUTER_KEY`, no `OPENROUTER_API_KEY`. Usar `OPENROUTER_API_KEY` sobrescribirá el endpoint de OpenAI para usar OpenRouter también.

</Callout>

**3. Añade el endpoint a `librechat.yaml`:**

```yaml filename="librechat.yaml"
version: 1.3.5
cache: true
endpoints:
  custom:
    - name: "OpenRouter"
      apiKey: "${OPENROUTER_KEY}"
      baseURL: "https://openrouter.ai/api/v1"
      models:
        default: ["meta-llama/llama-3-70b-instruct"]
        fetch: true
      titleConvo: true
      titleModel: "meta-llama/llama-3-70b-instruct"
      dropParams: ["stop"]
      modelDisplayLabel: "OpenRouter"
```

**4. Reinicie LibreChat** (consulte los comandos de reinicio anteriores) y seleccione OpenRouter en el selector de modelos.

Para obtener el archivo de configuración anotado completo con más ejemplos de endpoint, consulte la [configuración de ejemplo](/docs/configuration/librechat_yaml/example).

## Solución de problemas [#troubleshooting]

### Los cambios no se reflejan en LibreChat [#change-does-not-show-in-librechat]

Si editaste `librechat.yaml` y no hubo cambios en la UI:

1. Confirme que el archivo se encuentra en la raíz del proyecto de LibreChat a menos que haya configurado `CONFIG_PATH`.
2. Para Docker, confirme que el archivo esté montado en `docker-compose.override.yml`.
3. Reinicie LibreChat con `docker compose down && docker compose up -d`.
4. Revisa los registros de la API con `docker compose logs api`.
5. Valide el archivo con el [YAML Validator](/toolkit/yaml_checker).

Los endpoints personalizados como OpenRouter solo aparecen después de que las tres partes sean correctas: `.env` contiene la clave, `librechat.yaml` define el endpoint y Docker puede leer el archivo de configuración montado.

### Validación de configuración [#configuration-validation]

<Callout type="error" title="Validación de configuración">

LibreChat se cierra con un error (código de salida 1) si `librechat.yaml` contiene errores de validación. Este comportamiento de fallo rápido detecta problemas de configuración de forma temprana.

Para validar tu sintaxis YAML antes de reiniciar, utiliza el [YAML Validator](/toolkit/yaml_checker) o [yamlchecker.com](https://yamlchecker.com/).

</Callout>

### El servidor se cierra inmediatamente al iniciar [#server-exits-immediately-on-startup]

Si tu servidor se cierra inmediatamente después de iniciarse, es probable que se trate de un error de validación de configuración.

**Para diagnosticar:**

1. Revise los registros del servidor: `docker compose logs api`
2. Valide su sintaxis YAML con el [YAML Validator](/toolkit/yaml_checker)
3. Errores comunes: sangría incorrecta, dos puntos faltantes, claves desconocidas, valores no válidos

**Solución temporal** (no recomendada para producción):

```bash filename=".env"
CONFIG_BYPASS_VALIDATION=true
```

<Callout type="warning" title="Advertencia">

`CONFIG_BYPASS_VALIDATION=true` hace que el servidor omita la validación y utilice la configuración predeterminada. Siempre corrija los errores de validación en su lugar.

</Callout>
