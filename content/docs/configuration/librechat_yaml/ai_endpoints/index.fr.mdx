## Intro [#intro]

- Cette section répertorie les endpoints d'IA connus et compatibles, également appelés « Custom Endpoints », avec des exemples de configuration pour le fichier `librechat.yaml`, aussi connu sous le nom de fichier [Custom Config](/docs/configuration/librechat_yaml).

- Dans tous les exemples, des noms de variables d'environnement arbitraires sont définis, mais vous pouvez utiliser le nom de votre choix, ainsi que modifier la valeur par `user_provided` pour permettre aux utilisateurs de soumettre leur propre clé API depuis l'interface web.

<Callout type="warning" title="Important : Paramètre de clé 'user_provided'">
Lorsque vous définissez des clés API sur "user_provided", cela permet aux utilisateurs de saisir leurs propres clés API via l'interface web. Ceci est différent des endpoints préconfigurés dans le fichier .env où vous définiriez `ENDPOINT_KEY=user_provided` (par exemple, `OPENAI_API_KEY=user_provided`).

Pour les endpoints personnalisés dans librechat.yaml, vous utiliseriez :
```yaml
endpoints:
  custom:
    - name: "Your Endpoint"
      apiKey: "user_provided"  # No need for ${} syntax here
```

Pour les variables d'environnement dans le fichier .env, vous utiliseriez :
```bash
OPENAI_API_KEY=user_provided
```
</Callout>

- Certains endpoints sont marqués comme **Known**, ce qui signifie qu'ils peuvent bénéficier d'un traitement spécial et/ou d'une icône déjà fournie dans l'application pour vous.

### Notes [#notes]

- Il est recommandé de suivre le [Guide de démarrage rapide des endpoints personnalisés](/docs/quick_start/custom_endpoints) avant de poursuivre avec les exemples ci-dessous.
- Important : assurez-vous de configurer correctement le fichier `librechat.yaml` : **[documentation de configuration](/docs/configuration/librechat_yaml)**.
