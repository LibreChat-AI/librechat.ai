TrueFoundry AI Gateway est la couche proxy qui se situe entre vos applications et les fournisseurs de LLM ainsi que les serveurs MCP. Il s'agit d'une plateforme de niveau entreprise qui permet aux utilisateurs d'accéder à plus de 1000 LLM via une interface unifiée, tout en assurant l'observabilité et la gouvernance.

## Détails de configuration [#configuration-details]

Pour utiliser [TrueFoundry's AI Gateway](https://www.truefoundry.com/ai-gateway), suivez le guide de démarrage rapide [ici](https://docs.truefoundry.com/gateway/quick-start).

### Obtention de l'URL de base et des noms de modèles [#getting-base-url-and-model-names]

- **Base URL et noms de modèles** : Obtenez votre URL d'endpoint TrueFoundry AI Gateway et le nom du modèle à partir de l'extrait de code unifié (assurez-vous d'avoir ajouté le même nom de modèle)
- **PAT** : Générez à partir de votre Personal Access Token [PAT](https://docs.truefoundry.com/gateway/authentication#personal-access-token-pat) de TrueFoundry

## Utilisation de TrueFoundry AI Gateway avec n'importe quel modèle LLM [#using-truefoundry-ai-gateway-with-any-llm-model]
```yaml filename="librechat.yaml"
      - name: "TrueFoundry"
        apiKey: "${TRUEFOUNDRY_API_KEY}"
        baseURL: "${TRUEFOUNDRY_GATEWAY_URL}"
        models:
            default: ["openai-main/gpt-4o-mini", "openai-main/gpt-4o"]
            fetch: true
        titleConvo: true
        titleModel: "current_model"
        summarize: false
        summaryModel: "current_model"
        modelDisplayLabel: "TrueFoundry:OpenAI"
```
Pour plus de détails, vous pouvez consulter : [TrueFoundry Docs](https://docs.truefoundry.com/docs/introduction)
