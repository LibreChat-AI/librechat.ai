TrueFoundry AI Gateway est une couche de proxy d'entreprise située entre vos applications et les fournisseurs de LLM, offrant un accès à plus de 1000 modèles via une interface unifiée compatible avec OpenAI, avec des fonctionnalités intégrées d'observabilité et de gouvernance.

## Obtenir une clé API [#get-an-api-key]

Authentifiez-vous avec un Personal Access Token. Générez-en un en suivant le [guide PAT](https://docs.truefoundry.com/gateway/authentication#personal-access-token-pat), puis ajoutez-le avec votre URL de passerelle à votre fichier `.env` :

```bash filename=".env"
TRUEFOUNDRY_API_KEY=your-api-key
TRUEFOUNDRY_GATEWAY_URL=your-gateway-url
```

Obtenez l'URL de la passerelle et les noms des modèles à partir de l'extrait de code unifié dans la console TrueFoundry. Utilisez les mêmes noms de modèles que ceux que vous avez ajoutés à la passerelle. Le [guide de démarrage rapide](https://docs.truefoundry.com/gateway/quick-start) détaille la configuration.

## Configuration [#configuration]

Ajoutez l'endpoint sous `endpoints.custom` dans votre `librechat.yaml` :

```yaml filename="librechat.yaml"
    - name: "TrueFoundry"
      apiKey: "${TRUEFOUNDRY_API_KEY}"
      baseURL: "${TRUEFOUNDRY_GATEWAY_URL}"
      models:
        default: ["openai-main/gpt-4o-mini", "openai-main/gpt-4o"]
        fetch: true
      titleConvo: true
      titleModel: "current_model"
      summarize: false
      summaryModel: "current_model"
      modelDisplayLabel: "TrueFoundry:OpenAI"
```

## Notes [#notes]

- Les noms de modèles sont soumis à un espace de noms selon l'intégration que vous avez configurée dans la passerelle, comme `openai-main/gpt-4o`. Faites-les correspondre aux noms affichés dans votre console TrueFoundry.
- Pour plus de détails, consultez la [documentation TrueFoundry](https://docs.truefoundry.com/docs/introduction).
