## Intro [#intro]

- Questa sezione elenca gli endpoint AI noti e compatibili, noti anche come "Custom Endpoints", con esempi di configurazione per il file `librechat.yaml`, noto anche come file [Custom Config](/docs/configuration/librechat_yaml).

- In tutti gli esempi, vengono definiti nomi di variabili d'ambiente arbitrari, ma è possibile utilizzare qualsiasi nome desiderato, così come modificare il valore in `user_provided` per consentire agli utenti di inserire la propria chiave API dall'interfaccia web.

<Callout type="warning" title="Importante: Impostazione della chiave 'user_provided'">
Quando si impostano le chiavi API su "user_provided", si consente agli utenti di inserire le proprie chiavi API tramite l'interfaccia web. Questo è diverso dagli endpoint preconfigurati nel file .env in cui si imposterebbe `ENDPOINT_KEY=user_provided` (ad esempio, `OPENAI_API_KEY=user_provided`).

Per gli endpoint personalizzati in librechat.yaml, dovresti usare:
```yaml
endpoints:
  custom:
    - name: "Your Endpoint"
      apiKey: "user_provided"  # No need for ${} syntax here
```

Per le variabili d'ambiente nel file .env, dovresti usare:
```bash
OPENAI_API_KEY=user_provided
```
</Callout>

- Alcuni degli endpoint sono contrassegnati come **Known**, il che significa che potrebbero avere una gestione speciale e/o un'icona già fornita nell'app per te.

### Note [#notes]

- Si consiglia di seguire la [Guida rapida agli endpoint personalizzati](/docs/quick_start/custom_endpoints) prima di procedere con gli esempi seguenti.
- Importante: assicurati di configurare correttamente il file `librechat.yaml`: **[documentazione di configurazione](/docs/configuration/librechat_yaml)**.
