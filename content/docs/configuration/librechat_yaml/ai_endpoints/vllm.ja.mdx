> vLLMは、LLMのための高スループットかつメモリ効率に優れた推論およびサービングエンジンです。

**注記:**

- **不明:** アイコンは提供されていませんが、ローカルのvLLMサーバーから利用可能なモデルを取得するために、モデルリストのフェッチを推奨します。

- `titleMessageRole`は重要です。一部のローカルLLMでは、タイトルメッセージ（デフォルト設定）に対してシステムメッセージロールを受け付けないためです。

- この設定は、指定されたbaseURLでローカルにvLLMサーバーが実行されていることを前提としています。

```yaml
    - name: "vLLM"
      apiKey: "vllm"
      baseURL: "http://127.0.0.1:8023/v1"
      models:
        default: ['google/gemma-3-27b-it']
        fetch: true
      titleConvo: true
      titleModel: "current_model"
      titleMessageRole: "user"
      summarize: false
      summaryModel: "current_model"
```

上記の構成は、LibreChatをポート8023で実行されているローカルのvLLMサーバーに接続します。デフォルトモデルとしてGemma 3 27Bモデルを使用しますが、vLLMサーバーから利用可能なすべてのモデルを取得します。

## 主要な設定オプション [#key-configuration-options]

- `apiKey`: vLLM用のシンプルなプレースホルダー値（ローカルデプロイメントでは通常、認証は不要です）
- `baseURL`: vLLMサーバーが実行されているURL
- `titleMessageRole`: デフォルトの "system" ではなく "user" に設定します。一部のローカルLLMはシステムメッセージをサポートしていないためです。
