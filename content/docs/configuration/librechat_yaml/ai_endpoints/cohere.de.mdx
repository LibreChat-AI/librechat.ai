> Cohere API-Schlüssel: [dashboard.cohere.com](https://dashboard.cohere.com/)

**Hinweise:**

- **Bekannt:** Symbol bereitgestellt.
- Experimentell: folgt nicht der OpenAI-spec, verwendet eine neue Methode für die endpoint-Kompatibilität und weist einige Ähnlichkeiten und Parameter auf.
- Eine vollständige Liste der Cohere-spezifischen Parameter finden Sie in der [Cohere API documentation](https://docs.cohere.com/reference/chat).
- Hinweis: Die folgenden Parameter werden sowohl von OpenAI als auch von Cohere erkannt. Die meisten wurden in der Beispielkonfiguration unten entfernt, um die Standardeinstellungen von Cohere zu bevorzugen:
    - `stop`: zugeordnet zu `stopSequences`
    - `top_p`: zugeordnet zu `p`, abweichende min/max-Werte
    - `frequency_penalty`: wird auf `frequencyPenalty` gemappt, unterschiedliche Min/Max-Werte
    - `presence_penalty`: wird auf `presencePenalty` gemappt, abweichende Min-/Max-Werte
    - `model`: geteilt, standardmäßig enthalten.
    - `stream`: geteilt, standardmäßig enthalten.
    - `max_tokens`: gemeinsam genutzt, auf `maxTokens` gemappt, standardmäßig nicht enthalten.


```yaml filename="librechat.yaml"
    - name: "cohere"
      apiKey: "${COHERE_API_KEY}"
      baseURL: "https://api.cohere.ai/v1"
      models:
        default: ["command-r","command-r-plus","command-light","command-light-nightly","command","command-nightly"]
        fetch: false
      modelDisplayLabel: "cohere"
      titleModel: "command"
      dropParams: ["stop", "user", "frequency_penalty", "presence_penalty", "temperature", "top_p"]
```
