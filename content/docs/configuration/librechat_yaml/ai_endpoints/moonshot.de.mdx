Moonshot AI stellt seine Kimi-Modelle über eine OpenAI-kompatible API bereit, die Sie in LibreChat als benutzerdefinierten endpoint hinzufügen können.

## Einen API-Schlüssel erhalten [#get-an-api-key]

Erstellen Sie einen Schlüssel auf der [Moonshot platform](https://platform.moonshot.ai). Fügen Sie ihn Ihrer `.env` Datei hinzu:

```bash filename=".env"
MOONSHOT_API_KEY=your-api-key
```

## Konfiguration [#configuration]

Fügen Sie den endpoint unter `endpoints.custom` in Ihrer `librechat.yaml` hinzu:

```yaml filename="librechat.yaml"
    - name: "Moonshot"
      apiKey: "${MOONSHOT_API_KEY}"
      baseURL: "https://api.moonshot.ai/v1"
      models:
        default: ["kimi-k2.5"]
        fetch: true
      titleConvo: true
      titleModel: "current_model"
      modelDisplayLabel: "Moonshot"
```

## Hinweise [#notes]

- Für Modelle mit Reasoning-Fähigkeiten wie `kimi-k2.5` und `kimi-k2-thinking` muss der `name` des endpoint auf `"Moonshot"` (Groß-/Kleinschreibung wird ignoriert) gesetzt werden, damit verschachteltes Reasoning mit Tool-Aufrufen funktioniert. Ein anderer Name führt zu Fehlern wie `thinking is enabled but reasoning_content is missing in assistant tool call message`. Weitere Details finden Sie in der [Dokumentation von Moonshot](https://platform.moonshot.ai/docs/guide/use-kimi-k2-thinking-model#frequently-asked-questions).
