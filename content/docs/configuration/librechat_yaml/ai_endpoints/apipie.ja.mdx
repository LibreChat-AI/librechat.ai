APIpieは、多くのプロバイダーのモデルを単一のOpenAI互換endpointを通じて公開するアグリゲーターです。

## APIキーを取得する [#get-an-api-key]

[apipie.ai/dashboard/profile/api-keys](https://apipie.ai/dashboard/profile/api-keys) からキーを作成し、`.env` ファイルに追加します：

```bash filename=".env"
APIPIE_API_KEY=your-api-key
```

## 設定 [#configuration]

`librechat.yaml` の `endpoints.custom` の下にエンドポイントを追加します：

```yaml filename="librechat.yaml"
    - name: "APIpie"
      apiKey: "${APIPIE_API_KEY}"
      baseURL: "https://apipie.ai/v1/"
      models:
        default: [
          "gpt-4",
          "gpt-4-turbo",
          "gpt-3.5-turbo",
          "claude-3-opus",
          "claude-3-sonnet",
          "claude-3-haiku",
          "llama-3-70b-instruct",
          "llama-3-8b-instruct",
          "gemini-pro-1.5",
          "gemini-pro",
          "mistral-large",
          "mistral-medium",
          "mistral-small",
          "mistral-tiny",
          "mixtral-8x22b",
          ]
        fetch: false
      titleConvo: true
      titleModel: "claude-3-haiku"
      summarize: false
      summaryModel: "claude-3-haiku"
      dropParams: ["stream"]
      modelDisplayLabel: "APIpie"
```

<Callout type="tip" title="モデルを取得して並べ替える" collapsible>
このPythonスクリプトを使用すると、LLMモデルを取得して順序付けすることができます。出力は `models.txt` に保存され、yaml設定へ簡単に組み込める形式でフォーマットされます。

```py filename="fetch.py"
import json
import requests

def fetch_and_order_models():
    # API endpoint
    url = "https://apipie.ai/models"

    # headers as per request example
    headers = {"Accept": "application/json"}

    # request parameters
    params = {"type": "llm"}

    # make request
    response = requests.get(url, headers=headers, params=params)

    # parse JSON response
    data = response.json()

    # extract an ordered list of unique model IDs
    model_ids = sorted(set([model["id"] for model in data]))

    # write result to a text file
    with open("models.txt", "w") as file:
        json.dump(model_ids, file, indent=2)

# execute the function
if __name__ == "__main__":
    fetch_and_order_models()
```
</Callout>

## 注記 [#notes]

- モデルの自動取得はサポートされていないため、`fetch` は `false` に設定され、モデルリストはインラインで定義されています。新しいモデルが追加された際は、上記のスクリプトを使用してリストを再生成してください。
- 会話タイトルの生成結果が不安定になることがあります。
- `dropParams: ["stream"]` は、現在サポートされていない `stream` パラメータを削除します。
