Databricks héberge des modèles de fondation ainsi que vos propres modèles affinés via Mosaic AI Model Serving, qui expose un endpoint de service compatible avec OpenAI.

## Obtenir une clé API [#get-an-api-key]

[Inscrivez-vous à Databricks](https://www.databricks.com/try-databricks#account) et générez un jeton d'accès personnel depuis votre espace de travail. Ajoutez-le à votre fichier `.env` :

```bash filename=".env"
DATABRICKS_API_KEY=your-api-key
```

## Configuration [#configuration]

Ajoutez l'endpoint sous `endpoints.custom` dans votre `librechat.yaml` :

```yaml filename="librechat.yaml"
    - name: 'Databricks'
      apiKey: '${DATABRICKS_API_KEY}'
      baseURL: 'https://your_databricks_serving_endpoint_url_here_ending_with/invocations'
      models:
        default: [
          "databricks-meta-llama-3-70b-instruct",
        ]
        fetch: false
      titleConvo: true
      titleModel: 'current_model'
      directEndpoint: true # required
      titleMessageRole: 'user' # required
```

## Notes [#notes]

- Databricks expose un endpoint de complétion complet se terminant par `invocations` pour les `serving-endpoints`, donc [directEndpoint](/docs/configuration/librechat_yaml/object_structure/custom_endpoint#directendpoint) est requis.
- Définissez [titleMessageRole](/docs/configuration/librechat_yaml/object_structure/custom_endpoint#titlemessagerole) sur `user` pour la génération de titres. Un message `system` autonome n'est pas pris en charge.
