## Einführung [#intro]

- Dieser Abschnitt listet bekannte, kompatible AI Endpoints auf, die auch als „Custom Endpoints“ bezeichnet werden, zusammen mit Beispielkonfigurationen für die `librechat.yaml` Datei, auch bekannt als [Custom Config](/docs/configuration/librechat_yaml) Datei.

- In allen Beispielen werden beliebige Umgebungsvariablennamen definiert, aber Sie können jeden gewünschten Namen verwenden sowie den Wert in `user_provided` ändern, um Benutzern zu ermöglichen, ihren eigenen API-Schlüssel über die Web-UI einzugeben.

<Callout type="warning" title="Wichtig: Einstellung des 'user_provided'-Schlüssels">
Wenn API-Schlüssel auf "user_provided" gesetzt werden, ermöglicht dies Benutzern, ihre eigenen API-Schlüssel über die Weboberfläche einzugeben. Dies unterscheidet sich von den vorkonfigurierten Endpoints in der .env Datei, bei denen Sie `ENDPOINT_KEY=user_provided` setzen würden (z. B. `OPENAI_API_KEY=user_provided`).

Für benutzerdefinierte endpoints in librechat.yaml verwenden Sie:
```yaml
endpoints:
  custom:
    - name: "Your Endpoint"
      apiKey: "user_provided"  # No need for ${} syntax here
```

Für Umgebungsvariablen in der .env Datei verwenden Sie:
```bash
OPENAI_API_KEY=user_provided
```
</Callout>

- Einige der endpoints sind als **Known** markiert, was bedeutet, dass sie möglicherweise eine spezielle Handhabung und/oder ein bereits in der App für dich bereitgestelltes Icon haben.

### Hinweise [#notes]

- Es wird empfohlen, dass Sie dem [Custom Endpoints Quick Start Guide](/docs/quick_start/custom_endpoints) folgen, bevor Sie mit den unten stehenden Beispielen fortfahren.
- Wichtig: Stellen Sie sicher, dass Sie die `librechat.yaml` Datei korrekt einrichten: **[Einrichtungsdokumentation](/docs/configuration/librechat_yaml)**.
