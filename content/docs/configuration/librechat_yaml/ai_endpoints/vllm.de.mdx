> vLLM ist eine leistungsstarke und speichereffiziente Inferenz- und Serving-Engine für LLMs.

**Hinweise:**

- **Nicht bekannt:** Icon nicht bereitgestellt, aber das Abrufen der Modellliste wird empfohlen, um die verfügbaren Modelle von Ihrem lokalen vLLM-Server zu erhalten.

- Die `titleMessageRole` ist wichtig, da einige lokale LLMs keine System-Nachrichtenrollen für Titel-Nachrichten (was der Standard ist) akzeptieren.

- Diese Konfiguration setzt voraus, dass Sie einen vLLM-Server lokal unter der angegebenen baseURL ausführen.

```yaml
    - name: "vLLM"
      apiKey: "vllm"
      baseURL: "http://127.0.0.1:8023/v1"
      models:
        default: ['google/gemma-3-27b-it']
        fetch: true
      titleConvo: true
      titleModel: "current_model"
      titleMessageRole: "user"
      summarize: false
      summaryModel: "current_model"
```

Die obige Konfiguration verbindet LibreChat mit einem lokalen vLLM-Server, der auf Port 8023 läuft. Sie verwendet das Gemma 3 27B-Modell als Standardmodell, ruft jedoch alle verfügbaren Modelle von Ihrem vLLM-Server ab.

## Wichtige Konfigurationsoptionen [#key-configuration-options]

- `apiKey`: Ein einfacher Platzhalterwert für vLLM (lokale Bereitstellungen erfordern normalerweise keine Authentifizierung)
- `baseURL`: Die URL, unter der Ihr vLLM-Server läuft
- `titleMessageRole`: Auf "user" anstatt des Standardwerts "system" setzen, da einige lokale LLMs keine Systemnachrichten unterstützen
