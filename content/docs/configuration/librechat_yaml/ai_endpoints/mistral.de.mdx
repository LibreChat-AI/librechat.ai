Mistral bietet eine eigene Familie von Chat- und Embedding-Modellen an, die in LibreChat als benutzerdefinierter endpoint verwendet werden.

## Einen API-Schlüssel erhalten [#get-an-api-key]

Erstellen Sie einen Schlüssel über die [Mistral console](https://console.mistral.ai/). Fügen Sie ihn Ihrer `.env` Datei hinzu:

```bash filename=".env"
MISTRAL_API_KEY=your-api-key
```

## Konfiguration [#configuration]

Fügen Sie den endpoint unter `endpoints.custom` in Ihrer `librechat.yaml` hinzu:

```yaml filename="librechat.yaml"
    - name: "Mistral"
      apiKey: "${MISTRAL_API_KEY}"
      baseURL: "https://api.mistral.ai/v1"
      models:
        default: ["mistral-tiny", "mistral-small", "mistral-medium", "mistral-large-latest"]
        fetch: true
      titleConvo: true
      titleModel: "mistral-tiny"
      modelDisplayLabel: "Mistral"
      dropParams: ["stop", "user", "frequency_penalty", "presence_penalty"]
```

## Hinweise [#notes]

- Die Mistral API erlaubt ein System-Message nur am Anfang des messages-Payloads.
- Die API ist streng bei nicht erkannten Parametern und ihre Fehlermeldungen sind nicht aussagekräftig (oft nur "no body"). Die Verwendung von [`dropParams`](/docs/configuration/librechat_yaml/object_structure/custom_endpoint#dropparams), um `user`, `frequency_penalty` und `presence_penalty` zu entfernen, ist erforderlich.
- `stop` ist kein Standardparameter mehr und muss daher nicht in [`dropParams`](/docs/configuration/librechat_yaml/object_structure/custom_endpoint#dropparams) enthalten sein, es sei denn, Sie möchten verhindern, dass Benutzer ihn konfigurieren.
- Das Abrufen der Modellliste wird unterstützt, aber achten Sie darauf, keine Embedding-Modelle für den Chat auszuwählen.
