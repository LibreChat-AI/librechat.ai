## Intro [#intro]

- Deze sectie bevat een lijst met bekende, compatibele AI Endpoints, ook wel "Custom Endpoints" genoemd, met voorbeeldconfiguraties voor het `librechat.yaml` bestand, ook wel bekend als het [Custom Config](/docs/configuration/librechat_yaml) bestand.

- In alle voorbeelden worden willekeurige omgevingsvariabelenamen gedefinieerd, maar je kunt elke gewenste naam gebruiken, evenals de waarde wijzigen naar `user_provided` om gebruikers in staat te stellen hun eigen API-sleutel in te dienen via de web-UI.

<Callout type="warning" title="Belangrijk: 'user_provided' sleutelinstelling">
Wanneer API-sleutels worden ingesteld op "user_provided", stelt dit gebruikers in staat om hun eigen API-sleutels in te voeren via de webinterface. Dit is anders dan de vooraf geconfigureerde endpoints in het .env bestand, waar je `ENDPOINT_KEY=user_provided` zou instellen (bijv. `OPENAI_API_KEY=user_provided`).

Voor aangepaste endpoints in librechat.yaml gebruik je:
```yaml
endpoints:
  custom:
    - name: "Your Endpoint"
      apiKey: "user_provided"  # No need for ${} syntax here
```

Voor omgevingsvariabelen in het .env bestand, gebruik je:
```bash
OPENAI_API_KEY=user_provided
```
</Callout>

- Sommige endpoints zijn gemarkeerd als **Known,** wat betekent dat ze mogelijk speciale afhandeling en/of een pictogram hebben dat al voor je in de app is voorzien.

### Opmerkingen [#notes]

- Het wordt aanbevolen om de [Custom Endpoints Quick Start Guide](/docs/quick_start/custom_endpoints) te volgen voordat je verdergaat met de onderstaande voorbeelden.
- Belangrijk: zorg ervoor dat je het `librechat.yaml` bestand correct instelt: **[installatiedocumentatie](/docs/configuration/librechat_yaml)**.
