## Giriş [#intro]

- Bu bölüm, [Custom Config](/docs/configuration/librechat_yaml) dosyası olarak da bilinen `librechat.yaml` dosyası için örnek kurulumlarla birlikte, bilinen ve uyumlu AI endpoint'lerini (diğer adıyla "Custom Endpoints") listeler.

- Tüm örneklerde, isteğe bağlı ortam değişkeni adları tanımlanmıştır ancak dilediğiniz ismi kullanabilir ve kullanıcıların kendi API anahtarlarını web arayüzünden göndermelerine olanak tanımak için değeri `user_provided` olarak değiştirebilirsiniz.

<Callout type="warning" title="Önemli: 'user_provided' Anahtar Ayarı">
API anahtarlarını "user_provided" olarak ayarlamak, kullanıcıların kendi API anahtarlarını web arayüzü üzerinden girmelerine olanak tanır. Bu, `ENDPOINT_KEY=user_provided` (örneğin, `OPENAI_API_KEY=user_provided`) şeklinde ayarlayacağınız .env dosyasındaki önceden yapılandırılmış uç noktalardan farklıdır.

librechat.yaml içindeki özel endpoint'ler için şunları kullanırsınız:
```yaml
endpoints:
  custom:
    - name: "Your Endpoint"
      apiKey: "user_provided"  # No need for ${} syntax here
```

.env dosyasındaki ortam değişkenleri için şunları kullanırsınız:
```bash
OPENAI_API_KEY=user_provided
```
</Callout>

- Bazı endpoint'ler **Known** (Bilinen) olarak işaretlenmiştir; bu, söz konusu endpoint'lerin özel bir işleme sahip olabileceği ve/veya uygulama içinde halihazırda sağlanan bir simgesinin bulunduğu anlamına gelir.

### Notlar [#notes]

- Aşağıdaki örneklere geçmeden önce [Özel Uç Noktalar Hızlı Başlangıç Kılavuzu](/docs/quick_start/custom_endpoints) bölümünü takip etmeniz önerilir.
- Önemli: `librechat.yaml` dosyasını doğru şekilde yapılandırdığınızdan emin olun: **[kurulum belgeleri](/docs/configuration/librechat_yaml)**.
