> Helicone API key: [https://us.helicone.ai/settings/api-keys](https://us.helicone.ai/settings/api-keys)

**注意：**

- **已知：** 已提供图标，建议获取模型列表，因为 Helicone 提供对来自多个提供商的 100 多种 AI 模型的访问权限。

- Helicone 是一个 AI 网关提供商，支持访问来自 OpenAI、Anthropic、Google、Meta、Mistral 及其他提供商的模型，并内置了可观测性和监控功能。

- **主要功能：**
  - 🚀 通过单一网关访问 100 多种 AI 模型
  - 📊 内置可观测性与监控
  - 🔄 多提供商支持
  - ⚡ 在 Helicone 仪表板中进行请求日志记录和分析

- **重要注意事项：**
  - 请确保您的 Helicone 账户中有足够的额度，以便访问这些模型。
  - 你可以在 [Helicone Model Library](https://helicone.ai/models) 中找到所有受支持的模型。
  - 您可以在 Helicone 仪表板中设置自己的速率限制和缓存策略。

```yaml
    - name: "Helicone"
      # For `apiKey` and `baseURL`, you can use environment variables that you define.
      # recommended environment variables:
      apiKey: "${HELICONE_KEY}"
      baseURL: "https://ai-gateway.helicone.ai"
      headers:
        x-librechat-body-parentmessageid: "{{LIBRECHAT_BODY_PARENTMESSAGEID}}"
      models:
        default: ["gpt-4o-mini", "claude-4.5-sonnet", "llama-3.1-8b-instruct", "gemini-2.5-flash-lite"]
        fetch: true
      titleConvo: true
      titleModel: "gpt-4o-mini"
      modelDisplayLabel: "Helicone"
      iconURL: "https://marketing-assets-helicone.s3.us-west-2.amazonaws.com/helicone.png"
```

**配置详情：**

- **apiKey:** 使用 `HELICONE_KEY` 环境变量来存储您的 Helicone API key。
- **baseURL:** Helicone AI Gateway 的 endpoint：`https://ai-gateway.helicone.ai`
- **headers:** `x-librechat-body-parentmessageid` 请求头对于消息追踪和对话连续性至关重要
- **models:** 设置默认模型，但通过启用 `fetch`，您将自动从 Helicone 的 API 获取所有可用模型。
- **fetch:** 设置为 `true` 以自动从 Helicone 的 API 获取可用模型

**设置步骤：**
1. 在 [helicone.ai](https://helicone.ai/) 注册一个 Helicone 账户
2. 从 [Helicone dashboard](https://us.helicone.ai/settings/api-keys) 生成您的 API key
3. 在您的 `.env` 文件中设置 `HELICONE_KEY` 环境变量
4. 将示例配置复制到您的 `librechat.yaml` 文件中
5. 如果使用 Docker 部署，请重新构建您的 Docker 容器
6. 重启 LibreChat 以加载新配置
7. 通过在提供商下拉菜单中选择 Helicone 进行测试
8. 前往 [Helicone dashboard](https://us.helicone.ai/dashboard) 查看您的使用情况和设置。

**潜在问题：**

- **模型访问 (Model Access)：** 请验证您在 Helicone 中拥有足够的额度，以便访问这些模型。
- **速率限制：** 您可以在 Helicone 仪表板内设置自己的速率限制和缓存策略。
- **环境变量：** 请仔细检查 `HELICONE_KEY` 是否已正确设置并可供您的 LibreChat 实例访问。

**测试：**
1. 配置完成后，从提供商下拉菜单中选择 Helicone
2. 验证模型是否出现在模型选择中
3. 发送一条测试消息并确认它出现在您的 Helicone 仪表板中
4. 检查对话线程是否通过父消息 ID 标头正常工作
