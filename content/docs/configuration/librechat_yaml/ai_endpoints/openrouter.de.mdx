[OpenRouter](https://openrouter.ai/) bietet Zugriff auf Hunderte von KI-Modellen über eine einzige API. Diese Anleitung führt Sie Schritt für Schritt durch die Einrichtung von OpenRouter als benutzerdefinierten endpoint in LibreChat.

<Callout type="info" title="Voraussetzungen">

Bevor Sie beginnen, stellen Sie sicher, dass Sie Folgendes haben:
- LibreChat installiert und ausgeführt (siehe [Docker setup](/docs/local/docker))
- Eine erstellte und eingebundene `librechat.yaml`-Datei (siehe [librechat.yaml guide](/docs/configuration/librechat_yaml))

</Callout>

## Einrichtung [#setup]

<Steps>
  <Step>

### Einen API-Schlüssel erhalten [#get-an-api-key]

Erstellen Sie ein Konto unter [openrouter.ai](https://openrouter.ai/) und generieren Sie einen API-Schlüssel auf der [Keys-Seite](https://openrouter.ai/keys).

Kopieren Sie den Schlüssel -- er beginnt mit `sk-or-v1-`.

  </Step>
  <Step>

### Fügen Sie den Schlüssel zu Ihrer .env Datei hinzu [#add-the-key-to-your-env-file]

Öffnen Sie Ihre `.env` Datei im Projektstammverzeichnis und fügen Sie Ihren OpenRouter API-Schlüssel hinzu:

```bash filename=".env"
OPENROUTER_KEY=sk-or-v1-your-key-here
```

<Callout type="error" title="Verwenden Sie OPENROUTER_KEY, nicht OPENROUTER_API_KEY">

Sie müssen `OPENROUTER_KEY` als Variablennamen verwenden. Die Verwendung von `OPENROUTER_API_KEY` würde auch den integrierten OpenAI endpoint überschreiben, um OpenRouter zu nutzen, was mit ziemlicher Sicherheit nicht das ist, was Sie beabsichtigen.

</Callout>

  </Step>
  <Step>

### Den Endpoint zu librechat.yaml hinzufügen [#add-the-endpoint-to-librechatyaml]

Fügen Sie Folgendes zu Ihrer `librechat.yaml` Datei hinzu. Falls die Datei bereits Inhalte enthält, führen Sie den `endpoints` Abschnitt mit Ihrer bestehenden Konfiguration zusammen:

```yaml filename="librechat.yaml"
version: 1.3.5
cache: true
endpoints:
  custom:
    - name: "OpenRouter"
      apiKey: "${OPENROUTER_KEY}"
      baseURL: "https://openrouter.ai/api/v1"
      models:
        default: ["meta-llama/llama-3-70b-instruct"]
        fetch: true
      titleConvo: true
      titleModel: "meta-llama/llama-3-70b-instruct"
      dropParams: ["stop"]
      modelDisplayLabel: "OpenRouter"
```

Erklärung der wichtigsten Felder:

| Feld | Zweck |
|-------|---------|
| `apiKey: "${OPENROUTER_KEY}"` | Referenziert die Umgebungsvariable aus Schritt 2. Die `${}`-Syntax weist LibreChat an, den Wert aus `.env` zu lesen. |
| `models.fetch: true` | Ruft die vollständige Modellliste von der API von OpenRouter ab, sodass neue Modelle automatisch erscheinen. |
| `dropParams: ["stop"]` | Entfernt den `stop`-Parameter aus Anfragen. OpenRouter-Modelle verwenden unterschiedliche Stop-Token; das Weglassen verhindert Kompatibilitätsprobleme. |
| `modelDisplayLabel: "OpenRouter"` | Der Name, der im endpoint-Auswahlmenü von LibreChat angezeigt wird. |

</Step>
  <Step>

### LibreChat neu starten [#restart-librechat]

<Tabs items={['Docker', 'Lokal']}>
  <Tabs.Tab>

```bash
docker compose down && docker compose up -d
```

  </Tabs.Tab>
  <Tabs.Tab>

Stoppen Sie den laufenden Prozess (Strg+C) und starten Sie ihn neu:

```bash
npm run backend
```

  </Tabs.Tab>
</Tabs>

  </Step>
  <Step>

### Überprüfen, ob es funktioniert [#verify-it-works]

Öffnen Sie LibreChat in Ihrem Browser. Sie sollten **OpenRouter** im Dropdown-Menü für die endpoint-Auswahl sehen. Wählen Sie es aus, um die verfügbaren Modelle zu sehen.

Wenn OpenRouter nicht erscheint, überprüfen Sie die Server-Logs auf Konfigurationsfehler:

```bash
docker compose logs api | grep -i "error\|openrouter"
```

  </Step>
</Steps>

## Fehlerbehebung [#troubleshooting]

### OpenRouter erscheint nicht in der Modellauswahl [#openrouter-does-not-appear-in-the-model-selector]

Überprüfen Sie diese Punkte in der angegebenen Reihenfolge:

1. `OPENROUTER_KEY` wird in `.env` festgelegt.
2. `librechat.yaml` enthält den OpenRouter-Eintrag unter `endpoints.custom`.
3. Docker-Benutzer haben `librechat.yaml` in `docker-compose.override.yml` gemountet.
4. LibreChat wurde nach dem Bearbeiten beider Dateien neu gestartet.
5. Die API-Logs zeigen keine YAML-Validierungsfehler an.

```bash
docker compose logs api
```

Wenn die Protokolle eine fehlende Umgebungsvariable erwähnen, überprüfen Sie, ob der Variablenname in `.env` mit der in `librechat.yaml` referenzierten Variable übereinstimmt. Für diese Anleitung bedeutet das `apiKey: "${OPENROUTER_KEY}"` in `librechat.yaml` und `OPENROUTER_KEY=...` in `.env`.

### OpenRouter gibt 402 Payment Required zurück [#openrouter-returns-402-payment-required]

Eine `402 Payment Required`-Antwort stammt von OpenRouter, nicht von LibreChat. Fügen Sie Guthaben hinzu oder wählen Sie ein kostenloses/verfügbares Modell in OpenRouter aus und versuchen Sie es dann erneut mit demselben endpoint in LibreChat.

## Anpassung [#customization]

### Verwendung eines user_provided API Key [#using-user_provided-api-key]

Anstatt den Schlüssel in `.env` zu speichern, können Sie jedem Benutzer erlauben, seinen eigenen Schlüssel über die LibreChat UI bereitzustellen:

```yaml
apiKey: "user_provided"
```

Benutzer sehen ein Schlüsseleingabefeld, wenn sie den OpenRouter endpoint auswählen.

### Begrenzung der verfügbaren Modelle [#limiting-available-models]

Anstatt alle Modelle abzurufen, können Sie eine feste Liste angeben:

```yaml
models:
  default: ["anthropic/claude-3.5-sonnet", "openai/gpt-4o", "meta-llama/llama-3-70b-instruct"]
  fetch: false
```
