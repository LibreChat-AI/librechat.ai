## Giới thiệu [#intro]

- Phần này liệt kê các AI Endpoints tương thích đã biết, còn được gọi là "Custom Endpoints", cùng với các thiết lập ví dụ cho tệp `librechat.yaml`, hay còn được gọi là tệp [Custom Config](/docs/configuration/librechat_yaml).

- Trong tất cả các ví dụ, các tên biến môi trường tùy ý được xác định nhưng bạn có thể sử dụng bất kỳ tên nào bạn muốn, cũng như thay đổi giá trị thành `user_provided` để cho phép người dùng gửi khóa API của riêng họ từ giao diện web.

<Callout type="warning" title="Quan trọng: Cài đặt khóa 'user_provided'">
Khi đặt các khóa API thành "user_provided", điều này cho phép người dùng nhập khóa API của riêng họ thông qua giao diện web. Điều này khác với các endpoint được cấu hình sẵn trong tệp .env, nơi bạn sẽ đặt `ENDPOINT_KEY=user_provided` (ví dụ: `OPENAI_API_KEY=user_provided`).

Đối với các endpoint tùy chỉnh trong librechat.yaml, bạn sẽ sử dụng:
```yaml
endpoints:
  custom:
    - name: "Your Endpoint"
      apiKey: "user_provided"  # No need for ${} syntax here
```

Đối với các biến môi trường trong tệp .env, bạn sẽ sử dụng:
```bash
OPENAI_API_KEY=user_provided
```
</Callout>

- Một số endpoint được đánh dấu là **Known,** nghĩa là chúng có thể đã có cách xử lý đặc biệt và/hoặc biểu tượng được cung cấp sẵn trong ứng dụng cho bạn.
