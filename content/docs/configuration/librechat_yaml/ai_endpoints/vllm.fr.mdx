> vLLM est un moteur d'inférence et de service à haut débit et économe en mémoire pour les LLM.

**Notes :**

- **Non connu :** icône non fournie, mais il est recommandé de récupérer la liste des modèles pour obtenir les modèles disponibles depuis votre serveur vLLM local.

- Le `titleMessageRole` est important car certains LLM locaux n'acceptent pas les rôles de message système pour les messages de titre (ce qui est la valeur par défaut).

- Cette configuration suppose que vous avez un serveur vLLM en cours d'exécution localement à la baseURL spécifiée.

```yaml
    - name: "vLLM"
      apiKey: "vllm"
      baseURL: "http://127.0.0.1:8023/v1"
      models:
        default: ['google/gemma-3-27b-it']
        fetch: true
      titleConvo: true
      titleModel: "current_model"
      titleMessageRole: "user"
      summarize: false
      summaryModel: "current_model"
```

La configuration ci-dessus connecte LibreChat à un serveur vLLM local fonctionnant sur le port 8023. Elle utilise le modèle Gemma 3 27B comme modèle par défaut, mais récupérera tous les modèles disponibles depuis votre serveur vLLM.

## Options de configuration clés [#key-configuration-options]

- `apiKey` : Une valeur d'espace réservé simple pour vLLM (les déploiements locaux ne nécessitent généralement pas d'authentification)
- `baseURL` : L'URL où votre serveur vLLM est en cours d'exécution
- `titleMessageRole`: Défini sur "user" au lieu de la valeur par défaut "system", car certains LLM locaux ne prennent pas en charge les messages système
