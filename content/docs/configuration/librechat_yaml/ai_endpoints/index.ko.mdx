## 소개 [#intro]

- 이 섹션에서는 [Custom Config](/docs/configuration/librechat_yaml) 파일이라고도 하는 `librechat.yaml` 파일에 대한 예시 설정과 함께 "Custom Endpoints"라고도 알려진 호환 가능한 AI Endpoints 목록을 제공합니다.

- 모든 예제에서 임의의 환경 변수 이름이 정의되어 있지만, 원하는 이름을 사용할 수 있으며 값을 `user_provided`로 변경하여 사용자가 웹 UI에서 직접 자신의 API 키를 제출하도록 설정할 수도 있습니다.

<Callout type="warning" title="중요: 'user_provided' 키 설정">
API 키를 "user_provided"로 설정하면 사용자가 웹 인터페이스를 통해 자신의 API 키를 직접 입력할 수 있습니다. 이는 .env 파일에서 `ENDPOINT_KEY=user_provided`(예: `OPENAI_API_KEY=user_provided`)와 같이 설정하는 사전 구성된 endpoint와는 다릅니다.

librechat.yaml 내의 사용자 지정 endpoint의 경우, 다음을 사용합니다:
```yaml
endpoints:
  custom:
    - name: "Your Endpoint"
      apiKey: "user_provided"  # No need for ${} syntax here
```

.env 파일의 환경 변수에 대해서는 다음을 사용합니다:
```bash
OPENAI_API_KEY=user_provided
```
</Callout>

- 일부 endpoint는 **Known**으로 표시되어 있는데, 이는 해당 endpoint가 앱 내에서 이미 제공되는 특별한 처리 방식이나 아이콘을 가지고 있을 수 있음을 의미합니다.
