## Wprowadzenie [#intro]

- Ta sekcja zawiera listę znanych, kompatybilnych endpointów AI, znanych również jako "Custom Endpoints", wraz z przykładowymi konfiguracjami dla pliku `librechat.yaml`, znanego również jako plik [Custom Config](/docs/configuration/librechat_yaml).

- We wszystkich przykładach zdefiniowano dowolne nazwy zmiennych środowiskowych, ale możesz użyć dowolnej wybranej nazwy, a także zmienić wartość na `user_provided`, aby umożliwić użytkownikom przesyłanie własnego klucza API z poziomu interfejsu internetowego.

<Callout type="warning" title="Ważne: Ustawienie klucza 'user_provided'">
Ustawienie kluczy API na "user_provided" pozwala użytkownikom na wprowadzanie własnych kluczy API za pośrednictwem interfejsu internetowego. Różni się to od wstępnie skonfigurowanych endpointów w pliku .env, gdzie ustawia się `ENDPOINT_KEY=user_provided` (np. `OPENAI_API_KEY=user_provided`).

W przypadku niestandardowych endpoint w pliku librechat.yaml, należy użyć:
```yaml
endpoints:
  custom:
    - name: "Your Endpoint"
      apiKey: "user_provided"  # No need for ${} syntax here
```

Dla zmiennych środowiskowych w pliku .env, należy użyć:
```bash
OPENAI_API_KEY=user_provided
```
</Callout>

- Niektóre z endpoint są oznaczone jako **Known**, co oznacza, że mogą posiadać specjalną obsługę i/lub ikonę już dostarczoną dla Ciebie w aplikacji.

### Uwagi [#notes]

- Zaleca się zapoznanie z [Przewodnikiem szybkiego startu dla niestandardowych endpointów](/docs/quick_start/custom_endpoints) przed przejściem do poniższych przykładów.
- Ważne: upewnij się, że poprawnie skonfigurowałeś plik `librechat.yaml`: **[dokumentacja konfiguracji](/docs/configuration/librechat_yaml)**.
