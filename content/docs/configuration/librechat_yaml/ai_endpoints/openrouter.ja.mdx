[OpenRouter](https://openrouter.ai/) は、単一のAPIを通じて数百ものAIモデルへのアクセスを提供します。このガイドでは、LibreChatでOpenRouterをカスタムendpointとしてゼロから設定する方法を説明します。

<Callout type="info" title="前提条件">

始める前に、以下が準備されていることを確認してください：
- LibreChat がインストールされ、実行されていること（[Docker setup](/docs/local/docker) を参照）
- 作成およびマウントされた `librechat.yaml` ファイル（[librechat.yaml ガイド](/docs/configuration/librechat_yaml)を参照）

</Callout>

## セットアップ [#setup]

<Steps>
  <Step>

### APIキーを取得する [#get-an-api-key]

[openrouter.ai](https://openrouter.ai/) でアカウントを作成し、[Keys ページ](https://openrouter.ai/keys) から API キーを生成してください。

キーをコピーしてください -- `sk-or-v1-` で始まります。

  </Step>
  <Step>

### .env ファイルにキーを追加する [#add-the-key-to-your-env-file]

プロジェクトルートにある `.env` ファイルを開き、OpenRouter APIキーを追加してください：

```bash filename=".env"
OPENROUTER_KEY=sk-or-v1-your-key-here
```

<Callout type="error" title="OPENROUTER_API_KEY ではなく OPENROUTER_KEY を使用してください">

変数名には必ず `OPENROUTER_KEY` を使用してください。`OPENROUTER_API_KEY` を使用すると、組み込みの OpenAI endpoint も OpenRouter を使用するように上書きされてしまいますが、これはほぼ間違いなく意図した動作ではないはずです。

</Callout>

  </Step>
  <Step>

### librechat.yaml に endpoint を追加する [#add-the-endpoint-to-librechatyaml]

`librechat.yaml` ファイルに以下を追加してください。ファイルに既に内容が含まれている場合は、`endpoints` セクションを既存の設定とマージしてください。

```yaml filename="librechat.yaml"
version: 1.3.5
cache: true
endpoints:
  custom:
    - name: "OpenRouter"
      apiKey: "${OPENROUTER_KEY}"
      baseURL: "https://openrouter.ai/api/v1"
      models:
        default: ["meta-llama/llama-3-70b-instruct"]
        fetch: true
      titleConvo: true
      titleModel: "meta-llama/llama-3-70b-instruct"
      dropParams: ["stop"]
      modelDisplayLabel: "OpenRouter"
```

主要なフィールドの解説:

| フィールド | 目的 |
|-------|---------|
| `apiKey: "${OPENROUTER_KEY}"` | ステップ2で設定した環境変数を参照します。`${}` 構文は、LibreChatに対して `.env` から値を取得するよう指示します。 |
| `models.fetch: true` | OpenRouterのAPIからモデルリスト全体を取得し、新しいモデルが自動的に表示されるようにします。 |
| `dropParams: ["stop"]` | リクエストから `stop` パラメータを削除します。OpenRouterのモデルは多様な停止トークンを使用するため、これを削除することで互換性の問題を回避します。 |
| `modelDisplayLabel: "OpenRouter"` | LibreChatのendpointセレクターに表示される名前です。 |

</Step>
  <Step>


  </Step>

### 動作確認 [#verify-it-works]

ブラウザで LibreChat を開きます。エンドポイントセレクターのドロップダウンに **OpenRouter** が表示されているはずです。それを選択して、利用可能なモデルを確認してください。

OpenRouterが表示されない場合は、サーバーログで設定エラーを確認してください：

```bash
docker compose logs api | grep -i "error\|openrouter"
```

## トラブルシューティング [#troubleshooting]

### OpenRouter がモデルセレクターに表示されない [#openrouter-does-not-appear-in-the-model-selector]

以下の項目を順番に確認してください：

1. `OPENROUTER_KEY` は `.env` に設定します。
2. `librechat.yaml` には、`endpoints.custom` の下に OpenRouter のエントリが含まれています。
3. Dockerユーザーは、`docker-compose.override.yml` 内で `librechat.yaml` をマウントしています。
4. 両方のファイルを編集した後、LibreChatを再起動しました。
5. APIログにはYAMLのバリデーションエラーが表示されません。

```bash
docker compose logs api
```

ログに環境変数の不足が示されている場合は、`.env` 内の変数名が `librechat.yaml` で参照されている変数名と一致しているか確認してください。このガイドの場合、`librechat.yaml` 内の `apiKey: "${OPENROUTER_KEY}"` と `.env` 内の `OPENROUTER_KEY=...` が一致している必要があります。

### OpenRouterで402 Payment Requiredが返される場合 [#openrouter-returns-402-payment-required]

`402 Payment Required` というレスポンスは LibreChat ではなく OpenRouter から返されるものです。OpenRouter でクレジットを追加するか、無料または利用可能なモデルを選択してから、LibreChat で同じ endpoint を再試行してください。

## カスタマイズ [#customization]

### user_provided API Keyの使用 [#using-user_provided-api-key]

`.env` にキーを保存する代わりに、各ユーザーが LibreChat UI を通じて独自のキーを提供できるようにすることができます：

```yaml
apiKey: "user_provided"
```

OpenRouter endpointを選択すると、ユーザーにはキー入力フィールドが表示されます。

### 利用可能なモデルの制限 [#limiting-available-models]

すべてのモデルを取得する代わりに、固定のリストを指定することもできます：

```yaml
models:
  default: ["anthropic/claude-3.5-sonnet", "openai/gpt-4o", "meta-llama/llama-3-70b-instruct"]
  fetch: false
```
