## Pengantar [#intro]

- Bagian ini mencantumkan AI Endpoint yang kompatibel dan diketahui, yang juga dikenal sebagai "Custom Endpoints," beserta contoh pengaturan untuk file `librechat.yaml`, yang juga dikenal sebagai file [Custom Config](/docs/configuration/librechat_yaml).

- Dalam semua contoh tersebut, nama variabel lingkungan arbitrer didefinisikan, tetapi Anda dapat menggunakan nama apa pun yang Anda inginkan, serta mengubah nilainya menjadi `user_provided` untuk memungkinkan pengguna mengirimkan kunci API mereka sendiri dari UI web.

<Callout type="warning" title="Penting: Pengaturan Kunci 'user_provided'">
Saat mengatur kunci API ke "user_provided", ini memungkinkan pengguna untuk memasukkan kunci API mereka sendiri melalui antarmuka web. Hal ini berbeda dengan endpoint yang telah dikonfigurasi sebelumnya di file .env di mana Anda akan mengatur `ENDPOINT_KEY=user_provided` (contohnya, `OPENAI_API_KEY=user_provided`).

Untuk endpoint kustom di librechat.yaml, Anda akan menggunakan:
```yaml
endpoints:
  custom:
    - name: "Your Endpoint"
      apiKey: "user_provided"  # No need for ${} syntax here
```

Untuk variabel lingkungan di dalam file .env, Anda akan menggunakan:
```bash
OPENAI_API_KEY=user_provided
```
</Callout>

- Beberapa endpoint ditandai sebagai **Known,** yang berarti endpoint tersebut mungkin memiliki penanganan khusus dan/atau ikon yang sudah disediakan di dalam aplikasi untuk Anda.

### Catatan [#notes]

- Disarankan agar Anda mengikuti [Panduan Memulai Cepat Custom Endpoints](/docs/quick_start/custom_endpoints) sebelum melanjutkan dengan contoh-contoh di bawah ini.
- Penting: pastikan Anda mengatur file `librechat.yaml` dengan benar: **[dokumentasi pengaturan](/docs/configuration/librechat_yaml)**.
