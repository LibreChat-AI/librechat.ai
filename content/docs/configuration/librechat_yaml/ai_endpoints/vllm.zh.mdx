> vLLM 是一个针对 LLM 的高吞吐量且内存高效的推理与服务引擎。

**注意：**

- **未知：** 未提供图标，但建议获取模型列表以从您的本地 vLLM 服务器获取可用模型。

- `titleMessageRole` 非常重要，因为一些本地 LLM 不会接受系统消息角色作为标题消息（这是默认设置）。

- 此配置假设您在指定的 baseURL 处本地运行了 vLLM 服务器。

```yaml
    - name: "vLLM"
      apiKey: "vllm"
      baseURL: "http://127.0.0.1:8023/v1"
      models:
        default: ['google/gemma-3-27b-it']
        fetch: true
      titleConvo: true
      titleModel: "current_model"
      titleMessageRole: "user"
      summarize: false
      summaryModel: "current_model"
```

上述配置将 LibreChat 连接到运行在 8023 端口的本地 vLLM 服务器。它使用 Gemma 3 27B 模型作为默认模型，但会从您的 vLLM 服务器获取所有可用模型。

## 关键配置选项 [#key-configuration-options]

- `apiKey`: vLLM 的简单占位符值（本地部署通常不需要身份验证）
- `baseURL`: vLLM 服务器运行的 URL
- `titleMessageRole`: 设置为 "user" 而不是默认的 "system"，因为某些本地 LLM 不支持系统消息
