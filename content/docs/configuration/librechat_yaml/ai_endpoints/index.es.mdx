## Introducción [#intro]

- Esta sección enumera los AI Endpoints conocidos y compatibles, también conocidos como "Custom Endpoints", con configuraciones de ejemplo para el archivo `librechat.yaml`, también conocido como el archivo [Custom Config](/docs/configuration/librechat_yaml).

- En todos los ejemplos, se definen nombres de variables de entorno arbitrarios, pero puede usar cualquier nombre que desee, así como cambiar el valor a `user_provided` para permitir que los usuarios envíen su propia clave de API desde la interfaz web.

<Callout type="warning" title="Importante: Configuración de la clave 'user_provided'">
Al configurar las claves de API como "user_provided", esto permite a los usuarios ingresar sus propias claves de API a través de la interfaz web. Esto es diferente de los endpoints preconfigurados en el archivo .env donde se establecería `ENDPOINT_KEY=user_provided` (por ejemplo, `OPENAI_API_KEY=user_provided`).

Para endpoints personalizados en librechat.yaml, utilizarías:
```yaml
endpoints:
  custom:
    - name: "Your Endpoint"
      apiKey: "user_provided"  # No need for ${} syntax here
```

Para las variables de entorno en el archivo .env, deberías usar:
```bash
OPENAI_API_KEY=user_provided
```
</Callout>

- Algunos de los endpoint están marcados como **Known,** lo que significa que podrían tener un manejo especial y/o un icono ya proporcionado en la aplicación para ti.

### Notas [#notes]

- Se recomienda seguir la [Guía de inicio rápido de endpoints personalizados](/docs/quick_start/custom_endpoints) antes de continuar con los ejemplos a continuación.
- Importante: asegúrese de configurar el archivo `librechat.yaml` correctamente: **[documentación de configuración](/docs/configuration/librechat_yaml)**.
