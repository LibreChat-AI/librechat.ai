> Helicone API-Schlüssel: [https://us.helicone.ai/settings/api-keys](https://us.helicone.ai/settings/api-keys)

**Hinweise:**

- **Bekannt:** Icon bereitgestellt, das Abrufen der Modellliste wird empfohlen, da Helicone Zugriff auf über 100 KI-Modelle von verschiedenen Anbietern bietet.

- Helicone ist ein AI Gateway-Anbieter, der den Zugriff auf Modelle von OpenAI, Anthropic, Google, Meta, Mistral und anderen Anbietern mit integrierter Observability und Monitoring ermöglicht.

- **Hauptfunktionen:**
  - 🚀 Zugriff auf über 100 KI-Modelle über ein einziges Gateway
  - 📊 Integrierte Observability und Monitoring
  - 🔄 Unterstützung für mehrere Anbieter
  - ⚡ Anforderungsprotokollierung und Analysen im Helicone-Dashboard

- **Wichtige Überlegungen:**
  - Stellen Sie sicher, dass Ihr Helicone-Konto über Guthaben verfügt, damit Sie auf die Modelle zugreifen können.
  - Sie finden alle unterstützten Modelle in der [Helicone Model Library](https://helicone.ai/models).
  - Sie können Ihre eigenen Ratenbegrenzungen und Caching-Richtlinien innerhalb des Helicone-Dashboards festlegen.

```yaml
    - name: "Helicone"
      # For `apiKey` and `baseURL`, you can use environment variables that you define.
      # recommended environment variables:
      apiKey: "${HELICONE_KEY}"
      baseURL: "https://ai-gateway.helicone.ai"
      headers:
        x-librechat-body-parentmessageid: "{{LIBRECHAT_BODY_PARENTMESSAGEID}}"
      models:
        default: ["gpt-4o-mini", "claude-4.5-sonnet", "llama-3.1-8b-instruct", "gemini-2.5-flash-lite"]
        fetch: true
      titleConvo: true
      titleModel: "gpt-4o-mini"
      modelDisplayLabel: "Helicone"
      iconURL: "https://marketing-assets-helicone.s3.us-west-2.amazonaws.com/helicone.png"
```

**Konfigurationsdetails:**

- **apiKey:** Verwenden Sie die Umgebungsvariable `HELICONE_KEY`, um Ihren Helicone API-Schlüssel zu speichern.
- **baseURL:** Der Helicone AI Gateway endpoint: `https://ai-gateway.helicone.ai`
- **headers:** Der `x-librechat-body-parentmessageid` Header ist für die Nachrichtenverfolgung und die Kontinuität von Unterhaltungen unerlässlich.
- **models:** Legt Standardmodelle fest. Wenn Sie jedoch `fetch` aktivieren, werden automatisch alle verfügbaren Modelle von der API von Helicone abgerufen.
- **fetch:** Auf `true` setzen, um verfügbare Modelle automatisch von der API von Helicone abzurufen

**Einrichtungsschritte:**
1. Registrieren Sie sich für ein Helicone-Konto unter [helicone.ai](https://helicone.ai/)
2. Generieren Sie Ihren API-Schlüssel über das [Helicone dashboard](https://us.helicone.ai/settings/api-keys)
3. Setzen Sie die Umgebungsvariable `HELICONE_KEY` in Ihrer `.env` Datei
4. Kopieren Sie die Beispielkonfiguration in Ihre `librechat.yaml` Datei
5. Erstellen Sie Ihre Docker-Container neu, falls Sie eine Docker-Bereitstellung verwenden
6. Starten Sie LibreChat neu, um die neue Konfiguration zu laden
7. Testen Sie dies, indem Sie Helicone aus dem Anbieter-Dropdown-Menü auswählen
8. Besuchen Sie das [Helicone dashboard](https://us.helicone.ai/dashboard), um Ihre Nutzung und Einstellungen zu überprüfen.

**Mögliche Probleme:**

- **Modellzugriff:** Stellen Sie sicher, dass Sie über Guthaben bei Helicone verfügen, damit Sie auf die Modelle zugreifen können.
- **Rate Limiting:** Sie können Ihre eigenen Ratenbegrenzungen und Caching-Richtlinien innerhalb des Helicone-Dashboards festlegen.
- **Umgebungsvariablen:** Überprüfen Sie noch einmal, ob `HELICONE_KEY` korrekt gesetzt und für Ihre LibreChat-Instanz zugänglich ist.

**Testen:**
1. Wählen Sie nach der Konfiguration Helicone aus dem Anbieter-Dropdown-Menü aus.
2. Überprüfen Sie, ob die Modelle in der Modellauswahl erscheinen
3. Senden Sie eine Testnachricht und bestätigen Sie, dass sie in Ihrem Helicone-Dashboard erscheint
4. Überprüfen Sie, ob das Conversation Threading mit dem Header der übergeordneten Nachrichten-ID (parent message ID) korrekt funktioniert.
