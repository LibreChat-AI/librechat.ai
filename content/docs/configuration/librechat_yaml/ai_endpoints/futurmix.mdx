[FuturMix](https://futurmix.ai/) is a unified AI gateway providing access to 22+ leading AI models through a single OpenAI-compatible API. This guide walks you through setting up FuturMix as a custom endpoint in LibreChat from scratch.

<Callout type="info" title="Prerequisites">

Before starting, make sure you have:
- LibreChat installed and running (see [Docker setup](/docs/local/docker))
- A `librechat.yaml` file created and mounted (see [librechat.yaml guide](/docs/configuration/librechat_yaml))

</Callout>

## Setup

<Steps>
  <Step>

### Get an API Key

Create an account at [futurmix.ai](https://futurmix.ai/) and generate an API key from your dashboard.

Copy the key -- it starts with `sk-`.

  </Step>
  <Step>

### Add the Key to Your .env File

Open your `.env` file in the project root and add your FuturMix API key:

```bash filename=".env"
FUTURMIX_API_KEY=sk-your-key-here
```

  </Step>
  <Step>

### Add the Endpoint to librechat.yaml

Add the following to your `librechat.yaml` file. If the file already has content, merge the `endpoints` section with your existing configuration:

```yaml filename="librechat.yaml"
version: 1.3.5
cache: true
endpoints:
  custom:
    - name: "FuturMix"
      apiKey: "${FUTURMIX_API_KEY}"
      baseURL: "https://futurmix.ai/v1"
      models:
        default:
          [
            "claude-sonnet-4-20250514",
            "gpt-4o",
            "gemini-2.5-flash",
            "deepseek-chat",
          ]
        fetch: true
      titleConvo: true
      titleModel: "gpt-4o-mini"
      modelDisplayLabel: "FuturMix"
```

Key fields explained:

| Field                         | Purpose                                                                                                                                      |
| ----------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| `apiKey: "${FUTURMIX_API_KEY}"` | References the env var from Step 2. The `${}` syntax tells LibreChat to read the value from `.env`.                                         |
| `baseURL`                     | FuturMix's OpenAI-compatible API endpoint.                                                                                                   |
| `models.fetch: true`          | Fetches the full model list from FuturMix's API, so new models appear automatically.                                                        |
| `models.default`              | Initial models shown in the selector. You can customize this list with your preferred models.                                               |
| `titleModel`                  | The model used to generate conversation titles. Using `gpt-4o-mini` provides a good balance of quality and cost for title generation.       |
| `modelDisplayLabel`           | The name shown in LibreChat's endpoint selector.                                                                                             |

</Step>
  <Step>

### Restart LibreChat

<Tabs items={['Docker', 'Local']}>
  <Tabs.Tab>

```bash
docker compose down && docker compose up -d
```

  </Tabs.Tab>
  <Tabs.Tab>

Stop the running process (Ctrl+C) and restart:

```bash
npm run backend
```

  </Tabs.Tab>
</Tabs>

  </Step>
  <Step>

### Verify It Works

Open LibreChat in your browser. You should see **FuturMix** in the endpoint selector dropdown. Select it to see the available models.

If FuturMix does not appear, check the server logs for configuration errors:

```bash
docker compose logs api | grep -i "error\|futurmix"
```

  </Step>
</Steps>

## Customization

### Using user_provided API Key

Instead of storing the key in `.env`, you can let each user provide their own key through the LibreChat UI:

```yaml
apiKey: "user_provided"
```

Users will see a key input field when selecting the FuturMix endpoint.

### Limiting Available Models

Instead of fetching all models, you can specify a fixed list:

```yaml
models:
  default:
    [
      "claude-sonnet-4-20250514",
      "claude-opus-4-20250514",
      "gpt-4o",
      "gpt-4o-mini",
      "gemini-2.5-flash",
      "gemini-2.5-pro",
      "deepseek-chat",
    ]
  fetch: false
```

### Recommended Models by Use Case

FuturMix provides access to a wide range of models optimized for different scenarios:

**General Purpose**
- `claude-sonnet-4-20250514` - Best balance of capability and speed
- `gpt-4o` - Strong reasoning and multimodal capabilities
- `gemini-2.5-flash` - Fast and cost-effective

**Coding & Technical Tasks**
- `claude-sonnet-4-20250514` - Excellent code understanding and generation
- `deepseek-chat` - Specialized in code-related tasks

**Creative Writing**
- `claude-opus-4-20250514` - Superior creative capabilities
- `gpt-4o` - Versatile for various content types

**Cost-Effective Options**
- `gpt-4o-mini` - Budget-friendly for simple tasks
- `gemini-2.5-flash` - Fast inference at lower cost
