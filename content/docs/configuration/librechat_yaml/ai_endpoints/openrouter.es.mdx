[OpenRouter](https://openrouter.ai/) proporciona acceso a cientos de modelos de IA a través de una única API. Esta guía le explica cómo configurar OpenRouter como un endpoint personalizado en LibreChat desde cero.

<Callout type="info" title="Requisitos previos">

Antes de comenzar, asegúrese de tener:
- LibreChat instalado y en ejecución (consulta la [configuración de Docker](/docs/local/docker))
- Un archivo `librechat.yaml` creado y montado (ver [guía de librechat.yaml](/docs/configuration/librechat_yaml))

</Callout>

## Configuración [#setup]

<Steps>
  <Step>

### Obtener una API Key [#get-an-api-key]

Cree una cuenta en [openrouter.ai](https://openrouter.ai/) y genere una clave API desde la [página de Keys](https://openrouter.ai/keys).

Copia la clave: comienza con `sk-or-v1-`.

  </Step>
  <Step>

### Añade la clave a tu archivo .env [#add-the-key-to-your-env-file]

Abra su archivo `.env` en la raíz del proyecto y añada su clave de API de OpenRouter:

```bash filename=".env"
OPENROUTER_KEY=sk-or-v1-your-key-here
```

<Callout type="error" title="Use OPENROUTER_KEY, no OPENROUTER_API_KEY">

Debe usar `OPENROUTER_KEY` como nombre de la variable. Usar `OPENROUTER_API_KEY` anulará el endpoint de OpenAI integrado para usar también OpenRouter, lo cual casi con seguridad no es lo que desea.

</Callout>

  </Step>
  <Step>

### Agregar el endpoint a librechat.yaml [#add-the-endpoint-to-librechatyaml]

Agregue lo siguiente a su archivo `librechat.yaml`. Si el archivo ya tiene contenido, combine la sección `endpoints` con su configuración existente:

```yaml filename="librechat.yaml"
version: 1.3.5
cache: true
endpoints:
  custom:
    - name: "OpenRouter"
      apiKey: "${OPENROUTER_KEY}"
      baseURL: "https://openrouter.ai/api/v1"
      models:
        default: ["meta-llama/llama-3-70b-instruct"]
        fetch: true
      titleConvo: true
      titleModel: "meta-llama/llama-3-70b-instruct"
      dropParams: ["stop"]
      modelDisplayLabel: "OpenRouter"
```

Explicación de los campos clave:

| Campo | Propósito |
|-------|---------|
| `apiKey: "${OPENROUTER_KEY}"` | Hace referencia a la variable de entorno del Paso 2. La sintaxis `${}` le indica a LibreChat que lea el valor desde `.env`. |
| `models.fetch: true` | Obtiene la lista completa de modelos desde la API de OpenRouter, por lo que los nuevos modelos aparecen automáticamente. |
| `dropParams: ["stop"]` | Elimina el parámetro `stop` de las solicitudes. Los modelos de OpenRouter utilizan diversos tokens de parada, por lo que omitir esto evita problemas de compatibilidad. |
| `modelDisplayLabel: "OpenRouter"` | El nombre que se muestra en el selector de endpoint de LibreChat. |

</Step>
  <Step>

### Reiniciar LibreChat [#restart-librechat]

<Tabs items={['Docker', 'Local']}>
  <Tabs.Tab>

```bash
docker compose down && docker compose up -d
```

  </Tabs.Tab>
  <Tabs.Tab>

Detenga el proceso en ejecución (Ctrl+C) y reinicie:

```bash
npm run backend
```

  </Tabs.Tab>
</Tabs>

  </Step>
  <Step>

### Verificar que funciona [#verify-it-works]

Abra LibreChat en su navegador. Debería ver **OpenRouter** en el menú desplegable del selector de endpoint. Selecciónelo para ver los modelos disponibles.

Si OpenRouter no aparece, revisa los registros del servidor en busca de errores de configuración:

```bash
docker compose logs api | grep -i "error\|openrouter"
```

  </Step>
</Steps>

## Solución de problemas [#troubleshooting]

### OpenRouter no aparece en el selector de modelos [#openrouter-does-not-appear-in-the-model-selector]

Comprueba estos elementos en orden:

1. `OPENROUTER_KEY` se configura en `.env`.
2. `librechat.yaml` contiene la entrada de OpenRouter bajo `endpoints.custom`.
3. Los usuarios de Docker han montado `librechat.yaml` en `docker-compose.override.yml`.
4. LibreChat se reinició después de editar ambos archivos.
5. Los registros de la API no muestran errores de validación de YAML.

```bash
docker compose logs api
```

Si los registros mencionan una variable de entorno faltante, verifique que el nombre de la variable en `.env` coincida con la variable referenciada en `librechat.yaml`. Para esta guía, eso significa `apiKey: "${OPENROUTER_KEY}"` en `librechat.yaml` y `OPENROUTER_KEY=...` en `.env`.

### OpenRouter devuelve 402 Payment Required [#openrouter-returns-402-payment-required]

Una respuesta `402 Payment Required` proviene de OpenRouter, no de LibreChat. Añade créditos o elige un modelo gratuito/disponible en OpenRouter, luego vuelve a intentar el mismo endpoint en LibreChat.

## Personalización [#customization]

### Uso de una API Key proporcionada por el usuario (user_provided) [#using-user_provided-api-key]

En lugar de almacenar la clave en `.env`, puedes permitir que cada usuario proporcione su propia clave a través de la interfaz de usuario de LibreChat:

```yaml
apiKey: "user_provided"
```

Los usuarios verán un campo de entrada de clave al seleccionar el endpoint de OpenRouter.

### Limitación de modelos disponibles [#limiting-available-models]

En lugar de obtener todos los modelos, puedes especificar una lista fija:

```yaml
models:
  default: ["anthropic/claude-3.5-sonnet", "openai/gpt-4o", "meta-llama/llama-3-70b-instruct"]
  fetch: false
```
