Heliconeは、OpenAI、Anthropic、Google、Meta、Mistralなどのプロバイダーのモデルへのリクエストを、単一のOpenAI互換endpoint経由でルーティングするAIゲートウェイであり、リクエストのログ記録と使用状況の分析機能が組み込まれています。

## APIキーを取得する [#get-an-api-key]

[helicone.ai](https://helicone.ai/) でアカウントを作成し、[Settings → API Keys](https://us.helicone.ai/settings/api-keys) からキーを生成します。それを `.env` ファイルに追加してください：

```bash filename=".env"
HELICONE_KEY=your-api-key
```

アカウントにクレジットがあることを確認してください。そうでない場合、モデルのリクエストは失敗します。

## 設定 [#configuration]

`librechat.yaml` の `endpoints.custom` の下にエンドポイントを追加します：

```yaml filename="librechat.yaml"
    - name: "Helicone"
      apiKey: "${HELICONE_KEY}"
      baseURL: "https://ai-gateway.helicone.ai"
      headers:
        x-librechat-body-parentmessageid: "{{LIBRECHAT_BODY_PARENTMESSAGEID}}"
      models:
        default: ["gpt-4o-mini", "claude-4.5-sonnet", "llama-3.1-8b-instruct", "gemini-2.5-flash-lite"]
        fetch: true
      titleConvo: true
      titleModel: "gpt-4o-mini"
      modelDisplayLabel: "Helicone"
      iconURL: "https://marketing-assets-helicone.s3.us-west-2.amazonaws.com/helicone.png"
```

## 注記 [#notes]

- `fetch: true` を設定すると、LibreChat は Helicone からモデルの全リストを読み込むため、`default` 配列は初期選択のみとなります。[Helicone モデルライブラリ](https://helicone.ai/models) で利用可能なすべてのモデルを閲覧できます。
- `x-librechat-body-parentmessageid` ヘッダーは、各リクエストの親メッセージIDをHeliconeに渡すことで、ログを会話ごとにグループ化できるようにします。
- [Helicone ダッシュボード](https://us.helicone.ai/dashboard)から、レート制限、キャッシュポリシーの設定、および使用状況の確認を行います。
