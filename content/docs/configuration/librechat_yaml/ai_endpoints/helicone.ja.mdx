> Helicone API key: [https://us.helicone.ai/settings/api-keys](https://us.helicone.ai/settings/api-keys)

**注記:**

- **既知の事項:** アイコンが提供されています。Heliconeは複数のプロバイダーから100種類以上のAIモデルへのアクセスを提供しているため、モデルリストの取得を推奨します。

- Heliconeは、OpenAI、Anthropic、Google、Meta、Mistralなどのプロバイダーが提供するモデルへのアクセスを可能にし、オブザーバビリティ（可観測性）とモニタリング機能を備えたAIゲートウェイプロバイダーです。

- **主な機能:**
  - 🚀 単一のゲートウェイから100種類以上のAIモデルにアクセス
  - 📊 組み込みの可観測性とモニタリング
  - 🔄 マルチプロバイダーサポート
  - ⚡ Helicone ダッシュボードでのリクエストログ記録と分析

- **重要な考慮事項:**
  - モデルにアクセスできるよう、Heliconeアカウントにクレジットがあることを確認してください。
  - サポートされているすべてのモデルは、[Helicone Model Library](https://helicone.ai/models)で確認できます。
  - Heliconeダッシュボード内で、独自のレート制限とキャッシュポリシーを設定できます。

```yaml
    - name: "Helicone"
      # For `apiKey` and `baseURL`, you can use environment variables that you define.
      # recommended environment variables:
      apiKey: "${HELICONE_KEY}"
      baseURL: "https://ai-gateway.helicone.ai"
      headers:
        x-librechat-body-parentmessageid: "{{LIBRECHAT_BODY_PARENTMESSAGEID}}"
      models:
        default: ["gpt-4o-mini", "claude-4.5-sonnet", "llama-3.1-8b-instruct", "gemini-2.5-flash-lite"]
        fetch: true
      titleConvo: true
      titleModel: "gpt-4o-mini"
      modelDisplayLabel: "Helicone"
      iconURL: "https://marketing-assets-helicone.s3.us-west-2.amazonaws.com/helicone.png"
```

**設定の詳細:**

- **apiKey:** `HELICONE_KEY` 環境変数を使用して、Helicone APIキーを保存します。
- **baseURL:** Helicone AI Gatewayのendpoint: `https://ai-gateway.helicone.ai`
- **headers:** `x-librechat-body-parentmessageid` ヘッダーは、メッセージの追跡と会話の継続性に不可欠です。
- **models:** デフォルトのモデルを設定しますが、`fetch` を有効にすることで、HeliconeのAPIから利用可能なすべてのモデルを自動的に取得します。
- **fetch:** `true` に設定すると、HeliconeのAPIから利用可能なモデルを自動的に取得します。

**セットアップ手順:**
1. [helicone.ai](https://helicone.ai/) で Helicone アカウントにサインアップしてください。
2. [Helicone dashboard](https://us.helicone.ai/settings/api-keys) からAPIキーを生成してください。
3. `.env` ファイルに `HELICONE_KEY` 環境変数を設定します。
4. `librechat.yaml` ファイルに設定例をコピーしてください。
5. Dockerデプロイメントを使用している場合は、Dockerコンテナを再ビルドしてください
6. LibreChatを再起動して新しい設定を読み込みます
7. プロバイダのドロップダウンからHeliconeを選択してテストします
8. [Helicone dashboard](https://us.helicone.ai/dashboard) にアクセスして、使用状況と設定を確認してください。

**潜在的な問題:**

- **Model Access:** Helicone内でモデルにアクセスするためのクレジットがあることを確認してください。
- **レート制限:** Heliconeダッシュボード内で、独自のレート制限やキャッシュポリシーを設定できます。
- **環境変数:** `HELICONE_KEY` が適切に設定されており、LibreChat インスタンスからアクセス可能であることを再確認してください。

**テスト:**
1. 設定後、プロバイダーのドロップダウンからHeliconeを選択してください。
2. モデルがモデル選択に表示されることを確認してください
3. テストメッセージを送信し、Heliconeダッシュボードに表示されることを確認してください。
4. 親メッセージIDヘッダーを使用して、会話のスレッド化が正しく機能することを確認してください。
