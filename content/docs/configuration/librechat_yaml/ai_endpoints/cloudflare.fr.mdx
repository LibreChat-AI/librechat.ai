Cloudflare Workers AI exécute des modèles ouverts sur le réseau de Cloudflare, et son AI Gateway expose un endpoint compatible avec OpenAI limité à votre compte et à votre passerelle.

## Obtenir une clé API [#get-an-api-key]

Créez une AI Gateway en suivant le [guide de configuration d'AI Gateway](https://developers.cloudflare.com/ai-gateway/get-started/), puis ajoutez vos identifiants à votre fichier `.env` :

```bash filename=".env"
CF_API_TOKEN=your-api-key
CF_ACCOUNT_ID=your-account-id
CF_GATEWAY_ID=your-gateway-id
```

## Configuration [#configuration]

Ajoutez l'endpoint sous `endpoints.custom` dans votre `librechat.yaml`. Le `baseURL` intègre votre ID de compte et votre ID de passerelle :

```yaml filename="librechat.yaml"
    - name: "Cloudflare Workers AI"
      apiKey: "${CF_API_TOKEN}"
      baseURL: "https://gateway.ai.cloudflare.com/v1/${CF_ACCOUNT_ID}/${CF_GATEWAY_ID}/workers-ai/v1"
      models:
        default: [
          "@cf/google/gemma-3-12b-it",
          "@cf/meta/llama-4-scout-17b-16e-instruct",
          "@cf/qwen/qwq-32b",
          "@cf/qwen/qwen2.5-coder-32b-instruct",
          "@cf/deepseek-ai/deepseek-r1-distill-qwen-32b",
          "@cf/openai/gpt-oss-120b"
        ]
        fetch: false
      titleConvo: true
      titleModel: "@cf/google/gemma-3-12b-it"
      modelDisplayLabel: "Cloudflare AI"
```

## Notes [#notes]

- Ceci utilise l' [OpenAI-compatible endpoint](https://developers.cloudflare.com/ai-gateway/providers/workersai/#openai-compatible-endpoints) de Workers AI. Parcourez les modèles disponibles dans le [Workers AI model catalog](https://developers.cloudflare.com/workers-ai/models/).
- Les modèles `@cf/openai/gpt-oss-*` pourraient ne pas fonctionner en raison d'une incompatibilité avec les Model Specs.
- Les noms de variables `CF_API_TOKEN`, `CF_ACCOUNT_ID` et `CF_GATEWAY_ID` peuvent être renommés tant qu'ils correspondent entre votre `.env` et votre `librechat.yaml`.
