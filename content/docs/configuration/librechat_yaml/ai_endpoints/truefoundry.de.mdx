TrueFoundry AI Gateway ist die Proxy-Ebene, die zwischen Ihren Anwendungen und den LLM-Anbietern sowie MCP Servers fungiert. Es handelt sich um eine Plattform auf Unternehmensniveau, die es Benutzern ermöglicht, über eine einheitliche Schnittstelle auf über 1000 LLMs zuzugreifen, während gleichzeitig Observability und Governance gewährleistet werden.

## Konfigurationsdetails [#configuration-details]

Um das [AI Gateway von TrueFoundry](https://www.truefoundry.com/ai-gateway) zu nutzen, folgen Sie der Schnellstartanleitung [hier](https://docs.truefoundry.com/gateway/quick-start).

### Abrufen der Base URL und der Modellnamen [#getting-base-url-and-model-names]

- **Basis-URL und Modellnamen**: Rufen Sie Ihre TrueFoundry AI Gateway endpoint URL und den Modellnamen aus dem einheitlichen Code-Snippet ab (stellen Sie sicher, dass Sie denselben Modellnamen hinzugefügt haben)
- **PAT**: Generieren Sie diesen aus Ihrem TrueFoundry Personal Access Token [PAT](https://docs.truefoundry.com/gateway/authentication#personal-access-token-pat)

## Verwendung des TrueFoundry AI Gateway mit einem beliebigen LLM-Modell [#using-truefoundry-ai-gateway-with-any-llm-model]
```yaml filename="librechat.yaml"
      - name: "TrueFoundry"
        apiKey: "${TRUEFOUNDRY_API_KEY}"
        baseURL: "${TRUEFOUNDRY_GATEWAY_URL}"
        models:
            default: ["openai-main/gpt-4o-mini", "openai-main/gpt-4o"]
            fetch: true
        titleConvo: true
        titleModel: "current_model"
        summarize: false
        summaryModel: "current_model"
        modelDisplayLabel: "TrueFoundry:OpenAI"
```
Für weitere Details können Sie hier nachsehen: [TrueFoundry Docs](https://docs.truefoundry.com/docs/introduction)
