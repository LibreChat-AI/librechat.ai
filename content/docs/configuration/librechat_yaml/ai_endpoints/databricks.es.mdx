Databricks sirve modelos base y sus propios modelos ajustados a través de Mosaic AI Model Serving, el cual expone un endpoint de servicio compatible con OpenAI.

## Obtener una API key [#get-an-api-key]

[Regístrese en Databricks](https://www.databricks.com/try-databricks#account) y genere un token de acceso personal desde su espacio de trabajo. Añádalo a su archivo `.env`:

```bash filename=".env"
DATABRICKS_API_KEY=your-api-key
```

## Configuración [#configuration]

Agregue el endpoint bajo `endpoints.custom` en su `librechat.yaml`:

```yaml filename="librechat.yaml"
    - name: 'Databricks'
      apiKey: '${DATABRICKS_API_KEY}'
      baseURL: 'https://your_databricks_serving_endpoint_url_here_ending_with/invocations'
      models:
        default: [
          "databricks-meta-llama-3-70b-instruct",
        ]
        fetch: false
      titleConvo: true
      titleModel: 'current_model'
      directEndpoint: true # required
      titleMessageRole: 'user' # required
```

## Notas [#notes]

- Databricks expone un endpoint de completado completo que termina en `invocations` para `serving-endpoints`, por lo que [directEndpoint](/docs/configuration/librechat_yaml/object_structure/custom_endpoint#directendpoint) es necesario.
- Establezca [titleMessageRole](/docs/configuration/librechat_yaml/object_structure/custom_endpoint#titlemessagerole) en `user` para la generación de títulos. No se admite un mensaje `system` independiente.
