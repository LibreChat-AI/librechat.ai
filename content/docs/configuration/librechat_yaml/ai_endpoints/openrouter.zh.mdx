[OpenRouter](https://openrouter.ai/) 通过单一 API 提供对数百种 AI 模型的访问。本指南将引导您从零开始在 LibreChat 中将 OpenRouter 设置为自定义 endpoint。

<Callout type="info" title="先决条件">

在开始之前，请确保您已具备：
- LibreChat 已安装并运行（请参阅 [Docker setup](/docs/local/docker)）
- 已创建并挂载的 `librechat.yaml` 文件（请参阅 [librechat.yaml 指南](/docs/configuration/librechat_yaml)）

</Callout>

## 设置 [#setup]

<Steps>
  <Step>

### 获取 API Key [#get-an-api-key]

在 [openrouter.ai](https://openrouter.ai/) 创建一个账户，并从 [Keys page](https://openrouter.ai/keys) 生成一个 API key。

复制该密钥 -- 它以 `sk-or-v1-` 开头。

  </Step>
  <Step>

### 将密钥添加到您的 .env 文件 [#add-the-key-to-your-env-file]

在项目根目录下打开你的 `.env` 文件并添加你的 OpenRouter API key：

```bash filename=".env"
OPENROUTER_KEY=sk-or-v1-your-key-here
```

<Callout type="error" title="使用 OPENROUTER_KEY，而不是 OPENROUTER_API_KEY">

您必须使用 `OPENROUTER_KEY` 作为变量名。使用 `OPENROUTER_API_KEY` 也会覆盖内置的 OpenAI endpoint 并将其指向 OpenRouter，这几乎肯定不是您想要的结果。

</Callout>

  </Step>
  <Step>

### 将 Endpoint 添加到 librechat.yaml [#add-the-endpoint-to-librechatyaml]

将以下内容添加到您的 `librechat.yaml` 文件中。如果该文件已有内容，请将 `endpoints` 部分与您现有的配置合并：

```yaml filename="librechat.yaml"
version: 1.3.5
cache: true
endpoints:
  custom:
    - name: "OpenRouter"
      apiKey: "${OPENROUTER_KEY}"
      baseURL: "https://openrouter.ai/api/v1"
      models:
        default: ["meta-llama/llama-3-70b-instruct"]
        fetch: true
      titleConvo: true
      titleModel: "meta-llama/llama-3-70b-instruct"
      dropParams: ["stop"]
      modelDisplayLabel: "OpenRouter"
```

关键字段说明：

| 字段 | 用途 |
|-------|---------|
| `apiKey: "${OPENROUTER_KEY}"` | 引用第 2 步中的环境变量。`${}` 语法告知 LibreChat 从 `.env` 中读取该值。 |
| `models.fetch: true` | 从 OpenRouter 的 API 获取完整的模型列表，以便新模型自动显示。 |
| `dropParams: ["stop"]` | 从请求中移除 `stop` 参数。OpenRouter 模型使用不同的停止标记，因此移除此参数可避免兼容性问题。 |
| `modelDisplayLabel: "OpenRouter"` | 在 LibreChat 的 endpoint 选择器中显示的名称。 |

</Step>
  <Step>

### 重启 LibreChat [#restart-librechat]

<Tabs items={['Docker', '本地']}>
  <Tabs.Tab>

```bash
docker compose down && docker compose up -d
```

  </Tabs.Tab>
  <Tabs.Tab>

停止正在运行的进程 (Ctrl+C) 并重启：

```bash
npm run backend
```

  </Tabs.Tab>
</Tabs>

  </Step>
  <Step>

### 验证其是否正常工作 [#verify-it-works]

在浏览器中打开 LibreChat。你应该能在 endpoint 选择下拉菜单中看到 **OpenRouter**。选择它即可查看可用模型。

如果 OpenRouter 没有出现，请检查服务器日志以查看是否存在配置错误：

```bash
docker compose logs api | grep -i "error\|openrouter"
```

  </Step>
</Steps>

## 故障排除 [#troubleshooting]

### OpenRouter 未出现在模型选择器中 [#openrouter-does-not-appear-in-the-model-selector]

请按顺序检查以下项目：

1. `OPENROUTER_KEY` 在 `.env` 中设置。
2. `librechat.yaml` 在 `endpoints.custom` 下包含 OpenRouter 条目。
3. Docker 用户已在 `docker-compose.override.yml` 中挂载了 `librechat.yaml`。
4. 在编辑完这两个文件后，LibreChat 已重启。
5. API 日志中未显示 YAML 验证错误。

```bash
docker compose logs api
```

如果日志中提到缺少环境变量，请检查 `.env` 中的变量名是否与 `librechat.yaml` 中引用的变量匹配。对于本指南而言，这意味着 `librechat.yaml` 中的 `apiKey: "${OPENROUTER_KEY}"` 必须与 `.env` 中的 `OPENROUTER_KEY=...` 相对应。

### OpenRouter 返回 402 Payment Required [#openrouter-returns-402-payment-required]

`402 Payment Required` 响应来自 OpenRouter，而非 LibreChat。请在 OpenRouter 中充值或选择一个免费/可用的模型，然后在 LibreChat 中重试相同的 endpoint。

## 自定义 [#customization]

### 使用 user_provided API Key [#using-user_provided-api-key]

除了将密钥存储在 `.env` 中，您还可以让每个用户通过 LibreChat UI 提供他们自己的密钥：

```yaml
apiKey: "user_provided"
```

当用户选择 OpenRouter endpoint 时，将会看到一个密钥输入框。

### 限制可用模型 [#limiting-available-models]

与其获取所有模型，您可以指定一个固定列表：

```yaml
models:
  default: ["anthropic/claude-3.5-sonnet", "openai/gpt-4o", "meta-llama/llama-3-70b-instruct"]
  fetch: false
```
