> vLLM es un motor de inferencia y servicio de alto rendimiento y eficiente en memoria para LLMs.

**Notas:**

- **Desconocido:** no se proporcionó el icono, pero se recomienda obtener la lista de modelos para ver los modelos disponibles desde su servidor vLLM local.

- El `titleMessageRole` es importante ya que algunos LLM locales no aceptan roles de mensaje de sistema para los mensajes de título (que es el valor predeterminado).

- Esta configuración asume que tienes un servidor vLLM ejecutándose localmente en la baseURL especificada.

```yaml
    - name: "vLLM"
      apiKey: "vllm"
      baseURL: "http://127.0.0.1:8023/v1"
      models:
        default: ['google/gemma-3-27b-it']
        fetch: true
      titleConvo: true
      titleModel: "current_model"
      titleMessageRole: "user"
      summarize: false
      summaryModel: "current_model"
```

La configuración anterior conecta LibreChat a un servidor vLLM local ejecutándose en el puerto 8023. Utiliza el modelo Gemma 3 27B como modelo predeterminado, pero obtendrá todos los modelos disponibles desde su servidor vLLM.

## Opciones de configuración clave [#key-configuration-options]

- `apiKey`: Un valor de marcador de posición simple para vLLM (las implementaciones locales normalmente no requieren autenticación)
- `baseURL`: La URL donde se está ejecutando su servidor vLLM
- `titleMessageRole`: Establecido en "user" en lugar del valor predeterminado "system", ya que algunos LLM locales no admiten mensajes del sistema
