> Clave de API de Ollama: Requerida pero ignorada - [Ollama OpenAI Compatibility](https://github.com/ollama/ollama/blob/main/docs/openai.md)

**Notas:**

- **Conocido:** icono proporcionado.
- Descargue modelos con el comando ollama run. Consulte la [Ollama Library](https://ollama.com/library)
- Se recomienda utilizar el valor "current_model" para el `titleModel` a fin de evitar cargar más de 1 modelo por conversación.
    - Al hacer esto, se utilizará dinámicamente el modelo de conversación actual para la generación del título.
- El ejemplo incluye una lista de los 5 modelos más populares de la Ollama Library, la cual fue actualizada por última vez el 1 de marzo de 2024, para su conveniencia.

```yaml filename="librechat.yaml"
  custom:
    - name: "Ollama"
      apiKey: "ollama"
      # use 'host.docker.internal' instead of localhost if running LibreChat in a docker container
      baseURL: "http://localhost:11434/v1/" 
      models:
        default: [
          "llama2",
          "mistral",
          "codellama",
          "dolphin-mixtral",
          "mistral-openorca"
          ]
        # fetching list of models is supported but the `name` field must start
        # with `ollama` (case-insensitive), as it does in this example.
        fetch: true
      titleConvo: true
      titleModel: "current_model"
      summarize: false
      summaryModel: "current_model"
      modelDisplayLabel: "Ollama"
```

<Callout type="tip" title="Ollama -> llama3">
    
Nota: Una vez que se eliminó `stop` de los [parámetros predeterminados](/docs/configuration/librechat_yaml/object_structure/default_params), el problema resaltado a continuación ya no debería existir.

Sin embargo, en caso de que experimentes el comportamiento donde `llama3` no deja de generar, añade este bloque `addParams` a la configuración:

```yaml filename="librechat.yaml"
  custom:
    - name: "Ollama"
      apiKey: "ollama"
      baseURL: "http://host.docker.internal:11434/v1/"
      models:
        default: [
          "llama3"
        ]
        fetch: false # fetching list of models is not supported
      titleConvo: true
      titleModel: "current_model"
      summarize: false
      summaryModel: "current_model"
      modelDisplayLabel: "Ollama"
      addParams:
          "stop": [
              "<|start_header_id|>",
              "<|end_header_id|>",
              "<|eot_id|>",
              "<|reserved_special_token"
          ]
```

Si solo estás utilizando `llama3` con **Ollama**, está bien configurar el parámetro `stop` a nivel de configuración mediante `addParams`.

Sin embargo, si estás utilizando múltiples modelos, ahora se recomienda añadir secuencias de parada desde el frontend a través de los parámetros de conversación y los ajustes preestablecidos (presets).

Por ejemplo, podemos omitir `addParams`:

```yaml filename="librechat.yaml"
- name: "Ollama"
    apiKey: "ollama"
    baseURL: "http://host.docker.internal:11434/v1/" 
    models:
    default: [
        "llama3:latest",
        "mistral"
        ]
    fetch: false # fetching list of models is not supported
    titleConvo: true
    titleModel: "current_model"
    modelDisplayLabel: "Ollama"
```
