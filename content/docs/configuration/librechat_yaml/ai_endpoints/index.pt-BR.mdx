## Intro [#intro]

- Esta seção lista Endpoints de IA conhecidos e compatíveis, também conhecidos como "Custom Endpoints", com exemplos de configuração para o arquivo `librechat.yaml`, também conhecido como o arquivo [Custom Config](/docs/configuration/librechat_yaml).

- Em todos os exemplos, nomes arbitrários de variáveis de ambiente são definidos, mas você pode usar qualquer nome que desejar, bem como alterar o valor para `user_provided` para permitir que os usuários enviem sua própria chave de API a partir da interface web.

<Callout type="warning" title="Importante: Configuração da chave 'user_provided'">
Ao definir chaves de API como "user_provided", isso permite que os usuários insiram suas próprias chaves de API através da interface web. Isso é diferente dos endpoints pré-configurados no arquivo .env, onde você definiria `ENDPOINT_KEY=user_provided` (por exemplo, `OPENAI_API_KEY=user_provided`).

Para endpoints personalizados no librechat.yaml, você usaria:
```yaml
endpoints:
  custom:
    - name: "Your Endpoint"
      apiKey: "user_provided"  # No need for ${} syntax here
```

Para variáveis de ambiente no arquivo .env, você usaria:
```bash
OPENAI_API_KEY=user_provided
```
</Callout>

- Alguns dos endpoints estão marcados como **Known**, o que significa que eles podem ter um tratamento especial e/ou um ícone já fornecido no aplicativo para você.

### Notas [#notes]

- Recomendamos que você siga o [Guia de Início Rápido de Endpoints Personalizados](/docs/quick_start/custom_endpoints) antes de prosseguir com os exemplos abaixo.
- Importante: certifique-se de configurar o arquivo `librechat.yaml` corretamente: **[documentação de configuração](/docs/configuration/librechat_yaml)**.
