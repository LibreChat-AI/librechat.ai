## 简介 [#intro]

- 本节列出了已知的、兼容的 AI Endpoints（也称为“自定义端点”），并提供了 `librechat.yaml` 文件（也称为 [Custom Config](/docs/configuration/librechat_yaml) 文件）的配置示例。

- 在所有示例中，定义的都是任意环境变量名称，但你可以使用任何你想要的名称，也可以将值更改为 `user_provided`，以允许用户从 Web UI 提交他们自己的 API 密钥。

<Callout type="warning" title="重要提示：'user_provided' 键设置">
当将 API 密钥设置为 "user_provided" 时，这允许用户通过 Web 界面输入他们自己的 API 密钥。这与 .env 文件中预配置的 endpoint 不同，在 .env 文件中，你需要设置 `ENDPOINT_KEY=user_provided`（例如 `OPENAI_API_KEY=user_provided`）。

对于 librechat.yaml 中的自定义 endpoint，您可以使用：
```yaml
endpoints:
  custom:
    - name: "Your Endpoint"
      apiKey: "user_provided"  # No need for ${} syntax here
```

对于 .env 文件中的环境变量，您可以使用：
```bash
OPENAI_API_KEY=user_provided
```
</Callout>

- 部分 endpoint 被标记为 **Known**，这意味着它们可能具有特殊处理方式，或者应用程序中已经为您提供了相应的图标。

### 注意事项 [#notes]

- 建议您在继续阅读以下示例之前，先参考 [自定义 endpoint 快速入门指南](/docs/quick_start/custom_endpoints)。
- 重要提示：请确保正确设置 `librechat.yaml` 文件：**[设置文档](/docs/configuration/librechat_yaml)**。
