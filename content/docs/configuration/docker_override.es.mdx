En Docker Compose, un archivo de anulación (override file) es una característica potente que le permite modificar la configuración predeterminada proporcionada por el `docker-compose.yml` principal sin necesidad de editar directamente o duplicar todo el archivo. El uso principal del archivo de anulación es para personalizaciones de desarrollo local, y Docker Compose combina las configuraciones de los archivos `docker-compose.yml` y `docker-compose.override.yml` cuando ejecuta `docker compose up`.

Aquí tienes una guía rápida sobre cómo usar el `docker-compose.override.yml`:

> Nota: Por favor, consulte el `docker-compose.override.yml.example` para obtener más ejemplos

Consulta la documentación oficial de Docker para obtener más información:

- **[docker docs - understanding-multiple-compose-files](https://docs.docker.com/compose/multiple-compose-files/extends/#understanding-multiple-compose-files)**
- **[documentación de docker - merge-compose-files](https://docs.docker.com/compose/multiple-compose-files/merge/#merge-compose-files)**
- **[documentación de docker - especificar-múltiples-archivos-compose](https://docs.docker.com/compose/reference/#specifying-multiple-compose-files)**

## Paso 1: Crear un archivo `docker-compose.override.yml` [#step-1-create-a-docker-composeoverrideyml-file]

Si aún no tienes un archivo `docker-compose.override.yml`, puedes crear uno copiando el contenido del ejemplo de anulación:

```bash filename="Copying the example override file"
cp docker-compose.override.yml.example docker-compose.override.yml
```

Este archivo será detectado por Docker Compose automáticamente cuando ejecutes comandos de docker-compose.

## Paso 2: Editar el archivo de anulación [#step-2-edit-the-override-file]

Abra su archivo `docker-compose.override.yml` con vscode o cualquier editor de texto.

Realice los cambios deseados descomentando las secciones relevantes y personalizándolas según sea necesario.

> Advertencia: Solo puede especificar cada nombre de servicio una vez (api, mongodb, meilisearch, ...). Si desea anular varias configuraciones en un mismo servicio, deberá editarlas en consecuencia.

### Ejemplos [#examples]

Si quieres asegurarte de que Docker pueda usar tu archivo `librechat.yaml` para [Custom Endpoints & Configuration](/docs/configuration/librechat_yaml), se vería así:

```yaml filename="docker-compose.override.yml"
services:
  api:
    volumes:
      - ./librechat.yaml:/app/librechat.yaml
```

O, si desea compilar localmente la imagen para el servicio `api`, utilizar el archivo de configuración de LibreChat y usar la versión anterior de Mongo que no requiere soporte AVX, su `docker-compose.override.yml` podría verse así:

```yaml filename="docker-compose.override.yml"
services:
  api:
    volumes:
      - ./librechat.yaml:/app/librechat.yaml
    image: librechat
    build:
      context: .
      target: node

  mongodb:
    image: mongo:4.4.18
```

> Nota: Tenga cuidado si expone puertos para MongoDB o Meilisearch al público, ya que puede hacer que sus datos sean vulnerables.

## Paso 3: Aplicar los cambios [#step-3-apply-the-changes]

Para aplicar los cambios de configuración, simplemente ejecute Docker Compose como de costumbre. Docker Compose toma en cuenta automáticamente tanto el archivo `docker-compose.yml` como el `docker-compose.override.yml`:

```bash filename="Apply the changes"
docker compose up -d
```

## Paso 4: Verificar los cambios [#step-4-verify-the-changes]

Después de iniciar sus servicios con la configuración modificada, puede verificar que los cambios se han aplicado utilizando el comando `docker ps` para listar los contenedores en ejecución y sus propiedades, tales como los puertos.

## Consideraciones importantes [#important-considerations]

- **Orden de precedencia**: Los valores definidos en el archivo de anulación tienen prioridad sobre los especificados en el archivo `docker-compose.yml` original.
- **Seguridad**: Al personalizar puertos y exponer servicios públicamente, sea siempre consciente de las implicaciones de seguridad. Evite utilizar los valores predeterminados para entornos de producción o sensibles.

Al seguir estos pasos y consideraciones, puede modificar fácil y seguramente su configuración de Docker Compose sin alterar el archivo `docker-compose.yml` original, lo que facilita la gestión y el mantenimiento de diferentes entornos o personalizaciones locales.


## `deploy-compose.yml` [#deploy-composeyml]

Para utilizar un archivo de anulación con un archivo Docker Compose que no sea el predeterminado, como `deploy-compose.yml`, deberá especificar explícitamente ambos archivos al ejecutar comandos de Docker Compose.

Docker Compose le permite especificar múltiples opciones `-f` o `--file` para incluir varios archivos compose, donde los ajustes en los archivos posteriores sobrescriben o se añaden a los del primero.

Si utilizas `deploy-compose.yml` como tu configuración principal de Docker Compose y tienes un archivo de anulación llamado `docker-compose.override.yml` (puedes nombrar el archivo de anulación como desees, pero es posible que ya tengas este archivo específico), ejecutarías los comandos de Docker Compose de la siguiente manera:

```bash
docker compose -f deploy-compose.yml -f docker-compose.override.yml pull
docker compose -f deploy-compose.yml -f docker-compose.override.yml up
```
