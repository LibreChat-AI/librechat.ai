LibreChat utiliza cuatro archivos de configuración principales. Cada uno controla un aspecto diferente de la aplicación, desde variables de entorno hasta endpoints de IA personalizados y anulaciones de servicios de Docker.

## Flujo de trabajo de cambios comunes [#common-change-workflow]

La mayoría de los cambios de configuración siguen el mismo patrón:

1. Edite `.env` para secretos, claves de API y flags de características a nivel de servidor.
2. Edite `librechat.yaml` para endpoints personalizados, Model Specs, configuraciones de interfaz, servidores MCP, agentes y comportamiento avanzado de la aplicación.
3. Para Docker, asegúrate de que `librechat.yaml` esté montado a través de `docker-compose.override.yml` antes de esperar que LibreChat lo lea.
4. Reinicie LibreChat después de cada cambio de configuración.
5. Revisa los registros de la API si el cambio no aparece en la UI.

Por ejemplo, para habilitar OpenRouter, añade `OPENROUTER_KEY` a `.env`, añade un endpoint de OpenRouter en `librechat.yaml`, asegúrate de que Docker monte `librechat.yaml`, reinicia y luego selecciona OpenRouter desde el selector de endpoint.

## Archivos de configuración [#configuration-files]

**`.env`** -- Configuración a nivel de servidor: claves de API, cadenas de conexión a bases de datos, indicadores de funciones y secretos de autenticación. Este es el archivo de configuración principal para la mayoría de las implementaciones. Consulta la [referencia de .env](/docs/configuration/dotenv) para ver todas las variables disponibles.

**`librechat.yaml`** -- Endpoints de IA personalizados, configuraciones de modelos, opciones de interfaz y funciones avanzadas como servidores MCP y agentes. Este archivo es opcional: LibreChat funciona con los valores predeterminados si no existe. Consulta la [guía de librechat.yaml](/docs/configuration/librechat_yaml) para obtener instrucciones de configuración.

**`docker-compose.yml`** -- Define los servicios de Docker (servidor API, base de datos, búsqueda). No edite este archivo directamente; utilice un archivo de anulación (override) en su lugar para que sus cambios sobrevivan a las actualizaciones.

**`docker-compose.override.yml`** -- Tus personalizaciones locales para los servicios de Docker: montajes de volúmenes, mapeos de puertos, anulaciones de variables de entorno. Docker Compose fusiona esto automáticamente con el archivo principal. Consulta la [guía de anulación de Docker](/docs/configuration/docker_override).

## Aplicar cambios [#applying-changes]

<Callout type="warning" title="Se requiere reinicio">

Después de editar cualquier archivo de configuración, debes reiniciar LibreChat para que los cambios surtan efecto.

<Tabs items={['Docker', 'Local']}>
  <Tabs.Tab>

```bash
docker compose down && docker compose up -d
```

  </Tabs.Tab>
  <Tabs.Tab>

Detenga el proceso en ejecución (Ctrl+C) y reinicie:

```bash
npm run backend
```

  </Tabs.Tab>
</Tabs>

</Callout>
