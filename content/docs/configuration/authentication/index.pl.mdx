## Wygasanie sesji i token odświeżania [#session-expiry-and-refresh-token]

- Wartości domyślne: wygaśnięcie sesji: 15 minut, wygaśnięcie tokena odświeżania: 7 dni

## Zautomatyzowany system moderacji (opcjonalnie) [#automated-moderation-system-optional]

Zautomatyzowany system moderacji (Automated Moderation System) jest domyślnie włączony. Wykorzystuje on mechanizm punktacji do śledzenia naruszeń użytkowników. Gdy użytkownicy wykonują działania takie jak nadmierna liczba logowań, rejestracji lub wysyłanie wiadomości, gromadzą punkty naruszeń. Po osiągnięciu określonego progu użytkownik oraz jego adres IP są tymczasowo blokowani. System ten zapewnia bezpieczeństwo platformy poprzez monitorowanie i karanie gwałtownych lub podejrzanych aktywności.

Aby skonfigurować system modów, zapoznaj się z [przewodnikiem konfiguracji](/docs/configuration/mod_system).

> *Uwaga: Jeśli chcesz, aby to działało w trybie deweloperskim, musisz utworzyć plik o nazwie `.env.development` w katalogu głównym i ustawić `DOMAIN_CLIENT` na `http://localhost:3090` lub inny port udostępniony przez vite podczas uruchamiania `npm run frontend-dev`*

## Skrypty zarządzania użytkownikami [#user-management-scripts]

### Utwórz skrypt użytkownika [#create-user-script]

Skrypt create-user umożliwia dodawanie użytkowników bezpośrednio do bazy danych, nawet gdy rejestracja jest wyłączona. Oto jak go użyć:

1. Dla domyślnego `docker-compose.yml` (jeśli używasz `docker compose up`, aby uruchomić aplikację):
   ```bash
   docker compose exec api npm run create-user
   ```

2. Dla `deploy-compose.yml` (jeśli postępowałeś zgodnie z [Przewodnikiem Docker dla Ubuntu](/docs/remote/docker_linux)):
   ```bash
   docker exec -it LibreChat-API /bin/sh -c "cd .. && npm run create-user"
   ```

3. Do lokalnego programowania (z katalogu głównego projektu):
   ```bash
   npm run create-user
   ```

Postępuj zgodnie z instrukcjami, aby wprowadzić adres e-mail i hasło nowego użytkownika.

### Skrypt usuwania użytkownika [#delete-user-script]

Aby usunąć użytkownika, możesz użyć skryptu delete-user:

1. Dla domyślnego `docker-compose.yml` (jeśli używasz `docker compose up`, aby uruchomić aplikację):
   ```bash
   docker compose exec api npm run delete-user email@domain.com
   ```

2. Dla `deploy-compose.yml` (jeśli postępowałeś zgodnie z [Przewodnikiem Docker dla Ubuntu](/docs/remote/docker_linux)):
   ```bash
   docker exec -it LibreChat-API /bin/sh -c "cd .. && npm run delete-user email@domain.com"
   ```

3. Do lokalnego programowania (z katalogu głównego projektu):
   ```bash
   npm run delete-user email@domain.com
   ```

Zastąp `email@domain.com` adresem e-mail użytkownika, którego chcesz usunąć.
