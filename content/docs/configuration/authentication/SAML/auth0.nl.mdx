## Stap 1: Maak een SAML-applicatie aan in Auth0 [#step-1-create-a-saml-application-in-auth0]

1. Log in op je Auth0 Dashboard.
2. Navigeer naar `Applications > Applications`.
3. Klik op `Create Application`.
4. Voer een Applicatienaam in (bijv. `LibreChat`) en selecteer `Regular Web Application`.
5. Klik op `Create`.

![auth0-1](/images/docs/configuration/authentication/SAML/auth0/1.png)


## Stap 2: Configureer de SAML-add-on [#step-2-configure-the-saml-add-on]

1. Open de zojuist gemaakte applicatie in Auth0.
2. Ga naar het tabblad `Addons`.
3. Klik op de schuifregelaar om de `SAML2 Web App` in te schakelen.
4. Klik op het `SAML2 Web App` paneel.
5. Configureer de volgende instellingen:
   - **Application Callback URL**: Stel dit in op uw LibreChat SAML callback URL:
   `https://YOUR_DOMAIN/oauth/saml/callback`
   - **Instellingen (JSON-formaat)**: Gebruik de volgende configuratie:
        ```json
        {
            "mappings": {
                "email": "email",
                "name": "username"
            }
        }
        ```
        <Callout type="note" title="opmerking">
        Als uw applicatie aanvullende attributen vereist zoals `given_name`, `family_name`, `username` of `picture`, zorg er dan voor dat deze toewijzingen correct zijn geconfigureerd in de Auth0 SAML-instellingen.
        </Callout>
6. Klik op `Save`.

## Stap 3: Verkrijg de Auth0 SAML-metadata [#step-3-obtain-the-auth0-saml-metadata]

1. Zodra SAML is ingeschakeld, ga je terug naar de `SAML2 Web App` instellingen.
2. Ga naar het `Usage` tabblad.
3. Klik op `Identity Provider Certificate: Download Atuh0 certificate`.
4. Gebruik de `Issuer` voor `SAML_ISSUER`
5. Gebruik de `Identity Provider Login URL` voor `SAML_ENTRY_POINT`.
6. Kopieer het gedownloade cert-bestand naar je projectmap

![auth0-4](/images/docs/configuration/authentication/SAML/auth0/4a.png)

## Stap 4: Configureer LibreChat met SAML [#step-4-configure-librechat-with-saml]

Open het `.env` bestand in je projectmap en voeg de volgende variabelen toe:

  ```bash filename=".env"
  SAML_ENTRY_POINT=https://dev-xxxxx.us.auth0.com/samlp/aaaaaa
  SAML_ISSUER=urn:dev-xxxxx.us.auth0.com
  SAML_CERT=dev-xxxxx.pem
  SAML_CALLBACK_URL=/oauth/saml/callback
  SAML_SESSION_SECRET=[JustGenerateARandomSessionSecret]

  # Attribute mappings (optional)
  SAML_EMAIL_CLAIM=
  SAML_USERNAME_CLAIM=
  SAML_GIVEN_NAME_CLAIM=
  SAML_FAMILY_NAME_CLAIM=
  SAML_PICTURE_CLAIM=
  SAML_NAME_CLAIM=

  # Logint buttion settings (optional)
  SAML_BUTTON_LABEL=
  SAML_IMAGE_URL=

  # Whether the SAML Response should be signed.
  # - If "true", the entire `SAML Response` will be signed.
  # - If "false" or unset, only the `SAML Assertion` will be signed (default behavior).
  # SAML_USE_AUTHN_RESPONSE_SIGNED=
  ```
