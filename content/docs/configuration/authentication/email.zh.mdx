## 常规设置 [#general-setup]

LibreChat 支持多种电子邮件提供商：
- **Mailgun API** - 推荐用于屏蔽了 SMTP 端口的服务器
- **SMTP 服务** - 通过 Gmail、Outlook 或自定义邮件服务器进行传统邮件发送

这些变量同时被 Mailgun 和 SMTP 使用：

## 使用 Mailgun 进行设置 [#setup-with-mailgun]

要设置 Mailgun，请按照以下步骤操作：

1. 在 [mailgun.com](https://www.mailgun.com/) 注册一个 Mailgun 账户
2. 在 Mailgun 控制面板中添加并验证您的域名
3. 导航至 API Keys 部分并复制您的 Private API key
4. 在 `.env` 文件中，按如下方式修改变量：

<OptionTable
  options={[    
    ['MAILGUN_API_KEY', 'string', '您的 Mailgun 私有 API 密钥', 'MAILGUN_API_KEY=your-mailgun-api-key'],
    ['MAILGUN_DOMAIN', 'string', '您已验证的 Mailgun 域名', 'MAILGUN_DOMAIN=mg.yourdomain.com'],
    ['EMAIL_FROM', 'string', '发件人电子邮件地址', 'EMAIL_FROM=noreply@yourdomain.com'],
    ['EMAIL_FROM_NAME', 'string', '发送者名称', 'EMAIL_FROM_NAME=LibreChat'],
    ['MAILGUN_HOST', 'string', '（可选）针对欧盟地区', 'MAILGUN_HOST=https://api.eu.mailgun.net'],
  ]}
/>

<Callout type="info" title="注意">
如果您的 Mailgun 账户位于欧盟区域，请确保设置 `MAILGUN_HOST=https://api.eu.mailgun.net`
</Callout>

## 使用 Gmail 进行设置 [#setup-with-gmail]

要设置 Gmail，请按照以下步骤操作：

1. 创建一个 Google 账号并启用两步验证。
2. 在 **[Google Account settings](https://myaccount.google.com/)** 中，点击“安全性”选项卡并打开“两步验证”。
3. 向下滚动并打开“App passwords”。为应用选择“Mail”，为设备选择“Other”，然后为其指定一个任意名称。
4. 点击“Generate”生成密码，并复制生成的密码。
5. 在 `.env` 文件中，按如下方式修改变量：

<OptionTable
  options={[    
    ['EMAIL_SERVICE', 'string', 'Gmail', 'EMAIL_SERVICE=gmail'],
    ['EMAIL_USERNAME', 'string', '您的电子邮箱', 'EMAIL_USERNAME=your-email'],
    ['EMAIL_PASSWORD', 'string', '您的电子邮箱密码', 'EMAIL_PASSWORD=your-email-password'],
    ['EMAIL_FROM', 'string', '发件人字段的电子邮件地址，例如 noreply@librechat.ai', 'EMAIL_FROM=noreply@librechat.ai'],
    ['EMAIL_FROM_NAME', 'string', '我的 LibreChat 服务器', 'EMAIL_FROM_NAME=LibreChat'],
  ]}
/>

## 完整配置示例 [#complete-configuration-examples]

### 示例 1：Mailgun 配置 [#example-1-mailgun-configuration]

```bash
# ===================================
# Email Configuration - Mailgun
# ===================================
# Mailgun is recommended for servers that block SMTP ports

# Required Mailgun settings
MAILGUN_API_KEY=your-mailgun-api-key
MAILGUN_DOMAIN=mg.yourdomain.com

# Optional: For EU region
# MAILGUN_HOST=https://api.eu.mailgun.net

# Common email settings
EMAIL_FROM=noreply@yourdomain.com
EMAIL_FROM_NAME=LibreChat

# Enable password reset functionality
ALLOW_PASSWORD_RESET=true
```

### 示例 2：Gmail SMTP 配置 [#example-2-gmail-smtp-configuration]

```bash
# ===================================
# Email Configuration - Gmail SMTP
# ===================================
# Traditional SMTP configuration

# Gmail service configuration
EMAIL_SERVICE=gmail
EMAIL_USERNAME=your-email@gmail.com
EMAIL_PASSWORD=your-app-password

# Common email settings
EMAIL_FROM=your-email@gmail.com
EMAIL_FROM_NAME=LibreChat

# Enable password reset functionality
ALLOW_PASSWORD_RESET=true
```

### 示例 3：自定义 SMTP 服务器配置 [#example-3-custom-smtp-server-configuration]

```bash
# ===================================
# Email Configuration - Custom SMTP
# ===================================
# For custom mail servers

# SMTP server details
EMAIL_HOST=smtp.example.com
EMAIL_PORT=587
EMAIL_ENCRYPTION=starttls
EMAIL_USERNAME=username@example.com
EMAIL_PASSWORD=your-password

# Optional settings
# EMAIL_ENCRYPTION_HOSTNAME=
# EMAIL_ALLOW_SELFSIGNED=false

# Common email settings
EMAIL_FROM=noreply@example.com
EMAIL_FROM_NAME=LibreChat

# Enable password reset functionality
ALLOW_PASSWORD_RESET=true
```

## 故障排除 [#troubleshooting]

### Mailgun 问题 [#mailgun-issues]

1. **身份验证失败**：请确保您的 Mailgun API 密钥正确且拥有发送权限
2. **找不到域名**：请核实您的 Mailgun 域名是否已在您的 Mailgun 账户中正确配置
3. **EU 地区问题**：如果您的 Mailgun 账户位于 EU 地区，请确保设置 `MAILGUN_HOST=https://api.eu.mailgun.net`
4. **回退至 SMTP**：如果仅设置了 `MAILGUN_API_KEY` 或 `MAILGUN_DOMAIN` 中的一项，系统将回退至 SMTP 配置。

### SMTP 问题 [#smtp-issues]

1. **Connection Refused**: 请检查您的服务器是否允许在指定端口进行出站 SMTP 连接
2. **身份验证失败**：请核实您的用户名和密码是否正确
3. **Gmail 应用专用密码 (App Password)**：对于 Gmail，您必须使用应用专用密码，而不是您的常规密码
4. **自签名证书**：如果您的邮件服务器使用自签名证书，请设置 `EMAIL_ALLOW_SELFSIGNED=true`

### 常见问题 [#general-issues]

1. **未发送电子邮件**：检查 LibreChat 日志以获取错误消息
2. **未受保护的密码重置**：当 Mailgun 或 SMTP 未正确配置时，会出现此情况
3. **From 地址问题**：确保 `EMAIL_FROM` 地址有效，并已获得从您的邮件服务发送邮件的授权。
