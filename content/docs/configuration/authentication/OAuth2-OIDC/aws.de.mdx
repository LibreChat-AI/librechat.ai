## Erstellen eines neuen User Pools in Cognito [#create-a-new-user-pool-in-cognito]

- Besuchen Sie: **[https://console.aws.amazon.com/cognito/](https://console.aws.amazon.com/cognito/)**
- Melden Sie sich als Root User an
- Klicken Sie auf `Create user pool`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/e9b412c3-2cf1-4f54-998c-d1d6c12581a5)

## Anmeldeerfahrung konfigurieren [#configure-sign-in-experience]

Ihre Cognito-Benutzerpool-Anmeldeoptionen sollten `User Name` und `Email` enthalten.

![image](https://github.com/danny-avila/LibreChat/assets/32828263/d2cf362d-469e-4993-8466-10282da114c2)

## Sicherheitsanforderungen konfigurieren [#configure-security-requirements]

Sie können die Passwortanforderungen jetzt konfigurieren, wenn Sie dies wünschen.

![image](https://github.com/danny-avila/LibreChat/assets/32828263/e125e8f1-961b-4a38-a6b7-ed1faf29c4a3)

## Registrierungserfahrung konfigurieren [#configure-sign-up-experience]

Wählen Sie die bei der Registrierung erforderlichen Attribute aus. Das Minimum ist `name`. Wenn Sie möchten, dass Benutzer bei der Registrierung ihren vollständigen Namen angeben, verwenden Sie `given_name` und `family_name` als erforderliche Attribute.

![image](https://github.com/danny-avila/LibreChat/assets/32828263/558b8e2c-afbd-4dd1-87f3-c409463b5f7c)

## Nachrichtenzustellung konfigurieren [#configure-message-delivery]

Das Senden von E-Mails mit Cognito kann für bis zu 50 E-Mails pro Tag kostenlos genutzt werden.

![image](https://github.com/danny-avila/LibreChat/assets/32828263/fcb2323b-708e-488c-9420-7eb482974648)

## Integrieren Sie Ihre App [#integrate-your-app]

Wählen Sie `Use Cognitio Hosted UI` und einen Domainnamen aus

![image](https://github.com/danny-avila/LibreChat/assets/32828263/111b3dd4-3b20-4e3e-80e1-7167d2ad0f62)

Setzen Sie den App-Typ auf `Confidential client`.
Stellen Sie sicher, dass `Generate a client secret` aktiviert ist.
Setzen Sie die `Allowed callback URLs` auf `https://YOUR_DOMAIN/oauth/openid/callback`.

![image](https://github.com/danny-avila/LibreChat/assets/32828263/1f92a532-7c4d-4632-a55d-9d00bf77fc4d)

Stellen Sie unter `Advanced app client settings` sicher, dass `Profile` in den `OpenID Connect scopes` (ganz unten) enthalten ist.

![image](https://github.com/danny-avila/LibreChat/assets/32828263/5b035eae-4a8e-482c-abd5-29cee6502eeb)

## Holen Sie sich Ihre Umgebungsvariablen [#get-your-environment-variables]

1. Öffnen Sie Ihren User Pool

![image](https://github.com/danny-avila/LibreChat/assets/32828263/b658ff2a-d252-4f3d-90a7-9fbde42c01db)

2. Die `User Pool ID` und Ihre AWS-Region werden verwendet, um den `OPENID_ISSUER` zu erstellen (siehe unten).

![image](https://github.com/danny-avila/LibreChat/assets/32828263/dc8ae403-cbff-4aae-9eee-42d7cf3485e7)
![image](https://github.com/danny-avila/LibreChat/assets/32828263/d606f5c8-c60b-4d20-bdb2-d0d69e49ea1e)

3. Gehen Sie zum Tab `App Integrations`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/58713bdc-24bc-47de-bdca-020dc321e997)

4. Öffnen Sie den App-Client

![image](https://github.com/danny-avila/LibreChat/assets/32828263/271bf7d2-3df2-43a7-87fc-e50294e49b2e)

5. `Show Client Secret` umschalten

![image](https://github.com/danny-avila/LibreChat/assets/32828263/a844fe65-313d-4754-81b4-380336e0e336)

- Verwenden Sie die `Client ID` für `OPENID_CLIENT_ID`

- Verwenden Sie das `Client secret` für `OPENID_CLIENT_SECRET`

- Generieren Sie eine zufällige Zeichenfolge für das `OPENID_SESSION_SECRET`

> Die `OPENID_SCOPE` und `OPENID_CALLBACK_URL` sind mit den korrekten Werten vorkonfiguriert.

6. Öffnen Sie die `.env` Datei im Stammverzeichnis Ihres LibreChat Ordners und fügen Sie die folgenden Variablen mit den von Ihnen kopierten Werten hinzu:

```bash filename=".env"
DOMAIN_CLIENT=https://your-domain.com # use http://localhost:3080 if not using a custom domain
DOMAIN_SERVER=https://your-domain.com # use http://localhost:3080 if not using a custom domain

OPENID_CLIENT_ID=Your client ID
OPENID_CLIENT_SECRET=Your client secret
OPENID_ISSUER=https://cognito-idp.[AWS REGION].amazonaws.com/[USER POOL ID]/.well-known/openid-configuration
OPENID_SESSION_SECRET=Any random string
OPENID_SCOPE=openid profile email
OPENID_CALLBACK_URL=/oauth/openid/callback

# Optional: redirects the user to the end session endpoint after logging out
OPENID_USE_END_SESSION_ENDPOINT=true
# Optional: generates the nonce url parameter.
OPENID_GENERATE_NONCE=true
```

> [!WARNING]  
> Wenn Cognito mit einem OIDC-Anbieter konfiguriert ist, d. h. eine Föderation mit Entra besteht, ist `OPENID_GENERATE_NONCE=true` erforderlich. Andernfalls generiert Cognito diesen ungeachtet dessen, und die Validierung der Claims schlägt fehl, da der Client keinen bereitgestellt hat.

7. Speichere die .env Datei

> Hinweis: Wenn Sie Docker verwenden, führen Sie `docker compose up -d` aus, um die Konfigurationsänderungen in der .env anzuwenden.
