## Crea un nuovo User Pool in Cognito [#create-a-new-user-pool-in-cognito]

- Visita: **[https://console.aws.amazon.com/cognito/](https://console.aws.amazon.com/cognito/)**
- Accedi come Root User
- Fai clic su `Create user pool`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/e9b412c3-2cf1-4f54-998c-d1d6c12581a5)

## Configura l'esperienza di accesso [#configure-sign-in-experience]

Le opzioni di accesso al pool di utenti Cognito devono includere `User Name` ed `Email`.

![image](https://github.com/danny-avila/LibreChat/assets/32828263/d2cf362d-469e-4993-8466-10282da114c2)

## Configura i requisiti di sicurezza [#configure-security-requirements]

Puoi configurare i requisiti della password ora, se lo desideri.

![image](https://github.com/danny-avila/LibreChat/assets/32828263/e125e8f1-961b-4a38-a6b7-ed1faf29c4a3)

## Configura l'esperienza di registrazione [#configure-sign-up-experience]

Scegli gli attributi richiesti durante la registrazione. Il minimo richiesto è `name`. Se desideri che gli utenti utilizzino il proprio nome completo al momento della registrazione, utilizza `given_name` e `family_name` come attributi richiesti.

![image](https://github.com/danny-avila/LibreChat/assets/32828263/558b8e2c-afbd-4dd1-87f3-c409463b5f7c)

## Configurazione della consegna dei messaggi [#configure-message-delivery]

L'invio di email con Cognito può essere utilizzato gratuitamente fino a 50 email al giorno

![image](https://github.com/danny-avila/LibreChat/assets/32828263/fcb2323b-708e-488c-9420-7eb482974648)

## Integra la tua app [#integrate-your-app]

Seleziona `Use Cognitio Hosted UI` e scegli un nome di dominio

![image](https://github.com/danny-avila/LibreChat/assets/32828263/111b3dd4-3b20-4e3e-80e1-7167d2ad0f62)

Imposta il tipo di app su `Confidential client`
Assicurati che `Generate a client secret` sia impostato.
Imposta `Allowed callback URLs` su `https://YOUR_DOMAIN/oauth/openid/callback`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/1f92a532-7c4d-4632-a55d-9d00bf77fc4d)

Sotto `Advanced app client settings` assicurati che `Profile` sia incluso negli `OpenID Connect scopes` (in basso)

![image](https://github.com/danny-avila/LibreChat/assets/32828263/5b035eae-4a8e-482c-abd5-29cee6502eeb)

## Ottieni le tue variabili d'ambiente [#get-your-environment-variables]

1. Apri il tuo User Pool

![image](https://github.com/danny-avila/LibreChat/assets/32828263/b658ff2a-d252-4f3d-90a7-9fbde42c01db)

2. L'`User Pool ID` e la tua regione AWS saranno utilizzati per costruire l'`OPENID_ISSUER` (vedi sotto)

![image](https://github.com/danny-avila/LibreChat/assets/32828263/dc8ae403-cbff-4aae-9eee-42d7cf3485e7)
![image](https://github.com/danny-avila/LibreChat/assets/32828263/d606f5c8-c60b-4d20-bdb2-d0d69e49ea1e)

3. Vai alla scheda `App Integrations`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/58713bdc-24bc-47de-bdca-020dc321e997)

4. Apri il client dell'app

![image](https://github.com/danny-avila/LibreChat/assets/32828263/271bf7d2-3df2-43a7-87fc-e50294e49b2e)

5. Attiva `Show Client Secret`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/a844fe65-313d-4754-81b4-380336e0e336)

- Usa il `Client ID` per `OPENID_CLIENT_ID`

- Usa il `Client secret` per `OPENID_CLIENT_SECRET`

- Genera una stringa casuale per `OPENID_SESSION_SECRET`

> `OPENID_SCOPE` e `OPENID_CALLBACK_URL` sono preconfigurati con i valori corretti

6. Apri il file `.env` nella root della tua cartella LibreChat e aggiungi le seguenti variabili con i valori che hai copiato:

```bash filename=".env"
DOMAIN_CLIENT=https://your-domain.com # use http://localhost:3080 if not using a custom domain
DOMAIN_SERVER=https://your-domain.com # use http://localhost:3080 if not using a custom domain

OPENID_CLIENT_ID=Your client ID
OPENID_CLIENT_SECRET=Your client secret
OPENID_ISSUER=https://cognito-idp.[AWS REGION].amazonaws.com/[USER POOL ID]/.well-known/openid-configuration
OPENID_SESSION_SECRET=Any random string
OPENID_SCOPE=openid profile email
OPENID_CALLBACK_URL=/oauth/openid/callback

# Optional: redirects the user to the end session endpoint after logging out
OPENID_USE_END_SESSION_ENDPOINT=true
# Maximum logout URL length before using logout_hint instead of id_token_hint (default: 2000)
# OPENID_MAX_LOGOUT_URL_LENGTH=2000
# Optional: generates the nonce url parameter.
OPENID_GENERATE_NONCE=true
```

> [!WARNING]  
> Se Cognito è configurato con un provider OIDC, ad esempio la federazione con Entra, è necessario `OPENID_GENERATE_NONCE=true`. In caso contrario, Cognito lo genererà comunque e la convalida delle attestazioni fallirà poiché il client non ne ha fornito uno.

7. Salva il file .env

> Nota: Se si utilizza Docker, eseguire `docker compose up -d` per applicare le modifiche alla configurazione nel file .env
