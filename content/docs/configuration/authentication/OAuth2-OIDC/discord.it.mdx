## Crea una nuova Discord Application [#create-a-new-discord-application]

- Vai al **[Discord Developer Portal](https://discord.com/developers)**

- Crea una nuova Application e assegnale un nome

![image](https://github.com/danny-avila/LibreChat/assets/32828263/7e7cdfa0-d1d6-4b6b-a8a9-905aaa40d135)

## Configurazione dell'applicazione Discord [#discord-application-configuration]

- Nelle impostazioni generali di OAuth2, aggiungi un URL di reindirizzamento valido:
    - Esempio per localhost: `http://localhost:3080/oauth/discord/callback`
    - Esempio per un dominio: `https://example.com/oauth/discord/callback`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/6c56fb92-f4ab-43b9-981b-f98babeeb19d)

- In `Default Authorization Link`, seleziona `In-app Authorization` e imposta gli scope su `applications.commands`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/2ce94670-9422-48d2-97e9-ec40bd331573)

- Salva le modifiche e reimposta il Client Secret

![image](https://github.com/danny-avila/LibreChat/assets/32828263/3af164fc-66ed-4e5e-9f5a-9bcab3df37b4)
![image](https://github.com/danny-avila/LibreChat/assets/32828263/2ece3935-68e6-4f2e-8656-9721cba5388a)

## Configurazione di .env [#env-configuration]

- Incolla il tuo `Client ID` e `Client Secret` nel file `.env`:

```bash filename=".env"
DOMAIN_CLIENT=https://your-domain.com # use http://localhost:3080 if not using a custom domain
DOMAIN_SERVER=https://your-domain.com # use http://localhost:3080 if not using a custom domain

DISCORD_CLIENT_ID=your_client_id
DISCORD_CLIENT_SECRET=your_client_secret
DISCORD_CALLBACK_URL=/oauth/discord/callback
```

- Salva il file `.env`

> Nota: Se si utilizza Docker, eseguire `docker compose up -d` per applicare le modifiche alla configurazione nel file .env
