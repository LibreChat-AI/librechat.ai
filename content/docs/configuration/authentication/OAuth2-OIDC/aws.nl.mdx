- Meld u aan als Root User

Kies de vereiste kenmerken bij aanmelding. Het minimum dat vereist is, is `name`. Als je wilt dat gebruikers hun volledige naam gebruiken bij aanmelding, gebruik dan `given_name` en `family_name` als vereiste kenmerken.

## Integreer je app [#integrate-your-app]

Selecteer `Use Cognitio Hosted UI` en kies een domeinnaam

![image](https://github.com/danny-avila/LibreChat/assets/32828263/111b3dd4-3b20-4e3e-80e1-7167d2ad0f62)

Stel het app-type in op `Confidential client`
Zorg ervoor dat `Generate a client secret` is ingeschakeld.
Stel de `Allowed callback URLs` in op `https://YOUR_DOMAIN/oauth/openid/callback`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/1f92a532-7c4d-4632-a55d-9d00bf77fc4d)

Zorg er onder `Advanced app client settings` voor dat `Profile` is opgenomen in de `OpenID Connect scopes` (onderaan).

![image](https://github.com/danny-avila/LibreChat/assets/32828263/5b035eae-4a8e-482c-abd5-29cee6502eeb)

## Verkrijg je omgevingsvariabelen [#get-your-environment-variables]

1. Open uw User Pool

![image](https://github.com/danny-avila/LibreChat/assets/32828263/b658ff2a-d252-4f3d-90a7-9fbde42c01db)

2. De `User Pool ID` en uw AWS-regio worden gebruikt om de `OPENID_ISSUER` samen te stellen (zie hieronder)

![image](https://github.com/danny-avila/LibreChat/assets/32828263/dc8ae403-cbff-4aae-9eee-42d7cf3485e7)
![image](https://github.com/danny-avila/LibreChat/assets/32828263/d606f5c8-c60b-4d20-bdb2-d0d69e49ea1e)

3. Ga naar het tabblad `App Integrations`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/58713bdc-24bc-47de-bdca-020dc321e997)

4. Open de app-client

![image](https://github.com/danny-avila/LibreChat/assets/32828263/271bf7d2-3df2-43a7-87fc-e50294e49b2e)

5. Schakel `Show Client Secret` in/uit

![image](https://github.com/danny-avila/LibreChat/assets/32828263/a844fe65-313d-4754-81b4-380336e0e336)

- Gebruik de `Client ID` voor `OPENID_CLIENT_ID`

- Gebruik de `Client secret` voor `OPENID_CLIENT_SECRET`

- Genereer een willekeurige string voor de `OPENID_SESSION_SECRET`

> De `OPENID_SCOPE` en `OPENID_CALLBACK_URL` zijn vooraf geconfigureerd met de juiste waarden

6. Open het `.env` bestand in de hoofdmap van je LibreChat map en voeg de volgende variabelen toe met de waarden die je hebt gekopieerd:

```bash filename=".env"
DOMAIN_CLIENT=https://your-domain.com # use http://localhost:3080 if not using a custom domain
DOMAIN_SERVER=https://your-domain.com # use http://localhost:3080 if not using a custom domain

OPENID_CLIENT_ID=Your client ID
OPENID_CLIENT_SECRET=Your client secret
OPENID_ISSUER=https://cognito-idp.[AWS REGION].amazonaws.com/[USER POOL ID]/.well-known/openid-configuration
OPENID_SESSION_SECRET=Any random string
OPENID_SCOPE=openid profile email
OPENID_CALLBACK_URL=/oauth/openid/callback

# Optional: redirects the user to the end session endpoint after logging out
OPENID_USE_END_SESSION_ENDPOINT=true
# Maximum logout URL length before using logout_hint instead of id_token_hint (default: 2000)
# OPENID_MAX_LOGOUT_URL_LENGTH=2000
# Optional: generates the nonce url parameter.
OPENID_GENERATE_NONCE=true
```

> [!WARNING]  
> Als Cognito is geconfigureerd met een OIDC-provider, d.w.z. federatie naar Entra, dan is `OPENID_GENERATE_NONCE=true` vereist. Anders zal Cognito deze hoe dan ook genereren en zal de validatie van de claims mislukken omdat de client er geen heeft verstrekt.

7. Sla het .env bestand op

> Let op: Als je Docker gebruikt, voer dan `docker compose up -d` uit om de wijzigingen in de .env configuratie toe te passen.
