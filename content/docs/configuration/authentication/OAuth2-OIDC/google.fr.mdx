## Configuration de l'application Google [#google-application-configuration]

- Sélectionnez le projet que vous venez de créer et allez dans `APIs and Services`

![image](https://github.com/danny-avila/LibreChat/assets/138638445/c6265582-2cf6-430f-ae51-1edbdd9f2c48)

![image](https://github.com/danny-avila/LibreChat/assets/138638445/006e16ba-56b8-452d-b324-5f2d202637ab)

- Sélectionnez `Credentials` et cliquez sur `CONFIGURE CONSENT SCREEN`

![image](https://github.com/danny-avila/LibreChat/assets/138638445/e4285cbb-833f-4366-820d-addf04a2ad77)

- Sélectionnez `External` puis cliquez sur `CREATE`

![image](https://github.com/danny-avila/LibreChat/assets/138638445/232d46c0-dd00-4637-b538-3ba3bdbdc0b2)

- Remplissez vos informations d'application

> Remarque : Vous pouvez obtenir un logo depuis votre dossier LibreChat ici : `docs\assets\favicon_package\android-chrome-192x192.png`

![image](https://github.com/danny-avila/LibreChat/assets/138638445/e6c4c8ec-2f02-4af5-9458-c72394d0b7c5)

- Configurez votre `App domain` et ajoutez vos `Developer contact information`, puis cliquez sur `SAVE AND CONTINUE`

![image](https://github.com/danny-avila/LibreChat/assets/138638445/6c2aa557-9b9b-412d-bc2b-76a0dc11f394)

- Configure the `Sopes`
    - Ajoutez `email`, `profile` et `openid`
    - Cliquez sur `UPDATE` et `SAVE AND CONTINUE`

![image](https://github.com/danny-avila/LibreChat/assets/138638445/46af2fb9-8cfd-41c5-a763-814b308e45c3)

![image](https://github.com/danny-avila/LibreChat/assets/138638445/4e832970-d392-4c67-bb38-908a5c51660a)

- Cliquez sur `SAVE AND CONTINUE`
- Examinez votre application et retournez au tableau de bord

- Retournez à l'onglet `Credentials`, cliquez sur `+ CREATE CREDENTIALS` et sélectionnez `OAuth client ID`

![image](https://github.com/danny-avila/LibreChat/assets/138638445/beef1982-55a3-4837-8e8c-20bad8d846ba)

- Sélectionnez `Web application` et donnez-lui un nom

![image](https://github.com/danny-avila/LibreChat/assets/138638445/badde864-f6b5-468f-a72f-bac93326ffa5)

- Configurez les `Authorized JavaScript origins`, vous pouvez ajouter à la fois votre domaine et localhost si vous le souhaitez
    - Exemple pour localhost : `http://localhost:3080`
    - Exemple pour un domaine : `https://example.com`

![image](https://github.com/danny-avila/LibreChat/assets/138638445/f7e3763a-5f74-4850-8638-44f81693b9ac)

- Ajoutez des `Authorized redirect URIs` valides
    - Exemple pour localhost : `http://localhost:3080/oauth/google/callback`
    - Exemple pour un domaine : `https://example.com/oauth/google/callback`

![image](https://github.com/danny-avila/LibreChat/assets/138638445/0db34b19-d780-4651-9c2f-d33e24a74d55)

## Configuration de .env [#env-configuration]

- Cliquez sur `CREATE` et copiez votre `Client ID` et votre `Client secret`

![image](https://github.com/danny-avila/LibreChat/assets/138638445/fa8572bf-f482-457a-a285-aec7d41af76b)

- Ajoutez-les à votre fichier `.env` :

```bash filename=".env"
DOMAIN_CLIENT=https://your-domain.com # use http://localhost:3080 if not using a custom domain
DOMAIN_SERVER=https://your-domain.com # use http://localhost:3080 if not using a custom domain

GOOGLE_CLIENT_ID=your_client_id
GOOGLE_CLIENT_SECRET=your_client_secret
GOOGLE_CALLBACK_URL=/oauth/google/callback
```

- Enregistrez le fichier `.env`

> Remarque : Si vous utilisez Docker, exécutez `docker compose up -d` pour appliquer les modifications de configuration du fichier .env
