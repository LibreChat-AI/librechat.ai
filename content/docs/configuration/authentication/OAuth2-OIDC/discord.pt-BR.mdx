## Criar um novo Discord Application [#create-a-new-discord-application]

- Vá para o **[Discord Developer Portal](https://discord.com/developers)**

- Crie um novo Application e dê a ele um nome

![image](https://github.com/danny-avila/LibreChat/assets/32828263/7e7cdfa0-d1d6-4b6b-a8a9-905aaa40d135)

## Configuração do Aplicativo Discord [#discord-application-configuration]

- Nas configurações gerais do OAuth2, adicione uma URL de redirecionamento válida:
    - Exemplo para localhost: `http://localhost:3080/oauth/discord/callback`
    - Exemplo para um domínio: `https://example.com/oauth/discord/callback`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/6c56fb92-f4ab-43b9-981b-f98babeeb19d)

- Em `Default Authorization Link`, selecione `In-app Authorization` e defina os escopos como `applications.commands`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/2ce94670-9422-48d2-97e9-ec40bd331573)

- Salve as alterações e redefina o Client Secret

![image](https://github.com/danny-avila/LibreChat/assets/32828263/3af164fc-66ed-4e5e-9f5a-9bcab3df37b4)
![image](https://github.com/danny-avila/LibreChat/assets/32828263/2ece3935-68e6-4f2e-8656-9721cba5388a)

## Configuração do .env [#env-configuration]

- Cole o seu `Client ID` e `Client Secret` no arquivo `.env`:

```bash filename=".env"
DOMAIN_CLIENT=https://your-domain.com # use http://localhost:3080 if not using a custom domain
DOMAIN_SERVER=https://your-domain.com # use http://localhost:3080 if not using a custom domain

DISCORD_CLIENT_ID=your_client_id
DISCORD_CLIENT_SECRET=your_client_secret
DISCORD_CALLBACK_URL=/oauth/discord/callback
```

- Salve o arquivo `.env`

> Nota: Se estiver usando Docker, execute `docker compose up -d` para aplicar as alterações de configuração do .env
