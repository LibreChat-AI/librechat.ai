## Google Anwendungskonfiguration [#google-application-configuration]

- Wählen Sie das soeben erstellte Projekt aus und gehen Sie zu `APIs and Services`

![image](https://github.com/danny-avila/LibreChat/assets/138638445/c6265582-2cf6-430f-ae51-1edbdd9f2c48)

![image](https://github.com/danny-avila/LibreChat/assets/138638445/006e16ba-56b8-452d-b324-5f2d202637ab)

- Wählen Sie `Credentials` und klicken Sie auf `CONFIGURE CONSENT SCREEN`

![image](https://github.com/danny-avila/LibreChat/assets/138638445/e4285cbb-833f-4366-820d-addf04a2ad77)

- Wählen Sie `External` und klicken Sie dann auf `CREATE`

![image](https://github.com/danny-avila/LibreChat/assets/138638445/232d46c0-dd00-4637-b538-3ba3bdbdc0b2)

- Geben Sie Ihre App-Informationen ein

> Hinweis: Sie können ein Logo aus Ihrem LibreChat-Ordner hier abrufen: `docs\assets\favicon_package\android-chrome-192x192.png`

![image](https://github.com/danny-avila/LibreChat/assets/138638445/e6c4c8ec-2f02-4af5-9458-c72394d0b7c5)

- Konfigurieren Sie Ihre `App domain` und fügen Sie Ihre `Developer contact information` hinzu, klicken Sie dann auf `SAVE AND CONTINUE`

![image](https://github.com/danny-avila/LibreChat/assets/138638445/6c2aa557-9b9b-412d-bc2b-76a0dc11f394)

- Configure the `Sopes`
    - Fügen Sie `email`, `profile` und `openid` hinzu
    - Klicken Sie auf `UPDATE` und `SAVE AND CONTINUE`

![image](https://github.com/danny-avila/LibreChat/assets/138638445/46af2fb9-8cfd-41c5-a763-814b308e45c3)

![image](https://github.com/danny-avila/LibreChat/assets/138638445/4e832970-d392-4c67-bb38-908a5c51660a)

- Klicken Sie auf `SAVE AND CONTINUE`
- Überprüfen Sie Ihre App und kehren Sie zum Dashboard zurück

- Gehen Sie zurück zum Tab `Credentials`, klicken Sie auf `+ CREATE CREDENTIALS` und wählen Sie `OAuth client ID` aus.

![image](https://github.com/danny-avila/LibreChat/assets/138638445/beef1982-55a3-4837-8e8c-20bad8d846ba)

- Wählen Sie `Web application` aus und geben Sie ihr einen Namen

![image](https://github.com/danny-avila/LibreChat/assets/138638445/badde864-f6b5-468f-a72f-bac93326ffa5)

- Konfigurieren Sie die `Authorized JavaScript origins`; Sie können sowohl Ihre Domain als auch localhost hinzufügen, falls gewünscht.
    - Beispiel für localhost: `http://localhost:3080`
    - Beispiel für eine Domain: `https://example.com`

![image](https://github.com/danny-avila/LibreChat/assets/138638445/f7e3763a-5f74-4850-8638-44f81693b9ac)

- Fügen Sie eine gültige `Authorized redirect URIs` hinzu
    - Beispiel für localhost: `http://localhost:3080/oauth/google/callback`
    - Beispiel für eine Domain: `https://example.com/oauth/google/callback`

![image](https://github.com/danny-avila/LibreChat/assets/138638445/0db34b19-d780-4651-9c2f-d33e24a74d55)

## .env Konfiguration [#env-configuration]

- Klicken Sie auf `CREATE` und kopieren Sie Ihre `Client ID` und Ihr `Client secret`

![image](https://github.com/danny-avila/LibreChat/assets/138638445/fa8572bf-f482-457a-a285-aec7d41af76b)

- Fügen Sie diese Ihrer `.env` Datei hinzu:

```bash filename=".env"
DOMAIN_CLIENT=https://your-domain.com # use http://localhost:3080 if not using a custom domain
DOMAIN_SERVER=https://your-domain.com # use http://localhost:3080 if not using a custom domain

GOOGLE_CLIENT_ID=your_client_id
GOOGLE_CLIENT_SECRET=your_client_secret
GOOGLE_CALLBACK_URL=/oauth/google/callback
```

- Speichern Sie die `.env` Datei

> Hinweis: Wenn Sie Docker verwenden, führen Sie `docker compose up -d` aus, um die Konfigurationsänderungen in der .env anzuwenden.
