## Erstellen einer neuen Discord-Anwendung [#create-a-new-discord-application]

- Gehen Sie zum **[Discord Developer Portal](https://discord.com/developers)**

- Erstellen Sie eine neue Application und geben Sie ihr einen Namen

![image](https://github.com/danny-avila/LibreChat/assets/32828263/7e7cdfa0-d1d6-4b6b-a8a9-905aaa40d135)

## Konfiguration der Discord-Anwendung [#discord-application-configuration]

- Fügen Sie in den allgemeinen OAuth2-Einstellungen eine gültige Redirect-URL hinzu:
    - Beispiel für localhost: `http://localhost:3080/oauth/discord/callback`
    - Beispiel für eine Domain: `https://example.com/oauth/discord/callback`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/6c56fb92-f4ab-43b9-981b-f98babeeb19d)

- Wählen Sie unter `Default Authorization Link` die Option `In-app Authorization` und setzen Sie die Scopes auf `applications.commands`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/2ce94670-9422-48d2-97e9-ec40bd331573)

- Änderungen speichern und das Client Secret zurücksetzen

![image](https://github.com/danny-avila/LibreChat/assets/32828263/3af164fc-66ed-4e5e-9f5a-9bcab3df37b4)
![image](https://github.com/danny-avila/LibreChat/assets/32828263/2ece3935-68e6-4f2e-8656-9721cba5388a)

## .env Konfiguration [#env-configuration]

- Fügen Sie Ihre `Client ID` und Ihr `Client Secret` in die `.env` Datei ein:

```bash filename=".env"
DOMAIN_CLIENT=https://your-domain.com # use http://localhost:3080 if not using a custom domain
DOMAIN_SERVER=https://your-domain.com # use http://localhost:3080 if not using a custom domain

DISCORD_CLIENT_ID=your_client_id
DISCORD_CLIENT_SECRET=your_client_secret
DISCORD_CALLBACK_URL=/oauth/discord/callback
```

- Speichern Sie die `.env` Datei

> Hinweis: Wenn Sie Docker verwenden, führen Sie `docker compose up -d` aus, um die Konfigurationsänderungen in der .env anzuwenden.
