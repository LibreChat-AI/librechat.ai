## Membuat Aplikasi Discord baru [#create-a-new-discord-application]

- Buka **[Discord Developer Portal](https://discord.com/developers)**

- Buat Aplikasi baru dan beri nama

![image](https://github.com/danny-avila/LibreChat/assets/32828263/7e7cdfa0-d1d6-4b6b-a8a9-905aaa40d135)

## Konfigurasi Aplikasi Discord [#discord-application-configuration]

- Di pengaturan umum OAuth2, tambahkan URL pengalihan (redirect URL) yang valid:
    - Contoh untuk localhost: `http://localhost:3080/oauth/discord/callback`
    - Contoh untuk domain: `https://example.com/oauth/discord/callback`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/6c56fb92-f4ab-43b9-981b-f98babeeb19d)

- Pada `Default Authorization Link`, pilih `In-app Authorization` dan atur scope ke `applications.commands`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/2ce94670-9422-48d2-97e9-ec40bd331573)

- Simpan perubahan dan atur ulang Client Secret

![image](https://github.com/danny-avila/LibreChat/assets/32828263/3af164fc-66ed-4e5e-9f5a-9bcab3df37b4)
![image](https://github.com/danny-avila/LibreChat/assets/32828263/2ece3935-68e6-4f2e-8656-9721cba5388a)

## Konfigurasi .env [#env-configuration]

- Tempelkan `Client ID` dan `Client Secret` Anda di dalam file `.env`:

```bash filename=".env"
DOMAIN_CLIENT=https://your-domain.com # use http://localhost:3080 if not using a custom domain
DOMAIN_SERVER=https://your-domain.com # use http://localhost:3080 if not using a custom domain

DISCORD_CLIENT_ID=your_client_id
DISCORD_CLIENT_SECRET=your_client_secret
DISCORD_CALLBACK_URL=/oauth/discord/callback
```

- Simpan file `.env`

> Catatan: Jika menggunakan Docker, jalankan `docker compose up -d` untuk menerapkan perubahan konfigurasi .env
