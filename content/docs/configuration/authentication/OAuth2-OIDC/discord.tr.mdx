## Yeni bir Discord Uygulaması oluşturun [#create-a-new-discord-application]

- **[Discord Developer Portal](https://discord.com/developers)** adresine gidin

- Yeni bir Uygulama oluşturun ve bir ad verin

![image](https://github.com/danny-avila/LibreChat/assets/32828263/7e7cdfa0-d1d6-4b6b-a8a9-905aaa40d135)

## Discord Uygulama Yapılandırması [#discord-application-configuration]

- OAuth2 genel ayarlarında geçerli bir yönlendirme URL'si ekleyin:
    - localhost için örnek: `http://localhost:3080/oauth/discord/callback`
    - Bir alan adı için örnek: `https://example.com/oauth/discord/callback`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/6c56fb92-f4ab-43b9-981b-f98babeeb19d)

- `Default Authorization Link` içinde `In-app Authorization` seçeneğini belirleyin ve kapsamları (scopes) `applications.commands` olarak ayarlayın.

![image](https://github.com/danny-avila/LibreChat/assets/32828263/2ce94670-9422-48d2-97e9-ec40bd331573)

- Değişiklikleri kaydet ve Client Secret'ı sıfırla

![image](https://github.com/danny-avila/LibreChat/assets/32828263/3af164fc-66ed-4e5e-9f5a-9bcab3df37b4)
![image](https://github.com/danny-avila/LibreChat/assets/32828263/2ece3935-68e6-4f2e-8656-9721cba5388a)

## .env Yapılandırması [#env-configuration]

- `Client ID` ve `Client Secret` bilgilerinizi `.env` dosyasına yapıştırın:

```bash filename=".env"
DOMAIN_CLIENT=https://your-domain.com # use http://localhost:3080 if not using a custom domain
DOMAIN_SERVER=https://your-domain.com # use http://localhost:3080 if not using a custom domain

DISCORD_CLIENT_ID=your_client_id
DISCORD_CLIENT_SECRET=your_client_secret
DISCORD_CALLBACK_URL=/oauth/discord/callback
```

- `.env` dosyasını kaydedin

> Not: Docker kullanıyorsanız, .env yapılandırma değişikliklerini uygulamak için `docker compose up -d` komutunu çalıştırın.
