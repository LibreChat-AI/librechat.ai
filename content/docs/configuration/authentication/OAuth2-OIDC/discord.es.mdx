## Crear una nueva Discord Application [#create-a-new-discord-application]

- Ve al **[Discord Developer Portal](https://discord.com/developers)**

- Cree una nueva Application y asígnele un nombre

![image](https://github.com/danny-avila/LibreChat/assets/32828263/7e7cdfa0-d1d6-4b6b-a8a9-905aaa40d135)

## Configuración de la aplicación de Discord [#discord-application-configuration]

- En la configuración general de OAuth2, añade una URL de redirección válida:
    - Ejemplo para localhost: `http://localhost:3080/oauth/discord/callback`
    - Ejemplo para un dominio: `https://example.com/oauth/discord/callback`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/6c56fb92-f4ab-43b9-981b-f98babeeb19d)

- En `Default Authorization Link`, selecciona `In-app Authorization` y establece los scopes en `applications.commands`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/2ce94670-9422-48d2-97e9-ec40bd331573)

- Guardar cambios y restablecer el Client Secret

![image](https://github.com/danny-avila/LibreChat/assets/32828263/3af164fc-66ed-4e5e-9f5a-9bcab3df37b4)
![image](https://github.com/danny-avila/LibreChat/assets/32828263/2ece3935-68e6-4f2e-8656-9721cba5388a)

## Configuración de .env [#env-configuration]

- Pega tu `Client ID` y `Client Secret` en el archivo `.env`:

```bash filename=".env"
DOMAIN_CLIENT=https://your-domain.com # use http://localhost:3080 if not using a custom domain
DOMAIN_SERVER=https://your-domain.com # use http://localhost:3080 if not using a custom domain

DISCORD_CLIENT_ID=your_client_id
DISCORD_CLIENT_SECRET=your_client_secret
DISCORD_CALLBACK_URL=/oauth/discord/callback
```

- Guarda el archivo `.env`

> Nota: Si utiliza Docker, ejecute `docker compose up -d` para aplicar los cambios de configuración en el archivo .env
