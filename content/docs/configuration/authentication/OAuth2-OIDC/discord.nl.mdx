## Een nieuwe Discord-applicatie aanmaken [#create-a-new-discord-application]

- Ga naar de **[Discord Developer Portal](https://discord.com/developers)**

- Maak een nieuwe Application aan en geef deze een naam

![image](https://github.com/danny-avila/LibreChat/assets/32828263/7e7cdfa0-d1d6-4b6b-a8a9-905aaa40d135)

## Discord-applicatieconfiguratie [#discord-application-configuration]

- Voeg in de algemene OAuth2-instellingen een geldige redirect URL toe:
    - Voorbeeld voor localhost: `http://localhost:3080/oauth/discord/callback`
    - Voorbeeld voor een domein: `https://example.com/oauth/discord/callback`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/6c56fb92-f4ab-43b9-981b-f98babeeb19d)

- In `Default Authorization Link`, selecteer `In-app Authorization` en stel de scopes in op `applications.commands`

![image](https://github.com/danny-avila/LibreChat/assets/32828263/2ce94670-9422-48d2-97e9-ec40bd331573)

- Sla wijzigingen op en reset de Client Secret

![image](https://github.com/danny-avila/LibreChat/assets/32828263/3af164fc-66ed-4e5e-9f5a-9bcab3df37b4)
![image](https://github.com/danny-avila/LibreChat/assets/32828263/2ece3935-68e6-4f2e-8656-9721cba5388a)

## .env Configuratie [#env-configuration]

- Plak je `Client ID` en `Client Secret` in het `.env` bestand:

```bash filename=".env"
DOMAIN_CLIENT=https://your-domain.com # use http://localhost:3080 if not using a custom domain
DOMAIN_SERVER=https://your-domain.com # use http://localhost:3080 if not using a custom domain

DISCORD_CLIENT_ID=your_client_id
DISCORD_CLIENT_SECRET=your_client_secret
DISCORD_CALLBACK_URL=/oauth/discord/callback
```

- Sla het `.env` bestand op

> Let op: Als je Docker gebruikt, voer dan `docker compose up -d` uit om de wijzigingen in de .env configuratie toe te passen.
