- 请参阅：[LibreChat 中的搜索功能](/docs/features/search)以获取快速概览。

MeiliSearch 是一款功能强大的开源搜索引擎，它通过为过往对话提供全文搜索、容错纠错和即时搜索结果，增强了 LibreChat 的功能。

请按照以下步骤为 LibreChat 设置 MeiliSearch：

## 1. 下载 MeiliSearch [#1-download-meilisearch]
    - 前往 MeiliSearch GitHub 发布页面：https://github.com/meilisearch/meilisearch/releases
    - 下载适用于您操作系统的最新版 MeiliSearch（例如 Linux 使用 `meilisearch-linux-amd64.tar.gz`，macOS 使用 `meilisearch-macos-amd64`，或 Windows 使用 `meilisearch-windows-amd64.zip`）。

## 2. 提取/安装 MeiliSearch [#2-extractinstall-meilisearch]
    - Linux/macOS: 将下载的归档文件解压到您选择的目录中。
    - Windows：将 ZIP 文件解压到您选择的目录中。

## 3. 使 MeiliSearch 二进制文件可执行 (Linux/macOS) [#3-make-the-meilisearch-binary-executable-linuxmacos]
    - 打开终端并导航到您解压 MeiliSearch 的目录。
    - 运行以下命令使二进制文件可执行：
        ```sh filename="Make the MeiliSearch Binary Executable"
        chmod +x meilisearch
        ```

## 4. 生成主密钥 [#4-generate-a-master-key]
    - 打开终端（或 Windows 上的命令提示符）并导航到 MeiliSearch 目录。
    - 运行以下命令以生成 Master Key：
        ```sh filename="Generate a Master Key"
        ./meilisearch --generate-master-key
        ```
    - 复制生成的 Master Key，因为稍后会用到它。

## 5. 启动 MeiliSearch [#5-start-meilisearch]
    - 在同一个终端中，运行以下命令来启动 MeiliSearch，并将 `<your_master_key>` 替换为您在上一步中生成的 Master Key：
        ```sh filename="Start MeiliSearch"
        ./meilisearch --master-key=<your_master_key>
        ```
    - MeiliSearch 现在将在默认端口 (`7700`) 上运行。

## 6. 更新 LibreChat 的环境变量 [#6-update-librechats-environment-variables]
    - 打开 LibreChat 项目根目录下的 `.env` 文件。
    - 添加或更新以下行以配置您的 MeiliSearch：

        ```sh filename=".env"
        SEARCH=true
        MEILI_NO_ANALYTICS=true
        MEILI_HOST=http://localhost:7700
        MEILI_MASTER_KEY=<your_master_key>
        ```

    - 将 `<your_master_key>` 替换为您之前生成的 Master Key。

## 7. 启动/重启 LibreChat [#7-startrestart-librechat]
    - 启动或重启您的 LibreChat 应用程序。

就是这样！随着 MeiliSearch 的设置和配置完成，LibreChat 现在应该已经启用了对话搜索功能，允许用户执行全文搜索、利用容错机制，并获得即时的历史对话搜索结果体验。

注意：请确保 MeiliSearch 进程在后台运行，以使搜索功能正常工作。您可能需要设置脚本或服务来让 MeiliSearch 持久运行。

## 8. 可选：在多节点设置中禁用 Meilisearch/数据库同步 [#8-optional-disable-meilisearchdatabase-sync-in-a-multi-node-setup]

如果您在节点集群或多节点设置中运行 LibreChat，并希望禁用 MeiliSearch 索引同步，可以在您的 `.env` 文件中将 `MEILI_NO_SYNC` 环境变量设置为 `true`。这将防止您的数据库文档在多个 LibreChat 实例之间进行冗余同步，因为通常只需要一个实例负责索引，此举也能避免不必要的资源消耗。

        ```sh filename=".env"
        MEILI_NO_SYNC=true
        ```

## 9. 重置 MeiliSearch 同步 [#9-resetting-meilisearch-synchronization]

如果您遇到 MeiliSearch 数据被删除或损坏的问题，或者 LibreChat 在数据尚未同步完成时误认为所有数据已同步（例如，在升级 MeiliSearch 或删除其数据文件后），您可以使用重置同步脚本来强制进行全量重新同步。

此脚本会重置 MongoDB 中的同步标志，这将触发 LibreChat 在下次启动或同步检查时将所有对话和消息重新索引到 MeiliSearch 中。

### 运行重置脚本 [#running-the-reset-script]

```bash filename="To reset MeiliSearch synchronization flags, run:"
# Local Development
npm run reset-meili-sync

# Docker (default setup)
docker compose exec api npm run reset-meili-sync

# Docker (deployment setup)
docker exec -it LibreChat-API /bin/sh -c "cd .. && npm run reset-meili-sync"
```

### 脚本的功能 [#what-the-script-does]

1. 将 MongoDB 中所有消息和对话的 `_meiliIndex` 标志重置为 `false`
2. 显示已重置的文档数量以及待同步的文档总数
3. 提供用于控制同步行为的高级选项

### 何时使用此脚本 [#when-to-use-this-script]

- 在删除 MeiliSearch 数据文件后
- 当将 MeiliSearch 升级到需要重新索引的新版本时
- 如果 LibreChat 显示对话为“已完全同步”，但 MeiliSearch 中缺少数据
- 在恢复没有对应 MeiliSearch 数据的 MongoDB 备份后

### 高级同步选项 [#advanced-sync-options]

运行重置脚本后，您可以通过以下环境变量来控制同步行为：

- `MEILI_SYNC_BATCH_SIZE`: 每批同步的文档数量（默认值：100）
- `MEILI_SYNC_DELAY_MS`：同步批次之间的延迟（以毫秒为单位，默认值：100ms）
- `MEILI_SYNC_THRESHOLD`：触发同步前未同步文档的最小数量（默认值：1000）

注意：重置同步标志后，您需要重启 LibreChat 以开始重新同步。
